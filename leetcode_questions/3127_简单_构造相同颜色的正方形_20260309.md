## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

**核心要求**  
给定一个 3×3 的字符矩阵，每个格子是 `'B'`（黑色）或 `'W'`（白色）。最多可以修改一个格子的颜色，判断能否在矩阵中找到一个 2×2 的子正方形，其四个格子颜色完全相同。

**关键约束**  
- 矩阵固定为 3×3，因此只有 4 个可能的 2×2 正方形（左上、右上、左下、右下）。  
- 最多只能修改一个格子（也可以不修改）。  
- 修改后，只要存在一个 2×2 正方形颜色完全相同即可。

**输入输出**  
- 输入：`grid [][]byte` 或 `[][]string`（LeetCode 通常用 `[][]byte`）。  
- 输出：`bool`，表示是否可行。

## 2️⃣ 解题思路

### 解法一：暴力枚举修改位置
**核心思想**  
由于矩阵很小（3×3），我们可以枚举所有可能的修改位置（包括不修改），然后检查是否存在一个 2×2 正方形颜色相同。

**步骤**  
1. 遍历所有格子（共 9 个），对每个格子尝试修改为另一种颜色（或不修改）。  
2. 对于每种修改后的矩阵，检查 4 个可能的 2×2 正方形是否颜色相同。  
3. 如果找到符合条件的正方形，返回 `true`；否则返回 `false`。

**注意事项**  
- 修改后要恢复原矩阵，或者使用副本。  
- 不修改的情况也要检查。

### 解法二：直接检查 2×2 正方形
**核心思想**  
因为最多只能修改一个格子，所以我们可以直接检查每个 2×2 正方形，看它能否通过修改至多一个格子变成全同色。

**步骤**  
1. 遍历 4 个 2×2 正方形。  
2. 对每个正方形，统计其中 `'B'` 和 `'W'` 的数量。  
3. 如果某个正方形已经全同色（`countB == 4` 或 `countW == 4`），直接返回 `true`。  
4. 否则，如果某个正方形中某一种颜色出现 3 次，另一种颜色出现 1 次，那么修改那个不同的格子即可，返回 `true`。  
5. 如果所有正方形都不满足条件，返回 `false`。

**为什么这样可行？**  
- 如果正方形已经全同色，不需要修改。  
- 如果正方形中有 3 个相同颜色，1 个不同颜色，修改那个不同的格子即可。  
- 如果正方形中有 2 个 `'B'` 和 2 个 `'W'`，则至少需要修改两个格子才能全同色，不符合条件。

## 3️⃣ 代码实现

### 解法一：暴力枚举修改位置
```go
func canMakeSquare(grid [][]byte) bool {
    // 复制原矩阵，避免修改原数据
    copyGrid := func() [][]byte {
        g := make([][]byte, 3)
        for i := range g {
            g[i] = make([]byte, 3)
            copy(g[i], grid[i])
        }
        return g
    }

    // 检查是否存在全同色的 2x2 正方形
    check := func(g [][]byte) bool {
        // 四个可能的 2x2 正方形的左上角坐标
        positions := [][]int{{0, 0}, {0, 1}, {1, 0}, {1, 1}}
        for _, pos := range positions {
            r, c := pos[0], pos[1]
            color := g[r][c]
            if g[r][c+1] == color && g[r+1][c] == color && g[r+1][c+1] == color {
                return true
            }
        }
        return false
    }

    // 先检查不修改的情况
    if check(grid) {
        return true
    }

    // 枚举每个格子，尝试修改颜色
    for i := 0; i < 3; i++ {
        for j := 0; j < 3; j++ {
            g := copyGrid()
            // 修改颜色
            if g[i][j] == 'B' {
                g[i][j] = 'W'
            } else {
                g[i][j] = 'B'
            }
            if check(g) {
                return true
            }
        }
    }
    return false
}
```

### 解法二：直接检查 2×2 正方形
```go
func canMakeSquare(grid [][]byte) bool {
    // 四个可能的 2x2 正方形的左上角坐标
    positions := [][]int{{0, 0}, {0, 1}, {1, 0}, {1, 1}}
    for _, pos := range positions {
        r, c := pos[0], pos[1]
        // 统计当前正方形中 'B' 的数量
        countB := 0
        if grid[r][c] == 'B' {
            countB++
        }
        if grid[r][c+1] == 'B' {
            countB++
        }
        if grid[r+1][c] == 'B' {
            countB++
        }
        if grid[r+1][c+1] == 'B' {
            countB++
        }
        // 如果全黑或全白，直接返回 true
        if countB == 4 || countB == 0 {
            return true
        }
        // 如果有 3 个相同颜色，1 个不同颜色，修改一个格子即可
        if countB == 3 || countB == 1 {
            return true
        }
        // 如果 countB == 2，则需要修改至少两个格子，不符合条件
    }
    return false
}
```

## 4️⃣ 复杂度分析

### 解法一：暴力枚举修改位置
- ⏱️ **时间复杂度**：O(9 × 4) = O(36) ≈ O(1)。  
  枚举 9 个格子（包括不修改），每次检查 4 个正方形，每个正方形检查 4 个格子，总操作次数为常数。
- 💾 **空间复杂度**：O(1)（如果不考虑复制矩阵，则为 O(9) 的额外空间，仍是常数）。

### 解法二：直接检查 2×2 正方形
- ⏱️ **时间复杂度**：O(4 × 4) = O(16) ≈ O(1)。  
  遍历 4 个正方形，每个正方形统计 4 个格子。
- 💾 **空间复杂度**：O(1)，只用了常数个变量。

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 暴力枚举 | 思路直接，易于理解 | 需要复制矩阵，代码稍长 | 通用性强，适用于任意大小的矩阵（但复杂度会变高） |
| 直接检查 | 逻辑简洁，效率高 | 只适用于本题的特定条件（最多修改一个格子） | 本题的最优解 |

**推荐解法**：解法二（直接检查 2×2 正方形）。  
- 代码更简洁，效率更高。  
- 充分利用了题目条件（最多修改一个格子），逻辑更清晰。
