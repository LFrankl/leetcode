## 1️⃣ 题意理解

**题目重述**：
给定一个整数数组 `nums`，需要找到一个最长的子序列（不要求连续），使得子序列中相邻元素的绝对差构成的序列是**非递增**的（即后一个绝对差 ≤ 前一个绝对差）。

**关键约束**：
- 子序列不要求连续，但必须保持原数组中的相对顺序。
- 绝对差序列必须非递增。
- 数组长度 ≤ 10^4，元素值 ≤ 300。

**边界情况**：
- 数组长度为 2 时，直接返回 2。
- 所有元素相等时，绝对差为 0，满足非递增。
- 数组严格递增或递减时，需要动态规划处理。

**输入输出**：
- 输入：整数数组 `nums`
- 输出：最长满足条件的子序列长度

---

## 2️⃣ 解题思路

### 解法一：动态规划（朴素 DP）
**核心思想**：
定义 `dp[i][d]` 表示以 `nums[i]` 结尾，且最后一个绝对差为 `d` 的最长子序列长度。由于绝对差需要非递增，我们需要记录最后一个绝对差的值。

**步骤**：
1. 初始化 `dp[i][d] = 1`（至少包含自己）。
2. 遍历 `i` 从 0 到 n-1，对于每个 `i`，遍历 `j` 从 0 到 i-1：
   - 计算 `diff = |nums[i] - nums[j]|`
   - 对于所有可能的绝对差 `prevDiff`（0 ≤ prevDiff ≤ diff），更新：
     `dp[i][diff] = max(dp[i][diff], dp[j][prevDiff] + 1)`
3. 最终答案是所有 `dp[i][d]` 的最大值。

**坑点**：
- 绝对差最大为 299（因为元素值 ≤ 300），所以 `d` 的范围是 0~299。
- 需要遍历 `prevDiff`，导致复杂度较高。

---

### 解法二：动态规划 + 前缀最大值优化
**核心思想**：
在解法一的基础上，注意到对于固定的 `j`，我们需要的是 `dp[j][prevDiff]` 在 `prevDiff ≥ diff` 时的最大值。因此可以维护一个前缀最大值数组 `maxDiff[j][d]` 表示以 `nums[j]` 结尾，且最后一个绝对差 ≥ d 的最长子序列长度。

**步骤**：
1. 定义 `dp[i][d]` 同上。
2. 定义 `maxDiff[i][d]` 表示以 `nums[i]` 结尾，且最后一个绝对差 ≥ d 的最长子序列长度。
3. 遍历 `i`，对于每个 `i`，遍历 `j`：
   - 计算 `diff = |nums[i] - nums[j]|`
   - 从 `maxDiff[j][diff]` 中获取最优值（因为需要 `prevDiff ≥ diff`）
   - 更新 `dp[i][diff] = max(dp[i][diff], maxDiff[j][diff] + 1)`
4. 更新 `maxDiff[i][d]` 为后缀最大值。

**优化**：
- 将 `maxDiff[i][d]` 改为从大往小遍历，避免重复计算。

---

### 解法三：动态规划 + 差分数组优化（推荐）
**核心思想**：
由于元素值 ≤ 300，绝对差 ≤ 299，我们可以将状态定义为 `dp[i][d]` 表示以 `nums[i]` 结尾，且最后一个绝对差为 `d` 的最长子序列长度。对于每个 `i`，我们只需要知道所有 `j < i` 的 `dp[j][prevDiff]` 在 `prevDiff ≥ diff` 时的最大值。这可以通过维护一个二维数组 `best[j][d]` 实现，其中 `best[j][d]` 表示以 `nums[j]` 结尾，且最后一个绝对差 ≥ d 的最长子序列长度。

**步骤**：
1. 初始化 `dp[i][d] = 1`。
2. 对于每个 `i`，遍历 `j` 从 0 到 i-1：
   - 计算 `diff = |nums[i] - nums[j]|`
   - 从 `best[j][diff]` 中获取最大值（因为需要 `prevDiff ≥ diff`）
   - 更新 `dp[i][diff] = max(dp[i][diff], best[j][diff] + 1)`
3. 更新 `best[i][d]`：
   - 从 d = 299 到 0，`best[i][d] = max(dp[i][d], best[i][d+1])`
4. 最终答案是所有 `dp[i][d]` 的最大值。

## 3️⃣ 代码实现

### 解法三：动态规划 + 差分数组优化
```go
func longestDecreasingDiffSubsequence(nums []int) int {
    n := len(nums)
    if n == 0 {
        return 0
    }
    // 最大绝对差为 299
    maxDiff := 300
    // dp[i][d] 表示以 nums[i] 结尾，且最后一个绝对差为 d 的最长子序列长度
    dp := make([][]int, n)
    // best[i][d] 表示以 nums[i] 结尾，且最后一个绝对差 >= d 的最长子序列长度
    best := make([][]int, n)
    for i := 0; i < n; i++ {
        dp[i] = make([]int, maxDiff)
        best[i] = make([]int, maxDiff)
        for d := 0; d < maxDiff; d++ {
            dp[i][d] = 1 // 至少包含自己
        }
    }

    ans := 1
    for i := 0; i < n; i++ {
        for j := 0; j < i; j++ {
            diff := abs(nums[i] - nums[j])
            // 从 best[j][diff] 中获取最优值（prevDiff >= diff）
            dp[i][diff] = max(dp[i][diff], best[j][diff]+1)
        }
        // 更新 best[i][d] 为后缀最大值
        best[i][maxDiff-1] = dp[i][maxDiff-1]
        for d := maxDiff - 2; d >= 0; d-- {
            best[i][d] = max(dp[i][d], best[i][d+1])
        }
        // 更新答案
        for d := 0; d < maxDiff; d++ {
            ans = max(ans, dp[i][d])
        }
    }
    return ans
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

## 4️⃣ 复杂度分析

**解法三**：
- ⏱️ **时间复杂度**：O(n² * D)，其中 D = 300（最大绝对差）。内层循环中更新 `dp[i][diff]` 是 O(1)，更新 `best[i][d]` 是 O(D)，总复杂度 O(n² * D)。在本题中 n ≤ 10^4，D=300，最坏情况可能达到 10^8，但实际常数较小，可以通过。
- 💾 **空间复杂度**：O(n * D)，存储 `dp` 和 `best` 数组。

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 朴素 DP | 思路直观，易于实现 | 时间复杂度高 O(n³ * D) | 小规模数据 |
| 前缀最大值优化 | 优化了 prevDiff 的查找 | 实现稍复杂 | 中等规模数据 |
| 差分数组优化（推荐） | 利用元素值范围小的特点，高效处理绝对差 | 空间占用较大 | 本题数据范围（元素值 ≤ 300） |

**推荐解法三**，因为本题元素值范围小（≤300），绝对差范围有限，可以利用差分数组优化，将时间复杂度中的 D 因子降低到常数级别。
