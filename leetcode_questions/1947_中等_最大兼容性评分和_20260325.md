## 题目描述

<p>有一份由 <code>n</code> 个问题组成的调查问卷，每个问题的答案要么是 <code>0</code>（no，否），要么是 <code>1</code>（yes，是）。</p>

<p>这份调查问卷被分发给 <code>m</code> 名学生和 <code>m</code> 名导师，学生和导师的编号都是从 <code>0</code> 到 <code>m - 1</code> 。学生的答案用一个二维整数数组 <code>students</code> 表示，其中 <code>students[i]</code> 是一个整数数组，包含第 <code>i</code> 名学生对调查问卷给出的答案（<strong>下标从 0 开始</strong>）。导师的答案用一个二维整数数组 <code>mentors</code> 表示，其中 <code>mentors[j]</code> 是一个整数数组，包含第 <code>j</code> 名导师对调查问卷给出的答案（<strong>下标从 0 开始</strong>）。</p>

<p>每个学生都会被分配给 <strong>一名</strong> 导师，而每位导师也会分配到 <strong>一名</strong> 学生。配对的学生与导师之间的兼容性评分等于学生和导师答案相同的次数。</p>

<ul>
	<li>例如，学生答案为<code>[1, <strong><em>0</em></strong>, <strong><em>1</em></strong>]</code> 而导师答案为 <code>[0, <strong><em>0</em></strong>, <strong><em>1</em></strong>]</code> ，那么他们的兼容性评分为 2 ，因为只有第二个和第三个答案相同。</li>
</ul>

<p>请你找出最优的学生与导师的配对方案，以 <strong>最大程度上</strong> 提高 <strong>兼容性评分和</strong> 。</p>

<p>给你 <code>students</code> 和 <code>mentors</code> ，返回可以得到的<em> </em><strong>最大兼容性评分和</strong> 。</p>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre><strong>输入：</strong>students = [[1,1,0],[1,0,1],[0,0,1]], mentors = [[1,0,0],[0,0,1],[1,1,0]]
<strong>输出：</strong>8
<strong>解释：</strong>按下述方式分配学生和导师：
- 学生 0 分配给导师 2 ，兼容性评分为 3 。
- 学生 1 分配给导师 0 ，兼容性评分为 2 。
- 学生 2 分配给导师 1 ，兼容性评分为 3 。
最大兼容性评分和为 3 + 2 + 3 = 8 。</pre>

<p><strong>示例 2：</strong></p>

<pre><strong>输入：</strong>students = [[0,0],[0,0],[0,0]], mentors = [[1,1],[1,1],[1,1]]
<strong>输出：</strong>0
<strong>解释：</strong>任意学生与导师配对的兼容性评分都是 0 。
</pre>

<p>&nbsp;</p>

<p><strong>提示：</strong></p>

<ul>
	<li><code>m == students.length == mentors.length</code></li>
	<li><code>n == students[i].length == mentors[j].length</code></li>

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

**核心要求**：
- 有 `m` 个学生和 `m` 个导师，每个学生/导师对 `n` 个问题给出 0/1 答案。
- 需要将学生和导师一一配对。
- 每对（学生 i，导师 j）的兼容性评分 = 他们答案相同的题目数量。
- 目标是找到一种配对方式，使得所有配对的兼容性评分之和最大。

**关键约束**：
- `1 <= m, n <= 8`，数据规模很小。
- 每个答案数组长度相同，且元素为 0 或 1。
- 学生和导师数量相等，必须一一配对。

**边界情况**：
- `m = 1` 时只有一种配对方式。
- 所有答案完全相同或完全不同时，评分为 `m * n` 或 0。
- 由于 `m, n` 最大为 8，暴力搜索是可行的。

**输入输出**：
- 输入：`students [][]int`，`mentors [][]int`
- 输出：最大兼容性评分和（整数）

## 2️⃣ 解题思路

### 解法一：回溯法（DFS + 状态压缩）
**核心思想**：
- 将配对问题看作一个排列问题：为每个学生选择一个导师，且每个导师只能被选一次。
- 使用回溯法枚举所有可能的配对方案，计算每种方案的评分和，取最大值。
- 由于 `m <= 8`，总排列数为 `m!`，最大为 `40320`，完全可行。

**步骤**：
1. 预处理一个 `score[i][j]` 矩阵，表示学生 i 与导师 j 的兼容性评分。
2. 使用 DFS 递归地为每个学生选择导师，用 `used` 数组标记导师是否已被选。
3. 递归到第 `i` 个学生时，遍历所有未被选的导师 j，选择后继续递归。
4. 当所有学生都配对完成时，更新最大评分和。

**注意事项**：
- 注意递归边界条件。
- 可以加入剪枝优化（但本题规模小，非必需）。

### 解法二：状态压缩动态规划（DP + Bitmask）
**核心思想**：
- 将导师的选择状态压缩为一个二进制数 `mask`，其中第 `j` 位为 1 表示导师 j 已被选。
- 定义 `dp[mask]` 表示当前已选导师状态为 `mask` 时的最大评分和。
- 状态转移：`dp[mask] = max(dp[mask], dp[mask ^ (1<<j)] + score[count][j])`，其中 `count` 是已选学生数（即 `mask` 中 1 的个数），`j` 是当前选择的导师。

**步骤**：
1. 预处理 `score[i][j]` 矩阵。
2. 初始化 `dp[0] = 0`。
3. 遍历所有 `mask`，对于每个 `mask`，计算已选学生数 `cnt`，然后遍历所有导师 j，如果 j 未被选（`mask` 的第 j 位为 0），则尝试选择 j 进行转移。
4. 最终 `dp[(1<<m)-1]` 即为答案。

**注意事项**：
- 注意 `mask` 的遍历顺序，确保转移时子状态已计算（通常从小到大遍历即可）。
- `cnt` 可以通过 `bits.OnesCount(uint(mask))` 快速计算。

## 3️⃣ 代码实现

### 解法一：回溯法
```go
func maxCompatibilitySum(students [][]int, mentors [][]int) int {
    m, n := len(students), len(students[0])
    // 预处理评分矩阵
    score := make([][]int, m)
    for i := 0; i < m; i++ {
        score[i] = make([]int, m)
        for j := 0; j < m; j++ {
            s := 0
            for k := 0; k < n; k++ {
                if students[i][k] == mentors[j][k] {
                    s++
                }
            }
            score[i][j] = s
        }
    }

    used := make([]bool, m)
    maxSum := 0
    var dfs func(i, curSum int)
    dfs = func(i, curSum int) {
        if i == m {
            if curSum > maxSum {
                maxSum = curSum
            }
            return
        }
        for j := 0; j < m; j++ {
            if !used[j] {
                used[j] = true
                dfs(i+1, curSum+score[i][j])
                used[j] = false
            }
        }
    }
    dfs(0, 0)
    return maxSum
}
```

### 解法二：状态压缩 DP
```go
func maxCompatibilitySum(students [][]int, mentors [][]int) int {
    m, n := len(students), len(students[0])
    // 预处理评分矩阵
    score := make([][]int, m)
    for i := 0; i < m; i++ {
        score[i] = make([]int, m)
        for j := 0; j < m; j++ {
            s := 0
            for k := 0; k < n; k++ {
                if students[i][k] == mentors[j][k] {
                    s++
                }
            }
            score[i][j] = s
        }
    }

    // dp[mask] 表示当前已选导师状态为 mask 时的最大评分和
    dp := make([]int, 1<<m)
    for mask := 0; mask < (1 << m); mask++ {
        // 计算当前已选学生数（即 mask 中 1 的个数）
        cnt := bits.OnesCount(uint(mask))
        // 为第 cnt 个学生选择导师
        for j := 0; j < m; j++ {
            if mask&(1<<j) == 0 { // 导师 j 未被选
                newMask := mask | (1 << j)
                if dp[newMask] < dp[mask]+score[cnt][j] {
                    dp[newMask] = dp[mask] + score[cnt][j]
                }
            }
        }
    }
    return dp[(1<<m)-1]
}
```

## 4️⃣ 复杂度分析

### 解法一：回溯法
- ⏱️ **时间复杂度**：O(m! * n)
  - 共有 m! 种配对方案，每种方案需要计算 m 对评分，每对评分需要 O(n) 时间。
  - 由于我们预处理了评分矩阵，实际递归中每层只需 O(m) 时间选择导师，总时间 O(m! * m)。
  - 预处理评分矩阵 O(m² * n)。
- 💾 **空间复杂度**：O(m² + m)
  - 存储评分矩阵 O(m²)，递归栈深度 O(m)，used 数组 O(m)。

### 解法二：状态压缩 DP
- ⏱️ **时间复杂度**：O(m² * 2^m + m² * n)
  - 预处理评分矩阵 O(m² * n)。
  - DP 状态数 2^m，每个状态需要遍历 m 个导师进行转移，总 O(m * 2^m)。
- 💾 **空间复杂度**：O(m² + 2^m)
  - 评分矩阵 O(m²)，dp 数组 O(2^m)。

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 回溯法 | 思路直观，易于实现 | 当 m 较大时（如 m>10）会超时 | m ≤ 8 的小规模问题 |
| 状态压缩 DP | 效率更高，可处理稍大的 m（如 m≤16） | 代码稍复杂，需要理解位运算 | m ≤ 16 的中等规模问题 |

**推荐**：
- 本题 m ≤ 8，两种方法均可，但状态压缩 DP 更高效且更具通用性。
- 如果对位运算不熟悉，回溯法更易实现。
