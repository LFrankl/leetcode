# 使三个字符串相等 — 详细解题分析

## 题目分析

### 核心观察

1. **操作性质**：只能从右端删除字符，意味着最终结果必须是每个原始字符串的**前缀**
2. **可行性条件**：三个字符串必须共享至少长度为 1 的公共前缀（因为操作要求字符串长度 ≥ 2，无法将字符串删至空串）
3. **最优策略**：找到三个字符串的**最长公共前缀 (LCP)**，将每个字符串缩减到该前缀，这样删除的字符最少，操作次数最少

### 数学公式

设 LCP 长度为 `L`，则最小操作次数为：

``operations = (len(s1) - L) + (len(s2) - L) + (len(s3) - L)``

若 `L = 0`（首字符不匹配），返回 `-1`。

### 验证示例

| 示例 | 字符串 | LCP | 操作数 | 结果 |
|------|--------|-----|--------|------|
| 1 | "abc", "abb", "ab" | "ab" (L=2) | (3-2)+(3-2)+(2-2) = 2 | ✓ |
| 2 | "dac", "bac", "cac" | "" (L=0) | — | -1 ✓ |

## 解法一：逐字符比较（最优解法）

**思路**：从左到右逐字符比较三个字符串，找到最长公共前缀的长度。

```go
func findMinimumOperations(s1 string, s2 string, s3 string) int {
    minLen := min(len(s1), len(s2), len(s3))
    lcpLen := 0
    for i := 0; i < minLen; i++ {
        if s1[i] == s2[i] && s2[i] == s3[i] {
            lcpLen++
        } else {
            break
        }
    }
    if lcpLen == 0 {
        return -1
    }
    return (len(s1) - lcpLen) + (len(s2) - lcpLen) + (len(s3) - lcpLen)
}
```

```cpp
public:
    int findMinimumOperations(string s1, string s2, string s3) {
        int minLen = min({(int)s1.size(), (int)s2.size(), (int)s3.size()});
        int lcpLen = 0;
        for (int i = 0; i < minLen; i++) {
            if (s1[i] == s2[i] && s2[i] == s3[i]) {
                lcpLen++;
            } else {
                break;
            }
        }
        if (lcpLen == 0) return -1;
        return (s1.size() - lcpLen) + (s2.size() - lcpLen) + (s3.size() - lcpLen);
    }
};
```

|--------|------|
| 时间 | O(min(n₁, n₂, n₃))，最多比较最短字符串的长度 |
| 空间 | O(1)，仅用计数器 |

## 解法二：字符串切片 + 逐步比较

**思路**：逐长度递增地取出三个字符串的前缀，检查是否相等，找到最长匹配长度后立即终止。

```go
func findMinimumOperations(s1 string, s2 string, s3 string) int {
    minLen := min(len(s1), len(s2), len(s3))
    maxLCP := 0
    for l := 1; l <= minLen; l++ {
        // Go 的字符串切片不拷贝底层数据，效率较高
        if s1[:l] == s2[:l] && s2[:l] == s3[:l] {
            maxLCP = l
        } else {
            break // 一旦失配，前缀不可能再增长
        }
    }
    if maxLCP == 0 {
        return -1
    }
    return (len(s1) - maxLCP) + (len(s2) - maxLCP) + (len(s3) - maxLCP)
}
```

```cpp
class Solution {
public:
    int findMinimumOperations(string s1, string s2, string s3) {
        int minLen = min({(int)s1.size(), (int)s2.size(), (int)s3.size()});
        int maxLCP = 0;
        for (int l = 1; l <= minLen; l++) {
            // C++ substr 会产生拷贝，但数据量小可忽略
            if (s1.substr(0, l) == s2.substr(0, l) &&
                s2.substr(0, l) == s3.substr(0, l)) {
                maxLCP = l;
            } else {
                break;
            }
        }
        if (maxLCP == 0) return -1;
        return (s1.size() - maxLCP) + (s2.size() - maxLCP) + (s3.size() - maxLCP);
    }
};
```

| 复杂度 | 分析 |
|--------|------|
| 时间 | O(L²)，每次切片比较耗时 O(l)，总计 O(1+2+…+L) ≈ O(L²) |
| 空间 | Go: O(1)（切片共享底层数据）；C++: O(L)（substr 拷贝） |

> **注意**：虽然理论复杂度更高，但 L 最大为 100，实际运行时间差异极小。

## 解法三：两步分解法 — 先求两串 LCP，再验证第三串

**思路**：分解问题为两步——先找 s1 和 s2 的最长公共前缀，再将该前缀与 s3 比较，确定最终 LCP 长度。

```go
func lcpLength(a, b string) int {
    minLen := min(len(a), len(b))
    length := 0
    for i := 0; i < minLen; i++ {
        if a[i] == b[i] {
            length++
        } else {
            break
        }
    }
    return length
}

func findMinimumOperations(s1 string, s2 string, s3 string) int {
    // 第一步：找 s1 和 s2 的 LCP
    l12 := lcpLength(s1, s2)
    if l12 == 0 {
        return -1
    }
    // 第二步：用 s1 的前 l12 个字符与 s3 比较
    l123 := lcpLength(s1[:l12], s3)
    if l123 == 0 {
        return -1
    }
    return (len(s1) - l123) + (len(s2) - l123) + (len(s3) - l123)
}
```

```cpp
class Solution {
public:
    int lcpLength(const string& a, const string& b) {
        int minLen = min(a.size(), b.size());
        int len = 0;
        for (int i = 0; i < minLen; i++) {
            if (a[i] == b[i]) len++;
            else break;
        }
        return len;
    }

    int findMinimumOperations(string s1, string s2, string s3) {
        // 第一步：找 s1 和 s2 的 LCP
        int l12 = lcpLength(s1, s2);
        if (l12 == 0) return -1;
        // 第二步：与 s3 比较
        int l123 = lcpLength(s1.substr(0, l12), s3);
        if (l123 == 0) return -1;
        return (s1.size() - l123) + (s2.size() - l123) + (s3.size() - l123);
    }
};
```

| 复杂度 | 分析 |
|--------|------|
| 时间 | O(min(n₁,n₂) + min(L₁₂, n₃)) = O(min(n₁, n₂, n₃)) |
| 空间 | O(1)（Go）；O(L₁₂)（C++ substr） |

## 三种解法对比

| 维度 | 解法一（逐字符） | 解法二（切片比较） | 解法三（两步分解） |
|------|------------------|-------------------|-------------------|
| **时间复杂度** | O(min(n₁,n₂,n₃)) | O(L²) | O(min(n₁,n₂,n₃)) |
| **空间复杂度** | O(1) | O(1)~O(L) | O(1)~O(L) |
| **代码简洁度** | ★★★★★ | ★★★★ | ★★★ |
| **可扩展性** | 一般（固定三串） | 一般 | ★★★★★（可推广到 K 个串） |
| **推荐程度** | ✅ **首选** | 可用但非最优 | 适合推广到多串场景 |

### 关键结论

- **日常做题**：解法一最优——代码最简洁、复杂度最低、一步到位
- **扩展到 K 个字符串**：解法三更具优势——可以复用 `lcpLength` 函数，对 K 个字符串依次两两求 LCP
- **解法二**虽然理论复杂度略高，但在本题约束下（字符串长度 ≤ 100）完全够用，且思路直观易懂

### 易错点提醒

1. **空串陷阱**：不能将字符串删为空串（操作要求长度 ≥ 2），所以 LCP 长度必须 ≥ 1
2. **边界情况**：当某个字符串长度为 1 时，不能再对其进行操作，其他字符串也只能缩减到长度 1
3. **提前终止**：一旦发现字符不匹配立即 `break`，无需继续比较
