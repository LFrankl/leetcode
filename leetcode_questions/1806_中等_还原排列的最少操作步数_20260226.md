## 题目描述

<p>给你一个偶数 <code>n</code>​​​​​​ ，已知存在一个长度为 <code>n</code> 的排列 <code>perm</code> ，其中 <code>perm[i] == i</code>​（下标 <strong>从 0 开始</strong> 计数）。</p>

<p>一步操作中，你将创建一个新数组 <code>arr</code> ，对于每个 <code>i</code> ：</p>

<ul>
	<li>如果 <code>i % 2 == 0</code> ，那么 <code>arr[i] = perm[i / 2]</code></li>
	<li>如果 <code>i % 2 == 1</code> ，那么 <code>arr[i] = perm[n / 2 + (i - 1) / 2]</code></li>
</ul>

<p>然后将 <code>arr</code>​​ 赋值​​给 <code>perm</code> 。</p>

<p>要想使 <code>perm</code> 回到排列初始值，至少需要执行多少步操作？返回最小的 <strong>非零</strong> 操作步数。</p>

<p> </p>

<p><strong>示例 1：</strong></p>

<pre>
<strong>输入：</strong>n = 2
<strong>输出：</strong>1
<strong>解释：</strong>最初，perm = [0,1]
第 1 步操作后，perm = [0,1]
所以，仅需执行 1 步操作</pre>

<p><strong>示例 2：</strong></p>

<pre>
<strong>输入：</strong>n = 4
<strong>输出：</strong>2
<strong>解释：</strong>最初，perm = [0,1,2,3]
第 1 步操作后，perm = [0,2,1,3]
第 2 步操作后，perm = [0,1,2,3]
所以，仅需执行 2 步操作</pre>

<p><strong>示例 3：</strong></p>

<pre>
<strong>输入：</strong>n = 6
<strong>输出：</strong>4
</pre>

<p> </p>

<p><strong>提示：</strong></p>

<ul>
	<li><code>2 <= n <= 1000</code></li>
	<li><code>n</code>​​​​​​ 是一个偶数</li>
</ul>

## 题目分析

题目描述了一个特定的变换规则：
- 初始排列：`perm = [0, 1, 2, ..., n-1]`
- 每次操作根据规则生成新数组 `arr`：
  - 如果 `i` 是偶数：`arr[i] = perm[i/2]`
  - 如果 `i` 是奇数：`arr[i] = perm[n/2 + (i-1)/2]`
- 然后将 `arr` 赋值给 `perm`

我们需要找到最小的正整数 `k`，使得经过 `k` 次变换后，`perm` 恢复到初始状态。

## 关键观察

1. **变换规则的本质**：
   - 偶数位置 `i` 的元素来自原数组前半部分 `perm[i/2]`
   - 奇数位置 `i` 的元素来自原数组后半部分 `perm[n/2 + (i-1)/2]`
   - 这实际上是一个**洗牌操作**：将数组分成前后两半，然后交错合并

2. **数学表示**：
   - 对于位置 `i`，变换后的新位置 `j` 满足：
     - 如果 `i < n/2`，则 `j = 2*i`
     - 如果 `i ≥ n/2`，则 `j = 2*(i - n/2) + 1`
   - 这可以统一表示为：`j = (2*i) % (n-1)`，对于 `i ∈ [0, n-2]`
   - 注意：最后一个元素 `n-1` 始终保持在位置 `n-1`

3. **问题转化**：
   - 每个元素的位置变化形成一个置换（排列）
   - 我们需要找到这个置换的阶（所有循环长度的最小公倍数）
   - 特别地，元素 `0` 和 `n-1` 的位置不变，所以只需考虑 `1` 到 `n-2` 的元素

## 解法一：模拟变换过程

### 思路
直接模拟变换过程，直到数组恢复到初始状态。

### 复杂度分析
- 时间复杂度：O(n*k)，其中 k 是答案
- 空间复杂度：O(n)

### 代码实现

#### Go
```go
func reinitializePermutation(n int) int {
    perm := make([]int, n)
    for i := 0; i < n; i++ {
        perm[i] = i
    }
    
    arr := make([]int, n)
    steps := 0
    
    for {
        // 执行一次变换
        for i := 0; i < n; i++ {
            if i%2 == 0 {
                arr[i] = perm[i/2]
            } else {
                arr[i] = perm[n/2 + (i-1)/2]
            }
        }
        
        steps++
        
        // 检查是否恢复
        restored := true
        for i := 0; i < n; i++ {
            if arr[i] != i {
                restored = false
                break
            }
        }
        
        if restored {
            return steps
        }
        
        // 准备下一次迭代
        copy(perm, arr)
    }
}
```

#### C++
```cpp
class Solution {
public:
    int reinitializePermutation(int n) {
        vector<int> perm(n);
        for (int i = 0; i < n; i++) {
            perm[i] = i;
        }
        
        vector<int> arr(n);
        int steps = 0;
        
        while (true) {
            // 执行一次变换
            for (int i = 0; i < n; i++) {
                if (i % 2 == 0) {
                    arr[i] = perm[i / 2];
                } else {
                    arr[i] = perm[n / 2 + (i - 1) / 2];
                }
            }
            
            steps++;
            
            // 检查是否恢复
            bool restored = true;
            for (int i = 0; i < n; i++) {
                if (arr[i] != i) {
                    restored = false;
                    break;
                }
            }
            
            if (restored) {
                return steps;
            }
            
            // 准备下一次迭代
            perm = arr;
        }
    }
};
```

## 解法二：跟踪单个元素

### 思路
由于变换是确定性的，我们可以只跟踪一个元素（比如元素 `1`）的位置变化，直到它回到原始位置。因为所有元素都在各自的循环中，整个排列恢复的步数是所有循环长度的最小公倍数，但跟踪单个元素可以得到答案。

### 复杂度分析
- 时间复杂度：O(k)，其中 k 是答案
- 空间复杂度：O(1)

### 代码实现

#### Go
```go
func reinitializePermutation(n int) int {
    if n == 2 {
        return 1
    }
    
    steps := 0
    pos := 1  // 跟踪元素1的位置
    
    for {
        // 根据当前位置计算下一个位置
        if pos < n/2 {
            pos = 2 * pos
        } else {
            pos = 2 * (pos - n/2) + 1
        }
        
        steps++
        
        // 当元素1回到位置1时，整个排列恢复
        if pos == 1 {
            return steps
        }
    }
}
```

#### C++
```cpp
class Solution {
public:
    int reinitializePermutation(int n) {
        if (n == 2) return 1;
        
        int steps = 0;
        int pos = 1;  // 跟踪元素1的位置
        
        do {
            // 根据当前位置计算下一个位置
            if (pos < n / 2) {
                pos = 2 * pos;
            } else {
                pos = 2 * (pos - n / 2) + 1;
            }
            steps++;
        } while (pos != 1);
        
        return steps;
    }
};
```

## 解法三：数学公式法

### 思路
观察变换公式：`f(i) = (2*i) % (n-1)`，对于 `i ∈ [1, n-2]`
- 元素 `0` 和 `n-1` 始终不变
- 我们需要找到最小的 `k` 使得 `(2^k * 1) % (n-1) = 1`

这等价于求 `2^k ≡ 1 (mod n-1)` 的最小正整数解 `k`。

### 复杂度分析
- 时间复杂度：O(log n)
- 空间复杂度：O(1)

### 代码实现

#### Go
```go
func reinitializePermutation(n int) int {
    if n == 2 {
        return 1
    }
    
    steps := 1
    powerOfTwo := 2 % (n - 1)
    
    for powerOfTwo != 1 {
        powerOfTwo = (powerOfTwo * 2) % (n - 1)
        steps++
    }
    
    return steps
}
```

#### C++
```cpp
class Solution {
public:
    int reinitializePermutation(int n) {
        if (n == 2) return 1;
        
        int steps = 1;
        int powerOfTwo = 2 % (n - 1);
        
        while (powerOfTwo != 1) {
            powerOfTwo = (powerOfTwo * 2) % (n - 1);
            steps++;
        }
        
        return steps;
    }
};
```

## 解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|-----------|-----------|------|------|
| 模拟变换 | O(n*k) | O(n) | 直观，易于理解 | 效率低，n较大时慢 |
| 跟踪单个元素 | O(k) | O(1) | 效率高，空间优 | 需要数学洞察 |
| 数学公式法 | O(log n) | O(1) | 效率最高 | 需要数学推导，不易理解 |

## 推荐解法

**解法二（跟踪单个元素）** 是最佳选择：
1. 效率高：O(k) 时间复杂度，k 通常远小于 n
2. 空间优：O(1) 空间复杂度
3. 易于实现：代码简洁，逻辑清晰
4. 不需要复杂的数学推导

对于本题，由于 n 最大为 1000，所有解法都能通过，但解法二在效率和实现难度上取得了最佳平衡。
