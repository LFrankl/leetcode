## 题目描述

<p>给你一个下标从 <strong>0</strong>&nbsp;开始且长度为 <code>n</code>&nbsp;的整数数组&nbsp;<code>nums</code>&nbsp;。<strong>分割</strong>&nbsp;数组 <code>nums</code>&nbsp;的方案数定义为符合以下两个条件的 <code>pivot</code>&nbsp;数目：</p>

<ul>
	<li><code>1 &lt;= pivot &lt; n</code></li>
	<li><code>nums[0] + nums[1] + ... + nums[pivot - 1] == nums[pivot] + nums[pivot + 1] + ... + nums[n - 1]</code></li>
</ul>

<p>同时给你一个整数&nbsp;<code>k</code>&nbsp;。你可以将&nbsp;<code>nums</code>&nbsp;中&nbsp;<strong>一个</strong>&nbsp;元素变为&nbsp;<code>k</code>&nbsp;或&nbsp;<strong>不改变</strong>&nbsp;数组。</p>

<p>请你返回在 <strong>至多</strong>&nbsp;改变一个元素的前提下，<strong>最多</strong>&nbsp;有多少种方法 <strong>分割</strong>&nbsp;<code>nums</code>&nbsp;使得上述两个条件都满足。</p>

<p>&nbsp;</p>

## 1️⃣ 题意理解

**题目重述**：
给定一个整数数组 `nums` 和一个整数 `k`，我们可以选择将数组中的**最多一个**元素修改为 `k`（也可以不修改）。要求计算在修改后，数组有多少个位置 `pivot`（1 ≤ pivot < n）满足：`nums[0] + ... + nums[pivot-1] == nums[pivot] + ... + nums[n-1]`。目标是最大化满足条件的 `pivot` 数量。

**关键约束**：
- 只能修改**最多一个**元素（可以不改）
- pivot 的取值范围是 `[1, n-1]`
- 数组长度 n 最大为 10^5，需要 O(n) 或 O(n log n) 算法
- 元素值范围较大，注意整数溢出（使用 int64）

**输入输出**：
- 输入：`nums []int`, `k int`
- 输出：修改后最多可能的方案数（int）

## 2️⃣ 解题思路

### 解法一：前缀和 + 哈希表统计差值变化
**核心思想**：
- 设原数组总和为 `total`，前缀和为 `prefix`，对于每个位置 `i`（作为 pivot），左半部分和为 `prefix[i]`，右半部分和为 `total - prefix[i]`。
- 分割条件：`prefix[i] == total - prefix[i]` → `2 * prefix[i] == total`。
- 如果修改某个位置 `j` 的值为 `k`，则数组总和变化为 `delta = k - nums[j]`，新的总和为 `total + delta`。
- 修改后，对于每个 pivot 位置 `i`，新的分割条件为：
  - 如果 `j < i`（修改在左半部分）：`2 * (prefix[i] + delta) == total + delta`
  - 如果 `j ≥ i`（修改在右半部分）：`2 * prefix[i] == total + delta`
- 我们可以枚举每个位置 `i` 作为 pivot，计算修改每个位置 `j` 后是否满足条件，但这样是 O(n²)，不可行。
- 优化：固定修改位置 `j`，计算它对所有 pivot 的影响。但我们可以反过来，对于每个 pivot 位置 `i`，推导出修改哪个位置 `j` 可以使得条件成立，然后统计每个 `j` 的贡献。
- 具体推导：
  - 设原数组在 pivot `i` 处不满足条件，即 `2 * prefix[i] != total`。
  - 修改后新总和为 `total' = total + delta`，新前缀和为 `prefix'[i]`。
  - 情况1：修改在左半部分（j < i）：
    - `prefix'[i] = prefix[i] + delta`
    - 条件：`2 * (prefix[i] + delta) == total + delta`
    - 解得：`delta = total - 2 * prefix[i]`
    - 即需要将某个左半部分的元素 `nums[j]` 改为 `k`，使得 `k - nums[j] = delta` → `nums[j] = k - delta`
  - 情况2：修改在右半部分（j ≥ i）：
    - `prefix'[i] = prefix[i]`
    - 条件：`2 * prefix[i] == total + delta`
    - 解得：`delta = 2 * prefix[i] - total`
    - 即需要将某个右半部分的元素 `nums[j]` 改为 `k`，使得 `k - nums[j] = delta` → `nums[j] = k - delta`
- 因此，对于每个 pivot `i`，我们可以计算出：
  - 如果修改左半部分，需要左半部分存在值为 `k - (total - 2*prefix[i])` 的元素
  - 如果修改右半部分，需要右半部分存在值为 `k - (2*prefix[i] - total)` 的元素
- 我们可以用哈希表统计左半部分和右半部分的元素值出现次数，遍历 pivot 时动态更新，从而 O(1) 判断是否存在所需值。
- 同时，还需要统计不修改任何元素时，满足条件的 pivot 数量（即 `2*prefix[i] == total` 的数量）。

**步骤**：
1. 计算前缀和数组 `prefix` 和总和 `total`。
2. 统计不修改时的方案数 `base`。
3. 初始化右半部分哈希表 `rightCount`，包含所有元素。
4. 初始化左半部分哈希表 `leftCount` 为空。
5. 遍历每个 pivot 位置 `i`（1 ≤ i < n）：
   - 将 `nums[i-1]` 从 `rightCount` 移到 `leftCount`（因为 pivot 移动，左半部分增加一个元素）。
   - 计算左半部分修改所需值 `needLeft = k - (total - 2*prefix[i])`，如果 `leftCount[needLeft] > 0`，则修改左半部分某个元素可使 pivot `i` 成立。
   - 计算右半部分修改所需值 `needRight = k - (2*prefix[i] - total)`，如果 `rightCount[needRight] > 0`，则修改右半部分某个元素可使 pivot `i` 成立。
   - 用哈希表 `changeCount` 记录每个位置 `j` 被修改后能新增的 pivot 数量（注意：一个 pivot 可能通过修改多个不同位置实现，但一次只能改一个位置，所以每个 pivot 只能贡献给一个修改位置）。
6. 遍历 `changeCount`，找到最大值 `maxChange`。
7. 最终答案为 `base + maxChange`（注意：如果修改后使得原来成立的 pivot 失效？不会，因为我们可以选择不改那个位置，所以 base 一定保留）。

**坑点**：
- 注意整数溢出，使用 int64。
- 修改左半部分时，所需值 `needLeft` 可能不在左半部分；修改右半部分同理。
- 一个 pivot 可能同时通过修改左半部分和右半部分实现，但一次只能改一个位置，所以每个 pivot 只能算一次贡献给某个修改位置。我们需要选择最优的修改位置，因此统计每个修改位置能新增的 pivot 数，取最大值。
- 实际上，我们可以直接统计每个 pivot 能否通过修改某个位置实现，然后累加该位置的贡献。但要注意，一个 pivot 可能对应多个可修改位置，我们只能选一个，所以需要取最大值。但更简单的方法是：对于每个 pivot，如果它能通过修改某个位置实现，就给那个位置 +1，最后取最大值。因为每个 pivot 只贡献给一个位置（如果多个位置可选，我们任选一个即可，因为目标是最大化总方案数，我们只关心最大值）。

**优化**：
- 不需要显式存储 `changeCount` 哈希表，可以在遍历时实时更新最大值。
- 对于每个 pivot，如果左半部分和右半部分都有可修改位置，我们选择能增加更多方案数的位置？但每个 pivot 只能被满足一次，所以只需给任意一个可修改位置 +1 即可。为了最大化，我们优先给左半部分？但这样可能不是全局最优。实际上，我们需要统计每个修改位置能覆盖多少 pivot，然后取最大值。所以还是需要哈希表记录每个修改位置的覆盖数。

**修正**：
- 对于每个 pivot，计算 `needLeft` 和 `needRight`，如果左半部分存在 `needLeft`，则所有左半部分值为 `needLeft` 的元素都可以通过修改为 `k` 使该 pivot 成立。因此，我们可以给这些元素的位置的覆盖数 +1。右半部分同理。
- 但一个 pivot 可能同时被左半部分和右半部分的修改覆盖，如果同时给两边 +1，则相当于这个 pivot 被算了两次，而实际上一次修改只能覆盖一次。所以我们需要避免重复计数。
- 解决方案：对于每个 pivot，如果左半部分和右半部分都有可修改位置，我们只选择其中一个（比如左半部分）来贡献，以确保每个 pivot 只被计数一次。但这样可能不是最优，因为可能左半部分的那个位置覆盖的 pivot 少，右半部分的覆盖多。所以更好的方法是：分别统计左半部分和右半部分的贡献，然后取最大值。因为一次修改只能在左半部分或右半部分，所以我们可以分别考虑修改左半部分和修改右半部分能新增的 pivot 数，取最大值。
- 更直接的方法：枚举修改位置 `j`，计算修改后能满足的 pivot 数量。这样是 O(n²)，但我们可以优化：修改位置 `j` 后，对于所有 pivot `i`，新的分割条件可以推导为：
  - 如果 `j < i`：`2*(prefix[i] + delta) == total + delta` → `2*prefix[i] == total - delta`
  - 如果 `j ≥ i`：`2*prefix[i] == total + delta`
  其中 `delta = k - nums[j]`。
  所以对于每个修改位置 `j`，我们可以计算它对应的两个等式，然后统计有多少 pivot 满足等式。但这样还是需要遍历所有 pivot。
- 进一步优化：将等式变形为 `2*prefix[i]` 的表达式，然后用哈希表统计 `2*prefix[i]` 的出现次数。对于每个修改位置 `j`，我们可以 O(1) 得到满足条件的 pivot 数量。
- 具体步骤：
  1. 计算所有 `2*prefix[i]`（i 从 1 到 n-1）的出现次数，存入哈希表 `count2p`。
  2. 计算不修改时的方案数 `base`（即 `2*prefix[i] == total` 的数量）。
  3. 对于每个位置 `j`，计算 `delta = k - nums[j]`。
      - 如果修改 `j`，则新的总和为 `total + delta`。
      - 对于 pivot `i`，如果 `j < i`，需要满足 `2*prefix[i] == total - delta`。
      - 如果 `j ≥ i`，需要满足 `2*prefix[i] == total + delta`。
      - 但同一个 pivot 可能同时满足两个条件？不可能，因为 `j` 相对于 `i` 的位置是确定的。
      - 因此，修改 `j` 后新增的 pivot 数量 = `count2p[total - delta]`（对应 `j < i`） + `count2p[total + delta]`（对应 `j ≥ i`）。
      - 注意：如果 `2*prefix[i]` 原本就等于 `total`（即原本就满足），则修改后可能仍然满足，但我们已经计入 `base`，所以这里不需要重复计算。但我们的目标是最大化总方案数，所以应该计算修改后总共满足的 pivot 数，然后取最大值。因此，对于每个修改位置 `j`，总方案数 = `count2p[total - delta] + count2p[total + delta]`。
      - 但这样会重复计数吗？不会，因为 `j < i` 和 `j ≥ i` 是互斥的。
  4. 遍历所有 `j`，计算总方案数，取最大值。
  5. 注意：还需要考虑不修改的情况，即 `base`。所以最终答案是 `max(base, maxChange)`？不对，因为修改后可能使原来满足的 pivot 不再满足，所以不能直接加。我们需要重新计算修改后的总方案数。
  6. 实际上，修改位置 `j` 后，原来满足的 pivot 可能不再满足，所以不能直接用 `base` 加上新增的。因此，对于每个修改位置 `j`，我们应该计算修改后**新满足**的 pivot 数量，即原来不满足但修改后满足的。但这样计算复杂。
  7. 更简单：对于每个修改位置 `j`，我们直接计算修改后满足的 pivot 总数（包括原来就满足的和新增的），然后取所有修改位置的最大值，再与不修改的情况比较取最大值。因为不修改的情况就是 `base`。
  8. 如何计算修改后满足的 pivot 总数？我们已经有了 `count2p[x]` 表示 `2*prefix[i] == x` 的 pivot 数量。修改后，对于 pivot `i`，如果 `j < i`，需要满足 `2*prefix[i] == total - delta`；如果 `j ≥ i`，需要满足 `2*prefix[i] == total + delta`。所以修改后满足的 pivot 总数 = `count2p[total - delta] + count2p[total + delta]`。但这里有一个问题：同一个 pivot 可能被两个条件都计数？不会，因为 `j` 相对于 `i` 的位置是唯一的。
  9. 因此，算法为：
      - 计算 `count2p`。
      - 计算 `base = count2p[total]`。
      - 对于每个位置 `j`，计算 `delta = k - nums[j]`，然后 `cur = count2p[total - delta] + count2p[total + delta]`，更新最大值 `maxChange`。
      - 答案 = `max(base, maxChange)`。
  10. 但这样对吗？考虑一个 pivot `i`，原本 `2*prefix[i] == total`，修改后如果 `j < i`，需要 `2*prefix[i] == total - delta`，即 `total == total - delta`，所以 `delta=0`，即 `k == nums[j]`。如果 `k == nums[j]`，则修改相当于没改，所以仍然满足。如果 `k != nums[j]`，则原来满足的 pivot 可能不再满足。但在我们的计算中，如果 `delta != 0`，则 `count2p[total - delta]` 不包含原来满足的 pivot，所以修改后满足的 pivot 总数可能小于 `base`。这符合实际情况。
  11. 因此，最终算法正确。

**最终算法步骤**：
1. 计算前缀和 `prefix`，总和 `total`。
2. 构建哈希表 `count2p`，键为 `2*prefix[i]`（i 从 1 到 n-1），值为出现次数。
3. 计算不修改的方案数 `base = count2p[total]`。
4. 初始化 `maxChange = 0`。
5. 遍历每个位置 `j`（0 ≤ j < n）：
   - `delta = int64(k) - int64(nums[j])`
   - `cur = count2p[total - delta] + count2p[total + delta]`
   - 更新 `maxChange = max(maxChange, cur)`
6. 返回 `max(base, maxChange)`。

**注意事项**：
- 使用 int64 防止溢出。
- 哈希表用 `map[int64]int`。
- 注意 `total - delta` 和 `total + delta` 可能超出 int64 范围？题目中元素值范围 ±10^5，n ≤ 10^5，总和最大约 10^10，在 int64 范围内。

## 3️⃣ 代码实现

### 解法一：哈希表统计 2*prefix 出现次数
```go
func waysToPartition(nums []int, k int) int {
    n := len(nums)
    // 前缀和
    prefix := make([]int64, n)
    prefix[0] = int64(nums[0])
    for i := 1; i < n; i++ {
        prefix[i] = prefix[i-1] + int64(nums[i])
    }
    total := prefix[n-1]
    
    // 统计 2*prefix[i] 的出现次数（i 从 1 到 n-1）
    count2p := make(map[int64]int)
    for i := 0; i < n-1; i++ {
        key := 2 * prefix[i]
        count2p[key]++
    }
    
    // 不修改的方案数
    base := count2p[total]
    
    // 枚举修改每个位置 j
    maxChange := 0
    for j := 0; j < n; j++ {
        delta := int64(k) - int64(nums[j])
        cur := count2p[total - delta] + count2p[total + delta]
        if cur > maxChange {
            maxChange = cur
        }
    }
    
    // 返回最大值
    if base > maxChange {
        return base
    }
    return maxChange
}
```

  - 哈希表操作平均 O(1)
- **空间复杂度**：O(n)

  - 哈希表存储 n-1 个键值对，O(n)

## 5️⃣ 解法对比

本题只有一种高效解法（O(n)），其他暴力或枚举 pivot 的解法都会超时。该解法巧妙地将问题转化为统计 `2*prefix[i]` 的出现次数，然后通过修改位置 j 推导出新的等式，利用哈希表快速计算满足条件的 pivot 数量。
