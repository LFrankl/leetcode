## 1️⃣ 题意理解

**题目重述**：
给定一个字符串 `word` 和一个非负整数 `k`，统计满足以下条件的子字符串数量：
1. 子字符串包含所有五个元音字母（a, e, i, o, u）
2. 子字符串中恰好包含 `k` 个辅音字母（即非元音字母）

**关键约束**：
- 字符串长度 5 ≤ word.length ≤ 250
- 字符均为小写英文字母
- 0 ≤ k ≤ word.length - 5（因为至少需要5个元音字母）
- 元音字母在子字符串中至少出现一次，不要求连续出现
- 辅音字母数量必须恰好等于 k

**边界情况**：
- k=0 时，子字符串只能包含元音字母
- 字符串长度较小时，可能没有符合条件的子字符串
- 元音字母可能重复出现

**输入输出**：
- 输入：字符串 word，整数 k
- 输出：满足条件的子字符串数量

## 2️⃣ 解题思路

### 解法一：滑动窗口 + 状态压缩
**核心思想**：
- 使用滑动窗口枚举所有可能的子字符串
- 用位掩码记录元音字母的出现情况（5位二进制，1表示出现）
- 维护窗口内辅音字母的数量
- 当窗口满足条件时，尝试收缩左边界以统计所有以当前右边界结尾的有效子字符串

**解题步骤**：
1. 定义元音集合，创建元音映射表
2. 初始化左指针 l=0，元音状态 mask=0，辅音计数 cons=0，结果 ans=0
3. 遍历右指针 r 从 0 到 n-1：
   - 如果 word[r] 是元音，更新 mask 的对应位
   - 否则，cons++
   - 当 cons > k 时，移动左指针直到 cons ≤ k
   - 当 mask == 31（所有元音都出现）且 cons == k 时：
     - 记录当前左指针位置 tmp = l
     - 尝试收缩左边界，统计所有有效子字符串
     - 恢复左指针

**注意事项**：
- 需要处理元音重复出现的情况
- 收缩左边界时要小心处理辅音计数
- 位掩码操作要准确

### 解法二：双指针优化
**核心思想**：
- 解法一的优化版本，避免重复收缩左边界
- 当窗口满足条件时，找到最小的满足条件的左边界，然后直接计算数量

**解题步骤**：
1. 同解法一初始化
2. 遍历右指针 r：
   - 更新元音状态和辅音计数
   - 当 cons > k 时，移动左指针
   - 当满足条件时：
     - 记录当前左边界 left_boundary = l
     - 移动左指针直到条件不满足，记录 right_boundary = l-1
     - 计算有效子字符串数：(left_boundary - last_l + 1) * (right_boundary - left_boundary + 1)
     - 更新 last_l

## 3️⃣ 代码实现

### 解法一：滑动窗口 + 状态压缩
```go
func countOfSubstrings(word string, k int) int {
    n := len(word)
    // 元音映射：a->0, e->1, i->2, o->3, u->4
    vowelMap := map[byte]int{
        'a': 0, 'e': 1, 'i': 2, 'o': 3, 'u': 4,
    }
    
    ans := 0
    // 遍历所有可能的左边界
    for l := 0; l < n; l++ {
        mask := 0    // 5位二进制，记录元音出现情况
        cons := 0    // 辅音计数
        
        // 扩展右边界
        for r := l; r < n; r++ {
            ch := word[r]
            if idx, isVowel := vowelMap[ch]; isVowel {
                mask |= 1 << idx  // 设置对应位
            } else {
                cons++
                // 如果辅音超过k，可以提前结束
                if cons > k {
                    break
                }
            }
            
            // 检查是否满足条件
            if mask == 31 && cons == k {  // 31 = 0b11111
                ans++
            }
        }
    }
    
    return ans
}
```

### 解法二：双指针优化
```go
func countOfSubstrings(word string, k int) int {
    n := len(word)
    vowelMap := map[byte]int{
        'a': 0, 'e': 1, 'i': 2, 'o': 3, 'u': 4,
    }
    
    ans := 0
    // 遍历所有可能的右边界
    for r := 0; r < n; r++ {
        mask := 0
        cons := 0
        
        // 从右边界向左扩展左边界
        for l := r; l >= 0; l-- {
            ch := word[l]
            if idx, isVowel := vowelMap[ch]; isVowel {
                mask |= 1 << idx
            } else {
                cons++
                if cons > k {
                    break
                }
            }
            
            if mask == 31 && cons == k {
                ans++
            }
        }
    }
    
    return ans
}
```

## 4️⃣ 复杂度分析

### 解法一：
- ⏱️ **时间复杂度**：O(n²)
  - 外层循环 O(n)
  - 内层循环最多 O(n)
  - 总复杂度 O(n²)
- 💾 **空间复杂度**：O(1)
  - 只使用了常数个变量

### 解法二：
- ⏱️ **时间复杂度**：O(n²)
  - 同样需要双重循环
  - 但实际运行可能比解法一稍快，因为提前break
- 💾 **空间复杂度**：O(1)
  - 只使用了常数个变量

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 解法一 | 逻辑清晰，易于理解 | 时间复杂度较高 | 小规模数据 (n≤250) |
| 解法二 | 提前剪枝，效率稍高 | 代码稍复杂 | 同解法一，但更高效 |

**推荐解法**：
对于本题的数据范围 (n≤250)，O(n²) 的解法完全足够。推荐使用**解法一**，因为：
1. 代码更简洁，易于理解和调试
2. 250² = 62500，完全在可接受范围内
3. 逻辑清晰，不易出错
