# 寻找可见山的数量

## 简要思路

每座山由峰坐标 `(x, y)` 定义，其底边区间为 `[x-y, x+y]`。一座山被另一座山"完全遮挡"的条件是：另一座山的底边区间完全覆盖它的区间（左端点更小、右端点更大，且高度必然更高）。因此问题转化为：**统计不被任何其他区间完全覆盖的区间数量**，同时完全相同的山互相遮挡，均不可见。

---

### 解法一：区间排序 + 遍历判定

**思路：** 将每座山转换为区间 `[x-y, x+y]`，统计每个区间出现次数（处理重复）。按左端点升序、右端点降序排序后遍历，维护已见最大右端点 `maxRight`。若当前区间右端点 > `maxRight`，则未被覆盖，候选可见；但若该区间出现次数 > 1（有重复山），仍不可见。

```go
func visibleMountains(peaks [][]int) int {
    n := len(peaks)
    intervals := make([][2]int, n)
    cnt := map[[2]int]int{}

    for i, p := range peaks {
        l, r := p[0]-p[1], p[0]+p[1]
        intervals[i] = [2]int{l, r}
        cnt[[2]int{l, r}]++
    }

    // 按左端点升序，右端点降序排序
    // 同一左端点时，更宽的区间先处理，确保宽区间先"占据"maxRight
    sort.Slice(intervals, func(i, j int) bool {
        if intervals[i][0] != intervals[j][0] {
            return intervals[i][0] < intervals[j][0]
        }
        return intervals[i][1] > intervals[j][1]
    })

    maxRight := -1
    visible := 0
    for _, iv := range intervals {
        if iv[1] > maxRight {
            // 右端点突破已有最大右边界 → 未被完全覆盖
            // 但若有重复山（同一区间出现多次），互相遮挡，不可见
            if cnt[iv] == 1 {
                visible++
            }
            maxRight = iv[1]
        }
    }

    return visible
}
```

**复杂度：** 时间 O(n log n)（排序），空间 O(n)（区间数组 + 计数map）

### 解法二：单调栈（去重后）

**思路：** 先去重（标记重复区间），再对剩余区间按左端点升序排序，使用单调栈维护"可见"候选：新区间入栈时，弹出被其完全覆盖的旧区间（左端点 ≥ 栈顶左端点 且 右端点 ≤ 栈顶右端点 → 栈顶被覆盖弹出）。最后栈中元素即为可见山。

```go
func visibleMountains(peaks [][]int) int {
    cnt := map[[2]int]int{}
    for _, p := range peaks {
        l, r := p[0]-p[1], p[0]+p[1]
        cnt[[2]int{l, r}]++
    }

    // 去重：只保留出现1次的区间候选
    unique := [][2]int{}
    for iv, c := range cnt {
        if c == 1 {
            unique = append(unique, iv)
        }
    }

    sort.Slice(unique, func(i, j int) bool {
        if unique[i][0] != unique[j][0] {
            return unique[i][0] < unique[j][0]
        }
        return unique[i][1] > unique[j][1]
    })

    // 单调栈：栈顶右端点单调递增
    // 新区间入栈时，弹出被新区间完全覆盖的栈顶
    stack := [][2]int{}
    for _, iv := range unique {
        for len(stack) > 0 && stack[len(stack)-1][1] <= iv[1] && stack[len(stack)-1][0] >= iv[0] {
            stack = stack[:len(stack)-1]
        }
        stack = append(stack, iv)
    }

    return len(stack)
}
```

**复杂度：** 时间 O(n log n)（排序），空间 O(n)（计数map + 栈）

---

> **推荐解法一**，逻辑更简洁，一次遍历即可完成判定，无需额外的栈操作。核心关键点：**按左升序右降序排序**，保证同起点时宽区间优先处理，使被覆盖的窄区间自然被跳过。
