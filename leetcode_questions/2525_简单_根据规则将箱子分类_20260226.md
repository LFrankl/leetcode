## 题目描述

<p>给你四个整数&nbsp;<code>length</code>&nbsp;，<code>width</code>&nbsp;，<code>height</code>&nbsp;和&nbsp;<code>mass</code>&nbsp;，分别表示一个箱子的三个维度和质量，请你返回一个表示箱子 <strong>类别</strong> 的字符串。</p>

<ul>
	<li>如果满足以下条件，那么箱子是&nbsp;<code>"Bulky"</code>&nbsp;的：

	<ul>
		<li>箱子 <strong>至少有一个</strong> 维度大于等于 <code>10<sup>4</sup></code>&nbsp;。</li>
		<li>或者箱子的 <strong>体积</strong> 大于等于&nbsp;<code>10<sup>9</sup></code>&nbsp;。</li>
	</ul>
	</li>
	<li>如果箱子的质量大于等于&nbsp;<code>100</code>&nbsp;，那么箱子是&nbsp;<code>"Heavy"</code>&nbsp;的。</li>
	<li>如果箱子同时是&nbsp;<code>"Bulky"</code> 和&nbsp;<code>"Heavy"</code>&nbsp;，那么返回类别为&nbsp;<code>"Both"</code>&nbsp;。</li>
	<li>如果箱子既不是&nbsp;<code>"Bulky"</code>&nbsp;，也不是&nbsp;<code>"Heavy"</code>&nbsp;，那么返回类别为&nbsp;<code>"Neither"</code>&nbsp;。</li>
	<li>如果箱子是&nbsp;<code>"Bulky"</code>&nbsp;但不是&nbsp;<code>"Heavy"</code>&nbsp;，那么返回类别为&nbsp;<code>"Bulky"</code>&nbsp;。</li>
	<li>如果箱子是&nbsp;<code>"Heavy"</code>&nbsp;但不是&nbsp;<code>"Bulky"</code>&nbsp;，那么返回类别为&nbsp;<code>"Heavy"</code>&nbsp;。</li>
</ul>

<p><strong>注意</strong>，箱子的体积等于箱子的长度、宽度和高度的乘积。</p>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre>
<b>输入：</b>length = 1000, width = 35, height = 700, mass = 300
<b>输出：</b>"Heavy"
<b>解释：</b>
箱子没有任何维度大于等于 10<sup>4 </sup>。
体积为 24500000 &lt;= 10<sup>9 </sup>。所以不能归类为 "Bulky" 。
但是质量 &gt;= 100 ，所以箱子是 "Heavy" 的。
由于箱子不是 "Bulky" 但是是 "Heavy" ，所以我们返回 "Heavy" 。</pre>

<p><strong>示例 2：</strong></p>

<pre>
<b>输入：</b>length = 200, width = 50, height = 800, mass = 50
<b>输出：</b>"Neither"
<b>解释：</b>
箱子没有任何维度大于等于 10<sup>4</sup>&nbsp;。
体积为 8 * 10<sup>6</sup> &lt;= 10<sup>9</sup>&nbsp;。所以不能归类为 "Bulky" 。
质量小于 100 ，所以不能归类为 "Heavy" 。
由于不属于上述两者任何一类，所以我们返回 "Neither" 。</pre>

<p>&nbsp;</p>

<p><strong>提示：</strong></p>

<ul>
	<li><code>1 &lt;= length, width, height &lt;= 10<sup>5</sup></code></li>
	<li><code>1 &lt;= mass &lt;= 10<sup>3</sup></code></li>
</ul>

## 题目分析

题目要求根据箱子的尺寸（长、宽、高）和质量（mass）将其分类为以下四种情况之一：
1. **Both**：箱子既是"Bulky"又是"Heavy"
2. **Bulky**：箱子是"Bulky"但不是"Heavy"
3. **Heavy**：箱子是"Heavy"但不是"Bulky"
4. **Neither**：箱子既不是"Bulky"也不是"Heavy"

**判断条件**：
- **Bulky**：至少有一个维度 ≥ 10⁴ **或者** 体积 ≥ 10⁹
- **Heavy**：质量 ≥ 100

**注意**：
- 体积 = length × width × height
- 维度比较时，每个维度单独与 10⁴ 比较
- 体积可能很大（最大 10¹⁵），需要注意整数溢出问题

---

## 解法一：直接判断法（推荐）

### 解题思路
直接按照题目描述的条件进行判断：
1. 判断是否为 Bulky：
   - 检查 length、width、height 是否 ≥ 10⁴
   - 计算体积并检查是否 ≥ 10⁹
2. 判断是否为 Heavy：
   - 检查 mass 是否 ≥ 100
3. 根据两个布尔值的组合返回对应字符串

### 代码实现

#### Go 实现
```go
func categorizeBox(length int, width int, height int, mass int) string {
    bulky := false
    heavy := mass >= 100
    
    // 检查维度
    if length >= 10000 || width >= 10000 || height >= 10000 {
        bulky = true
    }
    
    // 检查体积（注意类型转换避免溢出）
    volume := int64(length) * int64(width) * int64(height)
    if volume >= 1000000000 {
        bulky = true
    }
    
    // 分类
    if bulky && heavy {
        return "Both"
    } else if bulky && !heavy {
        return "Bulky"
    } else if !bulky && heavy {
        return "Heavy"
    } else {
        return "Neither"
    }
}
```

#### C++ 实现
```cpp
class Solution {
public:
    string categorizeBox(int length, int width, int height, int mass) {
        bool bulky = false;
        bool heavy = mass >= 100;
        
        // 检查维度
        if (length >= 10000 || width >= 10000 || height >= 10000) {
            bulky = true;
        }
        
        // 检查体积（使用 long long 避免溢出）
        long long volume = (long long)length * width * height;
        if (volume >= 1000000000) {
            bulky = true;
        }
        
        // 分类
        if (bulky && heavy) return "Both";
        if (bulky && !heavy) return "Bulky";
        if (!bulky && heavy) return "Heavy";
        return "Neither";
    }
};
```

### 复杂度分析
- **时间复杂度**：O(1)，只有常数次比较和乘法运算
- **空间复杂度**：O(1)，只使用了几个变量

## 解法二：位运算标记法

### 解题思路
使用两个二进制位来标记状态：
- 第 0 位：是否为 Bulky
- 第 1 位：是否为 Heavy
然后通过状态值直接映射到结果

### 代码实现

#### Go 实现
```go
func categorizeBox(length int, width int, height int, mass int) string {
    status := 0
    
    // 设置 Heavy 位
    if mass >= 100 {
        status |= 2  // 二进制 10
    }
    
    // 设置 Bulky 位
    if length >= 10000 || width >= 10000 || height >= 10000 {
        status |= 1  // 二进制 01
    } else {
        volume := int64(length) * int64(width) * int64(height)
        if volume >= 1000000000 {
            status |= 1
        }
    }
    
    // 映射结果
    switch status {
    case 0: return "Neither"
    case 1: return "Bulky"
    case 2: return "Heavy"
    case 3: return "Both"
    }
    return "" // 不会执行到这里
}
```

#### C++ 实现

## 解法三：提前计算体积（优化版）

### 解题思路
先计算体积，然后一次性判断所有 Bulky 条件，减少条件判断次数

### 代码实现

#### Go 实现
```go
func categorizeBox(length int, width int, height int, mass int) string {
    // 提前计算体积
    volume := int64(length) * int64(width) * int64(height)
    
    // 判断 Bulky
    bulky := length >= 10000 || width >= 10000 || height >= 10000 || volume >= 1000000000
    
    // 判断 Heavy
    heavy := mass >= 100
    
    // 使用二维数组映射结果
    results := [2][2]string{
        {"Neither", "Bulky"},
        {"Heavy", "Both"},
    }
    
    bulkyIdx := 0
    if bulky {
        bulkyIdx = 1
    }
    
    heavyIdx := 0
    if heavy {
        heavyIdx = 1
    }
    
    return results[heavyIdx][bulkyIdx]
}
```

#### C++ 实现
```cpp
class Solution {
public:
    string categorizeBox(int length, int width, int height, int mass) {
        // 提前计算体积
        long long volume = (long long)length * width * height;
        
        // 判断 Bulky
        bool bulky = (length >= 10000 || width >= 10000 || height >= 10000 || volume >= 1000000000);
        
        // 判断 Heavy
        bool heavy = mass >= 100;
        
        // 使用二维数组映射结果
        string results[2][2] = {
            {"Neither", "Bulky"},
            {"Heavy", "Both"}
        };
        
        return results[heavy][bulky];
    }
};
```

### 复杂度分析
- **时间复杂度**：O(1)
- **空间复杂度**：O(1)

## 不同解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **解法一：直接判断法** | 逻辑清晰，易于理解和维护 | 有多个条件判断分支 | 通用场景，推荐使用 |
| **解法二：位运算标记法** | 代码简洁，状态映射高效 | 可读性稍差 | 对性能有极致要求的场景 |
| **解法三：提前计算体积** | 减少条件判断次数 | 可能提前计算不必要的体积 | 维度经常满足条件的情况 |

### 推荐
对于 LeetCode 题目，**解法一（直接判断法）** 是最佳选择，因为：
1. 逻辑最清晰，符合题目描述
2. 易于理解和维护
3. 性能已经足够（O(1) 时间复杂度）
4. 避免了不必要的优化带来的复杂性

### 注意事项
1. **整数溢出问题**：计算体积时，三个 int 相乘可能溢出，需要转换为 long long（C++）或 int64（Go）
2. **边界条件**：注意 ≥ 和 > 的区别，题目使用的是 ≥
3. **条件顺序**：Bulky 的两个条件是"或"关系，只要满足一个就是 Bulky

### 扩展思考
如果维度数量增加到 n 个，可以使用循环检查每个维度是否 ≥ 10⁴，但核心逻辑不变。
