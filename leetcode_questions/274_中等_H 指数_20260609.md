## 题目描述

<p>给你一个整数数组 <code>citations</code> ，其中 <code>citations[i]</code> 表示研究者的第 <code>i</code> 篇论文被引用的次数。计算并返回该研究者的 <strong><code>h</code><em>&nbsp;</em>指数</strong>。</p>

<p>根据维基百科上&nbsp;<a href="https://baike.baidu.com/item/h-index/3991452?fr=aladdin" target="_blank">h 指数的定义</a>：<code>h</code> 代表“高引用次数” ，一名科研人员的 <code>h</code><strong> 指数 </strong>是指他（她）至少发表了 <code>h</code> 篇论文，并且&nbsp;<strong>至少&nbsp;</strong>有 <code>h</code> 篇论文被引用次数大于等于 <code>h</code> 。如果 <code>h</code><em> </em>有多种可能的值，<strong><code>h</code> 指数 </strong>是其中最大的那个。</p>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre>
<strong>输入：</strong><code>citations = [3,0,6,1,5]</code>
<strong>输出：</strong>3 
<strong>解释：</strong>给定数组表示研究者总共有 <code>5</code> 篇论文，每篇论文相应的被引用了 <code>3, 0, 6, 1, 5</code> 次。
&nbsp;    由于研究者有 <code>3 </code>篇论文每篇 <strong>至少 </strong>被引用了 <code>3</code> 次，其余两篇论文每篇被引用 <strong>不多于</strong> <code>3</code> 次，所以她的 <em>h </em>指数是 <code>3</code>。</pre>

<p><strong>示例 2：</strong></p>

<pre>
<strong>输入：</strong>citations = [1,3,1]
<strong>输出：</strong>1
</pre>

<p>&nbsp;</p>

<p><strong>提示：</strong></p>

<ul>
	<li><code>n == citations.length</code></li>
	<li><code>1 &lt;= n &lt;= 5000</code></li>
	<li><code>0 &lt;= citations[i] &lt;= 1000</code></li>
</ul>

## AI 解答 (DeepSeek)

### 解法一：排序法

思路：将 citations 从大到小排序，然后从前往后扫描，找到最大的 h 使得至少有 h 篇论文的引用次数 ≥ h。排序后，第 i 个位置（从0开始）表示第 i+1 篇最高引用的论文，若 citations[i] ≥ i+1，说明前 i+1 篇论文都满足引用次数 ≥ i+1，继续增大 h；否则 break，当前 h 即为答案。

```go
func hIndex(citations []int) int {
    sort.Sort(sort.Reverse(sort.IntSlice(citations)))
    h := 0
    for i, c := range citations {
        if c >= i+1 {
            h = i + 1
        } else {
            break
        }
    }
    return h
}
```

复杂度：时间 O(n log n)，空间 O(1)（如果排序不算额外空间）/ O(log n)（排序栈空间）

---

### 解法二：计数排序法（最优）

思路：h 指数最大不会超过论文总数 n，所以对于引用次数 > n 的论文，统一归入 n 这个桶即可。用长度为 n+1 的数组 count 记录每个引用次数对应的论文数量，然后从大到小累加论文数，当累计论文数 ≥ 当前引用次数时，该引用次数即为 h 指数。

```go
func hIndex(citations []int) int {
    n := len(citations)
    count := make([]int, n+1) // count[i]: 引用次数为i的论文数
    for _, c := range citations {
        if c >= n {
            count[n]++
        } else {
            count[c]++
        }
    }
    total := 0 // 累计论文数
    for i := n; i >= 0; i-- {
        total += count[i]
        if total >= i {
            return i
        }
    }
    return 0
}
```

复杂度：时间 O(n)，空间 O(n)

> **核心直觉**：h 指数的定义是"至少 h 篇论文引用 ≥ h"，排序法把"引用 ≥ h"直观呈现，计数法则利用 h ≤ n 的性质避免了排序，将复杂度从 O(n log n) 优化到 O(n)。
