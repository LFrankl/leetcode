# 2911. 得到 K 个半回文串的最少修改次数

**标签**: 双指针, 字符串, 动态规划

## 题目描述

<p>给你一个字符串&nbsp;<code>s</code>&nbsp;和一个整数&nbsp;<code>k</code>&nbsp;，请你将&nbsp;<code>s</code> 分成&nbsp;<code>k</code>&nbsp;个<strong>&nbsp;子字符串</strong>&nbsp;，使得每个 <strong>子字符串</strong>&nbsp;变成&nbsp;<strong>半回文串</strong>&nbsp;需要修改的字符数目最少。</p>

<p>请你返回一个整数，表示需要修改的 <strong>最少</strong>&nbsp;字符数目。</p>

<p><strong>注意：</strong></p>

<ul>
	<li>如果一个字符串从左往右和从右往左读是一样的，那么它是一个 <strong>回文串</strong>&nbsp;。</li>
	<li>如果长度为 <code>len</code>&nbsp;的字符串存在一个满足&nbsp;<code>1 &lt;= d &lt; len</code>&nbsp;的正整数&nbsp;<code>d</code>&nbsp;，<code>len % d == 0</code>&nbsp;成立且所有对 <code>d</code>&nbsp;做除法余数相同的下标对应的字符连起来得到的字符串都是 <strong>回文串</strong>&nbsp;，那么我们说这个字符串是 <strong>半回文串</strong>&nbsp;。比方说&nbsp;<code>"aa"</code>&nbsp;，<code>"aba"</code> ，<code>"adbgad"</code>&nbsp;和&nbsp;<code>"abab"</code>&nbsp;都是 <strong>半回文串</strong>&nbsp;，而&nbsp;<code>"a"</code>&nbsp;，<code>"ab"</code>&nbsp;和&nbsp;<code>"abca"</code>&nbsp;不是。</li>
	<li><strong>子字符串</strong>&nbsp;指的是一个字符串中一段连续的字符序列。</li>
</ul>

<p>&nbsp;</p>

<pre>

该方案是将 s 分成 2 个子字符串的前提下，得到 2 个半回文子字符串需要的最少修改次数。所以答案为 2 。</pre>

字符串 "aa" 和 "bb" 都已经是半回文串了。所以答案为 0 。
</pre>

## 提示

1. Define <code>dp[i][j]</code> as the minimum count of letter changes needed to split the suffix of string <code>s</code> starting from <code>s[i]</code> into <code>j</code> valid parts.
2. We have <code>dp[i][j] = min(dp[x + 1][j - 1] + v[i][x])</code>. Here <code>v[i][x]</code> is the minimum number of letter changes to change substring <code>s[i..x]</code> into semi-palindrome.
3. <code>v[i][j]</code> can be calculated separately by <b>brute-force</b>. We can create a table of <code>v[i][j]</code> independently to improve the complexity. Also note that semi-palindrome’s length is at least <code>2</code>.

## 1️⃣ 题意理解

**题目重述**  
给定字符串 `s` 和整数 `k`，要求将 `s` 分割成 `k` 个连续非空子串，使得每个子串变成 **半回文串** 所需修改的字符总数最少，返回这个最少修改次数。

**半回文串定义**  
长度为 `len` 的字符串是半回文串，当且仅当存在一个正整数 `d`（`1 <= d < len` 且 `len % d == 0`），使得将所有下标按 `mod d` 的余数分组后，每个组内的字符按原顺序组成的字符串是回文串。

**关键约束**  
- `2 <= s.length <= 200`
- `1 <= k <= s.length / 2`
- 字符串只包含小写英文字母

**边界情况**  
- `k = 1` 时，整个字符串需要变成一个半回文串
- 子串长度可能为 1，但长度为 1 的串不是半回文串（因为 `d` 必须小于长度）
- 子串长度可能为质数，此时 `d` 只能为 1，即要求整个子串是回文串

**输入输出**  
- 输入：`s`（字符串），`k`（整数）
- 输出：最少修改次数（整数）

## 2️⃣ 解题思路

### 解法一：动态规划 + 预处理
**核心思想**  
1. 预处理所有子串 `s[i:j]` 变成半回文串的最小修改代价 `cost[i][j]`
2. 使用动态规划 `dp[t][i]` 表示将前 `i` 个字符分成 `t` 段的最小总代价
3. 状态转移：`dp[t][i] = min(dp[t-1][j] + cost[j][i])`，其中 `j` 是上一段的结束位置

**关键点**  
- 预处理 `cost` 时，需要枚举所有可能的 `d`（长度 `len` 的因子），计算每组字符变成回文的最小修改次数
- 动态规划时注意下标范围和初始化

**步骤**  
1. 预处理 `cost[i][j]`（`0 <= i < j <= n`）
   - 对于每个子串，枚举所有可能的 `d`（`len` 的因子）
   - 对于每个 `d`，将下标按余数分组，每组字符需要变成回文串，计算修改次数
   - 取所有 `d` 中的最小值作为该子串的代价
2. 动态规划
   - `dp[t][i]`：前 `i` 个字符分成 `t` 段的最小代价
   - 初始化 `dp[1][i] = cost[0][i]`
   - 转移：`dp[t][i] = min(dp[t-1][j] + cost[j][i])`，其中 `t-1 <= j < i`
3. 返回 `dp[k][n]`

**注意事项**  
- 子串长度至少为 2（因为 `d` 必须小于长度）
- 因子枚举时只需枚举到 `len/2`，因为 `d` 必须小于 `len`
- 计算每组字符变成回文的最小修改次数时，可以双指针比较对称位置

### 解法二：记忆化搜索（DFS + 记忆化）
**核心思想**  
- 从字符串开头开始，递归地选择下一段子串，同时记录当前已分割的段数和已用字符数
- 使用记忆化 `memo[pos][seg]` 记录从位置 `pos` 开始，还需要分割 `seg` 段的最小代价
- 在递归时枚举下一段的结束位置，加上该段的代价，继续递归

**优缺点**  
- 思路直观，但递归深度可能较大
- 需要预处理 `cost` 数组，避免重复计算

## 3️⃣ 代码实现

### 解法一：动态规划
```go
func minimumChanges(s string, k int) int {
    n := len(s)
    // 预处理 cost[i][j] 表示 s[i:j] 变成半回文串的最小修改次数
    cost := make([][]int, n)
    for i := range cost {
        cost[i] = make([]int, n+1)
        for j := range cost[i] {
            cost[i][j] = -1 // 未计算
        }
    }

    // 计算子串 s[l:r] 变成半回文串的最小代价
    var computeCost func(l, r int) int
    computeCost = func(l, r int) int {
        if cost[l][r] != -1 {
            return cost[l][r]
        }
        length := r - l
        if length < 2 {
            cost[l][r] = 0 // 长度小于2的串无法成为半回文串，但这里我们不会用到
            return 0
        }
        minCost := length // 最大代价不超过长度
        // 枚举所有可能的 d (1 <= d < length 且 length % d == 0)
        for d := 1; d < length; d++ {
            if length%d != 0 {
                continue
            }
            // 对于每个 d，计算修改代价
            curCost := 0
            // 对于每个余数分组
            for rem := 0; rem < d; rem++ {
                // 获取该组的所有字符
                left, right := l+rem, r-d+rem
                for left < right {
                    if s[left] != s[right] {
                        curCost++
                    }
                    left += d
                    right -= d
                }
            }
            if curCost < minCost {
                minCost = curCost
            }
        }
        cost[l][r] = minCost
        return minCost
    }

    // 动态规划
    // dp[t][i] 表示将前 i 个字符分成 t 段的最小代价
    dp := make([][]int, k+1)
    for i := range dp {
        dp[i] = make([]int, n+1)
        for j := range dp[i] {
            dp[i][j] = 1 << 30
        }
    }
    // 初始化：分成1段
    for i := 1; i <= n; i++ {
        dp[1][i] = computeCost(0, i)
    }
    // 状态转移
    for t := 2; t <= k; t++ {
        for i := t; i <= n; i++ { // 前 i 个字符至少分成 t 段，所以 i >= t
            for j := t - 1; j < i; j++ { // 上一段结束位置 j
                // 前 j 个字符分成 t-1 段，加上 s[j:i] 的代价
                dp[t][i] = min(dp[t][i], dp[t-1][j]+computeCost(j, i))
            }
        }
    }
    return dp[k][n]
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

## 4️⃣ 复杂度分析

**预处理 `cost` 数组**  
- 子串数量：O(n²)
- 对于每个子串，枚举因子 d：O(√len) ~ O(√n)
- 对于每个 d，计算分组回文代价：O(len/d * d) = O(len)
- 总时间复杂度：O(n³)（因为 len 最大为 n）
- 空间复杂度：O(n²) 存储 `cost` 数组

**动态规划**  
- 状态数：O(k * n)
- 转移枚举：O(n)
- 总时间复杂度：O(k * n²)
- 空间复杂度：O(k * n)

**整体复杂度**  
- ⏱️ **时间复杂度**：O(n³ + k * n²) ≈ O(n³)（因为 n ≤ 200，可接受）
- 💾 **空间复杂度**：O(n² + k * n) ≈ O(n²)

| 动态规划 | 思路清晰，代码结构规整 | 预处理代价高，O(n³) | n ≤ 200 时完全可行 |
