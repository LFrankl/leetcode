## 题目描述

<p>给你一个 <strong>有向无环图</strong>&nbsp;， <code>n</code>&nbsp;个节点编号为 <code>0</code>&nbsp;到 <code>n-1</code>&nbsp;，以及一个边数组 <code>edges</code>&nbsp;，其中 <code>edges[i] = [from<sub>i</sub>, to<sub>i</sub>]</code>&nbsp;表示一条从点&nbsp;&nbsp;<code>from<sub>i</sub></code>&nbsp;到点&nbsp;<code>to<sub>i</sub></code>&nbsp;的有向边。</p>

<p>找到最小的点集使得从这些点出发能到达图中所有点。题目保证解存在且唯一。</p>

<p>你可以以任意顺序返回这些节点编号。</p>

<p>&nbsp;</p>

<pre><strong>输入：</strong>n = 6, edges = [[0,1],[0,2],[2,5],[3,4],[4,2]]
<strong>输出：</strong>[0,3]
<strong>解释：</strong>从单个节点出发无法到达所有节点。从 0 出发我们可以到达 [0,1,2,5] 。从 3 出发我们可以到达 [3,4,2,5] 。所以我们输出 [0,3] 。</pre>

</ul>

## AI 解答 (DeepSeek)

### 解法一：入度为零的节点

思路：在有向无环图中，能到达所有点的最小点集就是所有**入度为 0** 的节点。因为入度为 0 的节点无法从其他任何节点到达，必须被选入起点集；而入度大于 0 的节点一定能从某个入度为 0 的节点出发到达（DAG 保证解存在）。

```go
func findSmallestSetOfVertices(n int, edges [][]int) []int {
    inDegree := make([]int, n)
    for _, e := range edges {
        inDegree[e[1]]++
    }
    ans := []int{}
    for i := 0; i < n; i++ {
        if inDegree[i] == 0 {
            ans = append(ans, i)
        }
    }
    return ans
}
```

复杂度：时间 O(n + E)，空间 O(n)，其中 E 为边数。

---

### 解法二：集合差集法

思路：所有节点的集合减去所有边的终点集合，剩下的就是没有入边的节点，即入度为 0 的节点。本质与解法一相同，但用集合运算来表达，更直观。

```go
func findSmallestSetOfVertices(n int, edges [][]int) []int {
    toSet := make(map[int]bool)
    for _, e := range edges {
        toSet[e[1]] = true
    }
    ans := []int{}
    for i := 0; i < n; i++ {
        if !toSet[i] {
            ans = append(ans, i)
        }
    }
    return ans
}
```

复杂度：时间 O(n + E)，空间 O(n)。

> **说明**：解法一用数组记录入度更紧凑高效，是推荐写法；解法二用 map 表达"终点集合"语义更清晰，但常数略大。两者思路一致——**入度为 0 的节点就是答案**。
