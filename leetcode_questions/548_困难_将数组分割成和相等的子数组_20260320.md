## 1️⃣ 题意理解

**题目重述**：  
给定一个整数数组 `nums`，判断是否能将其分割成两个非空子数组（连续子数组），使得两个子数组的和相等。  
注意：这里要求的是**连续**的子数组，而不是任意挑选元素组成的子集。

**关键约束**：  
- 数组长度 `n` 在 `[1, 10^5]` 范围内  
- 数组元素值在 `[-10^4, 10^4]` 范围内  
- 要求分割成两个**连续**的子数组，而不是任意子序列  
- 两个子数组都必须非空

**输入输出**：  
- 输入：整数数组 `nums`  
- 输出：布尔值，表示是否能分割成两个和相等的连续子数组

## 2️⃣ 解题思路

### 解法一：前缀和 + 哈希表
**核心思想**：  
先计算整个数组的总和 `total`，如果 `total` 为奇数，直接返回 `false`（因为无法平分）。  
然后遍历数组，计算前缀和 `prefixSum`，在遍历过程中，如果存在某个位置 `i`，使得：
```
prefixSum == total / 2
```
则说明从 `0` 到 `i` 的子数组和等于总和的一半，那么剩余部分（`i+1` 到 `n-1`）的和也必然等于一半，满足条件。

**步骤**：
1. 计算数组总和 `total`
2. 如果 `total` 为奇数，返回 `false`
3. 初始化 `prefixSum = 0`
4. 遍历数组，累加 `prefixSum`
5. 如果 `prefixSum == total/2`，返回 `true`
6. 遍历结束返回 `false`

**注意事项**：
- 由于要求两个子数组都非空，所以分割点不能是最后一个元素（即 `i < n-1`）
- 元素可能为负数，但前缀和思路依然适用

### 解法二：双指针（适用于非负数组）
**核心思想**：  
如果数组元素**全部非负**，可以使用双指针从两端向中间逼近，寻找分割点。  
但本题允许负数，因此双指针法可能失效（不推荐）。

## 3️⃣ 代码实现

### 解法一：前缀和
```go
func canSplitArray(nums []int) bool {
    n := len(nums)
    if n < 2 {
        return false // 至少需要两个元素才能分割成两个非空子数组
    }
    
    // 计算总和
    total := 0
    for _, num := range nums {
        total += num
    }
    
    // 总和为奇数，无法平分
    if total%2 != 0 {
        return false
    }
    
    target := total / 2
    prefixSum := 0
    
    // 遍历数组，计算前缀和
    for i := 0; i < n-1; i++ { // i < n-1 保证第二个子数组非空
        prefixSum += nums[i]
        if prefixSum == target {
            return true
        }
    }
    
    return false
}
```

## 4️⃣ 复杂度分析

### 解法一：前缀和
- ⏱️ **时间复杂度**：O(n)  
  只需要一次遍历计算总和，一次遍历检查前缀和，总共 O(n)
- 💾 **空间复杂度**：O(1)  
  只使用了常数个额外变量

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 前缀和 | 时间复杂度 O(n)，空间 O(1)，简单高效 | 无 | 通用，适用于所有情况（包括负数） |
| 双指针 | 可能更直观 | 仅适用于非负数组，本题不适用 | 元素非负的特殊情况 |

**推荐解法**：前缀和  
理由：时间复杂度最优，实现简单，适用于本题所有情况。
