## 题目描述

<p>已知一个由像素点组成的单色屏幕，每行均有 <code>w</code> 个像素点，所有像素点初始为 <code>0</code>，左上角位置为 <code>(0,0)</code>。</p>

<p>现将每行的像素点按照「每 <code>32</code> 个像素点」为一组存放在一个 <code>int</code> 中，再依次存入长度为 <code>length</code> 的一维数组中。</p>

<p>我们将在屏幕上绘制一条从点 <code>(x1,y)</code> 到点 <code>(x2,y)</code> 的直线（即像素点修改为 <code>1</code>），请返回绘制过后的数组。</p>

<p>&nbsp;</p>

<p><strong>注意：</strong></p>

<ul>
	<li>用例保证屏幕宽度 <code>w</code> 可被 32 整除（即一个 <code>int</code> 不会分布在两行上）</li>
</ul>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre>
<strong> 输入</strong>：length = 1, w = 32, x1 = 30, x2 = 31, y = 0
<strong> 输出</strong>：[3]
<strong> 解释</strong>：在第 0 行的第 30 位到第 31 位画一条直线，屏幕二进制形式表示为 [00000000000000000000000000000011]，因此返回 [3]
</pre>

<p><strong>示例 2：</strong></p>

<pre>
<strong> 输入</strong>：length = 3, w = 96, x1 = 0, x2 = 95, y = 0
<strong> 输出</strong>：[-1, -1, -1]
<strong> 解释</strong>：由于二进制 <strong>11111111111111111111111111111111</strong> 的 int 类型代表 -1，因此返回 [-1,-1,-1]</pre>

<p>&nbsp;</p>

<p><strong>提示：</strong></p>

<ul>
	<li><code>1 &lt;= length &lt;= 10^5</code></li>
	<li><code>1 &lt;= w &lt;= 3 * 10^5</code></li>
	<li><code>0 &lt;= x1 &lt;= x2 &lt; w</code></li>
	<li><code>0 &lt;= y &lt;= 10</code></li>
</ul>

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

### 核心要求
- 模拟一个单色屏幕，每行有 `w` 个像素点（每个像素为 0 或 1）
- 屏幕数据按行存储在一个一维数组 `screen` 中，每个 `int` 存储 32 个像素（从高位到低位对应像素从左到右）
- 需要绘制一条从 `(x1, y)` 到 `(x2, y)` 的水平直线（将像素点置为 1）
- 返回绘制后的数组

### 关键约束
- `w` 能被 32 整除 → 每行的像素正好占用 `w/32` 个 `int`
- `length` 是数组总长度，`w/32 * (y+1) ≤ length`（因为数组可能包含多行）
- `0 ≤ x1 ≤ x2 < w`，`0 ≤ y ≤ 10`（y 很小）
- 使用二进制位操作处理 `int` 中的像素

### 输入输出
- **输入**：`length`（数组长度），`w`（屏幕宽度），`x1, x2`（直线起止横坐标），`y`（纵坐标）
- **输出**：绘制后的 `[]int` 数组

## 2️⃣ 解题思路

### 解法一：逐位处理（朴素法）
- **核心思想**：直接计算每个受影响的 `int` 在数组中的索引，然后通过位运算设置对应位
- **步骤**：
  1. 计算每行包含的 `int` 数：`intsPerRow = w/32`
  2. 确定直线所在行的起始索引：`rowStart = y * intsPerRow`
  3. 对于 `x1` 到 `x2` 的每个像素：
     - 计算它属于该行的第几个 `int`：`intIndex = rowStart + pixelPos/32`
     - 计算在该 `int` 中的位偏移：`bitOffset = 31 - (pixelPos % 32)`（因为高位在左）
     - 设置对应位为 1
- **注意事项**：
  - 当 `x2-x1` 很大时效率低（最坏 O(w)）
  - 注意 `int` 中位的顺序（最高位对应最左像素）

### 解法二：批量处理（高效法）
- **核心思想**：将直线覆盖的连续像素分段处理，特别是完整 `int` 的部分直接设为全 1
- **步骤**：
  1. 计算每行 `int` 数：`intsPerRow = w/32`
  2. 计算直线所在行的起始索引：`rowStart = y * intsPerRow`
  3. 确定直线覆盖的第一个和最后一个 `int` 索引：
     - `firstInt = rowStart + x1/32`
     - `lastInt = rowStart + x2/32`
  4. 分三种情况处理：
     - **情况 1**：直线在一个 `int` 内（`firstInt == lastInt`）
       - 生成掩码：从 `x1%32` 到 `x2%32` 的位设为 1
     - **情况 2**：直线跨多个 `int`
       - 处理第一个 `int`：从 `x1%32` 到 31 的位设为 1
       - 处理中间完整 `int`：直接设为 `0xFFFFFFFF`（全 1）
       - 处理最后一个 `int`：从 0 到 `x2%32` 的位设为 1
- **优点**：时间复杂度与直线长度无关，只与跨越的 `int` 数相关

## 3️⃣ 代码实现

### 解法一：逐位处理
```go
func drawLine(length int, w int, x1 int, x2 int, y int) []int {
    // 初始化屏幕数组
    screen := make([]int, length)
    
    // 每行包含的 int 数
    intsPerRow := w / 32
    
    // 直线所在行的起始索引
    rowStart := y * intsPerRow
    
    // 遍历直线上的每个像素
    for x := x1; x <= x2; x++ {
        // 计算该像素在数组中的索引
        intIndex := rowStart + x/32
        
        // 计算在该 int 中的位偏移（最高位对应最左像素）
        bitOffset := 31 - (x % 32)
        
        // 设置对应位为 1
        screen[intIndex] |= (1 << bitOffset)
    }
    
    return screen
}
```

### 解法二：批量处理（推荐）
```go
func drawLine(length int, w int, x1 int, x2 int, y int) []int {
    // 初始化屏幕数组
    screen := make([]int, length)
    
    // 每行包含的 int 数
    intsPerRow := w / 32
    
    // 直线所在行的起始索引
    rowStart := y * intsPerRow
    
    // 直线覆盖的第一个和最后一个 int 索引
    firstInt := rowStart + x1/32
    lastInt := rowStart + x2/32
    
    // 情况 1：直线在一个 int 内
    if firstInt == lastInt {
        // 生成掩码：从 x1%32 到 x2%32 的位设为 1
        mask := 0
        for i := x1 % 32; i <= x2 % 32; i++ {
            mask |= 1 << (31 - i)
        }
        screen[firstInt] = mask
        return screen
    }
    
    // 情况 2：直线跨多个 int
    
    // 处理第一个 int：从 x1%32 到 31 的位设为 1
    firstMask := 0
    for i := x1 % 32; i < 32; i++ {
        firstMask |= 1 << (31 - i)
    }
    screen[firstInt] = firstMask
    
    // 处理中间完整 int：设为全 1
    for i := firstInt + 1; i < lastInt; i++ {
        screen[i] = -1 // 0xFFFFFFFF 在补码中表示 -1
    }
    
    // 处理最后一个 int：从 0 到 x2%32 的位设为 1
    lastMask := 0
    for i := 0; i <= x2 % 32; i++ {
        lastMask |= 1 << (31 - i)
    }
    screen[lastInt] = lastMask
    
    return screen
}
```

## 4️⃣ 复杂度分析

### 解法一：逐位处理
- ⏱️ **时间复杂度**：O(x2 - x1 + 1)
  - 需要遍历直线上的每个像素点
  - 最坏情况下 w 可达 3×10^5，但 y ≤ 10，所以最坏 O(3×10^5)
- 💾 **空间复杂度**：O(length)
  - 需要创建长度为 length 的数组
  - 额外空间只有几个变量

### 解法二：批量处理
- ⏱️ **时间复杂度**：O(k)，其中 k 是直线跨越的 int 数
  - 最多处理 w/32 个 int，即 O(w/32)
  - 最坏情况下 w=3×10^5，k=9375，远小于像素数
- 💾 **空间复杂度**：O(length)
  - 同样需要创建结果数组
  - 额外空间常数级别

## 5️⃣ 解法对比

| 特性 | 解法一（逐位处理） | 解法二（批量处理） |
|------|-------------------|-------------------|
| 时间复杂度 | O(L)（L=直线长度） | O(k)（k=跨越的int数） |
| 空间复杂度 | O(length) | O(length) |
| 代码复杂度 | 简单直观 | 稍复杂，需处理边界 |
| 性能 | 直线长时性能差 | 性能稳定高效 |
| 适用场景 | 直线很短时 | 通用，特别是直线长时 |

**推荐使用解法二**：
- 题目中 w 可达 3×10^5，直线可能很长
- 解法二的时间复杂度与直线长度无关，只与跨越的 int 数相关
- 实际工程中更高效，特别是处理大屏幕时
