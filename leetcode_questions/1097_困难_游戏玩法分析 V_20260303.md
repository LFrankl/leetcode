## 1️⃣ 题意理解

**题目重述**：
给定一个游戏活动记录表 `Activity`，包含玩家 ID、设备 ID、事件日期和游戏次数。需要计算：
1. 每个日期当天首次登录的玩家数
2. 这些首次登录玩家在第二天再次登录的比例（保留两位小数）
3. 按日期升序输出结果

**关键约束**：
- 玩家可能在多个日期登录
- 首次登录指玩家在整个游戏历史中第一次登录的日期
- 需要计算次日留存率 = 次日仍登录的玩家数 / 当日首次登录玩家数
- 结果按日期升序排列

**边界情况**：
- 玩家可能只登录一次（次日留存为0）
- 可能有多个玩家在同一天首次登录
- 数据可能为空

**输入输出格式**：
输入：Activity 表（player_id, device_id, event_date, games_played）
输出：包含三列的表（event_date, players, retention_rate）

## 2️⃣ 解题思路

### 解法一：窗口函数 + 子查询
**核心思想**：
1. 使用窗口函数找到每个玩家的首次登录日期
2. 统计每天首次登录的玩家数
3. 计算这些玩家次日是否登录
4. 计算留存率

**解题步骤**：
1. 使用 `MIN() OVER()` 或子查询找到每个玩家的首次登录日期
2. 筛选出所有首次登录记录
3. 按日期分组统计首次登录玩家数
4. 使用自连接或窗口函数判断次日是否登录
5. 计算留存率并格式化

**注意事项**：
- 日期计算要准确（DATE_ADD）
- 处理 NULL 值（次日未登录）
- 保留两位小数

### 解法二：CTE + 聚合函数
**核心思想**：
1. 使用 CTE 分步计算，提高可读性
2. 先找出所有首次登录记录
3. 再计算留存情况

## 3️⃣ 代码实现

### 解法一：窗口函数法
```sql
-- LeetCode 1097. 游戏玩法分析 V
SELECT 
    first_login_date AS event_date,
    COUNT(DISTINCT player_id) AS players,
    ROUND(
        SUM(CASE WHEN DATEDIFF(next_date, first_login_date) = 1 THEN 1 ELSE 0 END) 
        / COUNT(DISTINCT player_id),
        2
    ) AS retention_rate
FROM (
    SELECT 
        player_id,
        event_date AS first_login_date,
        LEAD(event_date) OVER(PARTITION BY player_id ORDER BY event_date) AS next_date
    FROM (
        SELECT 
            player_id,
            event_date,
            ROW_NUMBER() OVER(PARTITION BY player_id ORDER BY event_date) AS rn
        FROM Activity
    ) t1
    WHERE rn = 1  -- 只保留首次登录记录
) t2
GROUP BY first_login_date
ORDER BY event_date;
```

### 解法二：CTE 分步法
```sql
WITH FirstLogin AS (
    -- 找出每个玩家的首次登录日期
    SELECT 
        player_id,
        MIN(event_date) AS first_login_date
    FROM Activity
    GROUP BY player_id
),
FirstLoginWithNext AS (
    -- 获取首次登录及下一次登录日期
    SELECT 
        f.player_id,
        f.first_login_date,
        MIN(a.event_date) AS next_login_date
    FROM FirstLogin f
    LEFT JOIN Activity a ON f.player_id = a.player_id 
        AND a.event_date > f.first_login_date
    GROUP BY f.player_id, f.first_login_date
)
SELECT 
    first_login_date AS event_date,
    COUNT(player_id) AS players,
    ROUND(
        SUM(CASE WHEN DATEDIFF(next_login_date, first_login_date) = 1 THEN 1 ELSE 0 END)
        / COUNT(player_id),
        2
    ) AS retention_rate
FROM FirstLoginWithNext
GROUP BY first_login_date
ORDER BY event_date;
```

## 4️⃣ 复杂度分析

### 解法一：窗口函数法
- ⏱️ **时间复杂度**：O(n log n)
  - 窗口函数排序：O(n log n)
  - 分组聚合：O(n)
  - 总体：O(n log n)，n 为记录数
- 💾 **空间复杂度**：O(n)
  - 需要存储窗口函数的中间结果
  - 子查询创建临时表

### 解法二：CTE 分步法
- ⏱️ **时间复杂度**：O(n log n)
  - 首次登录分组：O(n log n)
  - 左连接操作：O(n log n)
  - 最终分组：O(n)
- 💾 **空间复杂度**：O(n)
  - 需要存储 CTE 的中间结果
  - 连接操作可能产生临时表

## 5️⃣ 解法对比

| 特性 | 解法一（窗口函数） | 解法二（CTE） |
|------|-------------------|--------------|
| **代码简洁性** | 较简洁 | 较清晰，分步明确 |
| **可读性** | 中等，需要理解窗口函数 | 高，逻辑分步清晰 |
| **性能** | 较好，一次扫描 | 稍差，多次操作 |
| **适用场景** | 数据量大，追求性能 | 逻辑复杂，需要调试 |
| **推荐程度** | ★★★★☆ | ★★★☆☆ |

**推荐解法一**：
- 窗口函数在大多数数据库优化良好
- 代码更简洁
- 性能通常更好
