## 题目描述

<p><strong>独一无二的字符串</strong>&nbsp;指的是在一个数组中只出现过 <strong>一次</strong>&nbsp;的字符串。</p>

<p>给你一个字符串数组&nbsp;<code>arr</code>&nbsp;和一个整数&nbsp;<code>k</code>&nbsp;，请你返回&nbsp;<code>arr</code>&nbsp;中第&nbsp;<code>k</code>&nbsp;个&nbsp;<strong>独一无二的字符串</strong>&nbsp;。如果&nbsp;<strong>少于</strong>&nbsp;<code>k</code>&nbsp;个独一无二的字符串，那么返回&nbsp;<strong>空字符串</strong>&nbsp;<code>""</code>&nbsp;。</p>

<p>注意，按照字符串在原数组中的 <strong>顺序</strong>&nbsp;找到第 <code>k</code>&nbsp;个独一无二字符串。</p>

<p>&nbsp;</p>

<p><strong>示例 1:</strong></p>

<pre><b>输入：</b>arr = ["d","b","c","b","c","a"], k = 2
<b>输出：</b>"a"
<strong>解释：</strong>
arr 中独一无二字符串包括 "d" 和 "a"<code>&nbsp;。</code>
"d" 首先出现，所以它是第 1 个独一无二字符串。
"a" 第二个出现，所以它是 2 个独一无二字符串。
由于 k == 2 ，返回 "a" 。
</pre>

<p><strong>示例 2:</strong></p>

<pre><b>输入：</b>arr = ["aaa","aa","a"], k = 1
<b>输出：</b>"aaa"
<strong>解释：</strong>
arr 中所有字符串都是独一无二的，所以返回第 1 个字符串 "aaa" 。
</pre>

<p><strong>示例 3：</strong></p>

<pre><b>输入：</b>arr = ["a","b","a"], k = 3
<b>输出：</b>""
<strong>解释：</strong>
唯一一个独一无二字符串是 "b" 。由于少于 3 个独一无二字符串，我们返回空字符串 "" 。
</pre>

<p>&nbsp;</p>

<p><strong>提示：</strong></p>

<ul>
	<li><code>1 &lt;= k &lt;= arr.length &lt;= 1000</code></li>
	<li><code>1 &lt;= arr[i].length &lt;= 5</code></li>
	<li><code>arr[i]</code>&nbsp;只包含小写英文字母。</li>
</ul>

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

**核心要求**：  
给定一个字符串数组 `arr` 和一个整数 `k`，需要返回数组中第 `k` 个**只出现一次**的字符串（按原数组顺序）。如果这样的字符串少于 `k` 个，则返回空字符串 `""`。

**关键约束**：
- `1 <= k <= arr.length <= 1000`
- `1 <= arr[i].length <= 5`
- `arr[i]` 只包含小写英文字母
- 需要**保持原数组顺序**来统计第 k 个独一无二的字符串

**边界情况**：
- 所有字符串都是唯一的 → 直接返回第 k 个
- 所有字符串都重复 → 返回 `""`
- k 大于唯一字符串的数量 → 返回 `""`
- 数组长度为 1 的情况

**输入输出**：
- 输入：`arr []string`, `k int`
- 输出：`string`

## 2️⃣ 解题思路

### 解法一：哈希表两次遍历
**核心思想**：  
1. 第一次遍历统计每个字符串出现的次数
2. 第二次遍历找出第 k 个出现次数为 1 的字符串

**步骤**：
1. 使用哈希表记录每个字符串的出现次数
2. 再次遍历数组，维护一个计数器，当遇到出现次数为 1 的字符串时计数器加 1
3. 当计数器等于 k 时，返回当前字符串
4. 如果遍历结束计数器仍小于 k，返回空字符串

**注意事项**：
- 需要保持原数组顺序，所以不能直接对哈希表操作
- 字符串长度很短（≤5），哈希操作效率高

### 解法二：哈希表记录索引（优化版）
**核心思想**：  
使用哈希表记录字符串的状态：未出现、出现一次、重复出现。通过一次遍历完成统计和查找。

**步骤**：
1. 创建哈希表，值可以是整数状态：-1 表示未出现，索引表示第一次出现的位置，-2 表示已重复
2. 遍历数组，更新哈希表状态
3. 同时维护一个列表，记录所有只出现一次的字符串及其顺序
4. 最后直接取第 k-1 个元素（如果存在）

**注意事项**：
- 需要额外空间存储唯一字符串列表
- 但代码逻辑更清晰

## 3️⃣ 代码实现

### 解法一：哈希表两次遍历
```go
func kthDistinct(arr []string, k int) string {
    // 第一次遍历：统计每个字符串出现的次数
    count := make(map[string]int)
    for _, s := range arr {
        count[s]++
    }
    
    // 第二次遍历：按顺序找到第k个只出现一次的字符串
    distinctCount := 0
    for _, s := range arr {
        if count[s] == 1 {
            distinctCount++
            if distinctCount == k {
                return s
            }
        }
    }
    
    // 如果少于k个唯一字符串，返回空字符串
    return ""
}
```

### 解法二：哈希表记录状态（一次遍历）
```go
func kthDistinct(arr []string, k int) string {
    // 使用map记录字符串状态：
    // 值 = -1: 未出现过
    // 值 = index: 第一次出现的索引
    // 值 = -2: 已重复出现
    status := make(map[string]int)
    // 存储所有唯一字符串，保持顺序
    uniques := make([]string, 0)
    
    for i, s := range arr {
        if val, exists := status[s]; !exists {
            // 第一次出现，记录索引
            status[s] = i
            uniques = append(uniques, s)
        } else if val != -2 {
            // 第二次出现，标记为重复
            status[s] = -2
            // 从uniques中移除（实际不移除，只是标记）
            // 我们会在最后过滤掉重复的
        }
        // 已经是-2的情况不需要处理
    }
    
    // 过滤出真正的唯一字符串
    distinctCount := 0
    for _, s := range uniques {
        if status[s] != -2 {
            distinctCount++
            if distinctCount == k {
                return s
            }
        }
    }
    
    return ""
}
```

## 4️⃣ 复杂度分析

### 解法一：哈希表两次遍历
- ⏱️ **时间复杂度**：O(n)  
  其中 n 是数组长度。需要两次遍历，每次 O(n)，总体 O(2n) = O(n)
- 💾 **空间复杂度**：O(n)  
  哈希表最多存储 n 个字符串的计数

### 解法二：哈希表记录状态
- ⏱️ **时间复杂度**：O(n)  
  一次遍历构建状态，一次遍历过滤结果，总体 O(2n) = O(n)
- 💾 **空间复杂度**：O(n)  
  哈希表存储 n 个字符串状态，uniques 列表最多存储 n 个字符串

## 5️⃣ 解法对比

| 特性 | 解法一（两次遍历） | 解法二（状态记录） |
|------|-------------------|-------------------|
| **时间复杂度** | O(n) | O(n) |
| **空间复杂度** | O(n) | O(n) |
| **代码简洁性** | 简单直观 | 稍复杂 |
| **实际性能** | 两次遍历，但每次操作简单 | 一次遍历但逻辑稍复杂 |
| **适用场景** | 通用场景 | 需要一次遍历完成的场景 |

**推荐解法**：  
推荐使用**解法一（两次遍历）**，因为：
1. 代码更简洁，易于理解和维护
2. 实际性能差异不大（都是 O(n)）
3. 逻辑清晰，不易出错
4. 在本题数据规模（n ≤ 1000）下，性能足够
