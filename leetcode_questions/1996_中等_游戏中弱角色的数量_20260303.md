## 题目描述

<p>你正在参加一个多角色游戏，每个角色都有两个主要属性：<strong>攻击</strong> 和 <strong>防御</strong> 。给你一个二维整数数组 <code>properties</code> ，其中 <code>properties[i] = [attack<sub>i</sub>, defense<sub>i</sub>]</code> 表示游戏中第 <code>i</code> 个角色的属性。</p>

<p>如果存在一个其他角色的攻击和防御等级 <strong>都严格高于</strong> 该角色的攻击和防御等级，则认为该角色为 <strong>弱角色</strong> 。更正式地，如果认为角色 <code>i</code> <strong>弱于</strong> 存在的另一个角色 <code>j</code> ，那么 <code>attack<sub>j</sub> &gt; attack<sub>i</sub></code> 且 <code>defense<sub>j</sub> &gt; defense<sub>i</sub></code> 。</p>

<p>返回 <strong>弱角色</strong> 的数量。</p>

<p>&nbsp;</p>

<pre>
<strong>输入：</strong>properties = [[5,5],[6,3],[3,6]]
<strong>输出：</strong>0
<strong>解释：</strong>不存在攻击和防御都严格高于其他角色的角色。
</pre>

<p><strong>示例 2：</strong></p>

<pre>
<strong>输入：</strong>properties = [[2,2],[3,3]]
<strong>输出：</strong>1
<strong>解释：</strong>第一个角色是弱角色，因为第二个角色的攻击和防御严格大于该角色。
</pre>

<p><strong>示例 3：</strong></p>

<pre>
<strong>输入：</strong>properties = [[1,5],[10,4],[4,3]]
<strong>输出：</strong>1
<strong>解释：</strong>第三个角色是弱角色，因为第二个角色的攻击和防御严格大于该角色。
</pre>

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

**题目重述**：  
给定一个角色列表，每个角色有攻击力（attack）和防御力（defense）两个属性。如果一个角色的攻击力和防御力都**严格小于**另一个角色的攻击力和防御力，则该角色是“弱角色”。要求统计所有弱角色的数量。

**关键约束**：  
- 攻击力和防御力都严格大于才构成“弱于”关系。
- 数据规模：`properties.length ≤ 10^5`，要求算法时间复杂度优于 O(n²)。
- 攻击力和防御力的值范围：`[1, 10^5]`。

**输入输出格式**：  
- 输入：二维整数数组 `properties`，其中 `properties[i] = [attack_i, defense_i]`。
- 输出：整数，表示弱角色的数量。

## 2️⃣ 解题思路

### 解法一：排序 + 遍历（贪心）
**核心思想**：  
将角色按攻击力降序排序，攻击力相同时按防御力升序排序。这样在遍历时，保证攻击力相同的情况下，防御力高的角色不会误判为弱角色。遍历过程中维护当前遇到的最大防御力，若当前角色的防御力小于该最大值，则说明存在一个攻击力更高（或相等但防御力更高）的角色，当前角色为弱角色。

**步骤**：
1. 对 `properties` 排序：攻击力降序，攻击力相同时防御力升序。
2. 初始化 `maxDefense = 0`，用于记录遍历过程中遇到的最大防御力。
3. 遍历排序后的数组：
   - 若当前防御力 < `maxDefense`，则当前角色为弱角色，计数加一。
   - 否则更新 `maxDefense` 为当前防御力。
4. 返回计数。

**注意事项**：
- 攻击力相同时，必须按防御力升序排序，避免攻击力相同的角色之间误判。
- 由于攻击力降序，遍历时攻击力非递增，因此只要防御力小于之前遇到的最大防御力，就说明存在一个攻击力更高（或相等但防御力更高）的角色。

### 解法二：排序 + 单调栈（扩展思路）
**核心思想**：  
按攻击力升序排序，攻击力相同时按防御力降序排序。然后使用单调栈维护防御力的递减序列，遇到比栈顶防御力大的角色时，弹出栈中所有防御力小于当前防御力的角色，这些被弹出的角色就是弱角色。

**步骤**：
1. 对 `properties` 排序：攻击力升序，攻击力相同时防御力降序。
2. 初始化一个单调栈（存储防御力），栈中元素防御力递减。
3. 遍历排序后的数组：
   - 若栈非空且当前防御力 > 栈顶防御力，则栈顶角色为弱角色，弹出并计数。
   - 将当前防御力压入栈中。
4. 返回计数。

**注意事项**：
- 攻击力相同时按防御力降序排序，确保攻击力相同的角色不会互相误判。
- 单调栈维护的是防御力的递减序列，当遇到防御力更大的角色时，说明栈中防御力较小的角色遇到了攻击力更高（或相等但防御力更高）的角色。

## 3️⃣ 代码实现

### 解法一：排序 + 遍历（贪心）
```go
func numberOfWeakCharacters(properties [][]int) int {
    // 按攻击力降序排序，攻击力相同时按防御力升序排序
    sort.Slice(properties, func(i, j int) bool {
        if properties[i][0] == properties[j][0] {
            return properties[i][1] < properties[j][1] // 攻击力相同，防御力升序
        }
        return properties[i][0] > properties[j][0] // 攻击力降序
    })

    maxDefense := 0
    count := 0
    for _, p := range properties {
        if p[1] < maxDefense {
            count++
        } else {
            maxDefense = p[1]
        }
    }
    return count
}
```

### 解法二：排序 + 单调栈
```go
func numberOfWeakCharacters(properties [][]int) int {
    // 按攻击力升序排序，攻击力相同时按防御力降序排序
    sort.Slice(properties, func(i, j int) bool {
        if properties[i][0] == properties[j][0] {
            return properties[i][1] > properties[j][1] // 攻击力相同，防御力降序
        }
        return properties[i][0] < properties[j][0] // 攻击力升序
    })

    stack := []int{}
    count := 0
    for _, p := range properties {
        // 当栈非空且当前防御力大于栈顶防御力时，栈顶角色为弱角色
        for len(stack) > 0 && p[1] > stack[len(stack)-1] {
            stack = stack[:len(stack)-1]
            count++
        }
        stack = append(stack, p[1])
    }
    return count
}
```

## 4️⃣ 复杂度分析

### 解法一：排序 + 遍历
- ⏱️ **时间复杂度**：O(n log n)，主要来自排序操作，遍历 O(n)。
- 💾 **空间复杂度**：O(log n) 或 O(n)，取决于排序算法的空间复杂度（Go 的 `sort.Slice` 使用快速排序，空间复杂度 O(log n)）。

### 解法二：排序 + 单调栈
- ⏱️ **时间复杂度**：O(n log n)，排序 O(n log n)，遍历 O(n)，每个元素最多入栈出栈一次。
- 💾 **空间复杂度**：O(n)，最坏情况下所有角色都入栈。

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 解法一（排序+遍历） | 代码简洁，只需一次遍历，空间复杂度低 | 需要理解排序规则的巧妙性 | 通用场景，推荐使用 |
| 解法二（排序+单调栈） | 思路直观，单调栈常见于此类问题 | 空间复杂度较高，代码稍复杂 | 适合需要维护序列递减/递增的场景 |

**推荐解法**：解法一（排序+遍历），因为其代码简洁、效率高，且空间复杂度低，是本题的最优解。
