# 判定是否互为字符重排 - 解题分析

## 1️⃣ 题意理解

**核心要求**：判断两个字符串是否可以通过重新排列其中一个字符串的字符来得到另一个字符串。

**关键约束**：
1. 字符串仅包含小写字母
2. 字符串长度范围为 0 ≤ len(s) ≤ 100
3. 如果长度不同，直接返回 false

**边界情况**：
- 两个空字符串（互为重排）
- 一个空字符串和一个非空字符串（不互为重排）
- 字符串长度不同（直接返回 false）
- 字符串包含重复字符

**输入输出**：
- 输入：两个字符串 s1 和 s2
- 输出：布尔值，表示是否互为字符重排

## 2️⃣ 解题思路

### 解法一：哈希表计数法
**核心思想**：统计两个字符串中每个字符的出现次数，如果所有字符的出现次数都相同，则互为重排。

**解题步骤**：
1. 检查两个字符串长度是否相等，不等直接返回 false
2. 创建两个长度为 26 的数组（对应 26 个小写字母）
3. 遍历两个字符串，统计每个字符出现的次数
4. 比较两个计数数组是否完全相同

### 解法二：排序比较法
**核心思想**：将两个字符串排序后比较是否相等。

**解题步骤**：
1. 检查两个字符串长度是否相等
2. 将两个字符串转换为字符数组
3. 对两个字符数组进行排序
4. 比较排序后的字符数组是否相等

### 解法三：单哈希表优化法
**核心思想**：使用一个哈希表，先加后减，最后检查所有计数是否为 0。

**解题步骤**：
1. 检查长度是否相等
2. 创建长度为 26 的计数数组
3. 遍历 s1，对每个字符计数加 1
4. 遍历 s2，对每个字符计数减 1
5. 检查计数数组是否全为 0

## 3️⃣ 代码实现

### 解法一：哈希表计数法
```go
func CheckPermutation(s1 string, s2 string) bool {
    // 长度不同直接返回 false
    if len(s1) != len(s2) {
        return false
    }
    
    // 创建两个计数数组，对应 26 个小写字母
    count1 := make([]int, 26)
    count2 := make([]int, 26)
    
    // 统计 s1 中每个字符的出现次数
    for i := 0; i < len(s1); i++ {
        count1[s1[i]-'a']++
    }
    
    // 统计 s2 中每个字符的出现次数
    for i := 0; i < len(s2); i++ {
        count2[s2[i]-'a']++
    }
    
    // 比较两个计数数组是否完全相同
    for i := 0; i < 26; i++ {
        if count1[i] != count2[i] {
            return false
        }
    }
    
    return true
}
```

### 解法二：排序比较法
```go
import "sort"

func CheckPermutation(s1 string, s2 string) bool {
    // 长度不同直接返回 false
    if len(s1) != len(s2) {
        return false
    }
    
    // 将字符串转换为字符切片
    chars1 := []byte(s1)
    chars2 := []byte(s2)
    
    // 对字符切片进行排序
    sort.Slice(chars1, func(i, j int) bool { return chars1[i] < chars1[j] })
    sort.Slice(chars2, func(i, j int) bool { return chars2[i] < chars2[j] })
    
    // 比较排序后的字符串
    return string(chars1) == string(chars2)
}
```

### 解法三：单哈希表优化法
```go
func CheckPermutation(s1 string, s2 string) bool {
    // 长度不同直接返回 false
    if len(s1) != len(s2) {
        return false
    }
    
    // 创建计数数组，对应 26 个小写字母
    count := make([]int, 26)
    
    // 遍历 s1，增加计数
    for i := 0; i < len(s1); i++ {
        count[s1[i]-'a']++
    }
    
    // 遍历 s2，减少计数
    for i := 0; i < len(s2); i++ {
        count[s2[i]-'a']--
        // 如果某个字符的计数变为负数，说明 s2 中该字符比 s1 中多
        if count[s2[i]-'a'] < 0 {
            return false
        }
    }
    
    // 检查所有计数是否为 0（理论上不需要，因为长度相同且没有负数）
    for i := 0; i < 26; i++ {
        if count[i] != 0 {
            return false
        }
    }
    
    return true
}
```

## 4️⃣ 复杂度分析

### 解法一：哈希表计数法
- ⏱️ **时间复杂度**：O(n + m + 26) = O(n)，其中 n 和 m 是字符串长度
  - 遍历 s1：O(n)
  - 遍历 s2：O(m)
  - 比较 26 个计数：O(26) = O(1)
- 💾 **空间复杂度**：O(52) = O(1)
  - 两个长度为 26 的数组，固定大小

### 解法二：排序比较法
- ⏱️ **时间复杂度**：O(n log n + m log m)
  - 排序两个字符串：O(n log n + m log m)
  - 比较字符串：O(n)
- 💾 **空间复杂度**：O(n + m)
  - 需要额外的空间存储字符切片
  - 排序算法可能需要额外空间（取决于具体实现）

### 解法三：单哈希表优化法
- ⏱️ **时间复杂度**：O(n + m + 26) = O(n)
  - 遍历 s1：O(n)
  - 遍历 s2：O(m)
  - 检查 26 个计数：O(26) = O(1)
- 💾 **空间复杂度**：O(26) = O(1)
  - 一个长度为 26 的数组，固定大小

**推荐解法**：**解法三（单哈希表优化法）**

- 空间复杂度最优：O(1)
