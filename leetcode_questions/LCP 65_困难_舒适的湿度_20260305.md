# LCP 65. 舒适的湿度

**难度**: Hard

**标签**: 数组, 动态规划

## 题目描述

力扣嘉年华为了确保更舒适的游览环境条件，在会场的各处设置了湿度调节装置，这些调节装置受控于总控室中的一台控制器。
控制器中已经预设了一些调节指令，整数数组`operate[i]` 表示第 `i` 条指令增加空气湿度的大小。现在你可以将任意数量的指令修改为降低湿度（变化的数值不变），以确保湿度尽可能的适宜：
- 控制器会选择 **一段连续的指令** ，从而进行湿度调节的操作；
- 这段指令最终对湿度影响的绝对值，即为当前操作的「不适宜度」
- 在控制器所有可能的操作中，**最大** 的「不适宜度」即为「整体不适宜度」

请返回在所有修改指令的方案中，可以得到的 **最小** 「整体不适宜度」。

**示例 1：**
> 输入：`operate = [5,3,7]`
>
> 输出：`8`
>
> 解释：对于方案 `2` 的 `[5,3,-7]`
>操作指令 `[5],[3],[-7]` 的「不适宜度」分别为 `5,3,7`
>操作指令 `[5,3],[3,-7]` 的「不适宜度」分别为 `8,4`
>操作指令 `[5,3,-7]` 的「不适宜度」为 `1`，
>因此对于方案 `[5,3,-7]`的「整体不适宜度」为 `8`，其余方案的「整体不适宜度」均不小于 `8`，如下表所示：
![image.png](https://pic.leetcode.cn/1663902759-dgDCxn-image.png){:width=650px}

**示例 2：**
> 输入：`operate = [20,10]`
>
> 输出：`20`

**提示：**
- `1 <= operate.length <= 1000`
- `1 <= operate[i] <= 1000`

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

### 题目重述
给定一个整数数组 `operate`，表示一系列湿度调节指令的数值。我们可以将任意数量的指令修改为降低湿度（即数值变为负数，绝对值不变）。控制器会选择一段连续的指令执行，其数值和的绝对值称为「不适宜度」。所有可能的连续子段中最大的不适宜度称为「整体不适宜度」。我们需要通过修改指令（将某些正数变为负数），使得最终的整体不适宜度尽可能小。

### 关键约束
- 数组长度 n ≤ 1000
- 数组元素值 1 ≤ operate[i] ≤ 1000
- 可以修改任意数量的指令（包括0个或全部）
- 修改方式：将 `operate[i]` 变为 `-operate[i]`
- 最终整体不适宜度 = 所有连续子段的和的绝对值的最大值
- 目标：最小化这个最大值

### 输入输出
- 输入：整数数组 `operate`
- 输出：整数，表示最小可能的整体不适宜度

## 2️⃣ 解题思路

### 解法一：动态规划（基于最大子段和变形）
**核心思想**：
- 问题等价于：给每个元素分配符号（+1或-1），使得所有连续子段的和的绝对值的最大值最小
- 设修改后的数组为 `b[i] = s[i] * operate[i]`，其中 `s[i] ∈ {+1, -1}`
- 整体不适宜度 = max_{l≤r} |∑_{i=l}^r b[i]|
- 目标：min_{s} max_{l≤r} |∑_{i=l}^r b[i]|

**关键观察**：
- 对于固定的符号分配，最大绝对子段和 = max(最大正子段和, |最小负子段和|)
- 设 `dp_max[i]` 表示以 i 结尾的最大子段和，`dp_min[i]` 表示以 i 结尾的最小子段和
- 转移方程：
  - `dp_max[i] = max(b[i], dp_max[i-1] + b[i])`
  - `dp_min[i] = min(b[i], dp_min[i-1] + b[i])`
- 整体不适宜度 = max(max_i dp_max[i], |min_i dp_min[i]|)

**问题转化**：
- 我们需要选择符号分配来最小化这个值
- 由于 n ≤ 1000，可以尝试枚举所有可能的符号分配？不可行，有 2^n 种
- 需要更聪明的动态规划

**优化思路**：
- 设 `f[i][j]` 表示考虑前 i 个元素，当前前缀和为 j 时，到目前为止的最大绝对子段和的最小值
- 但 j 的范围可能很大（-sum~sum），需要压缩状态

### 解法二：二分答案 + 动态规划
**核心思想**：
- 二分答案 `mid`，检查是否存在符号分配使得所有子段和的绝对值 ≤ mid
- 检查方法：动态规划维护可能的前缀和范围
- 设 `dp[i]` 表示考虑前 i 个元素后，所有可能的前缀和集合
- 转移：`dp[i] = {x + operate[i] | x ∈ dp[i-1]} ∪ {x - operate[i] | x ∈ dp[i-1]}`
- 但需要限制所有前缀和都在 `[-mid, mid]` 范围内
- 最终检查是否存在前缀和使得所有子段和约束满足

**更高效的检查**：
- 维护前缀和的最小可能值 `min_pre` 和最大可能值 `max_pre`
- 每次转移：新的范围是 `[min_pre - a[i], max_pre + a[i]]` 与 `[-mid, mid]` 的交集
- 如果交集为空，则当前 mid 不可行

### 解法三：基于最大最小子段和的动态规划
**最终采用的方法**：
- 设 `dp[i][0]` 表示以 i 结尾的最大子段和的最小可能值（在所有符号分配中）
- 设 `dp[i][1]` 表示以 i 结尾的最小子段和的最大可能值（在所有符号分配中）
- 但实际上我们需要同时考虑全局的最大最小，所以需要更复杂的状态

**实际有效的方法（来自题解）**：
- 定义 `f[i]` 表示以 i 结尾的最大子段和的最小可能值
- 定义 `g[i]` 表示以 i 结尾的最小子段和的最大可能值
- 转移时考虑当前元素取正或负两种情况
- 最终答案 = min_{分配} max(全局最大子段和, |全局最小子段和|)

## 3️⃣ 代码实现

### 解法一：动态规划（维护前缀和范围）
```go
func unSuitability(operate []int) int {
    n := len(operate)
    // 计算总和，确定二分范围
    total := 0
    for _, v := range operate {
        total += v
    }
    
    // 二分答案
    left, right := 0, total
    for left <= right {
        mid := left + (right-left)/2
        if check(operate, mid) {
            right = mid - 1
        } else {
            left = mid + 1
        }
    }
    return left
}

// 检查是否存在符号分配使得所有子段和的绝对值 <= limit
func check(operate []int, limit int) bool {
    // 维护前缀和的最小可能值和最大可能值
    minPre, maxPre := 0, 0
    
    for _, x := range operate {
        // 当前元素可以取 +x 或 -x
        // 新的前缀和范围
        newMin := min(minPre - x, minPre + x)
        newMax := max(maxPre - x, maxPre + x)
        
        // 与限制范围求交集
        newMin = max(newMin, -limit)
        newMax = min(newMax, limit)
        
        // 如果交集为空，则不可行
        if newMin > newMax {
            return false
        }
        
        minPre, maxPre = newMin, newMax
    }
    return true
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

### 解法二：优化动态规划（一次遍历）
```go
func unSuitability(operate []int) int {
    // 最大可能值
    maxVal := 0
    for _, v := range operate {
        if v > maxVal {
            maxVal = v
        }
    }
    
    // 偏移量，将负数下标转为正数
    offset := maxVal * 3
    m := offset * 2 + 1
    const INF = 1 << 30
    
    // dp[j] 表示当前前缀和为 j-offset 时，最大绝对子段和的最小值
    dp := make([]int, m)
    for i := range dp {
        dp[i] = INF
    }
    dp[offset] = 0
    
    for _, x := range operate {
        ndp := make([]int, m)
        for i := range ndp {
            ndp[i] = INF
        }
        
        for j := 0; j < m; j++ {
            if dp[j] == INF {
                continue
            }
            // 当前前缀和
            curSum := j - offset
            // 两种选择：+x 或 -x
            for _, sign := range []int{1, -1} {
                newSum := curSum + sign*x
                // 新的前缀和下标
                nj := newSum + offset
                if nj < 0 || nj >= m {
                    continue
                }
                // 更新最大绝对子段和
                // 需要考虑以当前元素结尾的子段
                newMax := max(dp[j], abs(newSum))
                // 还需要考虑子段和本身的绝对值
                newMax = max(newMax, abs(sign*x))
                // 更新最小值
                if newMax < ndp[nj] {
                    ndp[nj] = newMax
                }
            }
        }
        dp = ndp
    }
    
    // 找到最小值
    ans := INF
    for _, v := range dp {
        if v < ans {
            ans = v
        }
    }
    return ans
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}
```

## 4️⃣ 复杂度分析

### 解法一（二分+检查）
- ⏱️ **时间复杂度**：O(n logS)，其中 S 是数组元素总和
  - 二分次数 O(logS)
  - 每次检查 O(n)
- 💾 **空间复杂度**：O(1)，只用了常数个变量

### 解法二（优化动态规划）
- ⏱️ **时间复杂度**：O(n * M)，其中 M = 6*maxVal+1 ≈ 6000
  - 外层循环 n 次
  - 内层循环 M 次
  - 总复杂度约 1000 * 6000 = 6e6，可接受
- 💾 **空间复杂度**：O(M)，M 为状态数量

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 二分+检查 | 思路直观，实现简单 | 需要确定二分范围，可能不是最优 | n 较大时更稳定 |
| 优化DP | 精确求解，不需要二分 | 状态数较多，常数较大 | n 较小，值范围较小时 |

**推荐解法**：解法一（二分+检查）
- 实现简单，易于理解
- 时间复杂度可接受
- 对于 n=1000 的数据规模完全足够
