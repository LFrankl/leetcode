## 1️⃣ 题意理解

**题目重述**：
题目要求从 `Orders` 表中，按月统计每个月的订单总数和下单的顾客数。其中，顾客数需要去重统计（即同一个顾客在一个月内多次下单只算一次）。统计结果按月份升序排列。

**关键约束与边界**：
- 输入表 `Orders` 包含字段：`order_id`, `customer_id`, `order_date`。
- 统计月份格式为 `'YYYY-MM'`。
- 需要处理可能存在的重复订单（同月同一顾客多次下单）。
- 如果某个月没有订单，则不需要出现在结果中。
- 结果按月份升序排序。

**输入输出格式**：
- 输入：`Orders` 表（假设已存在）。
- 输出：包含三列的表：`month`（月份），`order_count`（订单总数），`customer_count`（去重顾客数）。

## 2️⃣ 解题思路

### 解法一：基础聚合 + 日期格式化
**核心思想**：
1. 从 `order_date` 中提取出 `'YYYY-MM'` 格式的月份。
2. 按月份分组，统计每个月的订单总数（`COUNT(*)`）和去重顾客数（`COUNT(DISTINCT customer_id)`）。
3. 按月份升序排序。

**步骤**：
1. 使用 `DATE_FORMAT(order_date, '%Y-%m')` 或 `LEFT(order_date, 7)` 提取月份。
2. 按提取的月份分组。
3. 在分组内计算 `order_count` 和 `customer_count`。
4. 按月份排序。

**注意事项**：
- 确保 `order_date` 是日期或时间类型，否则需要转换。
- 使用 `DISTINCT` 对顾客去重。
- 月份格式必须为 `'YYYY-MM'`。

### 解法二：使用 YEAR 和 MONTH 函数
**核心思想**：
与解法一类似，但使用 `YEAR()` 和 `MONTH()` 函数分别提取年份和月份，然后拼接成 `'YYYY-MM'` 格式。

**步骤**：
1. 使用 `CONCAT(YEAR(order_date), '-', LPAD(MONTH(order_date), 2, '0'))` 生成月份字符串。
2. 后续步骤同解法一。

**注意事项**：
- 月份需要补零到两位（`LPAD` 或 `DATE_FORMAT`）。
- 确保拼接后的格式正确。

## 3️⃣ 代码实现

### 解法一：基础聚合 + 日期格式化
```sql
SELECT 
    DATE_FORMAT(order_date, '%Y-%m') AS month,
    COUNT(*) AS order_count,
    COUNT(DISTINCT customer_id) AS customer_count
FROM 
    Orders
GROUP BY 
    DATE_FORMAT(order_date, '%Y-%m')
ORDER BY 
    month;
```

### 解法二：使用 YEAR 和 MONTH 函数
```sql
SELECT 
    CONCAT(YEAR(order_date), '-', LPAD(MONTH(order_date), 2, '0')) AS month,
    COUNT(*) AS order_count,
    COUNT(DISTINCT customer_id) AS customer_count
FROM 
    Orders
GROUP BY 
    YEAR(order_date), MONTH(order_date)
ORDER BY 
    month;
```

## 4️⃣ 复杂度分析

### 解法一：
- ⏱️ **时间复杂度**：O(N)，其中 N 是 `Orders` 表的行数。需要扫描全表并进行分组聚合。
- 💾 **空间复杂度**：O(M)，其中 M 是不同月份的数量。需要存储分组中间结果。

### 解法二：
- ⏱️ **时间复杂度**：O(N)，与解法一相同。
- 💾 **空间复杂度**：O(M)，与解法一相同。

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 解法一 | 代码简洁，直接使用 `DATE_FORMAT` 函数 | 依赖数据库的日期格式化函数 | 大多数数据库支持 |
| 解法二 | 使用标准 SQL 函数，可移植性好 | 代码稍长，需要手动补零 | 需要跨数据库兼容时 |

**推荐解法**：解法一。因为代码更简洁，且 `DATE_FORMAT` 函数在主流数据库（如 MySQL）中广泛支持。如果考虑数据库兼容性，可选择解法二。
