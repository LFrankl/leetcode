## 网格中的最短路径（LeetCode 1293）

### 简要思路

这是一道带状态的 BFS 问题。关键在于状态不仅是 `(row, col)`，还需要记录**已消除的障碍物数量**，因为到达同一格子时消除次数不同，后续能力也不同。使用 BFS 按步数层级扩展，最早到达终点即为最短路径。

---

### 解法一：BFS + 三维状态数组

思路：将状态定义为 `(行, 列, 已消除障碍数)`，用三维 `visited` 数组避免重复访问同一状态。BFS 层级扩展保证首次到达终点时步数最少。额外优化：当 `k ≥ m+n-2` 时，任何路径上的障碍都可以消除，最短路径就是 `m+n-2`。

```go
func shortestPath(grid [][]int, k int) int {
    m, n := len(grid), len(grid[0])
    if m == 1 && n == 1 {
        return 0
    }
    // k 足够大时，最短路径就是 m+n-2（不绕路即可消除所有障碍）
    if k >= m + n - 2 {
        return m + n - 2
    }

    // visited[i][j][obstacles]：是否以 obstacles 消除次数到达过 (i,j)
    visited := make([][][]bool, m)
    for i := range visited {
        visited[i] = make([][]bool, n)
        for j := range visited[i] {
            visited[i][j] = make([]bool, k+1)
        }
    }

    dirs := [4][2]int{{0, 1}, {0, -1}, {1, 0}, {-1, 0}}
    // 队列元素：{row, col, obstaclesEliminated}
    queue := [][3]int{{0, 0, 0}}
    visited[0][0][0] = true
    steps := 0

    for len(queue) > 0 {
        size := len(queue)
        steps++
        for i := 0; i < size; i++ {
            r, c, ob := queue[i][0], queue[i][1], queue[i][2]
            for _, d := range dirs {
                nr, nc := r + d[0], c + d[1]
                if nr < 0 || nr >= m || nc < 0 || nc >= n {
                    continue
                }
                newOb := ob + grid[nr][nc]
                if newOb > k || visited[nr][nc][newOb] {
                    continue
                }
                if nr == m-1 && nc == n-1 {
                    return steps
                }
                visited[nr][nc][newOb] = true
                queue = append(queue, [3]int{nr, nc, newOb})
            }
        }
        queue = queue[size:]
    }
    return -1
}
```

复杂度：时间 O(m × n × k)，空间 O(m × n × k)

### 解法二：BFS + 二维优化剪枝（更优）

思路：用二维数组 `visited[i][j]` 记录到达 `(i,j)` 时**最少消除次数**。由于 BFS 按步数层级处理，若某状态到达同一格子时消除次数 ≥ 已记录值，说明该状态步数 ≥ 前次且剩余消除预算更少，**严格劣于**已访问状态，直接剪枝。空间从三维降为二维，实际运行效率也更优。

```go
func shortestPath(grid [][]int, k int) int {
    m, n := len(grid), len(grid[0])
    if m == 1 && n == 1 {
        return 0
    }
    if k >= m + n - 2 {
        return m + n - 2
    }

    // visited[i][j] = 到达 (i,j) 时的最少消除次数
    // 初始化为 k+1（不可能值），仅当更优时才更新
    visited := make([][]int, m)
    for i := range visited {
        visited[i] = make([]int, n)
        for j := range visited[i] {
            visited[i][j] = k + 1
        }
    }

    dirs := [4][2]int{{0, 1}, {0, -1}, {1, 0}, {-1, 0}}
    queue := [][3]int{{0, 0, 0}}
    visited[0][0] = 0
    steps := 0

    for len(queue) > 0 {
        size := len(queue)
        steps++
        for i := 0; i < size; i++ {
            r, c, ob := queue[i][0], queue[i][1], queue[i][2]
            for _, d := range dirs {
                nr, nc := r + d[0], c + d[1]
                if nr < 0 || nr >= m || nc < 0 || nc >= n {
                    continue
                }
                newOb := ob + grid[nr][nc]
                if newOb > k || newOb >= visited[nr][nc] {
                    continue // 消除次数不优于已有记录，剪枝
                }
                if nr == m-1 && nc == n-1 {
                    return steps
                }
                visited[nr][nc] = newOb
                queue = append(queue, [3]int{nr, nc, newOb})
            }
        }
        queue = queue[size:]
    }
    return -1
}
```

复杂度：时间 O(m × n × k)（理论上限，实际因剪枝远小于此），空间 O(m × n)

---

### 核心要点

1. **状态扩展**：普通 BFS 只记录位置，本题必须将"已消除障碍数"纳入状态，否则会错误跳过需要消除障碍的路径。
2. **剪枝原理**：BFS 保证步数递增，若到达同一位置时消除次数更多，则剩余预算更少、步数更多，**两维度都更差**，可安全剪枝。
3. **提前终止**：`k ≥ m+n-2` 时任何路径障碍都可消除，直接返回理论最短路径长度。
