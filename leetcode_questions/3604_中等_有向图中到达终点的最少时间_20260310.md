## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

**题目重述**  
我们有一个有向图，节点编号 0 到 n-1，每条边有可用时间窗口 `[start, end]`。从时间 0 在节点 0 出发，可以在任意时间停留在当前节点，也可以在时间 t 满足 `start ≤ t ≤ end` 时沿边移动（移动耗时 1 单位）。求到达节点 n-1 的最早时间，若无法到达则返回 -1。

**关键约束**  
- 边只能在时间窗口内使用，且移动耗时 1 单位。
- 可以任意等待（停留）。
- 节点数 n 和边数 m 最多 10^5，时间值可达 10^9。
- 边的时间窗口是闭区间 `[start, end]`。

**输入输出**  
- 输入：整数 n，边列表 edges，每条边为 `[u, v, start, end]`。
- 输出：到达 n-1 的最早时间（整数），或 -1。

## 2️⃣ 解题思路

### 解法一：Dijkstra 变种（时间窗口约束）
**核心思想**  
将问题转化为带时间窗口的最短路问题。每个节点记录到达该节点的最早时间，用优先队列（小顶堆）按时间排序进行 BFS。对于当前节点 u 在时间 t，考虑其出边 (u, v, s, e)：
- 若 t ≤ e：可以在 max(t, s) 时间出发，到达 v 的时间为 max(t, s) + 1。
- 若 t > e：无法使用该边（已过窗口）。
由于可以等待，我们总是可以等到 s 再出发，但若 t > e 则边不可用。

**步骤**  
1. 建邻接表。
2. 初始化 dist 数组，dist[0] = 0，其余为 INF。
3. 使用优先队列（按时间排序）存储 (time, node)。
4. 每次弹出最早时间节点，若 time > dist[node] 则跳过（旧状态）。
5. 遍历出边，计算到达时间 nextTime = max(time, start) + 1，若 nextTime ≤ end 且 nextTime < dist[v] 则更新并入队。
6. 最终返回 dist[n-1] 或 -1。

**注意事项**  
- 时间值可能很大，用 int64 存储。
- 优先队列需按时间排序，Go 中可用 heap 实现。
- 当 time > end 时边不可用，直接跳过。

### 解法二：BFS + 时间窗口贪心（不可行分析）
**核心思想**  
由于时间窗口的存在，BFS 按步数扩展可能错过更优解（等待策略）。BFS 无法处理“等待”操作，因此不适用。必须使用基于时间的优先队列（Dijkstra 思想）。

## 3️⃣ 代码实现

### 解法一：Dijkstra 变种
```go
func minTime(n int, edges [][]int) int {
    // 建邻接表
    graph := make([][][2]int, n) // 每个元素为 [v, start, end] 的切片
    for _, e := range edges {
        u, v, s, e := e[0], e[1], e[2], e[3]
        graph[u] = append(graph[u], [2]int{v, s, e})
    }

    const INF = int(1e18)
    dist := make([]int, n)
    for i := range dist {
        dist[i] = INF
    }
    dist[0] = 0

    // 优先队列，按时间排序
    type Item struct {
        time int
        node int
    }
    pq := &minHeap{}
    heap.Init(pq)
    heap.Push(pq, Item{0, 0})

    for pq.Len() > 0 {
        cur := heap.Pop(pq).(Item)
        t, u := cur.time, cur.node
        if t > dist[u] {
            continue // 旧状态
        }
        if u == n-1 {
            return t // 提前结束，因为优先队列保证最早时间
        }
        // 遍历出边
        for _, edge := range graph[u] {
            v, start, end := edge[0], edge[1], edge[2]
            if t > end {
                continue // 已过窗口
            }
            // 出发时间 = max(当前时间, 窗口开始)
            depart := max(t, start)
            nextTime := depart + 1
            if nextTime < dist[v] {
                dist[v] = nextTime
                heap.Push(pq, Item{nextTime, v})
            }
        }
    }

    if dist[n-1] == INF {
        return -1
    }
    return dist[n-1]
}

// 最小堆实现
type minHeap []Item

func (h minHeap) Len() int           { return len(h) }
func (h minHeap) Less(i, j int) bool { return h[i].time < h[j].time }
func (h minHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *minHeap) Push(x any)        { *h = append(*h, x.(Item)) }
func (h *minHeap) Pop() any {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

## 4️⃣ 复杂度分析

**解法一：Dijkstra 变种**
- ⏱️ **时间复杂度**：O((n + m) log n)  
  每个节点和每条边最多被处理一次，每次堆操作 O(log n)，其中 n 为节点数，m 为边数。
- 💾 **空间复杂度**：O(n + m)  
  邻接表存储所有边 O(m)，dist 数组 O(n)，堆最多 O(n)。

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| Dijkstra 变种 | 保证最优解，时间复杂度可接受 | 需要堆实现，代码稍复杂 | 通用，尤其适合时间窗口约束 |
| BFS + 贪心 | 简单 | 无法处理等待，可能错过最优解 | 不适用本题 |

**推荐解法**：Dijkstra 变种。因为本题本质是带时间约束的最短路，Dijkstra 能保证最早到达时间，且复杂度在可接受范围。
