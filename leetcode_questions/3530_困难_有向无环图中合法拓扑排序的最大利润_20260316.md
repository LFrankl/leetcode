## 题目描述

<p>给你一个由 <code>n</code> 个节点组成的<strong>有向无环图（DAG）</strong>，节点编号从 <code>0</code> 到 <code>n - 1</code>，通过二维数组 <code>edges</code> 表示，其中 <code>edges[i] = [u<sub>i</sub>, v<sub>i</sub>]</code> 表示一条从节点 <code>u<sub>i</sub></code> 指向节点 <code>v<sub>i</sub></code> 的有向边。每个节点都有一个对应的&nbsp;<strong>得分&nbsp;</strong>，由数组 <code>score</code> 给出，其中 <code>score[i]</code> 表示节点 <code>i</code> 的得分。</p>

<p>你需要以&nbsp;<strong>有效的拓扑排序&nbsp;</strong>顺序处理这些节点。每个节点在处理顺序中被分配一个编号从 <strong>1</strong> 开始的位置。</p>

<p>将每个节点的得分乘以其在拓扑排序中的位置，然后求和，得到的值称为&nbsp;<strong>利润</strong>。</p>

<p>请返回在所有合法拓扑排序中可获得的&nbsp;<strong>最大利润&nbsp;</strong>。</p>

<p><strong>拓扑排序&nbsp;</strong>是一个对 DAG 中所有节点的线性排序，使得每条有向边 <code>u → v</code> 中，节点 <code>u</code> 都出现在 <code>v</code> 之前。</p>

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

**核心要求**：
给定一个有向无环图（DAG），每个节点有一个得分。我们需要找到一个合法的拓扑排序顺序，使得每个节点的得分乘以其在排序中的位置（从1开始）的总和（称为“利润”）最大。

**关键约束**：
- 节点数 n ≤ 22（很小，暗示可能使用状态压缩动态规划）
- 得分值 score[i] ≥ 1
- 图保证是 DAG，无环
- 边无重复
- 拓扑排序必须满足所有边的方向约束（u 必须在 v 之前）

**输入输出**：
- 输入：n（节点数），edges（边列表），score（得分数组）
- 输出：最大利润（整数）

## 2️⃣ 解题思路

### 解法一：状态压缩 DP（推荐）
**核心思想**：
- 由于 n ≤ 22，可以用一个整数 mask 的二进制位表示已选择的节点集合。
- 定义 dp[mask] 表示已选择节点集合为 mask 时的最大利润。
- 转移时，考虑下一个可以选择的节点（即入度为0且未选择的节点），计算将其加入排序后的利润增量。
- 利润计算：每个节点的位置 = 已选节点数 + 1（因为位置从1开始）。

**步骤**：
1. 预处理每个节点的入度（原始入度）。
2. 初始化 dp[0] = 0。
3. 遍历所有 mask 状态（0 到 (1<<n)-1）。
4. 对于每个 mask，计算当前已选节点数 cnt = popcount(mask)。
5. 找出所有未选且入度为0的节点（需考虑已选节点的影响）。
6. 尝试将每个可选节点加入，更新 dp[mask|(1<<node)]。
7. 最终答案 = dp[(1<<n)-1]。

**注意事项**：
- 需要动态维护每个节点的入度：当选择节点 u 后，所有 u 指向的节点入度减1。
- 利润增量 = score[node] * (cnt + 1)。

### 解法二：基于优先队列的贪心（不可行）
- 贪心思路：每次选择入度为0且得分最高的节点。
- 反例：得分高的节点可能有很多后继，提前选择会限制后续选择。
- 结论：贪心不能保证全局最优，因为位置权重随时间增加。

## 3️⃣ 代码实现

### 解法一：状态压缩 DP
```go
func maxProfit(n int, edges [][]int, score []int) int64 {
    // 1. 预处理邻接表和入度
    indeg := make([]int, n)
    graph := make([][]int, n)
    for _, e := range edges {
        u, v := e[0], e[1]
        graph[u] = append(graph[u], v)
        indeg[v]++
    }
    
    // 2. DP 数组，dp[mask] 表示选择节点集合 mask 时的最大利润
    totalStates := 1 << n
    dp := make([]int64, totalStates)
    for i := range dp {
        dp[i] = -1
    }
    dp[0] = 0
    
    // 3. 遍历所有状态
    for mask := 0; mask < totalStates; mask++ {
        if dp[mask] < 0 {
            continue // 不可达状态
        }
        cnt := bits.OnesCount(uint(mask)) // 已选节点数
        // 4. 计算当前状态下每个节点的入度（考虑已选节点的影响）
        curIndeg := make([]int, n)
        copy(curIndeg, indeg)
        for u := 0; u < n; u++ {
            if mask>>u&1 == 1 { // u 已选
                for _, v := range graph[u] {
                    curIndeg[v]-- // u 已选，v 入度减1
                }
            }
        }
        // 5. 找出所有可选节点（入度为0且未选）
        for node := 0; node < n; node++ {
            if mask>>node&1 == 0 && curIndeg[node] == 0 {
                nextMask := mask | (1 << node)
                profit := dp[mask] + int64(score[node])*int64(cnt+1)
                if profit > dp[nextMask] {
                    dp[nextMask] = profit
                }
            }
        }
    }
    
    return dp[totalStates-1]
}
```

## 4️⃣ 复杂度分析

### 解法一：状态压缩 DP
- **时间复杂度**：O(2^n * n^2)
  - 状态数：2^n（n ≤ 22，最多约 4e6）
  - 每个状态需要 O(n) 计算当前入度（复制数组 O(n)，遍历邻接表 O(n)）
  - 总复杂度 O(2^n * n^2) ≈ 4e6 * 484 ≈ 1.9e9（最坏情况）
  - 实际运行中由于很多状态不可达，且 n=22 时 2^n=4e6 可接受（Go 约 1-2 秒）
- **空间复杂度**：O(2^n)
  - dp 数组大小 2^n
  - 辅助数组 O(n)

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 状态压缩 DP | 保证最优解，n≤22 可接受 | 时间复杂度高，n>22 不可用 | n ≤ 22 的 DAG |
| 贪心（不可行） | 简单快速 | 不能保证最优解 | 不适用本题 |

**推荐解法**：状态压缩 DP，因为 n 很小，且需要精确最优解。

输出：14

得分 [1,2,3] 降序排列 [3,2,1] 对应节点 [2,1,0]
利润=3*1+2*2+1*3=3+4+3=10？等等，我们检查：
正确降序应为得分高的放后面：位置权重越大，得分越高乘积越大。
所以应该按得分升序排列：节点0(1),节点1(2),节点2(3) → 利润=1*1+2*2+3*3=1+4+9=14
