# 2813. 子序列最大优雅度

**难度**: Hard

**标签**: 栈, 贪心, 数组, 哈希表, 排序, 堆（优先队列）

**链接**: https://leetcode.cn/problems/maximum-elegance-of-a-k-length-subsequence/

## 题目描述

<p>给你一个长度为 <code>n</code> 的二维整数数组 <code>items</code> 和一个整数 <code>k</code> 。</p>

<p><code>items[i] = [profit<sub>i</sub>, category<sub>i</sub>]</code>，其中 <code>profit<sub>i</sub></code> 和 <code>category<sub>i</sub></code> 分别表示第 <code>i</code> 个项目的利润和类别。</p>

<p>现定义&nbsp;<code>items</code> 的 <strong>子序列</strong> 的 <strong>优雅度</strong> 可以用 <code>total_profit + distinct_categories<sup>2</sup></code> 计算，其中 <code>total_profit</code> 是子序列中所有项目的利润总和，<code>distinct_categories</code> 是所选子序列所含的所有类别中不同类别的数量。</p>

<p>你的任务是从 <code>items</code> 所有长度为 <code>k</code> 的子序列中，找出 <strong>最大优雅度</strong> 。</p>

<p>用整数形式表示并返回 <code>items</code> 中所有长度恰好为 <code>k</code> 的子序列的最大优雅度。</p>

<p><strong>注意：</strong>数组的子序列是经由原数组删除一些元素（可能不删除）而产生的新数组，且删除不改变其余元素相对顺序。</p>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre>
<strong>输入：</strong>items = [[3,2],[5,1],[10,1]], k = 2
<strong>输出：</strong>17
<strong>解释：
</strong>在这个例子中，我们需要选出长度为 2 的子序列。
其中一种方案是 items[0] = [3,2] 和 items[2] = [10,1] 。
子序列的总利润为 3 + 10 = 13 ，子序列包含 2 种不同类别 [2,1] 。
因此，优雅度为 13 + 2<sup>2</sup> = 17 ，可以证明 17 是可以获得的最大优雅度。 
</pre>

<p><strong>示例 2：</strong></p>

<pre>
<strong>输入：</strong>items = [[3,1],[3,1],[2,2],[5,3]], k = 3
<strong>输出：</strong>19
<strong>解释：</strong>
在这个例子中，我们需要选出长度为 3 的子序列。 
其中一种方案是 items[0] = [3,1] ，items[2] = [2,2] 和 items[3] = [5,3] 。
子序列的总利润为 3 + 2 + 5 = 10 ，子序列包含 3 种不同类别 [1, 2, 3] 。 
因此，优雅度为 10 + 3<sup>2</sup> = 19 ，可以证明 19 是可以获得的最大优雅度。</pre>

<p><strong>示例 3：</strong></p>

<pre>
<strong>输入：</strong>items = [[1,1],[2,1],[3,1]], k = 3
<strong>输出：</strong>7
<strong>解释：
</strong>在这个例子中，我们需要选出长度为 3 的子序列。
我们需要选中所有项目。
子序列的总利润为 1 + 2 + 3 = 6，子序列包含 1 种不同类别 [1] 。
因此，最大优雅度为 6 + 1<sup>2</sup> = 7 。</pre>

## 题目分析

题目要求从 `items` 中选择恰好 `k` 个项目组成子序列，使得 `total_profit + distinct_categories²` 最大。其中：
- `total_profit` 是所选项目的利润总和
- `distinct_categories` 是所选项目中不同类别的数量

**关键观察**：
1. 利润总和越大越好，不同类别数量越多越好（因为平方项增长快）
2. 但高利润的项目可能集中在少数类别中
3. 我们需要在利润和类别多样性之间权衡

## 解题思路

### 核心思路
1. **贪心策略**：先按利润降序排序，选取前 k 个利润最高的项目
2. **问题**：这样可能忽略类别多样性，导致平方项较小
3. **优化**：用栈/堆维护已选项目，尝试用新类别替换重复类别的低利润项目

### 详细步骤
1. 将 `items` 按利润降序排序
2. 初始化：
   - 选择前 k 个项目
   - 计算初始利润总和
   - 记录每个类别的出现次数
   - 用栈保存重复类别的项目（利润低的在栈顶）
3. 从第 k+1 个项目开始遍历：
   - 如果当前项目的类别是新的（未出现过）
   - 从栈中弹出利润最低的重复类别项目
   - 用当前项目替换它，更新利润和类别数
   - 计算新的优雅度，更新最大值
4. 注意：替换的前提是栈不为空（有重复类别可替换）

## 解法一：排序 + 栈 + 贪心

### 思路
1. 按利润降序排序
2. 先选前 k 个利润最高的
3. 用栈保存重复类别的低利润项目
4. 遍历剩余项目，用新类别替换重复类别的低利润项目

### 代码实现

#### Go 实现
```go
func findMaximumElegance(items [][]int, k int) int64 {
    // 按利润降序排序
    sort.Slice(items, func(i, j int) bool {
        return items[i][0] > items[j][0]
    })
    
    totalProfit := int64(0)
    categoryMap := make(map[int]bool)
    duplicateStack := [][]int{} // 栈，存储重复类别的项目
    
    // 先选前 k 个利润最高的
    for i := 0; i < k; i++ {
        profit, category := items[i][0], items[i][1]
        totalProfit += int64(profit)
        
        if categoryMap[category] {
            // 重复类别，入栈
            duplicateStack = append(duplicateStack, items[i])
        } else {
            categoryMap[category] = true
        }
    }
    
    distinct := len(categoryMap)
    maxElegance := totalProfit + int64(distinct)*int64(distinct)
    
    // 尝试用剩余项目替换重复类别的项目
    for i := k; i < len(items); i++ {
        profit, category := items[i][0], items[i][1]
        
        // 如果当前类别是新的，且栈中有重复类别的项目可替换
        if !categoryMap[category] && len(duplicateStack) > 0 {
            // 弹出栈顶（利润最低的重复类别项目）
            popped := duplicateStack[len(duplicateStack)-1]
            duplicateStack = duplicateStack[:len(duplicateStack)-1]
            
            // 替换：减去旧利润，加上新利润
            totalProfit = totalProfit - int64(popped[0]) + int64(profit)
            // 添加新类别
            categoryMap[category] = true
            distinct++
            
            // 计算新的优雅度
            elegance := totalProfit + int64(distinct)*int64(distinct)
            if elegance > maxElegance {
                maxElegance = elegance
            }
        }
    }
    
    return maxElegance
}
```

#### C++ 实现
```cpp
#include <vector>
#include <algorithm>
#include <unordered_map>
#include <stack>
#include <functional>
using namespace std;

class Solution {
public:
    long long findMaximumElegance(vector<vector<int>>& items, int k) {
        // 按利润降序排序
        sort(items.begin(), items.end(), [](const vector<int>& a, const vector<int>& b) {
            return a[0] > b[0];
        });
        
        long long totalProfit = 0;
        unordered_map<int, bool> categoryMap;
        stack<vector<int>> duplicateStack; // 存储重复类别的项目
        
        // 先选前 k 个利润最高的
        for (int i = 0; i < k; i++) {
            int profit = items[i][0];
            int category = items[i][1];
            totalProfit += profit;
            
            if (categoryMap.find(category) != categoryMap.end()) {
                // 重复类别，入栈
                duplicateStack.push(items[i]);
            } else {
                categoryMap[category] = true;
            }
        }
        
        int distinct = categoryMap.size();
        long long maxElegance = totalProfit + (long long)distinct * distinct;
        
        // 尝试用剩余项目替换重复类别的项目
        for (int i = k; i < items.size(); i++) {
            int profit = items[i][0];
            int category = items[i][1];
            
            // 如果当前类别是新的，且栈中有重复类别的项目可替换
            if (categoryMap.find(category) == categoryMap.end() && !duplicateStack.empty()) {
                // 弹出栈顶（利润最低的重复类别项目）
                vector<int> popped = duplicateStack.top();
                duplicateStack.pop();
                
                // 替换：减去旧利润，加上新利润
                totalProfit = totalProfit - popped[0] + profit;
                // 添加新类别
                categoryMap[category] = true;
                distinct++;
                
                // 计算新的优雅度
                long long elegance = totalProfit + (long long)distinct * distinct;
                if (elegance > maxElegance) {
                    maxElegance = elegance;
                }
            }
        }
        
        return maxElegance;
    }
};
```

### 复杂度分析
- **时间复杂度**：O(n log n)，主要来自排序
- **空间复杂度**：O(n)，用于存储栈和哈希表

## 解法二：排序 + 最小堆优化

### 思路
与解法一类似，但使用最小堆（优先队列）来维护重复类别中的最小利润项目，便于快速获取最小值。

### 代码实现

#### Go 实现
```go
import (
    "container/heap"
    "sort"
)

type MinHeap []int

func (h MinHeap) Len() int            { return len(h) }
func (h MinHeap) Less(i, j int) bool  { return h[i] < h[j] }
func (h MinHeap) Swap(i, j int)       { h[i], h[j] = h[j], h[i] }
func (h *MinHeap) Push(x interface{}) { *h = append(*h, x.(int)) }
func (h *MinHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

func findMaximumElegance(items [][]int, k int) int64 {
    // 按利润降序排序
    sort.Slice(items, func(i, j int) bool {
        return items[i][0] > items[j][0]
    })
    
    totalProfit := int64(0)
    categoryMap := make(map[int]bool)
    duplicateHeap := &MinHeap{} // 最小堆，存储重复类别的利润
    heap.Init(duplicateHeap)
    
    // 先选前 k 个利润最高的
    for i := 0; i < k; i++ {
        profit, category := items[i][0], items[i][1]
        totalProfit += int64(profit)
        
        if categoryMap[category] {
            // 重复类别，利润加入最小堆
            heap.Push(duplicateHeap, profit)
        } else {
            categoryMap[category] = true
        }
    }
    
    distinct := len(categoryMap)
    maxElegance := totalProfit + int64(distinct)*int64(distinct)
    
    // 尝试用剩余项目替换重复类别的项目
    for i := k; i < len(items); i++ {
        profit, category := items[i][0], items[i][1]
        
        // 如果当前类别是新的，且堆中有重复类别的项目可替换
        if !categoryMap[category] && duplicateHeap.Len() > 0 {
            // 弹出最小利润的重复类别项目
            minProfit := heap.Pop(duplicateHeap).(int)
            
            // 替换：减去旧利润，加上新利润
            totalProfit = totalProfit - int64(minProfit) + int64(profit)
            // 添加新类别
            categoryMap[category] = true
            distinct++
            
            // 计算新的优雅度
            elegance := totalProfit + int64(distinct)*int64(distinct)
            if elegance > maxElegance {
                maxElegance = elegance
            }
        }
    }
    
    return maxElegance
}
```

## 解法三：动态规划（不可行说明）

### 为什么动态规划不可行？
1. 状态空间太大：需要记录已选项目数、利润总和、类别集合
2. 类别集合无法压缩：最多有 n 个不同类别，需要 2^n 的状态
3. n 最大为 10^5，无法使用状压 DP
4. 利润值范围大（1-10^9），无法作为 DP 维度

**结论**：对于本题的约束条件，动态规划不是可行解法。

## 解法对比

| 特性 | 解法一（栈） | 解法二（最小堆） |
|------|-------------|-----------------|
| 核心数据结构 | 栈 | 最小堆（优先队列） |
| 获取最小值效率 | O(1) 弹出栈顶 | O(log k) 弹出堆顶 |
| 代码复杂度 | 简单 | 稍复杂（需要堆实现） |
| 实际性能 | 较快 | 稍慢（堆操作有开销） |
| 适用场景 | 推荐 | 当需要频繁获取最小值时 |

### 优缺点分析

**解法一（栈）的优点**：
1. 实现简单直观
2. 由于栈顶就是最小利润（因为按利润排序后入栈），所以获取最小值是 O(1)
3. 实际运行效率高

**解法二（最小堆）的优点**：
1. 更通用的数据结构
2. 如果问题变体需要频繁获取最小值，堆更合适
3. 逻辑清晰，易于理解

**共同优点**：
1. 时间复杂度 O(n log n)，可以处理 n=10^5
2. 空间复杂度 O(n)，可以接受
3. 基于贪心策略，正确性有保障

### 推荐解法
推荐使用**解法一（栈）**，因为：
1. 本题中栈顶自然就是最小利润（排序后入栈）
2. 实现更简单
3. 运行效率更高

## 总结

本题的关键在于理解：
1. 优先选择高利润项目
2. 但也要考虑类别多样性带来的平方收益
3. 通过替换重复类别的低利润项目来增加类别数

贪心策略（先选利润最高的 k 个，然后尝试替换）是解决本题的最优方法，动态规划等其它方法在本题约束下不可行。
