## 题目描述

<p>给你一个下标从 <strong>0</strong>&nbsp;开始的数组&nbsp;<code>nums</code>&nbsp;，它包含 <code>n</code>&nbsp;个 <strong>互不相同</strong>&nbsp;的正整数。请你对这个数组执行 <code>m</code>&nbsp;个操作，在第 <code>i</code>&nbsp;个操作中，你需要将数字&nbsp;<code>operations[i][0]</code> 替换成&nbsp;<code>operations[i][1]</code>&nbsp;。</p>

<p>题目保证在第 <code>i</code>&nbsp;个操作中：</p>

<ul>
	<li><code>operations[i][0]</code>&nbsp;在&nbsp;<code>nums</code>&nbsp;中存在。</li>
	<li><code>operations[i][1]</code>&nbsp;在&nbsp;<code>nums</code>&nbsp;中不存在。</li>
</ul>

<p>请你返回执行完所有操作后的数组。</p>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre><b>输入：</b>nums = [1,2,4,6], operations = [[1,3],[4,7],[6,1]]
<b>输出：</b>[3,2,7,1]
<b>解释：</b>我们对 nums 执行以下操作：
- 将数字 1 替换为 3 。nums 变为 [<em><strong>3</strong></em>,2,4,6] 。
- 将数字 4 替换为 7 。nums 变为 [3,2,<em><strong>7</strong></em>,6] 。
- 将数字 6 替换为 1 。nums 变为 [3,2,7,<em><strong>1</strong></em>] 。
返回最终数组 [3,2,7,1] 。
</pre>

<p><strong>示例 2：</strong></p>

<pre><b>输入：</b>nums = [1,2], operations = [[1,3],[2,1],[3,2]]
<b>输出：</b>[2,1]
<b>解释：</b>我们对 nums 执行以下操作：
- 将数字 1 替换为 3 。nums 变为 [<em><strong>3</strong></em>,2] 。
- 将数字 2 替换为 1 。nums 变为 [3,<em><strong>1</strong></em>] 。
- 将数字 3 替换为 2 。nums 变为 [<em><strong>2</strong></em>,1] 。
返回最终数组 [2,1] 。
</pre>

<p>&nbsp;</p>

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

- **核心要求**：给定一个互不相同的正整数数组 `nums` 和一系列替换操作 `operations`，每个操作 `[old, new]` 表示将数组中的 `old` 值替换为 `new` 值。保证每次操作时 `old` 存在于数组中，`new` 不存在于数组中。执行所有操作后返回最终数组。
- **关键约束**：
  - 所有数字互不相同
  - 每次操作时 `old` 一定存在，`new` 一定不存在
  - 操作可能形成链式替换（如示例2中 1→3→2）
  - 数据规模：n, m ≤ 10^5
- **输入输出**：
  - 输入：`nums []int`, `operations [][]int`
  - 输出：执行所有操作后的 `[]int`

## 2️⃣ 解题思路

### 解法一：直接模拟（暴力替换）
- **核心思想**：对于每个操作，遍历数组找到 `old` 值并替换为 `new` 值
- **问题**：每次查找需要 O(n)，总复杂度 O(n×m) 会超时（10^10 级别）
- **不适用**：数据规模大时不可行

### 解法二：哈希映射法
- **核心思想**：使用哈希表记录每个值在数组中的索引，通过索引直接访问并修改
- **步骤**：
  1. 创建 `valueToIndex` 哈希表，记录每个值在 `nums` 中的索引
  2. 遍历每个操作 `[old, new]`：
     - 通过哈希表找到 `old` 的索引 `idx`
     - 将 `nums[idx]` 更新为 `new`
     - 更新哈希表：删除 `old` 的映射，添加 `new` → `idx` 的映射
  3. 返回 `nums`
- **注意事项**：
  - 需要同时更新数组和哈希表
  - 操作可能形成链式关系，但哈希表能正确处理

### 解法三：并查集/映射链法（优化链式操作）
- **核心思想**：对于链式替换 `a→b→c`，最终是 `a→c`，可以跳过中间映射
- **步骤**：
  1. 创建 `mapping` 哈希表，初始为空
  2. 逆序遍历操作（从最后一个操作向前）：
     - 对于 `[old, new]`，如果 `new` 在 `mapping` 中有映射，则建立 `old` → `mapping[new]` 的映射
     - 否则建立 `old` → `new` 的映射
  3. 遍历 `nums`，如果值在 `mapping` 中，则替换为最终映射值
- **优势**：避免多次更新数组，适合链式替换多的情况

## 3️⃣ 代码实现

### 解法二：哈希映射法
```go
func arrayChange(nums []int, operations [][]int) []int {
    // 创建值到索引的映射
    indexMap := make(map[int]int, len(nums))
    for i, num := range nums {
        indexMap[num] = i
    }
    
    // 执行所有操作
    for _, op := range operations {
        oldVal, newVal := op[0], op[1]
        idx := indexMap[oldVal]  // 获取旧值的索引
        
        // 更新数组
        nums[idx] = newVal
        
        // 更新映射：删除旧映射，添加新映射
        delete(indexMap, oldVal)
        indexMap[newVal] = idx
    }
    
    return nums
}
```

### 解法三：映射链法
```go
func arrayChange(nums []int, operations [][]int) []int {
    // 创建最终映射表
    finalMap := make(map[int]int)
    
    // 逆序遍历操作，建立最终映射关系
    for i := len(operations) - 1; i >= 0; i-- {
        oldVal, newVal := operations[i][0], operations[i][1]
        
        // 如果newVal有后续映射，则oldVal直接映射到最终值
        if finalVal, exists := finalMap[newVal]; exists {
            finalMap[oldVal] = finalVal
        } else {
            finalMap[oldVal] = newVal
        }
    }
    
    // 根据最终映射更新数组
    for i, num := range nums {
        if newVal, exists := finalMap[num]; exists {
            nums[i] = newVal
        }
    }
    
    return nums
}
```

## 4️⃣ 复杂度分析

### 解法二：哈希映射法
- ⏱️ **时间复杂度**：O(n + m)
  - 构建初始哈希表：O(n)
  - 执行 m 个操作：每个操作 O(1) 的查找和更新
  - 总计：O(n + m)
- 💾 **空间复杂度**：O(n)
  - 哈希表存储 n 个元素的索引映射

### 解法三：映射链法
- ⏱️ **时间复杂度**：O(n + m)
  - 逆序遍历 m 个操作：O(m)
  - 遍历 n 个元素更新：O(n)
  - 总计：O(n + m)
- 💾 **空间复杂度**：O(m)
  - 最坏情况下需要存储 m 个映射关系

## 5️⃣ 解法对比

| 特性 | 解法二（哈希映射） | 解法三（映射链） |
|------|-------------------|-----------------|
| 时间复杂度 | O(n+m) | O(n+m) |
| 空间复杂度 | O(n) | O(m) |
| 实现难度 | 简单直观 | 中等，需要逆序思维 |
| 适用场景 | 通用场景 | 链式替换多时更优 |
| 数组修改 | 实时修改 | 最后批量修改 |
| 推荐度 | ★★★★★ | ★★★★☆ |

**推荐解法二**：更直观易懂，实际性能相当，且能实时看到数组变化，调试方便。
