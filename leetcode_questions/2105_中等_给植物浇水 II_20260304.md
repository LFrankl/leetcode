## 题目描述

<p>Alice 和 Bob 打算给花园里的 <code>n</code> 株植物浇水。植物排成一行，从左到右进行标记，编号从 <code>0</code> 到 <code>n - 1</code> 。其中，第 <code>i</code> 株植物的位置是 <code>x = i</code> 。</p>

<p>每一株植物都需要浇特定量的水。Alice 和 Bob 每人有一个水罐，<strong>最初是满的 </strong>。他们按下面描述的方式完成浇水：</p>

<ul>
	<li>&nbsp;Alice 按 <strong>从左到右</strong> 的顺序给植物浇水，从植物 <code>0</code> 开始。Bob 按 <strong>从右到左</strong> 的顺序给植物浇水，从植物 <code>n - 1</code> 开始。他们 <strong>同时</strong> 给植物浇水。</li>
	<li>无论需要多少水，为每株植物浇水所需的时间都是相同的。</li>
	<li>如果 Alice/Bob 水罐中的水足以 <strong>完全</strong> 灌溉植物，他们 <strong>必须</strong> 给植物浇水。否则，他们 <strong>首先</strong>（立即）重新装满罐子，然后给植物浇水。</li>
	<li>如果 Alice 和 Bob 到达同一株植物，那么当前水罐中水 <strong>更多</strong> 的人会给这株植物浇水。如果他俩水量相同，那么 Alice 会给这株植物浇水。</li>
</ul>

<p>给你一个下标从 <strong>0</strong> 开始的整数数组 <code>plants</code> ，数组由 <code>n</code> 个整数组成。其中，<code>plants[i]</code> 为第 <code>i</code> 株植物需要的水量。另有两个整数 <code>capacityA</code> 和&nbsp;<code>capacityB</code> 分别表示 Alice 和 Bob 水罐的容量。返回两人浇灌所有植物过程中重新灌满水罐的 <strong>次数</strong> 。</p>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre>
<strong>输入：</strong>plants = [2,2,3,3], capacityA = 5, capacityB = 5
<strong>输出：</strong>1
<strong>解释：</strong>
- 最初，Alice 和 Bob 的水罐中各有 5 单元水。
- Alice 给植物 0 浇水，Bob 给植物 3 浇水。
- Alice 和 Bob 现在分别剩下 3 单元和 2 单元水。
- Alice 有足够的水给植物 1 ，所以她直接浇水。Bob 的水不够给植物 2 ，所以他先重新装满水，再浇水。
所以，两人浇灌所有植物过程中重新灌满水罐的次数 = 0 + 0 + 1 + 0 = 1 。</pre>

<p><strong>示例 2：</strong></p>

<pre>
<strong>输入：</strong>plants = [2,2,3,3], capacityA = 3, capacityB = 4
<strong>输出：</strong>2
<strong>解释：</strong>
- 最初，Alice 的水罐中有 3 单元水，Bob 的水罐中有 4 单元水。
- Alice 给植物 0 浇水，Bob 给植物 3 浇水。
- Alice 和 Bob 现在都只有 1 单元水，并分别需要给植物 1 和植物 2 浇水。
- 由于他们的水量均不足以浇水，所以他们重新灌满水罐再进行浇水。
所以，两人浇灌所有植物过程中重新灌满水罐的次数 = 0 + 1 + 1 + 0 = 2 。</pre>

<p><strong>示例 3：</strong></p>

<pre>
<strong>输入：</strong>plants = [5], capacityA = 10, capacityB = 8
<strong>输出：</strong>0
<strong>解释：</strong>
- 只有一株植物
- Alice 的水罐有 10 单元水，Bob 的水罐有 8 单元水。因此 Alice 的水罐中水更多，她会给这株植物浇水。
所以，两人浇灌所有植物过程中重新灌满水罐的次数 = 0 。</pre>

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

- **核心要求**：Alice 和 Bob 分别从左右两端开始给植物浇水，水罐有容量限制。当水罐中的水不够浇当前植物时，需要重新灌满水罐（次数+1）。如果两人相遇在同一株植物，则比较当前水罐水量，多者浇水；水量相同时 Alice 浇水。要求统计两人总共重新灌满水罐的次数。
- **关键约束**：
  - 两人同时开始浇水。
  - 浇水顺序：Alice 从左到右，Bob 从右到左。
  - 相遇时按水量决定谁浇水。
  - 每株植物必须被浇一次。
- **边界情况**：
  - 只有一株植物时，谁水量多谁浇（或 Alice 优先）。
  - 植物数量为奇数时，中间那株植物由相遇规则决定谁浇。
- **输入输出**：
  - 输入：`plants []int`（每株植物需水量），`capacityA int`（Alice 水罐容量），`capacityB int`（Bob 水罐容量）。
  - 输出：`int`（重新灌满水罐的总次数）。

---

## 2️⃣ 解题思路

### 解法一：双指针模拟
- **核心思想**：用两个指针分别模拟 Alice 和 Bob 的浇水过程，相遇时特殊处理。
- **步骤**：
  1. 初始化 `i=0`（Alice 位置），`j=n-1`（Bob 位置），`waterA=capacityA`，`waterB=capacityB`，`refill=0`。
  2. 当 `i <= j` 时循环：
     - 如果 `i == j`（相遇）：
       - 比较 `waterA` 和 `waterB`，谁多谁浇（或 Alice 优先）。
       - 如果水量不够，则对应的人需要重新灌满（次数+1），然后浇水。
     - 否则：
       - Alice 浇植物 `i`：如果 `waterA < plants[i]`，则重新灌满（次数+1），然后浇水。
       - Bob 浇植物 `j`：如果 `waterB < plants[j]`，则重新灌满（次数+1），然后浇水。
     - 更新指针：`i++`，`j--`。
  3. 返回总次数。
- **注意事项**：
  - 相遇时只浇一次水，不要重复计算。
  - 浇水后要扣除水量。

### 解法二：双指针优化（逻辑合并）
- **核心思想**：将相遇和分开浇水的情况合并处理，减少条件分支。
- **步骤**：
  1. 初始化同上。
  2. 循环条件 `i <= j`：
     - 如果 `i < j`：
       - 分别处理 Alice 和 Bob 的浇水逻辑（可能重新灌满）。
     - 如果 `i == j`：
       - 判断 `max(waterA, waterB)` 是否足够，不够则对应的人重新灌满（次数+1）。
     - 更新指针。
  3. 返回次数。
- **优点**：逻辑更清晰，减少重复判断。

## 3️⃣ 代码实现

### 解法一：双指针模拟
```go
func minimumRefill(plants []int, capacityA int, capacityB int) int {
    n := len(plants)
    i, j := 0, n-1
    waterA, waterB := capacityA, capacityB
    refill := 0

    for i <= j {
        if i == j { // 相遇
            if waterA >= waterB { // Alice 浇
                if waterA < plants[i] {
                    refill++
                    waterA = capacityA
                }
                waterA -= plants[i]
            } else { // Bob 浇
                if waterB < plants[j] {
                    refill++
                    waterB = capacityB
                }
                waterB -= plants[j]
            }
            i++
            j--
        } else {
            // Alice 浇植物 i
            if waterA < plants[i] {
                refill++
                waterA = capacityA
            }
            waterA -= plants[i]
            i++

            // Bob 浇植物 j
            if waterB < plants[j] {
                refill++
                waterB = capacityB
            }
            waterB -= plants[j]
            j--
        }
    }
    return refill
}
```

### 解法二：双指针优化
```go
func minimumRefill(plants []int, capacityA int, capacityB int) int {
    n := len(plants)
    i, j := 0, n-1
    waterA, waterB := capacityA, capacityB
    refill := 0

    for i <= j {
        if i < j {
            // Alice 浇
            if waterA < plants[i] {
                refill++
                waterA = capacityA
            }
            waterA -= plants[i]
            i++

            // Bob 浇
            if waterB < plants[j] {
                refill++
                waterB = capacityB
            }
            waterB -= plants[j]
            j--
        } else { // i == j
            // 相遇时，水量多的人浇（或 Alice 优先）
            if waterA >= waterB {
                if waterA < plants[i] {
                    refill++
                }
            } else {
                if waterB < plants[j] {
                    refill++
                }
            }
            break
        }
    }
    return refill
}
```

## 6️⃣ 测试用例

### 用例1（常规）
- 输入：`plants = [2,2,3,3]`, `capacityA = 5`, `capacityB = 5`
- 输出：`1`
- 解释：Alice 浇 0、1，Bob 浇 3、2，Bob 浇 2 时需要重新灌满一次。

### 用例2（相遇时水量相同）
- 输入：`plants = [2,2,3,3]`, `capacityA = 3`, `capacityB = 4`
- 输出：`2`
- 解释：Alice 浇 0 后剩 1，Bob 浇 3 后剩 1，两人浇 1 和 2 时都需要重新灌满。

### 用例3（单株植物）
- 输入：`plants = [5]`, `capacityA = 10`, `capacityB = 8`
- 输出：`0`
- 解释：Alice 水量多，直接浇水，无需重新灌满。

### 用例4（奇数株，相遇时水量不够）
- 输入：`plants = [1,2,3,4,5]`, `capacityA = 6`, `capacityB = 5`
- 输出：`2`
- 解释：Alice 浇 0、1，Bob 浇 4、3，相遇在植物 2，Alice 浇（水量多），但 Alice 水量不够，需重新灌满一次；Bob 浇 3 时也不够，需重新灌满一次。
