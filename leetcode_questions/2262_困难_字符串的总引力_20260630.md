## 题目描述

<p>字符串的 <strong>引力</strong> 定义为：字符串中 <strong>不同</strong> 字符的数量。</p>

<ul>
	<li>例如，<code>"abbca"</code> 的引力为 <code>3</code> ，因为其中有 <code>3</code> 个不同字符 <code>'a'</code>、<code>'b'</code> 和 <code>'c'</code> 。</li>
</ul>

<p>给你一个字符串 <code>s</code> ，返回 <strong>其所有子字符串的总引力</strong> <strong>。</strong></p>

<p><strong>子字符串</strong> 定义为：字符串中的一个连续字符序列。</p>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre><strong>输入：</strong>s = "abbca"
<strong>输出：</strong>28
<strong>解释：</strong>"abbca" 的子字符串有：
- 长度为 1 的子字符串："a"、"b"、"b"、"c"、"a" 的引力分别为 1、1、1、1、1，总和为 5 。
- 长度为 2 的子字符串："ab"、"bb"、"bc"、"ca" 的引力分别为 2、1、2、2 ，总和为 7 。
- 长度为 3 的子字符串："abb"、"bbc"、"bca" 的引力分别为 2、2、3 ，总和为 7 。
- 长度为 4 的子字符串："abbc"、"bbca" 的引力分别为 3、3 ，总和为 6 。
- 长度为 5 的子字符串："abbca" 的引力为 3 ，总和为 3 。
引力总和为 5 + 7 + 7 + 6 + 3 = 28 。
</pre>

<p><strong>示例 2：</strong></p>

<pre><strong>输入：</strong>s = "code"
<strong>输出：</strong>20
<strong>解释：</strong>"code" 的子字符串有：
- 长度为 1 的子字符串："c"、"o"、"d"、"e" 的引力分别为 1、1、1、1 ，总和为 4 。
- 长度为 2 的子字符串："co"、"od"、"de" 的引力分别为 2、2、2 ，总和为 6 。
- 长度为 3 的子字符串："cod"、"ode" 的引力分别为 3、3 ，总和为 6 。
- 长度为 4 的子字符串："code" 的引力为 4 ，总和为 4 。
引力总和为 4 + 6 + 6 + 4 = 20 。
</pre>

## 提示

1. Consider the set of substrings that end at a certain index i. Then, consider a specific alphabetic character. How do you count the number of substrings ending at index i that contain that character?
2. The number of substrings that contain the alphabetic character is equivalent to 1 plus the index of the last occurrence of the character before index i + 1.
3. The total appeal of all substrings ending at index i is the total sum of the number of substrings that contain each alphabetic character.
4. To find the total appeal of all substrings, we simply sum up the total appeal for each index.

## AI 解答 (DeepSeek)

### 解法一：逐字符贡献法（最优解）

思路：将总引力拆解为每个字符对总引力的"贡献"。字符 `s[i]` 对某个子串有贡献，当且仅当它是该子串中**首次出现**的该字符。设 `lastPos[c]` 为字符 `c` 上一次出现的位置，则以 `s[i]` 为首次出现的子串，起点范围为 `(lastPos[c]+1) ~ i`，终点范围为 `i ~ n-1`，贡献数为 `(i - lastPos[c]) × (n - i)`。遍历一遍累加即可。

```go
func totalGravity(s string) int64 {
    var ans int64
    n := len(s)
    lastPos := [26]int{} // 每个字符上次出现的位置，初始为 -1
    for i := range lastPos {
        lastPos[i] = -1
    }
    for i := 0; i < n; i++ {
        c := int(s[i] - 'a')
        ans += int64(i - lastPos[c]) * int64(n - i)
        lastPos[c] = i
    }
    return ans
}
```

复杂度：时间 **O(n)**，空间 **O(1)**（仅 26 个字母的 lastPos 数组）

---

### 解法二：增量递推法

思路：设 `g(i)` 表示以位置 `i` 为右端点的所有子串的引力之和。从 `g(i-1)` 递推到 `g(i)` 时，新增字符 `s[i]`：若 `s[i]` 之前未出现过，则所有以 `i` 结尾的子串引力各加 1，贡献 `i+1`；若 `s[i]` 上次出现在 `lastPos`，则仅从 `lastPos+1` 到 `i` 开头的子串引力加 1，贡献 `i - lastPos`。总答案为所有 `g(i)` 的累加。

```go
func totalGravity(s string) int64 {
    var ans int64
    var cur int64 // g(i)：以 i 为右端点的所有子串引力之和
    lastPos := [26]int{}
    for i := range lastPos {
        lastPos[i] = -1
    }
    for i := 0; i < len(s); i++ {
        c := int(s[i] - 'a')
        cur += int64(i - lastPos[c]) // 增量：s[i] 对以 i 结尾的子串的贡献
        ans += cur
        lastPos[c] = i
    }
    return ans
}
```

复杂度：时间 **O(n)**，空间 **O(1)**

> **两种解法本质相同**——解法一是从"全局贡献"角度直接计算，解法二是从"逐步累加"角度递推。核心都是维护每个字符的 `lastPos`，时间 O(n) 线性扫描，空间 O(1)。
