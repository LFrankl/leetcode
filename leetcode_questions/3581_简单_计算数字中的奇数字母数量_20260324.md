# 计算数字中的奇数字母数量

## 1️⃣ 题意理解

### 题目重述
给定一个整数 `n`，统计从 `1` 到 `n` 的所有整数中，每个数字的十进制表示中，出现次数为奇数的数字的个数之和。

例如：数字 `123` 中，数字 `1`、`2`、`3` 各出现 1 次（都是奇数次），所以贡献 3 个奇数字母。

### 关键约束
- 输入：整数 `n`，范围 `1 ≤ n ≤ 1000`
- 输出：一个整数，表示从 1 到 n 所有数字中奇数字母数量的总和
- 注意：统计的是每个数字中各个数字字符出现次数的奇偶性，不是数字本身的奇偶性

### 边界情况
- n = 1：只统计数字 1
- n = 1000：包含三位数和四位数
- 数字 0 也可能出现在数字中（如 101）

## 2️⃣ 解题思路

### 解法一：暴力枚举法
**核心思想**：遍历 1 到 n 的每个数字，统计每个数字中各个数字字符的出现次数，判断是否为奇数。

**步骤**：
1. 初始化结果变量 `total = 0`
2. 遍历 i 从 1 到 n：
   - 将 i 转换为字符串
   - 统计字符串中每个数字字符的出现次数
   - 遍历 0-9 的数字，统计出现次数为奇数的个数
   - 将个数加到 total 中
3. 返回 total

**注意事项**：
- 注意数字 0 的处理
- 字符串转换可能影响性能，但 n ≤ 1000 可以接受

### 解法二：数学优化法
**核心思想**：利用数字的规律，避免重复统计。对于每个数字位置，计算其贡献。

**步骤**：
1. 对于每个数字 d (0-9)，统计它在 1-n 中出现的总次数
2. 如果某个数字 d 出现的总次数是奇数，则贡献 1
3. 对所有 d 的贡献求和

**优化思路**：
- 使用数位 DP 的思想统计每个数字出现的次数
- 但本题 n ≤ 1000，暴力法已足够高效

## 3️⃣ 代码实现

### 解法一：暴力枚举法
```go
func countOddNumbers(n int) int {
    total := 0
    
    // 遍历从 1 到 n 的所有数字
    for i := 1; i <= n; i++ {
        // 将数字转换为字符串
        str := strconv.Itoa(i)
        
        // 统计当前数字中各个数字的出现次数
        freq := make([]int, 10) // 0-9 的数字频率
        
        // 遍历字符串的每个字符
        for _, ch := range str {
            digit := int(ch - '0') // 字符转数字
            freq[digit]++
        }
        
        // 统计出现次数为奇数的数字个数
        oddCount := 0
        for _, count := range freq {
            if count%2 == 1 {
                oddCount++
            }
        }
        
        total += oddCount
    }
    
    return total
}
```

### 解法二：优化统计法
```go
func countOddNumbersOptimized(n int) int {
    total := 0
    
    for i := 1; i <= n; i++ {
        num := i
        // 使用数组记录数字出现次数的奇偶性
        // 0 表示偶数次，1 表示奇数次
        parity := make([]int, 10)
        
        // 分解数字的每一位
        for num > 0 {
            digit := num % 10
            // 切换奇偶性：0->1, 1->0
            parity[digit] ^= 1
            num /= 10
        }
        
        // 处理前导零的情况（对于数字如 101，中间的0也要处理）
        // 但上面的循环已经处理了所有位，包括0
        
        // 统计奇数次出现的数字个数
        oddCount := 0
        for _, p := range parity {
            oddCount += p // p 为 1 表示奇数次
        }
        
        total += oddCount
    }
    
    return total
}
```

## 4️⃣ 复杂度分析

### 解法一：暴力枚举法
- ⏱️ **时间复杂度**：O(n × k)
  - n 是输入的数字
  - k 是数字的平均位数，最大为 4（n ≤ 1000）
  - 总体复杂度约为 O(4n) = O(n)
- 💾 **空间复杂度**：O(1)
  - 只使用了固定大小的数组（10个元素）
  - 字符串转换需要 O(k) 空间，但可重用

### 解法二：优化统计法
- ⏱️ **时间复杂度**：O(n × k)
  - 与解法一相同，但避免了字符串转换
  - 实际运行更快
- 💾 **空间复杂度**：O(1)
  - 使用固定大小的数组（10个元素）
