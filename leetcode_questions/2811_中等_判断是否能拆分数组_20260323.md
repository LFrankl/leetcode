## 题目描述

<p>给你一个长度为 <code>n</code> 的数组 <code>nums</code> 和一个整数 <code>m</code> 。请你判断能否执行一系列操作，将数组拆分成 <code>n</code> 个 <strong>非空 </strong>数组。</p>

<p>一个数组被称为 <strong>好</strong> 的，如果：</p>

<ul>
	<li>子数组的长度为 1 ，或者</li>
	<li>子数组元素之和 <strong>大于或等于</strong>&nbsp; <code>m</code> 。</li>
</ul>

<p>在每一步操作中，你可以选择一个 <strong>长度至少为 2</strong> 的现有数组（之前步骤的结果） 并将其拆分成 <strong>2</strong> 个子数组，而得到的 <strong>每个</strong> 子数组都需要是好的。</p>

<p>如果你可以将给定数组拆分成 <code>n</code> 个满足要求的数组，返回 <code>true</code><em> </em>；否则，返回 <code>false</code> 。</p>

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

### 题目重述
给定一个数组 `nums` 和整数 `m`，判断能否通过一系列拆分操作，将数组最终拆分成 `n` 个长度为 1 的子数组。  
每次拆分必须满足：**拆分出的两个子数组都是“好的”**。  
“好的”数组定义为：
1. 长度为 1，或者
2. 元素之和 ≥ m。

### 关键约束
- 初始数组长度 `n` 在 1 到 100 之间。
- 元素值在 1 到 100 之间。
- `m` 在 1 到 200 之间。
- 每次拆分必须将长度 ≥ 2 的数组拆成两个非空子数组。
- 拆分出的两个子数组必须都是“好的”。

### 边界情况
- 如果 `n == 1`，无需拆分，直接返回 `true`。
- 如果 `n == 2`，只需判断整个数组是否“好的”，且能拆成两个长度为 1 的子数组。
- 如果存在相邻两个元素之和 ≥ m，则可能更容易拆分。

### 输入输出
- 输入：`nums []int`, `m int`
- 输出：`bool`

## 2️⃣ 解题思路

### 解法一：区间 DP（动态规划）
**核心思想**  
问题本质是判断能否将整个数组拆分成单个元素，且每次拆分时两个子数组都是“好的”。  
这可以看作一个区间划分问题：  
定义 `dp[i][j]` 表示子数组 `nums[i:j+1]` 能否拆分成单个元素（满足拆分规则）。  
转移方程：  
`dp[i][j] = true` 当且仅当存在一个分割点 `k`（`i ≤ k < j`），使得：
1. `dp[i][k] == true` 且 `dp[k+1][j] == true`（左右两部分都能继续拆分）
2. 且当前拆分出的两个子数组 `nums[i:k+1]` 和 `nums[k+1:j+1]` 都是“好的”。

**优化**  
如果 `nums[i] + nums[j] ≥ m`，则整个区间 `[i, j]` 的和一定 ≥ m（因为所有元素为正），因此任意包含 `i` 和 `j` 的子数组都是“好的”。  
实际上，只要存在相邻两个元素之和 ≥ m，整个数组就一定能拆分成单个元素（因为每次拆分时，至少有一个子数组包含这两个相邻元素，从而满足“好的”条件）。

**结论**  
问题简化为：**是否存在相邻两个元素之和 ≥ m**，或者数组长度 ≤ 2 时特殊处理。

### 解法二：贪心（基于相邻元素和）
**核心思想**  
通过观察发现：
- 如果存在相邻两个元素之和 ≥ m，那么我们可以从这两个元素处开始拆分，逐步将整个数组拆分成单个元素。
- 如果不存在这样的相邻元素，且数组长度 > 2，则无法完成拆分（因为第一次拆分就无法得到两个“好的”子数组）。

**步骤**
1. 如果 `n == 1`，返回 `true`。
2. 如果 `n == 2`，返回 `true`（因为最终要拆成两个长度为 1 的子数组，初始数组必须“好的”）。
3. 遍历数组，检查是否存在相邻两个元素之和 ≥ m。
4. 如果存在，返回 `true`；否则返回 `false`。

**证明**  
- 如果存在相邻元素 `nums[i] + nums[i+1] ≥ m`，则我们可以先拆分成 `[0..i+1]` 和 `[i+2..n-1]`（或类似方式），使得每次拆分时至少有一个子数组包含这两个相邻元素，从而满足“好的”条件。
- 如果不存在这样的相邻元素，则第一次拆分时，无论怎么拆，两个子数组的和都 < m（因为每个子数组最多包含一个元素 ≥ m 的情况不可能出现，除非长度为 1），且长度 > 1，因此无法满足“好的”条件。

## 3️⃣ 代码实现

### 解法一：区间 DP
```go
func canSplitArray(nums []int, m int) bool {
    n := len(nums)
    if n <= 2 {
        return true
    }
    // 前缀和用于快速计算区间和
    prefix := make([]int, n+1)
    for i := 0; i < n; i++ {
        prefix[i+1] = prefix[i] + nums[i]
    }
    // dp[i][j] 表示 nums[i..j] 能否拆分成单个元素
    dp := make([][]bool, n)
    for i := range dp {
        dp[i] = make([]bool, n)
        dp[i][i] = true // 单个元素总是好的
    }
    // 枚举区间长度
    for length := 2; length <= n; length++ {
        for i := 0; i+length-1 < n; i++ {
            j := i + length - 1
            // 如果区间长度为 2，只需判断整个区间是否好的
            if length == 2 {
                dp[i][j] = (nums[i]+nums[j] >= m)
                continue
            }
            // 尝试所有分割点
            for k := i; k < j; k++ {
                leftGood := (k-i+1 == 1) || (prefix[k+1]-prefix[i] >= m)
                rightGood := (j-k == 1) || (prefix[j+1]-prefix[k+1] >= m)
                if leftGood && rightGood && dp[i][k] && dp[k+1][j] {
                    dp[i][j] = true
                    break
                }
            }
        }
    }
    return dp[0][n-1]
}
```

### 解法二：贪心（相邻元素和）
```go
func canSplitArray(nums []int, m int) bool {
    n := len(nums)
    if n <= 2 {
        return true
    }
    // 检查是否存在相邻两个元素之和 >= m
    for i := 0; i < n-1; i++ {
        if nums[i]+nums[i+1] >= m {
            return true
        }
    }
    return false
}
```

## 4️⃣ 复杂度分析

### 解法一：区间 DP
- ⏱️ **时间复杂度**：O(n³)  
  三重循环：区间长度 O(n) × 起点 O(n) × 分割点 O(n) = O(n³)。  
  n ≤ 100，因此最多 10⁶ 次操作，可接受。
- 💾 **空间复杂度**：O(n²)  
  用于存储 dp 数组和前缀和。

### 解法二：贪心
- ⏱️ **时间复杂度**：O(n)  
  只需一次遍历检查相邻元素和。
- 💾 **空间复杂度**：O(1)  
  只使用常数额外空间。

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 区间 DP | 通用性强，可处理更复杂的拆分规则 | 时间复杂度高，实现复杂 | 当拆分规则更复杂时 |
| 贪心 | 时间复杂度低，实现简单，效率高 | 依赖于本题的特殊性质 | 本题及类似简化场景 |

**推荐解法**  
本题推荐使用**解法二（贪心）**，因为：
1. 时间复杂度低，代码简洁。
2. 经过分析，本题的约束条件使得问题简化为检查相邻元素和。
3. 在 n ≤ 100 时，贪心解法足够高效且正确。
