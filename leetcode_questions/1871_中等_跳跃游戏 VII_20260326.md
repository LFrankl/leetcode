## 题目描述

<p>给你一个下标从 <strong>0 </strong>开始的二进制字符串 <code>s</code> 和两个整数 <code>minJump</code> 和 <code>maxJump</code> 。一开始，你在下标 <code>0</code> 处，且该位置的值一定为 <code>'0'</code> 。当同时满足如下条件时，你可以从下标 <code>i</code> 移动到下标 <code>j</code> 处：</p>

	<li><code>s[j] == '0'</code>.</li>
</ul>

<p>如果你可以到达 <code>s</code> 的下标<i> </i><code>s.length - 1</code> 处，请你返回 <code>true</code> ，否则返回 <code>false</code> 。</p>

	<li><code>s[0] == '0'</code></li>

## 1️⃣ 题意理解

**核心要求**：  
给定一个二进制字符串 `s`，初始位置在下标 `0`（保证 `s[0] == '0'`），每次可以从当前位置 `i` 跳跃到位置 `j`，当且仅当：
1. `i + minJump <= j <= min(i + maxJump, len(s)-1)`
2. `s[j] == '0'`

判断是否能从下标 `0` 到达最后一个下标 `len(s)-1`。

**关键约束**：
- `2 <= s.length <= 10^5`
- `s[i]` 为 `'0'` 或 `'1'`
- `s[0] == '0'`
- `1 <= minJump <= maxJump <= 10^5`

**边界情况**：
- 如果 `s[len(s)-1] == '1'`，直接返回 `false`。
- 如果 `minJump > len(s)-1`，无法到达。
- 需要高效处理大量跳跃区间，避免 O(n²) 的暴力检查。

**输入输出**：
- 输入：`s`（字符串）、`minJump`（整数）、`maxJump`（整数）
- 输出：`bool`（`true`/`false`）

## 2️⃣ 解题思路

### 解法一：BFS + 访问标记
- **核心思想**：将问题转化为图搜索，每个位置是节点，合法跳跃是边，用 BFS 遍历可达节点。
- **关键优化**：使用 `visited` 数组避免重复访问，并记录已访问过的最大下标，避免重复检查区间。
- **步骤**：
  1. 如果最后一个字符是 `'1'`，直接返回 `false`。
  2. 初始化队列，将下标 `0` 入队。
  3. 每次从队列取出 `i`，计算可跳跃的区间 `[left, right]`。
  4. 从 `max(left, 已访问最大下标+1)` 开始检查，将满足条件的位置入队。
  5. 如果遇到最后一个下标，返回 `true`。
- **坑点**：直接遍历区间会超时，需要跳过已访问的部分。

### 解法二：动态规划 + 前缀和优化
- **核心思想**：定义 `dp[i]` 表示能否到达位置 `i`。状态转移：`dp[i]` 为 `true` 当且仅当 `s[i]=='0'` 且存在 `j` 在 `[i-maxJump, i-minJump]` 范围内使得 `dp[j]==true`。
- **关键优化**：使用前缀和数组 `pre` 记录 `dp[0..i]` 中 `true` 的数量，从而 O(1) 判断区间内是否存在 `true`。
- **步骤**：
  1. 初始化 `dp[0]=true`，`pre[0]=1`。
  2. 遍历 `i` 从 `1` 到 `n-1`：
     - 如果 `s[i]=='1'`，`dp[i]=false`。
     - 否则，计算 `left = i-maxJump` 和 `right = i-minJump`，若 `right>=0` 且区间 `[max(0,left), right]` 内存在 `true`，则 `dp[i]=true`。
     - 更新前缀和 `pre[i] = pre[i-1] + dp[i]`。
  3. 返回 `dp[n-1]`。
- **坑点**：注意区间边界处理，避免越界。

## 3️⃣ 代码实现

### 解法一：BFS + 访问标记
```go
func canReach(s string, minJump int, maxJump int) bool {
    n := len(s)
    if s[n-1] == '1' {
        return false
    }
    
    queue := []int{0}
    visited := make([]bool, n)
    visited[0] = true
    maxVisited := 0 // 已访问的最大下标，用于跳过重复检查
    
    for len(queue) > 0 {
        i := queue[0]
        queue = queue[1:]
        
        // 计算可跳跃范围
        left := i + minJump
        right := i + maxJump
        if right >= n-1 && left <= n-1 {
            return true
        }
        // 从 max(left, maxVisited+1) 开始检查
        start := max(left, maxVisited+1)
        for j := start; j <= right && j < n; j++ {
            if !visited[j] && s[j] == '0' {
                if j == n-1 {
                    return true
                }
                visited[j] = true
                queue = append(queue, j)
            }
        }
        maxVisited = max(maxVisited, right)
    }
    return false
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

### 解法二：动态规划 + 前缀和优化
```go
func canReach(s string, minJump int, maxJump int) bool {
    n := len(s)
    if s[n-1] == '1' {
        return false
    }
    
    dp := make([]bool, n)
    pre := make([]int, n) // 前缀和，pre[i] 表示 dp[0..i] 中 true 的数量
    dp[0] = true
    pre[0] = 1
    
    for i := 1; i < n; i++ {
        if s[i] == '0' {
            left := i - maxJump
            right := i - minJump
            if right >= 0 {
                // 计算区间 [max(0, left), right] 内是否有 true
                cnt := pre[right]
                if left > 0 {
                    cnt -= pre[left-1]
                }
                if cnt > 0 {
                    dp[i] = true
                }
            }
        }
        pre[i] = pre[i-1]
        if dp[i] {
            pre[i]++
        }
    }
    return dp[n-1]
}
```

## 4️⃣ 复杂度分析

### 解法一：BFS + 访问标记
- ⏱️ **时间复杂度**：O(n)  
  每个位置最多入队一次，每次检查跳跃区间时，`maxVisited` 保证不会重复扫描已访问位置。
- 💾 **空间复杂度**：O(n)  
  用于存储 `visited` 数组和队列。

### 解法二：动态规划 + 前缀和优化
- ⏱️ **时间复杂度**：O(n)  
  只需一次遍历，每次 O(1) 判断区间内是否存在 `true`。
- 💾 **空间复杂度**：O(n)  
  用于存储 `dp` 和 `pre` 数组。

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| BFS + 访问标记 | 直观，易于理解 | 需要队列和 visited 数组，常数较大 | 适合需要显式路径的场景 |
| DP + 前缀和 | 高效，代码简洁 | 需要额外前缀和数组 | 大规模数据，追求最优时间复杂度 |

**推荐解法**：**解法二（DP + 前缀和）**  
理由：时间复杂度稳定 O(n)，空间复杂度 O(n)，代码简洁，适合 LeetCode 的大数据量限制。
