## 题目描述

<p>给定一个三角形 <code>triangle</code> ，找出自顶向下的最小路径和。</p>

<p>每一步只能移动到下一行中相邻的结点上。<strong>相邻的结点 </strong>在这里指的是 <strong>下标</strong> 与 <strong>上一层结点下标</strong> 相同或者等于 <strong>上一层结点下标 + 1</strong> 的两个结点。也就是说，如果正位于当前行的下标 <code>i</code> ，那么下一步可以移动到下一行的下标 <code>i</code> 或 <code>i + 1</code> 。</p>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre>
<strong>输入：</strong>triangle = [[2],[3,4],[6,5,7],[4,1,8,3]]
<strong>输出：</strong>11
<strong>解释：</strong>如下面简图所示：
   <strong>2</strong>
  <strong>3</strong> 4
 6 <strong>5</strong> 7
4 <strong>1</strong> 8 3
自顶向下的最小路径和为&nbsp;11（即，2&nbsp;+&nbsp;3&nbsp;+&nbsp;5&nbsp;+&nbsp;1&nbsp;= 11）。
</pre>

<p><strong>示例 2：</strong></p>

<pre>
<strong>输入：</strong>triangle = [[-10]]
<strong>输出：</strong>-10
</pre>

<p>&nbsp;</p>

<p><strong>提示：</strong></p>

<ul>
	<li><code>1 &lt;= triangle.length &lt;= 200</code></li>
	<li><code>triangle[0].length == 1</code></li>
	<li><code>triangle[i].length == triangle[i - 1].length + 1</code></li>
	<li><code>-10<sup>4</sup> &lt;= triangle[i][j] &lt;= 10<sup>4</sup></code></li>
</ul>

<p>&nbsp;</p>

<p><strong>进阶：</strong></p>

<ul>
	<li>你可以只使用 <code>O(n)</code>&nbsp;的额外空间（<code>n</code> 为三角形的总行数）来解决这个问题吗？</li>
</ul>

## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

**核心要求**：  
给定一个三角形数组 `triangle`，从顶部到底部寻找一条路径，使得路径上数字之和最小。每一步只能移动到下一行的相邻位置（当前下标 `i` 可以移动到下一行的 `i` 或 `i+1`）。

**关键约束**：
- 三角形行数 ≥ 1
- 每行长度递增 1
- 数字范围：[-10^4, 10^4]
- 要求最小路径和

**边界情况**：
- 只有一行时，直接返回该值
- 可能存在负数，路径和可能为负

**输入输出**：
- 输入：二维整数数组 `triangle`，表示三角形
- 输出：整数，表示最小路径和

## 2️⃣ 解题思路

### 解法一：自底向上动态规划（二维 DP）
**核心思想**：  
从三角形底部开始向上递推，定义 `dp[i][j]` 表示从位置 `(i, j)` 到底部的最小路径和。状态转移方程为：
```
dp[i][j] = triangle[i][j] + min(dp[i+1][j], dp[i+1][j+1])
```
最终 `dp[0][0]` 即为答案。

**步骤**：
1. 初始化 `dp` 数组，大小与 `triangle` 相同
2. 最后一行 `dp` 值等于 `triangle` 最后一行
3. 从倒数第二行向上遍历，计算每个位置的最小路径和
4. 返回 `dp[0][0]`

**注意事项**：
- 注意下标范围，避免越界
- 可以直接复用 `triangle` 作为 `dp` 数组以节省空间

### 解法二：自底向上动态规划（一维 DP 空间优化）
**核心思想**：  
由于 `dp[i][j]` 只依赖于下一行的 `dp[i+1][j]` 和 `dp[i+1][j+1]`，可以使用一维数组滚动更新，空间复杂度从 O(n²) 降为 O(n)。

**步骤**：
1. 初始化 `dp` 为三角形最后一行
2. 从倒数第二行向上遍历
3. 对于每行的每个位置 `j`，更新 `dp[j] = triangle[i][j] + min(dp[j], dp[j+1])`
4. 返回 `dp[0]`

**注意事项**：
- 更新顺序：从左到右，因为 `dp[j]` 依赖于 `dp[j]` 和 `dp[j+1]`，不会覆盖后续需要的值
- 一维 DP 是二维 DP 的空间优化版本，思路相同

### 解法三：自顶向下动态规划（带记忆化搜索）
**核心思想**：  
递归地从顶部向下探索所有路径，使用记忆化存储已计算的结果，避免重复计算。

**步骤**：
1. 定义递归函数 `dfs(i, j)` 返回从 `(i, j)` 到底部的最小路径和
2. 如果 `i` 是最后一行，直接返回 `triangle[i][j]`
3. 如果 `(i, j)` 已计算过，直接返回缓存值
4. 否则计算 `triangle[i][j] + min(dfs(i+1, j), dfs(i+1, j+1))` 并缓存
5. 返回 `dfs(0, 0)`

**注意事项**：
- 递归深度为行数 n，可能引起栈溢出（但本题 n ≤ 200，安全）
- 记忆化搜索更直观，但空间开销略大于自底向上

## 3️⃣ 代码实现

### 解法一：自底向上动态规划（二维 DP）
```go
func minimumTotal(triangle [][]int) int {
    n := len(triangle)
    if n == 0 {
        return 0
    }
    // 初始化 dp 数组
    dp := make([][]int, n)
    for i := range dp {
        dp[i] = make([]int, len(triangle[i]))
    }
    // 最后一行直接赋值
    for j := 0; j < len(triangle[n-1]); j++ {
        dp[n-1][j] = triangle[n-1][j]
    }
    // 自底向上递推
    for i := n - 2; i >= 0; i-- {
        for j := 0; j < len(triangle[i]); j++ {
            dp[i][j] = triangle[i][j] + min(dp[i+1][j], dp[i+1][j+1])
        }
    }
    return dp[0][0]
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

### 解法二：自底向上动态规划（一维 DP 空间优化）
```go
func minimumTotal(triangle [][]int) int {
    n := len(triangle)
    if n == 0 {
        return 0
    }
    // dp 初始化为最后一行
    dp := make([]int, n)
    for j := 0; j < n; j++ {
        dp[j] = triangle[n-1][j]
    }
    // 自底向上滚动更新
    for i := n - 2; i >= 0; i-- {
        for j := 0; j <= i; j++ {
            dp[j] = triangle[i][j] + min(dp[j], dp[j+1])
        }
    }
    return dp[0]
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

### 解法三：自顶向下动态规划（记忆化搜索）
```go
func minimumTotal(triangle [][]int) int {
    n := len(triangle)
    if n == 0 {
        return 0
    }
    // 初始化记忆化数组
    memo := make([][]int, n)
    for i := range memo {
        memo[i] = make([]int, len(triangle[i]))
        for j := range memo[i] {
            memo[i][j] = -1 // 标记未计算
        }
    }
    var dfs func(i, j int) int
    dfs = func(i, j int) int {
        if i == n-1 {
            return triangle[i][j]
        }
        if memo[i][j] != -1 {
            return memo[i][j]
        }
        left := dfs(i+1, j)
        right := dfs(i+1, j+1)
        memo[i][j] = triangle[i][j] + min(left, right)
        return memo[i][j]
    }
    return dfs(0, 0)
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

## 4️⃣ 复杂度分析

### 解法一：二维 DP
- ⏱️ **时间复杂度**：O(n²)  
  需要遍历三角形中的每个元素一次，总元素数为 1+2+...+n = n(n+1)/2 ≈ O(n²)
- 💾 **空间复杂度**：O(n²)  
  需要额外的二维数组存储 dp 状态

### 解法二：一维 DP
- ⏱️ **时间复杂度**：O(n²)  
  同样需要遍历每个元素
- 💾 **空间复杂度**：O(n)  
  只使用一维数组进行滚动更新，满足进阶要求

### 解法三：记忆化搜索
- ⏱️ **时间复杂度**：O(n²)  
  每个状态计算一次，共 O(n²) 个状态
- 💾 **空间复杂度**：O(n²)  
  需要存储记忆化数组，递归栈深度 O(n)

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 二维 DP | 思路直观，易于理解 | 空间复杂度高 | 教学、小规模数据 |
| 一维 DP | 空间最优，满足进阶要求 | 状态转移稍抽象 | 实际应用、大规模数据 |
| 记忆化搜索 | 自顶向下，符合问题描述 | 递归开销，空间略大 | 递归思维、状态转移复杂时 |

**推荐解法**：**解法二（一维 DP）**  
- 空间复杂度 O(n) 满足进阶要求
- 代码简洁，效率高
- 实际工程中最常用
