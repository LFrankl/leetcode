## 题目描述

<p>假设 力扣（LeetCode）即将开始 <strong>IPO</strong> 。为了以更高的价格将股票卖给风险投资公司，力扣 希望在 IPO 之前开展一些项目以增加其资本。 由于资源有限，它只能在 IPO 之前完成最多 <code>k</code> 个不同的项目。帮助 力扣 设计完成最多 <code>k</code> 个不同项目后得到最大总资本的方式。</p>

<p>给你 <code>n</code> 个项目。对于每个项目 <code>i</code><strong> </strong>，它都有一个纯利润 <code>profits[i]</code> ，和启动该项目需要的最小资本 <code>capital[i]</code> 。</p>

<p>最初，你的资本为 <code>w</code> 。当你完成一个项目时，你将获得纯利润，且利润将被添加到你的总资本中。</p>

<p>总而言之，从给定项目中选择 <strong>最多</strong> <code>k</code> 个不同项目的列表，以 <strong>最大化最终资本</strong> ，并输出最终可获得的最多资本。</p>

<p>答案保证在 32 位有符号整数范围内。</p>

<p>&nbsp;</p>

<p><strong>示例 1：</strong></p>

<pre>
<strong>输入：</strong>k = 2, w = 0, profits = [1,2,3], capital = [0,1,1]
<strong>输出：</strong>4
<strong>解释：
</strong>由于你的初始资本为 0，你仅可以从 0 号项目开始。
在完成后，你将获得 1 的利润，你的总资本将变为 1。
此时你可以选择开始 1 号或 2 号项目。
由于你最多可以选择两个项目，所以你需要完成 2 号项目以获得最大的资本。
因此，输出最后最大化的资本，为 0 + 1 + 3 = 4。
</pre>

</pre>

## IPO（LeetCode 502）解题分析

### 简要思路

这是一个**贪心+优先队列**问题。每次选择项目时，应该在当前资本能承担的所有项目中，选择利润最大的那个——因为先赚更多钱，后续能解锁更多项目，贪心策略最优。用最小资本排序项目，用最大堆维护当前可承担项目的利润，每轮从堆顶取最大利润即可。

---

### 解法一：贪心 + 最大堆（最优解法）

思路：将所有项目按 `capital` 升序排序，维护一个指针扫描可承担项目。维护一个最大堆存放当前资本 `w` 能承担的所有项目的利润。每轮从堆顶取出最大利润加入资本，重复 k 次或直到堆空。

```go
import "sort"

type Project struct {
    capital int
    profit  int
}

func findMaximizedCapital(k int, w int, profits []int, capital []int) int {
    n := len(profits)
    projects := make([]Project, n)
    for i := 0; i < n; i++ {
        projects[i] = Project{capital[i], profits[i]}
    }
    // 按 capital 升序排序
    sort.Slice(projects, func(i, j int) bool {
        return projects[i].capital < projects[j].capital
    })

    // 最大堆（用数组模拟，profit 越大越优先）
    heap := make([]int, 0)

    // 堆操作
    push := func(val int) {
        heap = append(heap, val)
        i := len(heap) - 1
        for i > 0 {
            parent := (i - 1) / 2
            if heap[parent] >= heap[i] {
                break
            }
            heap[parent], heap[i] = heap[i], heap[parent]
            i = parent
        }
    }
    pop := func() int {
        top := heap[0]
        heap[0] = heap[len(heap)-1]
        heap = heap[:len(heap)-1]
        i := 0
        for {
            left := 2*i + 1
            right := 2*i + 2
            largest := i
            if left < len(heap) && heap[left] > heap[largest] {
                largest = left
            }
            if right < len(heap) && heap[right] > heap[largest] {
                largest = right
            }
            if largest == i {
                break
            }
            heap[i], heap[largest] = heap[largest], heap[i]
            i = largest
        }
        return top
    }

    idx := 0
    for i := 0; i < k; i++ {
        // 将所有当前资本能承担的项目利润加入堆
        for idx < n && projects[idx].capital <= w {
            push(projects[idx].profit)
            idx++
        }
        // 堆空则无法继续
        if len(heap) == 0 {
            break
        }
        // 选择利润最大的项目
        w += pop()
    }
    return w
}
```

**复杂度：时间 O(nlogn + klogn)**，排序 O(nlogn)，每轮堆操作 O(logn)，最多 k 轮；**空间 O(n)**，堆最多存 n 个元素。

### 解法二：双数组排序 + 贪心（简洁写法，思路相同）

思路：与解法一核心逻辑一致，只是利用 Go 的 `container/heap` 包来简化最大堆的实现，代码更整洁。

```go
import (
    "container/heap"
    "sort"
)

type Project struct {
    capital int
    profit  int
}

// 最大堆
type MaxHeap []int

func (h MaxHeap) Len() int           { return len(h) }
func (h MaxHeap) Less(i, j int) bool { return h[i] > h[j] } // 最大堆
func (h MaxHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *MaxHeap) Push(x any)        { *h = append(*h, x.(int)) }
func (h *MaxHeap) Pop() any {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

func findMaximizedCapital(k int, w int, profits []int, capital []int) int {
    n := len(profits)
    projects := make([]Project, n)
    for i := 0; i < n; i++ {
        projects[i] = Project{capital[i], profits[i]}
    }
    sort.Slice(projects, func(i, j int) bool {
        return projects[i].capital < projects[j].capital
    })

    maxHeap := &MaxHeap{}
    heap.Init(maxHeap)

    idx := 0
    for i := 0; i < k; i++ {
        for idx < n && projects[idx].capital <= w {
            heap.Push(maxHeap, projects[idx].profit)
            idx++
        }
        if maxHeap.Len() == 0 {
            break
        }
        w += heap.Pop(maxHeap).(int)
    }
    return w
}
```

**复杂度：时间 O(nlogn + klogn)**，**空间 O(n)**。

### 关要点总结

| 要点 | 说明 |
|------|------|
| 贪心正确性 | 每步选最大利润，既增加资本又开启更多选项，不存在"放弃当前最大利润而选更小利润能获得更好结果"的情况 |
| 堆的作用 | 动态维护"当前可承担项目"的利润最大值，避免每轮重新遍历 |
| 排序+指针 | 按 capital 排序后用指针推进，每个项目只入堆一次，保证总效率 |
| 提前终止 | 堆空时说明无项目可做，直接 break |
