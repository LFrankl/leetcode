## AI 解答 (DeepSeek)

## 1️⃣ 题意理解

**核心要求**：
- 给定 `n` 种单位（编号 0 到 n-1）
- 给定 `n-1` 条转换关系 `[source, target, factor]`，表示 1 个 `source` 单位 = `factor` 个 `target` 单位
- 需要计算每个单位 `i` 相对于单位 0 的转换系数，即 1 个 0 类型单位 = ? 个 i 类型单位
- 结果需要对 10^9+7 取模

**关键约束**：
- 保证单位 0 可以通过**唯一**的转换路径转换为任何其他单位（不需要反向转换）
- 这意味着转换关系构成一棵以 0 为根的有向树
- 转换因子可能很大（最大 10^9），需要取模运算

**输入输出**：
- 输入：`conversions` 数组，长度 n-1
- 输出：长度为 n 的数组，`result[i]` 表示 1 个 0 单位 = ? 个 i 单位

## 2️⃣ 解题思路

### 解法一：DFS/BFS 遍历树
**核心思想**：
- 将转换关系看作有向边，构成一棵以 0 为根的有向树
- 从根节点 0 开始，DFS 或 BFS 遍历整棵树
- 对于每条边 `u → v`，如果已知 `1个0 = x个u`，那么 `1个0 = x * factor个v`
- 使用递归或栈/队列实现遍历

**解题步骤**：
1. 构建邻接表（因为是有向树，每个节点只有一个父节点，但可以有多个子节点）
2. 从节点 0 开始 DFS/BFS
3. 对于当前节点 u，遍历其所有子节点 v
4. 计算 `result[v] = result[u] * factor % MOD`
5. 递归/迭代处理子节点

**注意事项**：
- 需要取模，避免溢出
- 转换因子可能很大，乘法前先取模

### 解法二：并查集+路径压缩（不适用）
- 由于题目保证是树结构且有唯一路径，并查集不是最优解
- 但可以用于验证连通性

### 解法三：拓扑排序
- 由于是树结构，从根节点开始层次遍历即可
- 类似于 BFS 解法

## 3️⃣ 代码实现

### 解法一：DFS 递归
```go
func baseUnitConversion(conversions [][]int) []int {
    n := len(conversions) + 1
    MOD := int(1e9 + 7)
    
    // 构建邻接表：graph[u] = []{v, factor}
    graph := make([][][2]int, n)
    for _, conv := range conversions {
        u, v, factor := conv[0], conv[1], conv[2]
        graph[u] = append(graph[u], [2]int{v, factor})
    }
    
    result := make([]int, n)
    result[0] = 1 // 1个0单位 = 1个0单位
    
    // DFS 遍历
    var dfs func(int)
    dfs = func(u int) {
        for _, edge := range graph[u] {
            v, factor := edge[0], edge[1]
            // 计算转换系数：result[v] = result[u] * factor
            result[v] = (result[u] % MOD) * (factor % MOD) % MOD
            dfs(v)
        }
    }
    
    dfs(0)
    return result
}
```

### 解法二：BFS 迭代
```go
func baseUnitConversion(conversions [][]int) []int {
    n := len(conversions) + 1
    MOD := int(1e9 + 7)
    
    // 构建邻接表
    graph := make([][][2]int, n)
    for _, conv := range conversions {
        u, v, factor := conv[0], conv[1], conv[2]
        graph[u] = append(graph[u], [2]int{v, factor})
    }
    
    result := make([]int, n)
    result[0] = 1
    
    // BFS 队列
    queue := []int{0}
    
    for len(queue) > 0 {
        u := queue[0]
        queue = queue[1:]
        
        for _, edge := range graph[u] {
            v, factor := edge[0], edge[1]
            // 计算转换系数
            result[v] = (result[u] % MOD) * (factor % MOD) % MOD
            queue = append(queue, v)
        }
    }
    
    return result
}
```

## 4️⃣ 复杂度分析

### 解法一：DFS
- ⏱️ **时间复杂度**：O(n)
  - 每个节点访问一次，每条边访问一次
  - 总共 n 个节点，n-1 条边
- 💾 **空间复杂度**：O(n)
  - 邻接表存储 O(n)
  - 递归栈深度最坏 O(n)（链状树）

### 解法二：BFS
- ⏱️ **时间复杂度**：O(n)
  - 每个节点入队一次，每条边处理一次
- 💾 **空间复杂度**：O(n)
  - 邻接表 O(n)
  - 队列最坏 O(n)

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| DFS 递归 | 代码简洁，直观 | 递归深度可能过大（n最大10^5） | 树深度不大时 |
| BFS 迭代 | 无递归栈溢出风险 | 需要额外队列空间 | 通用，推荐使用 |

**推荐使用 BFS 解法**：
- 题目中 n 最大 10^5，递归可能导致栈溢出
- BFS 更稳定，适合大规模数据
- 时间复杂度相同，空间复杂度相近
