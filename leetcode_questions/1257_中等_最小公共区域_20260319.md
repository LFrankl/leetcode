# 最小公共区域 - 解题分析

## 1️⃣ 题意理解

### 题目重述
给定一个区域列表 `regions`，每个区域包含一个主区域和若干子区域。同时给定两个区域名 `region1` 和 `region2`，要求找到包含这两个区域的最小公共区域。

### 关键约束
1. 区域关系构成一棵树（有向无环图），每个区域最多有一个父区域
2. 区域名是唯一的字符串
3. 第一个区域是每个列表的主区域（父区域）
4. 保证 `region1` 和 `region2` 都在区域关系中
5. 保证存在公共区域

### 输入输出格式
**输入**：
- `regions`: `[][]string`，每个子数组第一个元素是主区域，后面是子区域
- `region1`: `string`，要查找的第一个区域
- `region2`: `string`，要查找的第二个区域

**输出**：
- `string`：最小公共区域的名称

## 2️⃣ 解题思路

### 解法一：哈希表 + 路径记录
**核心思想**：
1. 构建从子区域到父区域的映射关系
2. 分别找到 `region1` 和 `region2` 到根节点的路径
3. 从根节点开始比较两条路径，找到最后一个相同的节点

**解题步骤**：
1. 遍历 `regions`，为每个子区域建立到父区域的映射
2. 从 `region1` 开始向上遍历，记录到根节点的路径
3. 从 `region2` 开始向上遍历，记录到根节点的路径
4. 从路径末尾（根节点）开始向前比较，找到第一个不同的节点
5. 返回该节点的父节点（即最小公共区域）

**注意事项**：
- 路径记录时注意顺序（从当前节点到根节点）
- 比较路径时要从根节点开始比较
- 注意处理两个区域相同的情况

### 解法二：哈希表 + 集合查找
**核心思想**：
1. 构建从子区域到父区域的映射
2. 先遍历 `region1` 的所有祖先节点并存入集合
3. 再遍历 `region2` 的祖先节点，第一个在集合中出现的节点就是最小公共区域

**解题步骤**：
1. 构建父节点映射
2. 遍历 `region1` 的所有祖先节点，存入哈希集合
3. 遍历 `region2` 的祖先节点，检查是否在集合中
4. 第一个在集合中出现的节点即为答案

**注意事项**：
- 集合查找比路径比较更直观
- 注意遍历顺序，第一个找到的就是最小公共区域

## 3️⃣ 代码实现

### 解法一：路径记录法
```go
func findSmallestRegion(regions [][]string, region1 string, region2 string) string {
    // 构建子区域到父区域的映射
    parent := make(map[string]string)
    for _, regionList := range regions {
        mainRegion := regionList[0]
        for i := 1; i < len(regionList); i++ {
            parent[regionList[i]] = mainRegion
        }
    }
    
    // 获取 region1 到根节点的路径
    path1 := make([]string, 0)
    cur := region1
    for cur != "" {
        path1 = append(path1, cur)
        cur = parent[cur]
    }
    
    // 获取 region2 到根节点的路径
    path2 := make([]string, 0)
    cur = region2
    for cur != "" {
        path2 = append(path2, cur)
        cur = parent[cur]
    }
    
    // 从路径末尾（根节点）开始向前比较
    i, j := len(path1)-1, len(path2)-1
    for i >= 0 && j >= 0 && path1[i] == path2[j] {
        i--
        j--
    }
    
    // 返回最后一个相同的节点
    return path1[i+1]
}
```

### 解法二：集合查找法
```go
func findSmallestRegion(regions [][]string, region1 string, region2 string) string {
    // 构建子区域到父区域的映射
    parent := make(map[string]string)
    for _, regionList := range regions {
        mainRegion := regionList[0]
        for i := 1; i < len(regionList); i++ {
            parent[regionList[i]] = mainRegion
        }
    }
    
    // 记录 region1 的所有祖先节点
    ancestors := make(map[string]bool)
    cur := region1
    for cur != "" {
        ancestors[cur] = true
        cur = parent[cur]
    }
    
    // 查找 region2 的祖先节点中第一个在 ancestors 中的节点
    cur = region2
    for cur != "" {
        if ancestors[cur] {
            return cur
        }
        cur = parent[cur]
    }
    
    return "" // 根据题目保证，不会执行到这里
}
```

## 4️⃣ 复杂度分析

### 解法一：路径记录法
- ⏱️ **时间复杂度**：O(N + H)
  - N: 区域总数，用于构建映射 O(N)
  - H: 树的高度，最坏情况 O(N)，用于遍历路径 O(H)
  - 总复杂度 O(N)

- 💾 **空间复杂度**：O(N)
  - 父节点映射需要 O(N) 空间
  - 两条路径最多需要 O(H) 空间，最坏情况 O(N)

### 解法二：集合查找法
- ⏱️ **时间复杂度**：O(N + H)
  - N: 区域总数，用于构建映射 O(N)
  - H: 树的高度，最坏情况 O(N)，用于遍历祖先节点 O(H)
  - 总复杂度 O(N)

- 💾 **空间复杂度**：O(N)
  - 父节点映射需要 O(N) 空间
  - 祖先集合最多需要 O(H) 空间，最坏情况 O(N)

## 5️⃣ 解法对比

| 特性 | 解法一（路径记录） | 解法二（集合查找） |
|------|-------------------|-------------------|
| 思路复杂度 | 中等，需要理解路径比较 | 简单直观 |
| 代码实现 | 稍复杂，需要处理路径比较 | 简洁明了 |
| 空间使用 | 需要存储两条完整路径 | 只需要存储一条路径的集合 |
| 时间复杂度 | 相同 O(N) | 相同 O(N) |
| 适用场景 | 适合需要完整路径信息的场景 | 适合只需要找到公共节点的场景 |

**推荐解法**：解法二（集合查找法）
- 思路更直观，代码更简洁
- 实际运行效率相当
- 更容易理解和维护
