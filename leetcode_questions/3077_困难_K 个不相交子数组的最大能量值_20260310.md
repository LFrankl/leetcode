# 3077. K 个不相交子数组的最大能量值

**难度**: Hard

## 题目描述

<p>给你一个长度为 <code>n</code>&nbsp;下标从 <strong>0</strong>&nbsp;开始的整数数组&nbsp;<code>nums</code>&nbsp;和一个 <strong>正奇数</strong>&nbsp;整数&nbsp;<code>k</code>&nbsp;。</p>

<p><code>x</code> 个子数组的能量值定义为&nbsp;<code>strength = sum[1] * x - sum[2] * (x - 1) + sum[3] * (x - 2) - sum[4] * (x - 3) + ... + sum[x] * 1</code> ，其中&nbsp;<code>sum[i]</code>&nbsp;是第 <code>i</code>&nbsp;个子数组的和。更正式的，能量值是满足&nbsp;<code>1 &lt;= i &lt;= x</code>&nbsp;的所有&nbsp;<code>i</code>&nbsp;对应的&nbsp;<code>(-1)<sup>i+1</sup> * sum[i] * (x - i + 1)</code>&nbsp;之和。</p>

<p>你需要在 <code>nums</code>&nbsp;中选择 <code>k</code>&nbsp;个 <strong>不相交</strong><strong>子数组</strong>&nbsp;，使得&nbsp;<strong>能量值最大</strong>&nbsp;。</p>

<p>请你返回可以得到的 <strong>最大</strong><strong>能量值</strong>&nbsp;。</p>

<p><strong>注意</strong>，选出来的所有子数组&nbsp;<strong>不</strong>&nbsp;需要覆盖整个数组。</p>

<p>&nbsp;</p>

<p><strong class="example">示例 1：</strong></p>

<pre>
<b>输入：</b>nums = [1,2,3,-1,2], k = 3
<b>输出：</b>22
<b>解释：</b>选择 3 个子数组的最好方式是选择：nums[0..2] ，nums[3..3] 和 nums[4..4] 。能量值为 (1 + 2 + 3) * 3 - (-1) * 2 + 2 * 1 = 22 。
</pre>

<p><strong class="example">示例 2：</strong></p>

<pre>
<b>输入：</b>nums = [12,-2,-2,-2,-2], k = 5
<b>输出：</b>64
<b>解释：</b>唯一一种选 5 个不相交子数组的方案是：nums[0..0] ，nums[1..1] ，nums[2..2] ，nums[3..3] 和 nums[4..4] 。能量值为 12 * 5 - (-2) * 4 + (-2) * 3 - (-2) * 2 + (-2) * 1 = 64 。
</pre>

<p><strong class="example">示例 3：</strong></p>

<pre>
<b>输入：</b>nums = [-1,-2,-3], k = 1
<b>输出：</b>-1
<b>解释：</b>选择 1 个子数组的最优方案是：nums[0..0] 。能量值为 -1 。
</pre>

<p>&nbsp;</p>

	<li><code>1 &lt;= n &lt;= 10<sup>4</sup></code></li>
	<li><code>-10<sup>9</sup> &lt;= nums[i] &lt;= 10<sup>9</sup></code></li>

	<li><code>1 &lt;= n * k &lt;= 10<sup>6</sup></code></li>

## 提示

1. Let <code>dp[i][j][x == 0/1]</code> be the maximum strength to select <code>j</code> disjoint subarrays from the original array’s suffix (<code>nums[i..(n - 1)]</code>), x denotes whether we select the element or not.
2. Initially <code>dp[n][0][0] == 0</code>.
3. We have 
<code>dp[i][j][1] = nums[i] * get(j) + max(dp[i + 1][j - 1][0], dp[i + 1][j][1])</code> where <code>get(j) = j</code> if <code>j</code> is odd, otherwise <code>-j</code>.
4. We can select <code>nums[i]</code> as a separate subarray or select at least <code>nums[i]</code> and <code>nums[i + 1]</code> as the first subarray.
<code>dp[i][j][0] = max(dp[i + 1][j][0], dp[i][j][1])</code>.
5. The answer is <code>dp[0][k][0]</code>.

## 1️⃣ 题意理解

**核心要求**：
- 给定一个整数数组 `nums` 和一个正奇数 `k`
- 需要选择 **k 个不相交子数组**（子数组之间不重叠，且不需要覆盖整个数组）
- 定义这 k 个子数组的 **能量值** 为：
  ```
  strength = sum[1] * k - sum[2] * (k-1) + sum[3] * (k-2) - sum[4] * (k-3) + ... + sum[k] * 1
  ```
  即第 i 个子数组的贡献为 `(-1)^(i+1) * sum[i] * (k - i + 1)`
- 目标是 **最大化能量值**

**关键约束**：
- `k` 是正奇数，因此能量值公式中符号交替：奇数位置子数组贡献为正，偶数位置贡献为负
- 子数组不相交，但可以不连续
- `n * k ≤ 10^6`，这意味着我们可以使用 O(nk) 的算法
- 数组元素可为负数

**边界情况**：
- `k = 1` 时，能量值就是子数组和的最大值（乘以 1）
- 所有元素为负数时，需要选择单个最小负数子数组
- `k = n` 时，每个子数组只能包含一个元素

**输入输出**：
- 输入：`nums []int`, `k int`
- 输出：最大能量值 `int64`

## 2️⃣ 解题思路

### 解法一：动态规划（三维 DP）
**核心思想**：
- 定义 `dp[i][j][0/1]`：
  - `i`：考虑前 i 个元素（0-indexed）
  - `j`：已经选择了 j 个子数组
  - `0/1`：当前是否正在构建一个子数组（0=未在构建，1=正在构建）
- 由于能量值公式中有符号交替，需要记录当前是第几个子数组（奇偶性）
- 状态转移需要考虑：
  1. 不选当前元素
  2. 开始新的子数组
  3. 扩展当前子数组
  4. 结束当前子数组

**解题步骤**：
1. 初始化三维 DP 数组，大小为 `(n+1) × (k+1) × 2`
2. 初始状态：`dp[0][0][0] = 0`，其他为负无穷（表示不可达）
3. 遍历每个位置 i，对于每个已选子数组数 j：
   - 如果不在构建子数组（状态 0）：
     - 不选当前元素：`dp[i+1][j][0] = max(dp[i+1][j][0], dp[i][j][0])`
     - 开始新子数组（如果 j < k）：`dp[i+1][j+1][1] = max(dp[i+1][j+1][1], dp[i][j][0] + sign * (k-j) * nums[i])`
   - 如果在构建子数组（状态 1）：
     - 扩展当前子数组：`dp[i+1][j][1] = max(dp[i+1][j][1], dp[i][j][1] + sign * (k-j) * nums[i])`
     - 结束当前子数组：`dp[i+1][j][0] = max(dp[i+1][j][0], dp[i][j][1])`
4. 符号 sign 根据当前是第几个子数组决定：`sign = 1` 当 j 为奇数，`sign = -1` 当 j 为偶数
5. 最终答案为 `max(dp[n][k][0], dp[n][k][1])`

**注意事项**：
- 需要处理整数溢出，使用 int64
- 初始负无穷要足够小（如 -1e18）
- 由于 k 是奇数，最后一个子数组贡献为正

### 解法二：优化空间复杂度的 DP
**核心思想**：
- 观察发现 dp 只依赖前一行，可以滚动数组优化空间
- 进一步优化：由于每个位置要么不选，要么属于某个子数组，可以简化状态定义
- 定义 `dp[j][t]`：已选 j 个子数组，t=0/1 表示是否以当前位置结束一个子数组

**解题步骤**：
1. 初始化二维 DP 数组 `dp[k+1][2]`，`dp[0][0] = 0`，其他为负无穷
2. 遍历每个元素 nums[i]：
   - 创建新的 cur 数组用于更新
   - 对于每个 j 从 0 到 min(k, i+1)：
     - 计算符号 sign = 1 如果 j 为奇数，否则 -1
     - 系数 coeff = (k - j)  // 注意：这里 j 是已选数量
     - 状态转移：
       - 不选当前元素：`cur[j][0] = max(dp[j][0], dp[j][1])`
       - 开始/扩展子数组：`cur[j][1] = max(dp[j][1] + sign*coeff*nums[i], dp[j-1][0] + sign*coeff*nums[i])`
3. 更新 dp = cur
4. 最终答案为 `max(dp[k][0], dp[k][1])`

**优化点**：
- 空间复杂度从 O(nk) 降为 O(k)
- 注意 j 的遍历顺序，避免覆盖

## 3️⃣ 代码实现

### 解法一：三维 DP（基础版）
```go
func maximumStrength(nums []int, k int) int64 {
    n := len(nums)
    // dp[i][j][t]: 前i个元素，选了j个子数组，t=0未在构建，t=1正在构建
    dp := make([][][]int64, n+1)
    for i := range dp {
        dp[i] = make([][]int64, k+1)
        for j := range dp[i] {
            dp[i][j] = []int64{-1e18, -1e18}
        }
    }
    dp[0][0][0] = 0
    
    for i := 0; i < n; i++ {
        for j := 0; j <= k; j++ {
            // 当前符号：第j+1个子数组的符号（j从0开始）
            // 注意：当我们在构建第j+1个子数组时，它的符号由j的奇偶决定
            // j为偶数（0-based）对应第一个子数组，符号为正
            var sign int64 = 1
            if j%2 == 1 { // 偶数位置子数组（1-based）符号为负
                sign = -1
            }
            coeff := int64(k - j) // 系数
            
            // 状态0：不在构建子数组
            if dp[i][j][0] > -1e17 {
                // 不选当前元素
                dp[i+1][j][0] = max(dp[i+1][j][0], dp[i][j][0])
                // 开始新的子数组（如果还可以选）
                if j < k {
                    dp[i+1][j+1][1] = max(dp[i+1][j+1][1], dp[i][j][0] + sign*coeff*int64(nums[i]))
                }
            }
            
            // 状态1：正在构建子数组
            if dp[i][j][1] > -1e17 {
                // 扩展当前子数组
                dp[i+1][j][1] = max(dp[i+1][j][1], dp[i][j][1] + sign*coeff*int64(nums[i]))
                // 结束当前子数组
                dp[i+1][j][0] = max(dp[i+1][j][0], dp[i][j][1])
            }
        }
    }
    
    return max(dp[n][k][0], dp[n][k][1])
}

func max(a, b int64) int64 {
    if a > b {
        return a
    }
    return b
}
```

### 解法二：优化空间 DP（推荐）
```go
func maximumStrength(nums []int, k int) int64 {
    n := len(nums)
    // dp[j][t]: 已选j个子数组，t=0表示不以当前元素结束，t=1表示以当前元素结束（或正在构建）
    dp := make([][2]int64, k+1)
    for j := 0; j <= k; j++ {
        dp[j][0] = -1e18
        dp[j][1] = -1e18
    }
    dp[0][0] = 0
    
    for i := 0; i < n; i++ {
        cur := make([][2]int64, k+1)
        for j := 0; j <= k; j++ {
            cur[j][0] = -1e18
            cur[j][1] = -1e18
        }
        
        for j := 0; j <= k && j <= i+1; j++ {
            // 计算符号和系数
            var sign int64 = 1
            if j%2 == 1 { // 注意：j是已选数量，第j个子数组的符号
                sign = -1
            }
            coeff := int64(k - j)
            
            // 状态转移
            // 1. 不选当前元素（结束或不开始）
            if j <= i {
                cur[j][0] = max(cur[j][0], max(dp[j][0], dp[j][1]))
            }
            
            // 2. 选择当前元素
            // 2.1 作为新子数组的开始
            if j > 0 {
                cur[j][1] = max(cur[j][1], dp[j-1][0] + sign*coeff*int64(nums[i]))
            }
            // 2.2 扩展当前子数组
            cur[j][1] = max(cur[j][1], dp[j][1] + sign*coeff*int64(nums[i]))
        }
        
        dp = cur
    }
    
    return max(dp[k][0], dp[k][1])
}
```

## 4️⃣ 复杂度分析

### 解法一：三维 DP
- ⏱️ **时间复杂度**：O(nk)
  - 三层循环：外层 n，中层 k，内层常数
  - 总操作数约 2nk
- 💾 **空间复杂度**：O(nk)
  - dp 数组大小 (n+1)×(k+1)×2
  - 当 n=10^4, k=100 时，需要约 2×10^6 个 int64，约 16MB

### 解法二：优化空间 DP
- ⏱️ **时间复杂度**：O(nk)
  - 两层循环：外层 n，内层 min(k, i+1)
  - 总操作数约 nk
- 💾 **空间复杂度**：O(k)
  - 只维护两行 dp 数组，大小 2×(k+1)×2
  - 当 k=100 时，仅需约 400 个 int64，约 3.2KB

## 5️⃣ 解法对比

| 特性 | 解法一（三维DP） | 解法二（优化空间DP） |
|------|----------------|-------------------|
| 时间复杂度 | O(nk) | O(nk) |
| 空间复杂度 | O(nk) | O(k) |
| 代码复杂度 | 较高，状态转移复杂 | 中等，需要仔细处理状态 |
| 可读性 | 较好，状态定义清晰 | 一般，状态压缩后较难理解 |
| 适用场景 | n×k ≤ 10^6 时可用 | 推荐，满足题目约束 |

**推荐解法二的原因**：
1. 题目约束 `n*k ≤ 10^6`，但 n 最大可达 10^4，如果 k 较大，解法一可能内存超限
2. 解法二空间复杂度更低，更安全
3. 实际运行效率更高（缓存友好）
