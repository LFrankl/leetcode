## 题目描述

<p>你有 <code>n</code> 枚花的种子。每枚种子必须先种下，才能开始生长、开花。播种需要时间，种子的生长也是如此。给你两个下标从 <strong>0</strong> 开始的整数数组 <code>plantTime</code> 和 <code>growTime</code> ，每个数组的长度都是 <code>n</code> ：</p>

<ul>
	<li><code>plantTime[i]</code> 是 <strong>播种</strong> 第 <code>i</code> 枚种子所需的 <strong>完整天数</strong> 。每天，你只能为播种某一枚种子而劳作。<strong>无须</strong> 连续几天都在种同一枚种子，但是种子播种必须在你工作的天数达到 <code>plantTime[i]</code> 之后才算完成。</li>
	<li><code>growTime[i]</code> 是第 <code>i</code> 枚种子完全种下后生长所需的 <strong>完整天数 </strong>。在它生长的最后一天 <strong>之后</strong> ，将会开花并且永远 <strong>绽放</strong> 。</li>
</ul>

<p>从第 <code>0</code> 开始，你可以按 <strong>任意</strong> 顺序播种种子。</p>

<p>返回所有种子都开花的 <strong>最早</strong> 一天是第几天。</p>

## AI 解答 (DeepSeek)

### 解法一：贪心排序

思路：总播种时间是固定的（所有 plantTime 的总和），但播种顺序影响开花时间。关键观察：生长时间长的种子应该先种，因为生长时间长的种子需要更久才能开花，先种可以让它在其他种子播种期间就开始生长，从而减少最终等待时间。按 growTime 降序排序后，依次播种，计算每枚种子的开花时间 = 当前累计播种时间 + 该种子的 growTime，取最大值即为答案。

```go
func earliestFullBloom(plantTime []int, growTime []int) int {
    n := len(plantTime)
    indices := make([]int, n)
    for i := range indices {
        indices[i] = i
    }
    // 按 growTime 降序排序
    sort.Slice(indices, func(i, j int) bool {
        return growTime[indices[i]] > growTime[indices[j]]
    })

    curPlant := 0
    ans := 0
    for _, idx := range indices {
        curPlant += plantTime[idx]
        // 该种子开花时间 = 累计播种完成时间 + 生长时间
        bloomDay := curPlant + growTime[idx]
        if bloomDay > ans {
            ans = bloomDay
        }
    }
    return ans
}
```

复杂度：时间 O(n log n)，空间 O(n)

### 解法二：贪心排序（简化写法）

思路：与解法一相同，但用结构体数组替代索引数组，代码更简洁。

```go
func earliestFullBloom(plantTime []int, growTime []int) int {
    seeds := make([][2]int, len(plantTime))
    for i := range seeds {
        seeds[i] = [2]int{plantTime[i], growTime[i]}
    }
    // 按 growTime 降序排序
    sort.Slice(seeds, func(i, j int) bool {
        return seeds[i][1] > seeds[j][1]
    })

    curPlant, ans := 0, 0
    for _, s := range seeds {
        curPlant += s[0]
        ans = max(ans, curPlant + s[1])
    }
    return ans
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

复杂度：时间 O(n log n)，空间 O(n)

---

**核心证明**：为什么按 growTime 降序贪心最优？

考虑相邻两枚种子 i 和 j（按当前顺序依次播种）。交换它们的顺序时，其他种子的开花时间不变，只有 i 和 j 的开花时间可能变化。

- 原顺序：i 先种，j 后种。i 开花 = `pre + p[i] + g[i]`，j 开花 = `pre + p[i] + p[j] + g[j]`
- 交换后：j 先种，i 后种。j 开花 = `pre + p[j] + g[j]`，i 开花 = `pre + p[j] + p[i] + g[i]`

原顺序的最大值 vs 交换后的最大值中，`pre + p[i] + p[j] + g[j]` 和 `pre + p[j] + p[i] + g[i]` 两项不变，而另外两项取决于 `pre + p[i] + g[i]` vs `pre + p[j] + g[j]`。当 `g[i] ≥ g[j]` 时原顺序更优，因此 growTime 大的应优先播种。这构成交换论证，证明了贪心策略的正确性。
