## 1️⃣ 题意理解

**题目重述**：  
给定平面上 n 个互不相同的点，要求找出由其中四个点构成的、边与坐标轴平行的矩形，满足矩形内部（包括边界）不包含其他任何点，并返回所有可能矩形中的最大面积。若不存在这样的矩形，返回 -1。

**关键约束**：
1. 矩形必须由恰好四个点构成，且这四个点必须是给定点集中的点。
2. 矩形边与坐标轴平行 → 四个点必须满足：两个点有相同的 x 坐标（左竖边），两个点有相同的 x 坐标（右竖边），两个点有相同的 y 坐标（上横边），两个点有相同的 y 坐标（下横边）。
3. 矩形内部或边界上不能有其他点 → 矩形区域内不能有除四个顶点外的任何点。
4. 点坐标范围较大（0 到 8×10^7），但点数量 n ≤ 2×10^5。

**输入输出**：
- 输入：两个整数数组 `xCoord` 和 `yCoord`，长度相同，表示点集。
- 输出：最大矩形面积（整数），若无则返回 -1。

## 2️⃣ 解题思路

### 解法一：枚举对角点 + 哈希验证
**核心思想**：  
由于矩形边与坐标轴平行，其四个顶点可以表示为 `(x1, y1)`, `(x1, y2)`, `(x2, y1)`, `(x2, y2)`。我们可以枚举两个不同的 x 坐标 `x1 < x2` 作为矩形的左右边，然后找出所有同时拥有这两个 x 坐标的点对 `(y1, y2)`（即存在点 `(x1, y1)` 和 `(x2, y1)`，以及 `(x1, y2)` 和 `(x2, y2)`）。对于每一对 `(y1, y2)`（y1 < y2），检查矩形内部是否没有其他点。

**步骤**：
1. 将点按 x 坐标分组，每组内按 y 坐标排序。
2. 枚举所有 x 坐标对 `(x1, x2)`，其中 x1 < x2。
3. 对于每个 x 对，找出它们共有的 y 坐标列表（即同时出现在 x1 组和 x2 组中的 y 值），并排序。
4. 在共有 y 列表中枚举所有 y 对 `(y1, y2)`（y1 < y2），计算面积 `(x2 - x1) * (y2 - y1)`。
5. 检查矩形内部是否没有其他点：
   - 对于 x 在 (x1, x2) 之间的点，检查其 y 是否在 (y1, y2) 之间。
   - 由于点数量多，直接检查会超时，需要优化：使用二维偏序查询，例如用线段树或树状数组维护 y 坐标的区间存在性。
   - 更简单的方法：在枚举 y 对时，只考虑相邻的 y 对（因为如果矩形内部没有点，那么 y1 和 y2 之间不能有其他点，即 y1 和 y2 在共有 y 列表中必须是相邻的）。这样，矩形内部在左右两条竖边上就没有其他点，但还需要检查中间 x 坐标的点是否落在矩形内。

**坑点**：
- 点坐标范围大，需要离散化 y 坐标。
- 检查矩形内部点时，不能只检查左右两条边上的点，还要检查中间 x 坐标的点。
- 枚举所有 x 对可能太多（最坏 O(n^2)），但实际由于点分布稀疏，可以剪枝：如果某个 x 坐标的点数很少，枚举它作为 x1 或 x2 可能更高效。

### 解法二：枚举水平边 + 扫描线
**核心思想**：  
固定一对 y 坐标 `(y1, y2)` 作为矩形的上下边，然后找出所有同时拥有这两个 y 坐标的点对 `(x1, x2)`（即存在点 `(x1, y1)` 和 `(x1, y2)`，以及 `(x2, y1)` 和 `(x2, y2)`）。对于每一对 `(x1, x2)`，检查矩形内部是否没有其他点。

**步骤**：
1. 将点按 y 坐标分组，每组内按 x 坐标排序。
2. 枚举所有 y 坐标对 `(y1, y2)`，其中 y1 < y2。
3. 对于每个 y 对，找出它们共有的 x 坐标列表（即同时出现在 y1 组和 y2 组中的 x 值），并排序。
4. 在共有 x 列表中枚举所有 x 对 `(x1, x2)`（x1 < x2），计算面积。
5. 检查矩形内部是否没有其他点：
   - 对于 y 在 (y1, y2) 之间的点，检查其 x 是否在 (x1, x2) 之间。
   - 同样，如果 x1 和 x2 在共有 x 列表中相邻，则矩形内部在上下两条边上没有其他点，但还需检查中间 y 坐标的点。

**优化**：
- 由于 n ≤ 2×10^5，枚举所有 y 对可能太多，但实际中如果某个 y 坐标的点数很多，那么它的配对可能产生很多矩形，需要高效检查内部点。
- 可以使用扫描线：对于每个 y 区间，维护一个数据结构（如平衡树）来存储当前 y 区间内所有点的 x 坐标，快速查询区间内是否有 x 落在 (x1, x2) 之间。

### 解法三：枚举对角点 + 内部点检查优化（推荐）
**核心思想**：  
结合解法一和解法二，选择点数较少的方向进行枚举。例如，如果 x 坐标的种类数较少，则枚举 x 对；否则枚举 y 对。对于每个候选矩形，检查内部点时，可以预先将点按 x 排序，然后对于每个 x 区间，快速判断 y 区间内是否有其他点。

**步骤**：
1. 统计每个 x 坐标对应的 y 列表，每个 y 坐标对应的 x 列表。
2. 选择坐标种类数较少的方向进行枚举（假设选择 x 方向）。
3. 枚举所有 x 对 `(x1, x2)`，其中 x1 < x2。
4. 找出共有 y 列表，排序。
5. 枚举共有 y 列表中相邻的 y 对 `(y1, y2)`。
6. 检查矩形 `(x1, y1) - (x2, y2)` 内部是否有其他点：
   - 对于 x 在 (x1, x2) 之间的点，检查其 y 是否在 (y1, y2) 之间。
   - 由于点已按 x 排序，可以在所有点中二分查找 x > x1 且 x < x2 的点，然后检查它们的 y 坐标。
   - 如果存在这样的点，则矩形无效。
7. 若矩形有效，更新最大面积。

**注意事项**：
- 需要快速判断某个 x 区间内是否存在 y 在某个区间的点。可以预先对每个 x 坐标的点按 y 排序，然后对于 x 在 (x1, x2) 之间的每个 x，二分查找是否有 y 在 (y1, y2) 之间的点。
- 由于 x 区间可能很多，可以进一步优化：将所有点按 x 排序，然后对于每个 x 对，只检查 x 在 (x1, x2) 之间的点，这些点可以通过二分查找快速定位。

## 3️⃣ 代码实现

### 解法三：枚举对角点 + 内部点检查优化
```go
func maxRectangleArea(xCoord []int, yCoord []int) int {
    n := len(xCoord)
    if n < 4 {
        return -1
    }

    // 构建点集和映射
    points := make([][2]int, n)
    pointSet := make(map[[2]int]bool, n)
    for i := 0; i < n; i++ {
        points[i] = [2]int{xCoord[i], yCoord[i]}
        pointSet[points[i]] = true
    }

    // 按 x 分组，每组内 y 排序
    xMap := make(map[int][]int)
    for _, p := range points {
        x, y := p[0], p[1]
        xMap[x] = append(xMap[x], y)
    }
    for x := range xMap {
        sort.Ints(xMap[x])
    }

    // 获取所有 x 坐标并排序
    xs := make([]int, 0, len(xMap))
    for x := range xMap {
        xs = append(xs, x)
    }
    sort.Ints(xs)

    maxArea := -1

    // 枚举 x 对
    for i := 0; i < len(xs); i++ {
        x1 := xs[i]
        yList1 := xMap[x1]
        for j := i + 1; j < len(xs); j++ {
            x2 := xs[j]
            yList2 := xMap[x2]

            // 找出共有 y 坐标
            commonY := make([]int, 0)
            p1, p2 := 0, 0
            for p1 < len(yList1) && p2 < len(yList2) {
                if yList1[p1] == yList2[p2] {
                    commonY = append(commonY, yList1[p1])
                    p1++
                    p2++
                } else if yList1[p1] < yList2[p2] {
                    p1++
                } else {
                    p2++
                }
            }

            // 枚举相邻的 y 对
            for k := 1; k < len(commonY); k++ {
                y1 := commonY[k-1]
                y2 := commonY[k]

                // 检查矩形内部是否有其他点
                valid := true
                // 只需检查 x 在 (x1, x2) 之间的点是否落在 y 在 (y1, y2) 之间
                for x := x1 + 1; x < x2; x++ {
                    if ys, ok := xMap[x]; ok {
                        // 二分查找是否有 y 在 (y1, y2) 之间的点
                        idx := sort.SearchInts(ys, y1+1)
                        if idx < len(ys) && ys[idx] < y2 {
                            valid = false
                            break
                        }
                    }
                }

                if valid {
                    area := (x2 - x1) * (y2 - y1)
                    if area > maxArea {
                        maxArea = area
                    }
                }
            }
        }
    }

    return maxArea
}
```

## 4️⃣ 复杂度分析

**时间复杂度**：
- 构建映射和排序：O(n log n)
- 枚举 x 对：最坏 O(m^2)，其中 m 是不同 x 坐标的数量（m ≤ n）
- 对于每个 x 对，找出共有 y 列表：O(|yList1| + |yList2|)
- 枚举相邻 y 对：O(|commonY|)
- 检查内部点：对于每个矩形，需要检查 x 在 (x1, x2) 之间的所有 x 坐标，每个 x 坐标二分查找 y 区间。最坏情况下，每个 x 对可能涉及很多 x 坐标，但实际中由于点分布稀疏，检查次数有限。
总体最坏 O(n^2)，但实际运行由于剪枝和稀疏性，可以接受。

**空间复杂度**：
- 存储点集和映射：O(n)
- 存储 x 坐标列表和 y 列表：O(n)

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 枚举对角点 + 哈希验证 | 直观，易于实现 | 内部点检查可能超时，需要优化 | 点分布均匀，x 坐标种类少 |
| 枚举水平边 + 扫描线 | 对称思路，可能更高效 | 实现复杂，需要维护扫描线数据结构 | y 坐标种类少，点分布密集 |
| 枚举对角点 + 内部点检查优化（推荐） | 结合两者优点，选择较少方向枚举，检查优化 | 实现稍复杂，但效率较高 | 通用，适合大多数情况 |

**推荐解法三**：因为它自适应地选择枚举方向，并通过相邻 y 对剪枝和二分查找检查内部点，在时间和空间上都有较好平衡。
