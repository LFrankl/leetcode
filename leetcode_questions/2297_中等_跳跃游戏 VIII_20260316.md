## 提示

1. For each index, find the indexes that you can jump to.
2. We can do this by using a monotonic stack and iterating through the array backwards.
3. Create another integer array of length n called dp where dp[i] is the minimum cost of getting to index i.

## 1️⃣ 题意理解

**题目重述**：
给定一个整数数组 `nums` 和一个整数 `minJump`、`maxJump`。初始位置在索引 0 处。如果满足以下条件，可以从位置 `i` 跳跃到位置 `j`：
1. `i < j`
2. `minJump <= j - i <= maxJump`
3. `nums[j] == 0`

目标是判断是否能从位置 0 到达最后一个位置（`n-1`）。注意：位置 0 和位置 `n-1` 的 `nums` 值保证为 0。

**关键约束**：
- `2 <= nums.length <= 10^5`
- `nums[i]` 为 0 或 1
- `1 <= minJump <= maxJump <= nums.length`
- 只能跳到值为 0 的位置
- 跳跃距离必须在 `[minJump, maxJump]` 范围内

**边界情况**：
- 数组长度为 2 时，只需检查 `minJump` 是否 ≤ 1
- 如果 `nums[0] != 0` 或 `nums[n-1] != 0`，直接返回 false（但题目保证为 0）
- 连续多个位置为 1 可能导致无法到达

**输入输出**：
- 输入：`nums []int`, `minJump int`, `maxJump int`
- 输出：`bool`（是否能到达最后一个位置）

## 2️⃣ 解题思路

### 解法一：BFS + 滑动窗口优化
**核心思想**：将问题转化为图论中的可达性问题。每个位置是节点，合法的跳跃是边。使用 BFS 遍历，但需要优化避免重复访问。

**解题步骤**：
1. 初始化队列，将位置 0 入队
2. 使用 visited 数组记录已访问位置
3. 使用指针 `right` 记录当前已处理的最远右边界，避免重复入队
4. 对于每个出队的位置 `i`，计算可跳范围 `[l, r]`
5. 只处理 `[max(right+1, l), r]` 范围内的位置（避免重复）
6. 将范围内值为 0 的位置入队，更新 `right`
7. 如果遇到 `n-1` 则返回 true

**注意事项**：
- 需要及时更新 `right` 指针，否则会超时
- 位置 0 需要预先检查是否为 0

### 解法二：动态规划 + 前缀和优化
**核心思想**：定义 `dp[i]` 表示是否能到达位置 i。状态转移：`dp[i] = (nums[i]==0) && (dp[i-maxJump]到dp[i-minJump]中有true)`。使用前缀和优化区间查询。

**解题步骤**：
1. 初始化 `dp[0]=true`，`prefix[0]=1`
2. 遍历位置 i 从 1 到 n-1
3. 如果 `nums[i]==1`，则 `dp[i]=false`
4. 否则计算可转移的区间 `[i-maxJump, i-minJump]`
5. 使用前缀和快速判断区间内是否有 true
6. 更新前缀和数组
7. 返回 `dp[n-1]`

**注意事项**：
- 区间边界需要 clamp 到 [0, n-1]
- 前缀和数组大小应为 n+1 方便计算

## 3️⃣ 代码实现

### 解法一：BFS + 滑动窗口优化
```go
func canReach(nums []int, minJump int, maxJump int) bool {
    n := len(nums)
    if nums[0] == 1 || nums[n-1] == 1 {
        return false
    }
    
    queue := make([]int, 0)
    queue = append(queue, 0)
    visited := make([]bool, n)
    visited[0] = true
    
    // 记录已处理的最远右边界
    right := 0
    
    for len(queue) > 0 {
        i := queue[0]
        queue = queue[1:]
        
        // 计算当前可跳范围
        l := i + minJump
        r := min(i+maxJump, n-1)
        
        // 只处理 [max(right+1, l), r] 范围内的位置，避免重复
        start := max(right+1, l)
        for j := start; j <= r; j++ {
            if nums[j] == 0 && !visited[j] {
                if j == n-1 {
                    return true
                }
                visited[j] = true
                queue = append(queue, j)
            }
        }
        // 更新右边界
        right = max(right, r)
    }
    
    return visited[n-1]
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

### 解法二：动态规划 + 前缀和优化
```go
func canReach(nums []int, minJump int, maxJump int) bool {
    n := len(nums)
    if nums[0] == 1 || nums[n-1] == 1 {
        return false
    }
    
    // dp[i] 表示是否能到达位置 i
    dp := make([]bool, n)
    dp[0] = true
    
    // prefix[i] 表示 dp[0] 到 dp[i-1] 中 true 的数量
    prefix := make([]int, n+1)
    prefix[1] = 1 // dp[0] = true
    
    for i := 1; i < n; i++ {
        if nums[i] == 1 {
            dp[i] = false
            prefix[i+1] = prefix[i]
            continue
        }
        
        // 可转移的区间 [i-maxJump, i-minJump]
        l := max(0, i-maxJump)
        r := i - minJump
        
        if r < 0 {
            dp[i] = false
            prefix[i+1] = prefix[i]
            continue
        }
        
        // 使用前缀和判断区间内是否有 true
        // sum = prefix[r+1] - prefix[l]
        sum := prefix[r+1] - prefix[l]
        dp[i] = sum > 0
        
        // 更新前缀和
        if dp[i] {
            prefix[i+1] = prefix[i] + 1
        } else {
            prefix[i+1] = prefix[i]
        }
    }
    
    return dp[n-1]
}
```

## 4️⃣ 复杂度分析

### 解法一：BFS + 滑动窗口优化
- ⏱️ **时间复杂度**：O(n)
  - 每个位置最多入队一次，出队一次
  - 内层循环的 `right` 指针保证每个位置只被处理一次
- 💾 **空间复杂度**：O(n)
  - 队列最多存储 n 个元素
  - visited 数组需要 O(n) 空间

### 解法二：动态规划 + 前缀和优化
- ⏱️ **时间复杂度**：O(n)
  - 遍历数组一次，每次操作 O(1)
  - 前缀和查询是 O(1)
- 💾 **空间复杂度**：O(n)
  - dp 数组 O(n)
  - prefix 数组 O(n)

## 5️⃣ 解法对比

| 解法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| BFS+滑动窗口 | 直观，容易理解 | 需要维护指针，实现稍复杂 | 适合需要实际路径的场景 |
| DP+前缀和 | 代码简洁，效率稳定 | 需要额外空间存储前缀和 | 大规模数据，只需判断可达性 |

**推荐解法**：**解法二（DP+前缀和）**
- 时间复杂度相同，但实现更简洁
- 没有队列操作，常数因子更小
- 更适合本题只需判断是否可达的需求
