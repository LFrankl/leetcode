# 按序打印 - 多线程顺序控制

## 1️⃣ 题意理解

**核心要求**：
- 三个线程 A、B、C 分别调用 `first()`、`second()`、`third()` 方法
- 必须保证执行顺序：`first()` → `second()` → `third()`
- 无论操作系统如何调度线程，都要保证这个顺序

**关键约束**：
- 线程启动顺序不确定（由输入数组 `nums` 决定）
- 需要线程同步机制来保证执行顺序
- 不能简单地在方法内部使用 `sleep` 等不可靠方式

**输入输出**：
- 输入：数组 `nums`，是 `[1,2,3]` 的排列，表示三个线程的调用顺序
- 输出：字符串 "firstsecondthird"

## 2️⃣ 解题思路

### 解法一：信号量/条件变量
**核心思想**：使用两个同步原语（信号量或条件变量）作为"门"，控制线程的执行顺序。

**步骤**：
1. 创建两个同步原语（如信号量、互斥锁+条件变量）
2. `first()` 执行后，释放第一个信号，允许 `second()` 执行
3. `second()` 执行后，释放第二个信号，允许 `third()` 执行

### 解法二：原子变量+自旋锁
**核心思想**：使用原子变量作为计数器，线程通过检查计数器的值来决定是否执行。

**步骤**：
1. 使用原子变量记录当前应该执行哪个方法
2. 每个方法执行前检查计数器值
3. 执行完成后递增计数器

### 解法三：通道（Go特有）
**核心思想**：利用 Go 的通道（channel）作为同步机制，通道的阻塞特性天然适合线程同步。

**步骤**：
1. 创建两个无缓冲通道
2. `first()` 执行后向第一个通道发送信号
3. `second()` 等待第一个通道的信号，执行后向第二个通道发送信号
4. `third()` 等待第二个通道的信号

## 3️⃣ 代码实现

### 解法一：使用 sync.Cond（条件变量）
```go
package main

import (
	"fmt"
	"sync"
)

type Foo struct {
	firstDone  *sync.Cond
	secondDone *sync.Cond
}

func NewFoo() *Foo {
	return &Foo{
		firstDone:  sync.NewCond(&sync.Mutex{}),
		secondDone: sync.NewCond(&sync.Mutex{}),
	}
}

func (f *Foo) first(printFirst func()) {
	// 执行 first
	printFirst()
	
	// 通知 second 可以执行
	f.firstDone.L.Lock()
	f.firstDone.Broadcast()
	f.firstDone.L.Unlock()
}

func (f *Foo) second(printSecond func()) {
	// 等待 first 执行完成
	f.firstDone.L.Lock()
	f.firstDone.Wait()
	f.firstDone.L.Unlock()
	
	// 执行 second
	printSecond()
	
	// 通知 third 可以执行
	f.secondDone.L.Lock()
	f.secondDone.Broadcast()
	f.secondDone.L.Unlock()
}

func (f *Foo) third(printThird func()) {
	// 等待 second 执行完成
	f.secondDone.L.Lock()
	f.secondDone.Wait()
	f.secondDone.L.Unlock()
	
	// 执行 third
	printThird()
}
```

### 解法二：使用原子变量
```go
package main

import (
	"fmt"
	"sync/atomic"
)

type Foo struct {
	step int32 // 原子计数器，0:等待first，1:等待second，2:等待third
}

func NewFoo() *Foo {
	return &Foo{step: 0}
}

func (f *Foo) first(printFirst func()) {
	// 执行 first
	printFirst()
	
	// 将计数器设置为1，允许second执行
	atomic.StoreInt32(&f.step, 1)
}

func (f *Foo) second(printSecond func()) {
	// 自旋等待，直到step变为1
	for atomic.LoadInt32(&f.step) != 1 {
		// 让出CPU，避免忙等待消耗过多资源
		// runtime.Gosched()
	}
	
	// 执行 second
	printSecond()
	
	// 将计数器设置为2，允许third执行
	atomic.StoreInt32(&f.step, 2)
}

func (f *Foo) third(printThird func()) {
	// 自旋等待，直到step变为2
	for atomic.LoadInt32(&f.step) != 2 {
		// runtime.Gosched()
	}
	
	// 执行 third
	printThird()
}
```

### 解法三：使用通道（Go推荐方式）
```go
package main

import (
	"fmt"
)

type Foo struct {
	firstChan  chan struct{} // 无缓冲通道，用于 first -> second 同步
	secondChan chan struct{} // 无缓冲通道，用于 second -> third 同步
}

func NewFoo() *Foo {
	return &Foo{
		firstChan:  make(chan struct{}),
		secondChan: make(chan struct{}),
	}
}

func (f *Foo) first(printFirst func()) {
	// 执行 first
	printFirst()
	
	// 关闭通道，通知 second 可以执行
	// 关闭通道比发送数据更安全，避免重复发送
	close(f.firstChan)
}

func (f *Foo) second(printSecond func()) {
	// 等待 first 执行完成（阻塞直到通道关闭）
	<-f.firstChan
	
	// 执行 second
	printSecond()
	
	// 关闭通道，通知 third 可以执行
	close(f.secondChan)
}

func (f *Foo) third(printThird func()) {
	// 等待 second 执行完成
	<-f.secondChan
	
	// 执行 third
	printThird()
}
```

## 4️⃣ 复杂度分析

### 解法一：sync.Cond
- ⏱️ **时间复杂度**：O(1)
  - 每个方法都是常数时间操作
  - 等待操作由操作系统调度，不消耗CPU时间
- 💾 **空间复杂度**：O(1)
  - 只使用了两个条件变量和对应的互斥锁

### 解法二：原子变量
- ⏱️ **时间复杂度**：O(1) 平均情况
  - 每个方法执行时间固定
  - `second()` 和 `third()` 可能有自旋等待，但等待时间取决于线程调度
- 💾 **空间复杂度**：O(1)
  - 只使用了一个原子变量

### 解法三：通道
- ⏱️ **时间复杂度**：O(1)
  - 通道操作是常数时间
  - 等待时线程被阻塞，不消耗CPU
- 💾 **空间复杂度**：O(1)
  - 创建了两个无缓冲通道

## 5️⃣ 解法对比

| 特性 | sync.Cond | 原子变量 | 通道 |
|------|-----------|----------|------|
| **实现复杂度** | 中等 | 简单 | 简单 |
| **性能** | 高（操作系统级阻塞） | 中等（可能忙等待） | 高（Go运行时优化） |
| **资源消耗** | 低 | 可能高（忙等待时） | 低 |
| **可读性** | 一般 | 较好 | 最好 |
| **Go风格** | 一般 | 一般 | 最符合 |
| **适用场景** | 通用多线程 | 简单同步 | Go语言特有 |

**推荐使用解法三（通道）**：
1. 最符合 Go 语言的并发哲学
2. 代码简洁，易于理解和维护
3. 性能优秀，由 Go 运行时优化
4. 避免忙等待，节省CPU资源

            switch n {

                foo.second(func() { result += "second" })
            case 3:
                foo.third(func() { result += "third" })
            }

    }

    return result
