1606. 找到处理最多请求的服务器

难度: Hard

标签: 数组, 有序集合, 模拟, 堆（优先队列）

链接: https://leetcode.cn/problems/find-servers-that-handled-most-number-of-requests/

题目描述

你有 k 个服务器，编号为 0 到 k-1 ，它们可以同时处理多个请求组。每个服务器有无穷的计算能力但是 不能同时处理超过一个请求 。请求分配到服务器的规则如下：

  • 第 i （序号从 0 开始）个请求到达。
  • 如果所有服务器都已被占据，那么该请求被舍弃（完全不处理）。
  • 如果第 (i % k) 个服务器空闲，那么对应服务器会处理该请求。
  • 否则，将请求安排给下一个空闲的服务器（服务器构成一个环，必要的话可能从第 0 个服务器开始继续找下一个空闲的服务器）。比方说，如果第 i 个服务器在忙，那么会查看第 (i+1) 个服务器，第 (i+2) 个服务器等等。

给你一个 严格递增 的正整数数组 arrival ，表示第 i 个任务的到达时间，和另一个数组 load ，其中 load[i] 表示第 i 个请求的工作量（也就是服务器完成它所需要的时间）。你的任务是找到 最繁忙的服务器 。最繁忙定义为一个服务器处理的请求数是所有服务器里最多的。

请你返回包含所有 最繁忙服务器 序号的列表，你可以以任意顺序返回这个列表。

 

示例 1：

输入：k = 3, arrival = [1,2,3,4,5], load = [5,2,3,3,3] 
输出：[1] 
解释：
所有服务器一开始都是空闲的。
前 3 个请求分别由前 3 台服务器依次处理。
请求 3 进来的时候，服务器 0 被占据，所以它被安排到下一台空闲的服务器，也就是服务器 1 。
请求 4 进来的时候，由于所有服务器都被占据，该请求被舍弃。
服务器 0 和 2 分别都处理了一个请求，服务器 1 处理了两个请求。所以服务器 1 是最忙的服务器。

示例 2：

输入：k = 3, arrival = [1,2,3,4], load = [1,2,1,2]
输出：[0]
解释：
前 3 个请求分别被前 3 个服务器处理。
请求 3 进来，由于服务器 0 空闲，它被服务器 0 处理。
服务器 0 处理了两个请求，服务器 1 和 2 分别处理了一个请求。所以服务器 0 是最忙的服务器。

示例 3：

输入：k = 3, arrival = [1,2,3], load = [10,12,11]
输出：[0,1,2]
解释：每个服务器分别处理了一个请求，所以它们都是最忙的服务器。

示例 4：

输入：k = 3, arrival = [1,2,3,4,8,9,10], load = [5,2,10,3,1,2,2]
输出：[1]

示例 5：

输入：k = 1, arrival = [1], load = [1]
输出：[0]

  • 1 <= arrival.length, load.length <= 105
  • arrival.length == load.length
  • 1 <= arrival[i], load[i] <= 109
  • arrival 保证 严格递增 。

1️⃣ 题意理解

题目重述

我们有 k 个服务器，编号从 0 到 k-1。每个服务器一次只能处理一个请求。
请求按顺序到达，第 i 个请求的到达时间为 arrival[i]，处理时长为 load[i]。
分配规则： 1. 如果所有服务器都忙，请求被丢弃。 2. 否则，从 i % k 号服务器开始，按环状顺序（0→1→...→k-1→0→...）寻找第一个空闲的服务器来处理该请求。

目标：找出处理请求数最多的服务器（可能有多个），返回它们的编号列表。

关键约束

  • arrival 严格递增。
  • 服务器数量 k 可达 10^5。
  • 请求数量 n 可达 10^5。
  • 到达时间和负载可达 10^9。
  • 服务器空闲判断基于请求到达时间：如果某个服务器正在处理请求的完成时间 ≤ 当前请求到达时间，则它空闲。

输入输出

  • 输入：k（服务器数），arrival []int（到达时间），load []int（处理时长）
  • 输出：[]int（最繁忙服务器的编号列表）

2️⃣ 解题思路

解法一：有序集合 + 优先队列（堆）

核心思想： - 用优先队列（最小堆）维护正在忙碌的服务器，堆中元素为 (完成时间, 服务器编号)，按完成时间排序。 - 用有序集合（如红黑树）维护当前空闲的服务器编号，便于快速查找环状顺序中的下一个空闲服务器。 - 遍历每个请求，先将所有完成时间 ≤ 当前到达时间的服务器从堆中弹出，加入空闲集合。 - 如果空闲集合为空，丢弃请求。 - 否则，在空闲集合中查找 ≥ i % k 的第一个服务器，若找不到则取集合中最小编号（环状）。

步骤： 1. 初始化空闲集合（包含所有服务器编号）。 2. 初始化忙碌堆（空）。 3. 初始化计数器 count 记录每个服务器处理的请求数。 4. 遍历每个请求： - 弹出堆中所有完成时间 ≤ 当前到达时间的服务器，加入空闲集合。 - 如果空闲集合为空，跳过该请求。 - 在空闲集合中查找 ≥ i % k 的第一个服务器。 - 若找到，分配请求，更新计数器，将该服务器从空闲集合移除，推入堆中（完成时间 = 到达时间 + 负载）。 - 若未找到，取空闲集合中最小编号的服务器（环状回到开头）。 5. 遍历计数器，找出最大值对应的服务器编号列表。

注意事项： - 空闲集合需要支持有序查找和删除，Go 中可用 redblacktree 或手动用 TreeSet 模拟。 - 环状查找时注意边界情况（找不到时取最小编号）。

解法二：双优先队列（堆） + 懒惰删除

核心思想： - 用两个堆： - busy 堆：(完成时间, 服务器编号)，按完成时间排序。 - free 堆：存储空闲服务器编号，但为了环状查找，需要特殊处理。 - 由于环状查找需要从 i % k 开始找第一个空闲服务器，我们可以将空闲服务器编号分为两部分： - 编号 ≥ i % k 的 - 编号 < i % k 的 - 用两个有序结构（如两个小根堆）分别存储这两部分空闲服务器，查找时先找第一部分，再找第二部分。

步骤： 1. 初始化两个空闲堆：free1（存储 ≥ i % k 的编号），free2（存储 < i % k 的编号）。 2. 遍历请求时，先弹出 busy 堆中所有完成的服务器，根据其编号与当前 i % k 的关系放入 free1 或 free2。 3. 分配时，先尝试从 free1 取最小编号，若 free1 为空则从 free2 取最小编号（环状回绕）。 4. 若两者都空，丢弃请求。

优点： - 避免使用复杂的平衡树，仅用堆实现。 - 懒惰删除：从 busy 堆弹出的服务器直接加入空闲堆，无需额外标记。

3️⃣ 代码实现

解法一：有序集合 + 优先队列（使用 github.com/emirpasic/gods/trees/redblacktree）

import (
    "container/heap"
    "github.com/emirpasic/gods/trees/redblacktree"
)

func busiestServers(k int, arrival []int, load []int) []int {
    n := len(arrival)
    count := make([]int, k)

    // 空闲服务器集合（有序）
    free := redblacktree.NewWithIntComparator()
    for i := 0; i < k; i++ {
        free.Put(i, nil)
    }

    // 忙碌服务器堆（完成时间, 服务器编号）
    busy := &minHeap{}
    heap.Init(busy)

    for i := 0; i < n; i++ {
        start := arrival[i]
        end := start + load[i]

        // 释放已完成请求的服务器
        for busy.Len() > 0 && (*busy)[0].endTime <= start {
            node := heap.Pop(busy).(*server)
            free.Put(node.id, nil)
        }

        if free.Size() == 0 {
            continue // 无空闲服务器，丢弃请求
        }

        // 查找 >= i%k 的第一个空闲服务器
        target := i % k
        var assignNode *redblacktree.Node
        if node, found := free.Ceiling(target); found {
            assignNode = node
        } else {
            // 环状回绕，取最小的编号
            assignNode = free.Left()
        }

        assignID := assignNode.Key.(int)
        free.Remove(assignID)
        count[assignID]++
        heap.Push(busy, &server{endTime: end, id: assignID})
    }

    // 找出最大请求数的服务器
    maxReq := 0
    for _, c := range count {
        if c > maxReq {
            maxReq = c
        }
    }
    res := []int{}
    for i, c := range count {
        if c == maxReq {
            res = append(res, i)
        }
    }
    return res
}

type server struct {
    endTime int
    id      int
}

type minHeap []*server

func (h minHeap) Len() int           { return len(h) }
func (h minHeap) Less(i, j int) bool { return h[i].endTime < h[j].endTime }
func (h minHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *minHeap) Push(x interface{}) {
    *h = append(*h, x.(*server))
}
func (h *minHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

解法二：双优先队列（堆） + 懒惰删除（纯标准库）

import "container/heap"

func busiestServers(k int, arrival []int, load []int) []int {
    n := len(arrival)
    count := make([]int, k)

    // 空闲服务器堆：free1 存储 >= target 的编号，free2 存储 < target 的编号
    free1 := &intHeap{}
    free2 := &intHeap{}
    heap.Init(free1)
    heap.Init(free2)
    for i := 0; i < k; i++ {
        heap.Push(free1, i)
    }

    // 忙碌服务器堆
    busy := &serverHeap{}
    heap.Init(busy)

    for i := 0; i < n; i++ {
        start := arrival[i]
        // 释放已完成请求的服务器
        for busy.Len() > 0 && (*busy)[0].endTime <= start {
            node := heap.Pop(busy).(*serverNode)
            id := node.id
            target := i % k
            if id >= target {
                heap.Push(free1, id)
            } else {
                heap.Push(free2, id)
            }
        }

        // 选择服务器
        var assignID int
        if free1.Len() > 0 {
            assignID = heap.Pop(free1).(int)
        } else if free2.Len() > 0 {
            assignID = heap.Pop(free2).(int)
        } else {
            continue // 无空闲服务器
        }

        count[assignID]++
        heap.Push(busy, &serverNode{endTime: start + load[i], id: assignID})
    }

    // 找出最大请求数的服务器
    maxReq := 0
    for _, c := range count {
        if c > maxReq {
            maxReq = c
        }
    }
    res := []int{}
    for i, c := range count {
        if c == maxReq {
            res = append(res, i)
        }
    }
    return res
}

    id      int
}

func (h *serverHeap) Push(x interface{}) {
    *h = append(*h, x.(*serverNode))
}
func (h *serverHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

func (h intHeap) Len() int           { return len(h) }
func (h intHeap) Less(i, j int) bool { return h[i] < h[j] }
func (h intHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *intHeap) Push(x interface{}) {
    *h = append(*h, x.(int))
}
func (h *intHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

4️⃣ 复杂度分析

解法一（有序集合 + 优先队列）

  • ⏱️ 时间复杂度：O(n log k)
    每个请求最多入堆一次、出堆一次（O(log k)），在有序集合中查找和删除 O(log k)。总 O(n log k)。
  • 💾 空间复杂度：O(k)
    堆和有序集合最多存储 k 个元素，计数器 O(k)。

解法二（双优先队列 + 懒惰删除）

  • ⏱️ 时间复杂度：O(n log k)
    每个请求在堆中操作 O(log k)，查找空闲服务器 O(log k)。
  • 💾 空间复杂度：O(k)
    两个空闲堆和一个忙碌堆总大小 O(k)，计数器 O(k)。

5️⃣ 解法对比

解法 优点 缺点 适用场景
解法一（有序集合） 逻辑清晰，直接利用有序集合查找，代码简洁 依赖外部库（如 redblacktree）或需手动实现平衡树 需要快速查找和删除的场景，适合对代码简洁性要求高的情况
解法二（双堆） 仅用标准库，无需外部依赖，效率高 逻辑稍复杂，需要维护两个堆 竞赛或面试中要求纯标准库实现，性能要求高

推荐解法二，因为： 1. 仅用标准库，更通用。 2. 时间复杂度相同，实际运行效率高。 3. 面试/竞赛中更常见。
