题目描述

给你一个只包含 '(' 和 ')' 的字符串，找出最长有效（格式正确且连续）括号 子串 的长度。

左右括号匹配，即每个左括号都有对应的右括号将其闭合的字符串是格式正确的，比如 "(()())"。

解释：最长有效括号子串是 "()()"

示例 3：

输出：0

最长有效括号 - 专业解题分析

1️⃣ 题意理解

核心要求： 给定一个只包含 '(' 和 ')' 的字符串，找出其中最长的连续有效括号子串的长度。有效括号子串需要满足： 1. 每个左括号都有对应的右括号匹配 2. 括号匹配顺序正确（即不能出现 ")(" 这样的无效匹配）

关键约束： - 字符串长度：0 ≤ s.length ≤ 3×10⁴ - 只包含 '(' 和 ')' 两种字符 - 需要找的是连续的有效括号子串，不是所有有效括号的总数

边界情况： - 空字符串 → 返回 0 - 全为 '(' 或全为 ')' → 返回 0 - 已经全部匹配的字符串 → 返回字符串长度 - 嵌套括号和并列括号混合的情况

输入输出： - 输入：字符串 s - 输出：最长有效括号子串的长度（整数）

2️⃣ 解题思路

解法一：动态规划

核心思想： 使用动态规划数组 dp[i] 表示以 s[i] 结尾的最长有效括号子串长度。通过分析括号匹配的规律，建立状态转移方程。

解题步骤： 1. 初始化 dp 数组，长度与 s 相同，所有元素为 0 2. 遍历字符串，从索引 1 开始（因为单个字符不可能形成有效括号） 3. 当 s[i] = ')' 时，考虑两种情况： - 情况1：s[i-1] = '('，形成 "()" 匹配 - 情况2：s[i-1] = ')'，可能形成 "((...))" 的嵌套匹配 4. 根据状态转移方程计算 dp[i] 5. 记录 dp 数组中的最大值

注意事项： - 注意数组越界检查 - 嵌套匹配时需要找到对应的左括号位置 - dp[i] 只记录以当前位置结尾的有效长度

解法二：栈

核心思想： 使用栈来跟踪未匹配的括号索引，通过计算相邻匹配括号之间的距离得到有效长度。

解题步骤： 1. 初始化栈，先压入 -1 作为基准点 2. 遍历字符串： - 遇到 '('：将其索引压入栈 - 遇到 ')'：弹出栈顶元素 - 如果栈为空：将当前索引压入栈作为新的基准点 - 如果栈不为空：计算当前索引与栈顶元素的差值，更新最大长度 3. 返回最大长度

注意事项： - 栈中存储的是索引，不是字符 - 初始基准点 -1 的作用是方便计算长度 - 栈为空时压入当前索引，表示新的起始点

解法三：双向扫描

核心思想： 从左到右和从右到左各扫描一次，分别统计左右括号的数量，当左右括号数量相等时记录长度。

解题步骤： 1. 从左到右扫描： - 统计 '(' 和 ')' 的数量 - 当 left = right 时，更新最大长度 - 当 right > left 时，重置计数器（右括号太多） 2. 从右到左扫描： - 统计 '(' 和 ')' 的数量 - 当 left = right 时，更新最大长度 - 当 left > right 时，重置计数器（左括号太多） 3. 返回两次扫描中的最大值

注意事项： - 需要扫描两次，分别处理不同的情况 - 重置计数器的条件不同 - 这种方法可以避免使用额外空间

3️⃣ 代码实现

解法一：动态规划

func longestValidParentheses(s string) int {
    n := len(s)
    if n < 2 {
        return 0
    }

    // dp[i] 表示以 s[i] 结尾的最长有效括号子串长度
    dp := make([]int, n)
    maxLen := 0

    for i := 1; i < n; i++ {
        if s[i] == ')' {
            // 情况1：前一个字符是 '('，形成 "()"
            if s[i-1] == '(' {
                if i >= 2 {
                    dp[i] = dp[i-2] + 2
                } else {
                    dp[i] = 2
                }
            } 
            // 情况2：前一个字符是 ')'，形成 "))"，需要找到对应的 '('
            else if i-dp[i-1] > 0 && s[i-dp[i-1]-1] == '(' {
                if i-dp[i-1] >= 2 {
                    dp[i] = dp[i-1] + dp[i-dp[i-1]-2] + 2
                } else {
                    dp[i] = dp[i-1] + 2
                }
            }

            if dp[i] > maxLen {
                maxLen = dp[i]
            }
        }
    }

    return maxLen
}

解法二：栈

func longestValidParentheses(s string) int {
    maxLen := 0
    stack := []int{-1} // 栈底放-1作为基准点

    for i := 0; i < len(s); i++ {
        if s[i] == '(' {
            // 遇到左括号，压入索引
            stack = append(stack, i)
        } else {
            // 遇到右括号，弹出栈顶
            stack = stack[:len(stack)-1]

            if len(stack) == 0 {
                // 栈为空，将当前索引作为新的基准点
                stack = append(stack, i)
            } else {
                // 计算当前有效长度
                length := i - stack[len(stack)-1]
                if length > maxLen {
                    maxLen = length
                }
            }
        }
    }

    return maxLen
}

解法三：双向扫描

func longestValidParentheses(s string) int {
    maxLen := 0

    // 从左到右扫描
    left, right := 0, 0
    for i := 0; i < len(s); i++ {
        if s[i] == '(' {
            left++
        } else {
            right++
        }

        if left == right {
            if left*2 > maxLen {
                maxLen = left * 2
            }
        } else if right > left {
            left, right = 0, 0
        }
    }

    // 从右到左扫描
    left, right = 0, 0
    for i := len(s) - 1; i >= 0; i-- {
        if s[i] == '(' {
            left++
        } else {
            right++
        }

        if left == right {
            if left*2 > maxLen {
                maxLen = left * 2
            }
        } else if left > right {
            left, right = 0, 0
        }
    }

    return maxLen
}

4️⃣ 复杂度分析

解法一：动态规划

  • ⏱️ 时间复杂度：O(n)
    • 只需要一次遍历字符串
    • 每个位置的计算是常数时间
  • 💾 空间复杂度：O(n)
    • 需要 dp 数组存储中间结果
    • 数组长度与输入字符串相同

解法二：栈

  • ⏱️ 时间复杂度：O(n)
    • 遍历字符串一次
    • 每个字符最多入栈出栈一次
  • 💾 空间复杂度：O(n)
    • 最坏情况下所有字符都是 '('，全部入栈
    • 栈的大小最多为 n+1（包含基准点）

解法三：双向扫描

  • ⏱️ 时间复杂度：O(n)
    • 遍历字符串两次
    • 每次遍历都是线性时间
  • 💾 空间复杂度：O(1)
    • 只使用了常数个变量
    • 不需要额外的数据结构
