2908. 元素和最小的山形三元组 I

难度: Easy

标签: 数组

链接: https://leetcode.cn/problems/minimum-sum-of-mountain-triplets-i/

题目描述

给你一个下标从 0 开始的整数数组 nums 。

如果下标三元组 (i, j, k) 满足下述全部条件，则认为它是一个 山形三元组 ：

  • i < j < k
  • nums[i] < nums[j] 且 nums[k] < nums[j]

请你找出 nums 中 元素和最小 的山形三元组，并返回其 元素和 。如果不存在满足条件的三元组，返回 -1 。

 

示例 1：

输入：nums = [8,6,1,5,3]
输出：9
解释：三元组 (2, 3, 4) 是一个元素和等于 9 的山形三元组，因为： 
- 2 < 3 < 4
- nums[2] < nums[3] 且 nums[4] < nums[3]
这个三元组的元素和等于 nums[2] + nums[3] + nums[4] = 9 。可以证明不存在元素和小于 9 的山形三元组。

示例 2：

输入：nums = [5,4,8,7,10,2]
输出：13
解释：三元组 (1, 3, 5) 是一个元素和等于 13 的山形三元组，因为： 
- 1 < 3 < 5 
- nums[1] < nums[3] 且 nums[5] < nums[3]
这个三元组的元素和等于 nums[1] + nums[3] + nums[5] = 13 。可以证明不存在元素和小于 13 的山形三元组。

示例 3：

输入：nums = [6,5,4,3,4,5]
输出：-1
解释：可以证明 nums 中不存在山形三元组。

 

提示：

  • 3 <= nums.length <= 50
  • 1 <= nums[i] <= 50

解法一：暴力枚举

思路：由于 n ≤ 50，直接枚举所有三元组 (i, j, k)，检查是否满足山形条件 nums[i] < nums[j] 且 nums[k] < nums[j]，取最小元素和。

func minimumSum(nums []int) int {
    n := len(nums)
    ans := -1
    for i := 0; i < n; i++ {
        for j := i + 1; j < n; j++ {
            for k := j + 1; k < n; k++ {
                if nums[i] < nums[j] && nums[k] < nums[j] {
                    sum := nums[i] + nums[j] + nums[k]
                    if ans == -1 || sum < ans {
                        ans = sum
                    }
                }
            }
        }
    }
    return ans
}

复杂度：时间 O(n³)，空间 O(1)

解法二：前后缀最小值（最优解）

思路：对于中间元素 j，左侧需要最小值且小于 nums[j]，右侧同理。预计算 prefixMin[j]（nums[0..j-1] 的最小值）和 suffixMin[j]（nums[j+1..n-1] 的最小值）。若两侧最小值都小于 nums[j]，则组成最优山形三元组，枚举所有 j 取最小和。

func minimumSum(nums []int) int {
    n := len(nums)
    // 前缀最小值：prefixMin[j] = min(nums[0..j-1])
    prefixMin := make([]int, n)
    prefixMin[0] = nums[0]
    for i := 1; i < n; i++ {
        prefixMin[i] = min(prefixMin[i-1], nums[i])
    }
    // 后缀最小值：suffixMin[j] = min(nums[j+1..n-1])
    suffixMin := make([]int, n)
    suffixMin[n-1] = nums[n-1]
    for i := n - 2; i >= 0; i-- {
        suffixMin[i] = min(suffixMin[i+1], nums[i])
    }

    ans := -1
    for j := 1; j < n-1; j++ {
        left := prefixMin[j-1]   // j 左侧的最小值
        right := suffixMin[j+1]  // j 右侧的最小值
        if left < nums[j] && right < nums[j] {
            sum := left + nums[j] + right
            if ans == -1 || sum < ans {
                ans = sum
            }
        }
    }
    return ans
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}

复杂度：时间 O(n)，空间 O(n)
