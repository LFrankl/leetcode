长度为 K 的最大子数组 - 解题分析

1️⃣ 题意理解

题目重述： 给定一个整数数组 nums 和一个整数 k，需要找到数组中所有长度为 k 的子数组中，数值最大的那个子数组。这里的"数值最大"指的是按照字典序比较，即从左到右逐位比较，第一个不同数字更大的数组更大。

关键约束条件： - 1 <= k <= nums.length <= 10^5 - 1 <= nums[i] <= 10^9 - 数组元素都是整数 - 需要返回长度为 k 的最大子数组（按字典序比较）

边界情况： - k = 1 时，只需要找数组中的最大值 - k = nums.length 时，直接返回整个数组 - 数组中有相同元素时，需要比较后续元素

输入输出格式： - 输入：整数数组 nums，整数 k - 输出：长度为 k 的最大子数组

2️⃣ 解题思路

解法一：滑动窗口 + 直接比较

核心思想： 由于需要找到长度为 k 的最大子数组，我们可以使用滑动窗口遍历所有可能的子数组，并维护当前找到的最大子数组。通过直接比较两个数组的字典序来更新最大值。

解题步骤： 1. 初始化最大子数组为前 k 个元素 2. 从第 2 个窗口开始（索引从 1 到 n-k） 3. 对于每个窗口，与当前最大子数组比较字典序 4. 如果当前窗口更大，则更新最大子数组 5. 返回最大子数组

注意事项： - 字典序比较需要逐位比较，直到找到不同的元素 - 时间复杂度较高，但可以通过优化比较过程来减少不必要的比较

解法二：贪心 + 单调栈思想

核心思想： 要得到字典序最大的长度为 k 的子数组，实际上就是要在数组中找到第一个元素最大的位置。如果有多个相同最大元素，则比较第二个元素，依此类推。

解题步骤： 1. 遍历数组，维护当前找到的最大子数组起始位置 maxStart 2. 对于每个可能的起始位置 i（0 ≤ i ≤ n-k）： - 如果 nums[i] > nums[maxStart]，则更新 maxStart = i - 如果 nums[i] == nums[maxStart]，则需要比较后续元素 3. 当遇到相等元素时，逐位比较后续元素 4. 返回从 maxStart 开始的 k 个元素

优化： - 在比较相等元素时，可以跳过已经比较过的部分 - 使用双指针避免重复比较

3️⃣ 代码实现

解法一：滑动窗口 + 直接比较

func largestSubarray(nums []int, k int) []int {
    n := len(nums)
    if n == k {
        return nums
    }

    // 初始化最大子数组为第一个窗口
    maxStart := 0

    // 遍历所有可能的起始位置
    for i := 1; i <= n-k; i++ {
        // 比较当前窗口和最大窗口
        if compareWindows(nums, i, maxStart, k) > 0 {
            maxStart = i
        }
    }

    // 返回最大子数组
    return nums[maxStart:maxStart+k]
}

// 比较两个窗口的字典序
// 返回正数表示窗口1更大，负数表示窗口2更大，0表示相等
func compareWindows(nums []int, start1, start2, k int) int {
    for i := 0; i < k; i++ {
        diff := nums[start1+i] - nums[start2+i]
        if diff != 0 {
            return diff
        }
    }
    return 0
}

解法二：贪心优化

func largestSubarray(nums []int, k int) []int {
    n := len(nums)
    if n == k {
        return nums
    }

    maxStart := 0 // 最大子数组的起始位置

    for i := 1; i <= n-k; i++ {
        // 比较当前起始位置和最大起始位置的子数组
        if isLarger(nums, i, maxStart, k) {
            maxStart = i
        }
    }

    return nums[maxStart:maxStart+k]
}

// 判断从start1开始的子数组是否比从start2开始的子数组大
func isLarger(nums []int, start1, start2, k int) bool {
    for i := 0; i < k; i++ {
        if nums[start1+i] > nums[start2+i] {
            return true
        } else if nums[start1+i] < nums[start2+i] {
            return false
        }
        // 如果相等，继续比较下一个元素
    }
    return false // 两个子数组完全相等
}

解法三：进一步优化（一次遍历）

func largestSubarray(nums []int, k int) []int {
    n := len(nums)
    maxStart := 0

    for i := 1; i <= n-k; i++ {
        // 优化比较：当遇到相等元素时，可以跳过已经比较过的部分
        j := 0
        for j < k && nums[maxStart+j] == nums[i+j] {
            j++
        }

        // 如果j < k，说明找到了不同的元素
        if j < k && nums[i+j] > nums[maxStart+j] {
            maxStart = i
        }
        // 注意：这里不能简单跳过，因为需要处理所有可能的起始位置
    }

    return nums[maxStart:maxStart+k]
}

4️⃣ 复杂度分析

解法一：滑动窗口 + 直接比较

  • ⏱️ 时间复杂度：O(n × k)
    • 需要遍历 n-k+1 个窗口
    • 每个窗口比较需要 O(k) 时间
    • 最坏情况下，每次比较都需要比较所有 k 个元素
  • 💾 空间复杂度：O(1)
    • 只使用了常数级别的额外空间

解法二：贪心优化

  • ⏱️ 时间复杂度：O(n × k) 最坏情况，但实际平均情况更好
    • 仍然需要遍历所有起始位置
    • 比较过程在遇到不同元素时可以提前结束
    • 最坏情况是所有元素都相同，需要比较所有 k 个元素
  • 💾 空间复杂度：O(1)
    • 只使用了常数级别的额外空间

解法三：进一步优化

  • ⏱️ 时间复杂度：O(n × k) 最坏情况
    • 虽然尝试跳过相等元素，但最坏情况仍然是 O(n × k)
    • 对于随机数据，平均性能会更好
  • 💾 空间复杂度：O(1)
    • 只使用了常数级别的额外空间
