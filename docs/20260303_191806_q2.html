1410. HTML 实体解析器

难度: Medium

题目描述

「HTML 实体解析器」 是一种特殊的解析器，它将 HTML 代码作为输入，并用字符本身替换掉所有这些特殊的字符实体。

HTML 里这些特殊字符和它们对应的字符实体包括：

  • 双引号：字符实体为 &quot; ，对应的字符是 " 。
  • 单引号：字符实体为 &apos; ，对应的字符是 ' 。
  • 与符号：字符实体为 &amp; ，对应对的字符是 & 。
  • 大于号：字符实体为 &gt; ，对应的字符是 > 。
  • 小于号：字符实体为 &lt; ，对应的字符是 < 。
  • 斜线号：字符实体为 &frasl; ，对应的字符是 / 。

给你输入字符串 text ，请你实现一个 HTML 实体解析器，返回解析器解析后的结果。

 

示例 1：

输入：text = "&amp; is an HTML entity but &ambassador; is not."
输出："& is an HTML entity but &ambassador; is not."
解释：解析器把字符实体 &amp; 用 & 替换

示例 2：

输入：text = "and I quote: &quot;...&quot;"
输出："and I quote: \"...\""

示例 3：

输入：text = "Stay home! Practice on Leetcode :)"
输出："Stay home! Practice on Leetcode :)"

示例 4：

输入：text = "x &gt; y &amp;&amp; x &lt; y is always false"
输出："x > y && x < y is always false"

示例 5：

输入：text = "leetcode.com&frasl;problemset&frasl;all"
输出："leetcode.com/problemset/all"

 

提示：

  • 1 <= text.length <= 10^5
  • 字符串可能包含 256 个ASCII 字符中的任意字符。

1️⃣ 题意理解

题目核心要求：
将 HTML 文本中的特殊字符实体（如 &quot;、&amp; 等）替换为对应的字符（如 "、& 等），其他部分保持不变。

关键约束条件： 1. 输入字符串长度 ≤ 10^5，需要 O(n) 或 O(n log n) 算法。 2. 只处理 6 种预定义的字符实体，其他类似 &ambassador; 的实体不替换。 3. 实体以 & 开头，以 ; 结尾，中间为字母或数字。 4. 实体可能连续出现（如 &amp;&amp;），需要正确处理。

边界情况： - 输入中没有实体。 - 实体出现在字符串开头或结尾。 - 有 & 但后面没有 ; 或不是预定义实体。 - 实体嵌套（实际不会，因为实体本身不含 &）。

输入输出格式： - 输入：字符串 text - 输出：替换后的字符串

2️⃣ 解题思路

解法一：顺序扫描 + 哈希表匹配

核心思想：
遍历字符串，遇到 & 时尝试匹配预定义的实体，匹配成功则替换并跳过实体长度，否则直接追加字符。

步骤： 1. 构建实体到字符的映射表。 2. 遍历字符串，使用指针 i： - 如果当前字符不是 &，直接追加到结果。 - 如果是 &，向后查找直到 ; 或字符串结束，截取子串尝试匹配。 - 若匹配成功，追加对应字符，i 跳到 ; 之后。 - 若匹配失败，追加 &，i 前进 1。 3. 返回结果字符串。

注意事项： - 匹配时需确保子串以 & 开头、以 ; 结尾。 - 注意索引边界。

解法二：自动机 / 状态机

核心思想：
使用有限状态机，根据当前字符和状态决定是输出字符还是进入实体匹配状态。

步骤： 1. 定义状态：普通状态、& 状态、匹配实体状态。 2. 遍历字符，根据状态转移： - 普通状态遇到 & 进入实体匹配状态，记录起始位置。 - 实体匹配状态遇到 ; 尝试匹配实体，成功则输出替换字符，失败则输出原片段。 - 其他情况直接输出字符。 3. 返回结果。

注意事项： - 状态机需要正确处理匹配失败的回退。 - 实现稍复杂，但扩展性强。

解法三：使用 strings.Replace 多次替换

核心思想：
直接使用标准库的替换函数，按顺序替换所有实体。

步骤： 1. 按顺序调用 strings.Replace 或 strings.ReplaceAll 替换每个实体。 2. 注意替换顺序：先替换 &amp; 避免循环替换。

注意事项： - 需要确保 &amp; 最先替换，否则可能将其他实体中的 & 错误替换。 - 多次扫描字符串，效率较低。

3️⃣ 代码实现

解法一：顺序扫描 + 哈希表匹配

func entityParser(text string) string {
    // 实体到字符的映射
    entityMap := map[string]string{
        "&quot;":  "\"",
        "&apos;":  "'",
        "&amp;":   "&",
        "&gt;":    ">",
        "&lt;":    "<",
        "&frasl;": "/",
    }

    n := len(text)
    var res strings.Builder
    res.Grow(n) // 预分配空间，减少扩容开销

    for i := 0; i < n; i++ {
        if text[i] == '&' {
            // 查找可能的实体结束位置
            j := i
            for j < n && text[j] != ';' {
                j++
            }
            if j < n && text[j] == ';' {
                entity := text[i : j+1] // 包含分号
                if val, ok := entityMap[entity]; ok {
                    res.WriteString(val)
                    i = j // 跳过整个实体
                    continue
                }
            }
        }
        // 不是实体或未匹配，直接追加当前字符
        res.WriteByte(text[i])
    }

    return res.String()
}

解法二：自动机（简化版）

func entityParser(text string) string {
    entityMap := map[string]string{
        "&quot;":  "\"",
        "&apos;":  "'",
        "&amp;":   "&",
        "&gt;":    ">",
        "&lt;":    "<",
        "&frasl;": "/",
    }

    var res strings.Builder
    res.Grow(len(text))
    i := 0
    n := len(text)

    for i < n {
        if text[i] == '&' {
            start := i
            // 尝试匹配实体
            for i < n && text[i] != ';' {
                i++
            }
            if i < n && text[i] == ';' {
                entity := text[start : i+1]
                if val, ok := entityMap[entity]; ok {
                    res.WriteString(val)
                    i++
                    continue
                }
            }
            // 匹配失败，回退到 start+1
            i = start + 1
            res.WriteByte(text[start])
        } else {
            res.WriteByte(text[i])
            i++
        }
    }

    return res.String()
}

解法三：多次替换（不推荐，仅作对比）

func entityParser(text string) string {
    // 注意顺序：必须先替换 &amp;，否则会干扰其他实体
    text = strings.ReplaceAll(text, "&amp;", "&")
    text = strings.ReplaceAll(text, "&quot;", "\"")
    text = strings.ReplaceAll(text, "&apos;", "'")
    text = strings.ReplaceAll(text, "&gt;", ">")
    text = strings.ReplaceAll(text, "&lt;", "<")
    text = strings.ReplaceAll(text, "&frasl;", "/")
    return text
}

4️⃣ 复杂度分析

解法一：顺序扫描 + 哈希表匹配

  • ⏱️ 时间复杂度：O(n)，其中 n 是字符串长度。每个字符最多被访问一次，哈希表查找 O(1)。
  • 💾 空间复杂度：O(n)，存储结果字符串。哈希表大小固定 O(1)。

解法二：自动机

  • ⏱️ 时间复杂度：O(n)，每个字符处理一次。
  • 💾 空间复杂度：O(n)，存储结果。

解法三：多次替换

  • ⏱️ 时间复杂度：O(k * n)，其中 k 是实体数量（6），每次替换需要扫描整个字符串，实际为 O(n)，但常数较大。
  • 💾 空间复杂度：O(n)，每次替换可能生成新字符串。

5️⃣ 解法对比

解法 优点 缺点 适用场景
解法一 一次扫描，效率高；逻辑清晰 需要手动处理匹配 推荐，适合所有情况
解法二 状态机思路清晰，易扩展新实体 实现稍复杂，需要状态管理 实体数量多或规则复杂时
解法三 代码简单，直接调用库函数 效率低，多次扫描；需注意替换顺序 小规模数据或快速原型

推荐解法一：
- 效率最高，一次扫描完成。 - 逻辑清晰，易于理解和维护。 - 适合本题数据规模（10^5）。
