1️⃣ 题意理解

核心要求：
给定两个正整数 n 和 k，要求返回第 n 小的正整数，其二进制表示中恰好包含 k 个 1。
题目保证答案严格小于 2^50。

关键约束： - 1 <= n <= 250 - 1 <= k <= 50 - 答案 < 2^50（即二进制最多 50 位） - 需要按数值从小到大排序，找到第 n 个满足条件的数

边界情况： - k = 1 时，就是找第 n 个 2 的幂（1, 2, 4, 8...） - n = 1 时，返回最小的满足条件的数（即 (1 << k) - 1） - k 较大时，数字会比较大，但不超过 2^50

输入输出： - 输入：int n, int k - 输出：int（第 n 小的满足条件的正整数）

2️⃣ 解题思路

解法一：组合数学 + 逐位确定

核心思想：
利用组合数计算在固定长度下，包含 k 个 1 的二进制数的数量。
从高位到低位逐位确定每个 bit 是 0 还是 1： - 如果当前位设为 0，那么剩余位中还需要放 k 个 1，计算这样的组合数 C(剩余位数, k) - 如果这个组合数 >= n，说明第 n 小的数在当前位是 0 - 如果组合数 < n，说明当前位必须是 1，并且 n 减去这个组合数，继续确定下一位

步骤： 1. 预处理组合数 C(m, k)，m 从 0 到 50 2. 从最高位（第 49 位，因为答案 < 2^50）开始向下逐位确定 3. 对于每一位： - 如果剩余位数 < k 或 k < 0，直接返回 - 计算如果当前位设为 0，剩余位能形成的组合数 - 根据比较结果决定当前位是 0 还是 1 4. 最终得到二进制表示，转换为整数

注意事项： - 组合数可能很大，但 n <= 250，可以用 int 存储 - 注意边界：剩余位数不足 k 时无法放置 k 个 1 - 当 k 变为 0 时，剩余位只能全为 0

解法二：暴力枚举（小范围适用）

核心思想：
由于答案 < 2^50，可以枚举所有可能的值，统计其中二进制 1 的个数为 k 的数，取第 n 个。
但 2^50 太大，不可行。优化：只枚举长度不超过 50 位的数，但数量仍然巨大。
实际上由于 n 很小（<=250），我们可以从小到大生成满足条件的数。

步骤： 1. 从最小的满足条件的数开始（即 (1 << k) - 1） 2. 使用 Gosper's Hack 或类似方法生成下一个恰好有 k 个 1 的数 3. 生成 n 个这样的数，返回最后一个

注意事项： - 需要高效生成下一个有 k 个 1 的数 - 注意数值不能超过 2^50

3️⃣ 代码实现

解法一：组合数学 + 逐位确定

func findKthNumber(n int, k int) int {
    // 预处理组合数 C[m][k] 表示从 m 位中选 k 位放 1 的方案数
    C := make([][]int, 51)
    for i := range C {
        C[i] = make([]int, 51)
    }
    C[0][0] = 1
    for i := 1; i <= 50; i++ {
        C[i][0] = 1
        for j := 1; j <= i; j++ {
            C[i][j] = C[i-1][j-1] + C[i-1][j]
            // 题目保证 n <= 250，组合数不会超过这个范围
            if C[i][j] > 250 {
                C[i][j] = 250 // 截断，因为 n 最大 250
            }
        }
    }

    // 从高位到低位逐位确定
    result := 0
    // 从第 49 位开始（因为答案 < 2^50，最高位是第 49 位）
    for bit := 49; bit >= 0; bit-- {
        if k < 0 || k > bit+1 {
            break
        }
        // 如果当前位设为 0，剩余 bit 位中放 k 个 1 的组合数
        count := C[bit][k] // 注意：剩余位数是 bit（从0开始计数）
        if count >= n {
            // 当前位为 0，继续下一位
            continue
        } else {
            // 当前位必须为 1
            result |= 1 << bit
            n -= count // 跳过所有当前位为 0 的数
            k--        // 还需要放置的 1 减少一个
        }
    }
    return result
}

解法二：Gosper's Hack 生成

func findKthNumber(n int, k int) int {
    // 最小的有 k 个 1 的数
    x := (1 << k) - 1
    for i := 1; i < n; i++ {
        // 使用 Gosper's Hack 生成下一个有 k 个 1 的数
        // 1. 找到最右边的 01，变成 10
        // 2. 将右侧的所有 1 移到最右边
        c := x & -x           // 最右边的 1
        r := x + c            // 将最右边的 01 变成 10
        // 重新排列右侧的 1
        next := (((r ^ x) >> 2) / c) | r
        if next >= (1 << 50) {
            break // 超过范围
        }
        x = next
    }
    return x
}

4️⃣ 复杂度分析

解法一：组合数学

  • ⏱️ 时间复杂度：O(50 * 50 + 50) = O(1)
    预处理组合数 O(50*50)，逐位确定 O(50)，都是常数时间
  • 💾 空间复杂度：O(50 * 50) = O(1)
    存储组合数表

解法二：Gosper's Hack

  • ⏱️ 时间复杂度：O(n)
    需要生成 n 个数，每次生成 O(1)
  • 💾 空间复杂度：O(1)
    只使用常数额外空间
