1996. 游戏中弱角色的数量

难度: Medium

标签: 栈, 贪心, 数组, 排序, 单调栈

链接: https://leetcode.cn/problems/the-number-of-weak-characters-in-the-game/

题目描述

你正在参加一个多角色游戏，每个角色都有两个主要属性：攻击 和 防御 。给你一个二维整数数组 properties ，其中 properties[i] = [attacki, defensei] 表示游戏中第 i 个角色的属性。

如果存在一个其他角色的攻击和防御等级 都严格高于 该角色的攻击和防御等级，则认为该角色为 弱角色 。更正式地，如果认为角色 i 弱于 存在的另一个角色 j ，那么 attackj > attacki 且 defensej > defensei 。

返回 弱角色 的数量。

 

示例 1：

输入：properties = [[5,5],[6,3],[3,6]]
输出：0
解释：不存在攻击和防御都严格高于其他角色的角色。

示例 2：

输入：properties = [[2,2],[3,3]]
输出：1
解释：第一个角色是弱角色，因为第二个角色的攻击和防御严格大于该角色。

示例 3：

输入：properties = [[1,5],[10,4],[4,3]]
输出：1
解释：第三个角色是弱角色，因为第二个角色的攻击和防御严格大于该角色。

 

提示：

  • 2 <= properties.length <= 105
  • properties[i].length == 2
  • 1 <= attacki, defensei <= 105

1️⃣ 题意理解

题目重述：
给定一个角色列表，每个角色有攻击力（attack）和防御力（defense）两个属性。如果一个角色的攻击力和防御力都严格小于另一个角色的攻击力和防御力，则该角色是“弱角色”。要求统计所有弱角色的数量。

关键约束：
- 攻击力和防御力都严格大于才构成“弱于”关系。 - 数据规模：properties.length ≤ 10^5，要求算法时间复杂度优于 O(n²)。 - 攻击力和防御力的值范围：[1, 10^5]。

输入输出格式：
- 输入：二维整数数组 properties，其中 properties[i] = [attack_i, defense_i]。 - 输出：整数，表示弱角色的数量。

2️⃣ 解题思路

解法一：排序 + 遍历（贪心）

核心思想：
将角色按攻击力降序排序，攻击力相同时按防御力升序排序。这样在遍历时，保证攻击力相同的情况下，防御力高的角色不会误判为弱角色。遍历过程中维护当前遇到的最大防御力，若当前角色的防御力小于该最大值，则说明存在一个攻击力更高（或相等但防御力更高）的角色，当前角色为弱角色。

步骤： 1. 对 properties 排序：攻击力降序，攻击力相同时防御力升序。 2. 初始化 maxDefense = 0，用于记录遍历过程中遇到的最大防御力。 3. 遍历排序后的数组： - 若当前防御力 < maxDefense，则当前角色为弱角色，计数加一。 - 否则更新 maxDefense 为当前防御力。 4. 返回计数。

注意事项： - 攻击力相同时，必须按防御力升序排序，避免攻击力相同的角色之间误判。 - 由于攻击力降序，遍历时攻击力非递增，因此只要防御力小于之前遇到的最大防御力，就说明存在一个攻击力更高（或相等但防御力更高）的角色。

解法二：排序 + 单调栈（扩展思路）

核心思想：
按攻击力升序排序，攻击力相同时按防御力降序排序。然后使用单调栈维护防御力的递减序列，遇到比栈顶防御力大的角色时，弹出栈中所有防御力小于当前防御力的角色，这些被弹出的角色就是弱角色。

步骤： 1. 对 properties 排序：攻击力升序，攻击力相同时防御力降序。 2. 初始化一个单调栈（存储防御力），栈中元素防御力递减。 3. 遍历排序后的数组： - 若栈非空且当前防御力 > 栈顶防御力，则栈顶角色为弱角色，弹出并计数。 - 将当前防御力压入栈中。 4. 返回计数。

注意事项： - 攻击力相同时按防御力降序排序，确保攻击力相同的角色不会互相误判。 - 单调栈维护的是防御力的递减序列，当遇到防御力更大的角色时，说明栈中防御力较小的角色遇到了攻击力更高（或相等但防御力更高）的角色。

3️⃣ 代码实现

解法一：排序 + 遍历（贪心）

func numberOfWeakCharacters(properties [][]int) int {
    // 按攻击力降序排序，攻击力相同时按防御力升序排序
    sort.Slice(properties, func(i, j int) bool {
        if properties[i][0] == properties[j][0] {
            return properties[i][1] < properties[j][1] // 攻击力相同，防御力升序
        }
        return properties[i][0] > properties[j][0] // 攻击力降序
    })

    maxDefense := 0
    count := 0
    for _, p := range properties {
        if p[1] < maxDefense {
            count++
        } else {
            maxDefense = p[1]
        }
    }
    return count
}

解法二：排序 + 单调栈

func numberOfWeakCharacters(properties [][]int) int {
    // 按攻击力升序排序，攻击力相同时按防御力降序排序
    sort.Slice(properties, func(i, j int) bool {
        if properties[i][0] == properties[j][0] {
            return properties[i][1] > properties[j][1] // 攻击力相同，防御力降序
        }
        return properties[i][0] < properties[j][0] // 攻击力升序
    })

    stack := []int{}
    count := 0
    for _, p := range properties {
        // 当栈非空且当前防御力大于栈顶防御力时，栈顶角色为弱角色
        for len(stack) > 0 && p[1] > stack[len(stack)-1] {
            stack = stack[:len(stack)-1]
            count++
        }
        stack = append(stack, p[1])
    }
    return count
}

4️⃣ 复杂度分析

解法一：排序 + 遍历

  • ⏱️ 时间复杂度：O(n log n)，主要来自排序操作，遍历 O(n)。
  • 💾 空间复杂度：O(log n) 或 O(n)，取决于排序算法的空间复杂度（Go 的 sort.Slice 使用快速排序，空间复杂度 O(log n)）。

解法二：排序 + 单调栈

  • ⏱️ 时间复杂度：O(n log n)，排序 O(n log n)，遍历 O(n)，每个元素最多入栈出栈一次。
  • 💾 空间复杂度：O(n)，最坏情况下所有角色都入栈。
