2848. 与车相交的点

难度: Easy

标签: 数组, 哈希表, 前缀和

链接: https://leetcode.cn/problems/points-that-intersect-with-cars/

题目描述

给你一个下标从 0 开始的二维整数数组 nums 表示汽车停放在数轴上的坐标。对于任意下标 i，nums[i] = [starti, endi] ，其中 starti 是第 i 辆车的起点，endi 是第 i 辆车的终点。

返回数轴上被车 任意部分 覆盖的整数点的数目。

 

示例 1：

输入：nums = [[3,6],[1,5],[4,7]]
输出：7
解释：从 1 到 7 的所有点都至少与一辆车相交，因此答案为 7 。

示例 2：

输入：nums = [[1,3],[5,8]]
输出：7
解释：1、2、3、5、6、7、8 共计 7 个点满足至少与一辆车相交，因此答案为 7 。

 

提示：

  • 1 <= nums.length <= 100
  • nums[i].length == 2
  • 1 <= starti <= endi <= 100

与车相交的点 - 解题分析

1️⃣ 题意理解

核心要求： 给定多个区间 [start_i, end_i]，每个区间表示一辆车在数轴上的覆盖范围（包含起点和终点）。需要计算所有区间合并后覆盖的整数点总数。

关键约束： 1. 区间包含起点和终点（闭区间） 2. 需要计算的是整数点，不是区间长度 3. 数据规模较小：1 <= nums.length <= 100，1 <= start_i <= end_i <= 100

输入输出： - 输入：二维整数数组 nums，每个元素 [start_i, end_i] - 输出：被至少一个区间覆盖的整数点总数

2️⃣ 解题思路

解法一：直接标记法

核心思想： 由于坐标范围很小（1-100），可以直接创建一个布尔数组标记每个点是否被覆盖。

步骤： 1. 创建长度为101的布尔数组（索引0不使用） 2. 遍历每个区间，将区间内所有点标记为已覆盖 3. 统计被标记的点数

注意事项： - 注意区间是闭区间，包含两端点 - 数组大小要足够容纳最大坐标值

解法二：区间合并法

核心思想： 先对所有区间排序，然后合并重叠区间，最后计算合并后各区间的点数总和。

步骤： 1. 按起点升序排序区间 2. 遍历排序后的区间，合并重叠区间 3. 计算每个合并后区间的点数并累加

注意事项： - 合并时要正确处理端点重叠的情况 - 计算点数时注意是闭区间

3️⃣ 代码实现

解法一：直接标记法

func numberOfPoints(nums [][]int) int {
    // 由于坐标最大为100，创建长度为101的数组（索引0不使用）
    covered := make([]bool, 101)

    // 遍历所有区间，标记覆盖的点
    for _, interval := range nums {
        start, end := interval[0], interval[1]
        for i := start; i <= end; i++ {
            covered[i] = true
        }
    }

    // 统计被覆盖的点数
    count := 0
    for i := 1; i <= 100; i++ {
        if covered[i] {
            count++
        }
    }

    return count
}

解法二：区间合并法

func numberOfPoints(nums [][]int) int {
    // 1. 按起点升序排序
    sort.Slice(nums, func(i, j int) bool {
        return nums[i][0] < nums[j][0]
    })

    // 2. 合并区间
    merged := [][]int{}
    for _, interval := range nums {
        // 如果merged为空或当前区间与最后一个合并区间不重叠
        if len(merged) == 0 || interval[0] > merged[len(merged)-1][1] {
            merged = append(merged, []int{interval[0], interval[1]})
        } else {
            // 有重叠，更新最后一个合并区间的终点
            if interval[1] > merged[len(merged)-1][1] {
                merged[len(merged)-1][1] = interval[1]
            }
        }
    }

    // 3. 计算总点数
    total := 0
    for _, interval := range merged {
        // 闭区间点数 = end - start + 1
        total += interval[1] - interval[0] + 1
    }

    return total
}

4️⃣ 复杂度分析

解法一：直接标记法

  • ⏱️ 时间复杂度：O(n × k)，其中 n 是区间数量，k 是平均区间长度。最坏情况下 O(100 × 100) = O(10000)
  • 💾 空间复杂度：O(1)，固定使用长度为101的布尔数组

复杂度来源： - 外层循环遍历所有区间：O(n) - 内层循环标记区间内所有点：O(k) - 统计循环：O(100)

解法二：区间合并法

  • ⏱️ 时间复杂度：O(n log n)，主要来自排序操作
  • 💾 空间复杂度：O(n)，用于存储合并后的区间

复杂度来源： - 排序：O(n log n) - 合并区间：O(n) - 计算点数：O(m)，其中 m 是合并后的区间数
