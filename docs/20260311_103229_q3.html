1298. 你能从盒子里获得的最大糖果数

难度: Hard

标签: 广度优先搜索, 图, 数组

链接: https://leetcode.cn/problems/maximum-candies-you-can-get-from-boxes/

题目描述

给你 n 个盒子，每个盒子的格式为 [status, candies, keys, containedBoxes] ，其中：

  • 状态字 status[i]：整数，如果 box[i] 是开的，那么是 1 ，否则是 0 。
  • 糖果数 candies[i]: 整数，表示 box[i] 中糖果的数目。
  • 钥匙 keys[i]：数组，表示你打开 box[i] 后，可以得到一些盒子的钥匙，每个元素分别为该钥匙对应盒子的下标。
  • 内含的盒子 containedBoxes[i]：整数，表示放在 box[i] 里的盒子所对应的下标。

给你一个整数数组 initialBoxes，包含你最初拥有的盒子。你可以拿走每个 已打开盒子 里的所有糖果，并且可以使用其中的钥匙去开启新的盒子，并且可以使用在其中发现的其他盒子。

请你按照上述规则，返回可以获得糖果的 最大数目 。

 

示例 1：

输入：status = [1,0,1,0], candies = [7,5,4,100], keys = [[],[],[1],[]], containedBoxes = [[1,2],[3],[],[]], initialBoxes = [0]
输出：16
解释：
一开始你有盒子 0 。你将获得它里面的 7 个糖果和盒子 1 和 2。
盒子 1 目前状态是关闭的，而且你还没有对应它的钥匙。所以你将会打开盒子 2 ，并得到里面的 4 个糖果和盒子 1 的钥匙。
在盒子 1 中，你会获得 5 个糖果和盒子 3 ，但是你没法获得盒子 3 的钥匙所以盒子 3 会保持关闭状态。
你总共可以获得的糖果数目 = 7 + 4 + 5 = 16 个。

示例 2：

输入：status = [1,0,0,0,0,0], candies = [1,1,1,1,1,1], keys = [[1,2,3,4,5],[],[],[],[],[]], containedBoxes = [[1,2,3,4,5],[],[],[],[],[]], initialBoxes = [0]
输出：6
解释：
你一开始拥有盒子 0 。打开它你可以找到盒子 1,2,3,4,5 和它们对应的钥匙。
打开这些盒子，你将获得所有盒子的糖果，所以总糖果数为 6 个。

示例 3：

输入：status = [1,1,1], candies = [100,1,100], keys = [[],[0,2],[]], containedBoxes = [[],[],[]], initialBoxes = [1]
输出：1

示例 4：

输入：status = [1], candies = [100], keys = [[]], containedBoxes = [[]], initialBoxes = []
输出：0

示例 5：

输入：status = [1,1,1], candies = [2,3,2], keys = [[],[],[]], containedBoxes = [[],[],[]], initialBoxes = [2,1,0]
输出：7

 

提示：

  • 1 <= status.length <= 1000
  • status.length == candies.length == keys.length == containedBoxes.length == n
  • status[i] 要么是 0 要么是 1 。
  • 1 <= candies[i] <= 1000
  • 0 <= keys[i].length <= status.length
  • 0 <= keys[i][j] < status.length
  • keys[i] 中的值都是互不相同的。
  • 0 <= containedBoxes[i].length <= status.length
  • 0 <= containedBoxes[i][j] < status.length
  • containedBoxes[i] 中的值都是互不相同的。
  • 每个盒子最多被一个盒子包含。
  • 0 <= initialBoxes.length <= status.length
  • 0 <= initialBoxes[i] < status.length

1️⃣ 题意理解

题目重述
我们有一组盒子，每个盒子有四种属性：
- status[i]：盒子初始是否打开（1开/0关）
- candies[i]：盒子中的糖果数
- keys[i]：打开该盒子后能获得的钥匙（指向其他盒子的索引）
- containedBoxes[i]：该盒子中包含的其他盒子索引

初始时我们拥有 initialBoxes 中列出的盒子（但不一定都能立即打开）。
我们可以：
1. 打开已拥有且状态为打开的盒子
2. 打开盒子后，获得其中的糖果、钥匙（可改变其他盒子的状态）、以及内含的盒子
3. 重复此过程直到无法再打开新盒子

求最终能获得的最大糖果总数。

关键约束
- 盒子总数 n ≤ 1000
- 糖果数 ≥ 1
- 钥匙和内含盒子列表可能为空
- 初始拥有的盒子可能为空
- 盒子状态可能因获得钥匙而改变（从关→开）
- 一个盒子可能被多个钥匙指向，但只能被一个盒子包含

边界情况
- 初始没有盒子 → 糖果数为 0
- 拥有盒子但无法打开（没钥匙） → 无法获得其糖果
- 钥匙可能指向已拥有或未拥有的盒子

输入输出格式
- 输入：status[], candies[], keys[][], containedBoxes[][], initialBoxes[]
- 输出：一个整数，表示可获得的最大糖果数

2️⃣ 解题思路

解法一：BFS + 状态维护

核心思想
模拟开盒过程，维护三个集合：
1. 已拥有的盒子
2. 已获得的钥匙
3. 已打开的盒子

每次从已拥有且可打开的盒子中选择一个打开，更新状态，直到没有盒子可打开。

解题步骤
1. 初始化队列（存放当前可打开的盒子）
2. 初始化三个集合：
- owned：已拥有的盒子（包括初始拥有和后续获得的）
- keys：已获得的钥匙
- opened：已打开的盒子（避免重复打开）
3. 将初始拥有的盒子中状态为打开的直接加入队列
4. BFS循环：
- 取出一个盒子，若已打开过则跳过
- 获得糖果、钥匙、内含盒子
- 对于每个钥匙：
- 若对应盒子已拥有且未打开，且现在状态变为可打开，则加入队列
- 对于每个内含盒子：
- 若未拥有则加入owned
- 若现在可打开（已拥有且状态为开或已有钥匙）且未打开过，则加入队列
5. 循环直到队列为空

注意事项
- 盒子状态可能因获得钥匙而改变，需动态判断是否可打开
- 已打开的盒子不能重复打开
- 内含盒子可能已在owned中，需去重

解法二：模拟 + 循环处理

核心思想
不使用队列，而是循环遍历所有已拥有但未打开的盒子，直到一轮中没有新盒子被打开。

解题步骤
1. 初始化owned、keys、opened集合
2. 将初始盒子加入owned
3. 循环：
- 标记changed = false
- 遍历所有owned中未打开的盒子：
- 若盒子可打开（状态为开或有钥匙）：
- 打开它，获得糖果、钥匙、内含盒子
- 更新changed = true
- 若一轮中无变化，结束循环
4. 返回糖果总数

优缺点
- 实现简单，但可能效率较低（每轮遍历所有盒子）
- 适合数据规模较小的情况

3️⃣ 代码实现

解法一：BFS + 状态维护

func maxCandies(status []int, candies []int, keys [][]int, containedBoxes [][]int, initialBoxes []int) int {
    n := len(status)
    owned := make([]bool, n)      // 已拥有的盒子
    hasKey := make([]bool, n)     // 已获得的钥匙
    opened := make([]bool, n)     // 已打开的盒子
    queue := make([]int, 0)       // 当前可打开的盒子队列

    // 初始化：拥有初始盒子
    for _, box := range initialBoxes {
        owned[box] = true
        // 如果初始就是打开的，加入队列
        if status[box] == 1 {
            queue = append(queue, box)
        }
    }

    totalCandies := 0

    for len(queue) > 0 {
        // 取出当前可打开的盒子
        curr := queue[0]
        queue = queue[1:]

        // 如果已经打开过，跳过
        if opened[curr] {
            continue
        }

        // 打开盒子
        opened[curr] = true
        totalCandies += candies[curr]

        // 获得该盒子中的钥匙
        for _, key := range keys[curr] {
            hasKey[key] = true
            // 如果拥有对应盒子且未打开，且现在可以打开了，加入队列
            if owned[key] && !opened[key] && (status[key] == 1 || hasKey[key]) {
                queue = append(queue, key)
            }
        }

        // 获得该盒子中的内含盒子
        for _, innerBox := range containedBoxes[curr] {
            if !owned[innerBox] {
                owned[innerBox] = true
                // 如果现在可以打开（状态为开或有钥匙），加入队列
                if !opened[innerBox] && (status[innerBox] == 1 || hasKey[innerBox]) {
                    queue = append(queue, innerBox)
                }
            }
        }
    }

    return totalCandies
}

解法二：模拟 + 循环处理

func maxCandies2(status []int, candies []int, keys [][]int, containedBoxes [][]int, initialBoxes []int) int {
    n := len(status)
    owned := make([]bool, n)
    hasKey := make([]bool, n)
    opened := make([]bool, n)

    // 初始化拥有盒子
    for _, box := range initialBoxes {
        owned[box] = true
    }

    totalCandies := 0
    changed := true

    // 循环直到没有新盒子被打开
    for changed {
        changed = false

        // 遍历所有已拥有但未打开的盒子
        for box := 0; box < n; box++ {
            if owned[box] && !opened[box] {
                // 检查是否可以打开：初始状态为开或已有钥匙
                canOpen := status[box] == 1 || hasKey[box]
                if canOpen {
                    // 打开盒子
                    opened[box] = true
                    changed = true
                    totalCandies += candies[box]

                    // 获得钥匙
                    for _, key := range keys[box] {
                        hasKey[key] = true
                    }

                    // 获得内含盒子
                    for _, innerBox := range containedBoxes[box] {
                        owned[innerBox] = true
                    }
                }
            }
        }
    }

    return totalCandies
}

4️⃣ 复杂度分析

解法一：BFS + 状态维护

  • ⏱️ 时间复杂度：O(n + m + k)

    • n：盒子数量
    • m：所有盒子的内含盒子总数（边数）
    • k：所有盒子的钥匙总数
    • 每个盒子最多被处理一次，每条边（内含关系）和每个钥匙最多被处理一次

  • 💾 空间复杂度：O(n)

    • 存储owned、hasKey、opened数组：O(3n)
    • 队列最坏情况：O(n)
    • 总体：O(n)

解法二：模拟 + 循环处理

  • ⏱️ 时间复杂度：O(n² + n*(m+k))

    • 最坏情况下每轮只能打开一个盒子，需要n轮
    • 每轮遍历所有n个盒子，检查每个盒子的钥匙和内含盒子
    • 总体：O(n² + n*(m+k))

  • 💾 空间复杂度：O(n)

    • 存储三个状态数组：O(3n)

5️⃣ 解法对比

特性 解法一（BFS） 解法二（模拟循环）
时间复杂度 O(n+m+k) O(n²+n*(m+k))
空间复杂度 O(n) O(n)
实现难度 中等 简单
性能 高效 较低效
适用场景 数据规模大时 数据规模小或作为参考实现

推荐解法：解法一（BFS）
- 时间复杂度更优，适合n≤1000的约束
- 模拟过程更自然，符合"打开盒子→获得新资源"的流程
- 实际运行效率更高
