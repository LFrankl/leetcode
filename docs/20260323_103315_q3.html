1️⃣ 题意理解

题目重述： 给定两个字符串 start 和 end，以及一个字符串列表 bank。每次操作可以将 start 中的一个字符替换为另一个字符（只能从 'A'、'C'、'G'、'T' 中选择），生成的新字符串必须存在于 bank 中。求从 start 转换到 end 所需的最少操作次数。如果无法转换，返回 -1。

关键约束： - 所有字符串长度相同，且只包含 'A'、'C'、'G'、'T' 四种字符 - bank 中的字符串互不相同 - 每次转换只能改变一个字符，且新字符串必须在 bank 中 - 如果 start == end，直接返回 0

边界情况： - bank 为空 - start 或 end 不在 bank 中（但 start 可以不在 bank 中） - 无法通过合法转换到达 end

输入输出格式： - 输入：start string, end string, bank []string - 输出：最少操作次数（int）

2️⃣ 解题思路

解法一：BFS（广度优先搜索）

核心思想： 将字符串转换过程看作图搜索问题： - 每个字符串是图中的一个节点 - 如果两个字符串只有一个字符不同，且目标字符串在 bank 中，则存在一条边 - 使用 BFS 寻找从 start 到 end 的最短路径

解题步骤： 1. 将 bank 转换为哈希集合，便于 O(1) 时间查询 2. 如果 end 不在 bank 中，直接返回 -1 3. 使用队列进行 BFS，记录当前字符串和步数 4. 对于当前字符串，遍历每个位置，尝试替换为其他三种字符 5. 如果新字符串在 bank 中且未被访问过，加入队列 6. 找到 end 时返回步数

注意事项： - 需要记录已访问的字符串，避免重复访问 - start 可能不在 bank 中，但这是允许的 - 字符替换时注意跳过与原字符相同的替换

解法二：双向 BFS

核心思想： 同时从 start 和 end 开始 BFS，当两个搜索相遇时停止。这可以减少搜索空间，提高效率。

解题步骤： 1. 同样将 bank 转为哈希集合 2. 初始化两个队列和两个 visited 集合 3. 每次选择当前层节点数较少的方向进行扩展 4. 检查新节点是否出现在另一个方向的 visited 集合中 5. 如果相遇，返回两个方向的步数之和

优势： - 减少搜索分支，时间复杂度更低 - 特别适合搜索空间较大的情况

解法一：单向 BFS

func minMutation(start string, end string, bank []string) int {
    // 将 bank 转换为哈希集合，便于快速查找
    bankSet := make(map[string]bool)
    for _, s := range bank {
        bankSet[s] = true
    }

    // 如果 end 不在 bank 中，无法转换
    if !bankSet[end] {
        return -1
    }

    // 基因字符集
    genes := []byte{'A', 'C', 'G', 'T'}

    // BFS 队列：存储当前基因序列和已用步数
    queue := []string{start}
    visited := make(map[string]bool)
    visited[start] = true

    steps := 0

    for len(queue) > 0 {
        // 处理当前层的所有节点
        levelSize := len(queue)
        for i := 0; i < levelSize; i++ {
            current := queue[0]
            queue = queue[1:]

            // 找到目标，返回步数
            if current == end {
                return steps
            }

            // 将当前字符串转为字节数组，便于修改
            currBytes := []byte(current)

            // 尝试修改每个位置的字符
            for j := 0; j < len(currBytes); j++ {
                originalChar := currBytes[j]

                // 尝试替换为其他三种字符
                for _, g := range genes {
                    if g == originalChar {
                        continue
                    }

                    currBytes[j] = g
                    newGene := string(currBytes)

                    // 如果新基因在 bank 中且未被访问过
                    if bankSet[newGene] && !visited[newGene] {
                        visited[newGene] = true
                        queue = append(queue, newGene)
                    }
                }

                // 恢复原字符
                currBytes[j] = originalChar
            }
        }

        // 每处理完一层，步数加1
        steps++
    }

    // BFS 结束仍未找到，返回 -1
    return -1
}

解法二：双向 BFS

func minMutation(start string, end string, bank []string) int {
    // 将 bank 转换为哈希集合
    bankSet := make(map[string]bool)
    for _, s := range bank {
        bankSet[s] = true
    }

    // 如果 end 不在 bank 中，无法转换
    if !bankSet[end] {
        return -1
    }

    // 基因字符集
    genes := []byte{'A', 'C', 'G', 'T'}

    // 双向 BFS 初始化
    startQueue := []string{start}
    endQueue := []string{end}

    startVisited := make(map[string]int)
    endVisited := make(map[string]int)

    startVisited[start] = 0
    endVisited[end] = 0

    // BFS 搜索
    for len(startQueue) > 0 && len(endQueue) > 0 {
        // 总是从节点数较少的方向开始扩展
        var steps int
        if len(startQueue) <= len(endQueue) {
            steps = extendQueue(&startQueue, startVisited, endVisited, bankSet, genes)
        } else {
            steps = extendQueue(&endQueue, endVisited, startVisited, bankSet, genes)
        }

        if steps != -1 {
            return steps
        }
    }

    return -1
}

// 扩展队列，返回相遇时的步数，未相遇返回 -1
func extendQueue(queue *[]string, visited, otherVisited map[string]int, 
                bankSet map[string]bool, genes []byte) int {
    current := (*queue)[0]
    *queue = (*queue)[1:]

    currSteps := visited[current]
    currBytes := []byte(current)

    // 尝试修改每个位置的字符
    for i := 0; i < len(currBytes); i++ {
        originalChar := currBytes[i]

        for _, g := range genes {
            if g == originalChar {
                continue
            }

            currBytes[i] = g
            newGene := string(currBytes)

            // 如果新基因不在 bank 中，跳过
            if !bankSet[newGene] {
                currBytes[i] = originalChar
                continue
            }

            // 如果新基因在另一个方向已访问过，相遇了！
            if otherSteps, ok := otherVisited[newGene]; ok {
                return currSteps + 1 + otherSteps
            }

            // 如果新基因在当前方向未访问过，加入队列
            if _, ok := visited[newGene]; !ok {
                visited[newGene] = currSteps + 1
                *queue = append(*queue, newGene)
            }
        }

        // 恢复原字符
        currBytes[i] = originalChar
    }

    return -1
}

4️⃣ 复杂度分析

解法一：单向 BFS

  • ⏱️ 时间复杂度：O(N × L × 4) = O(4NL)

    • N 是 bank 的大小
    • L 是字符串长度（固定为 8）
    • 每个节点最多扩展 3L 个新节点（每个位置尝试 3 种新字符）
    • 最坏情况需要访问所有节点

  • 💾 空间复杂度：O(N)

    • visited 集合存储最多 N 个节点
    • 队列最多存储 N 个节点

解法二：双向 BFS

  • ⏱️ 时间复杂度：O(N × L × 4) = O(4NL)

    • 理论上与单向 BFS 同阶
    • 但实际运行更快，因为搜索空间从两端同时缩小
    • 最坏情况仍需访问所有节点

  • 💾 空间复杂度：O(N)

    • 需要两个 visited 集合
    • 但每个集合存储的节点数通常少于单向 BFS

5️⃣ 解法对比

特性 单向 BFS 双向 BFS
实现难度 简单 中等
时间复杂度 O(4NL) O(4NL)（但常数更小）
空间复杂度 O(N) O(N)
适用场景 bank 规模较小 bank 规模较大
推荐程度 推荐初学者使用 推荐追求性能时使用

推荐建议： - 对于 LeetCode 的测试用例规模，两种方法都能通过 - 单向 BFS 实现简单，不易出错，适合面试快速实现 - 双向 BFS 在 bank 规模较大时性能优势明显 - 实际编码中，如果时间允许，推荐实现双向 BFS
