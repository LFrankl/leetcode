2448. 使数组相等的最小开销

难度: Hard

标签: 贪心, 数组, 二分查找, 前缀和, 排序

链接: https://leetcode.cn/problems/minimum-cost-to-make-array-equal/

题目描述

给你两个下标从 0 开始的数组 nums 和 cost ，分别包含 n 个 正 整数。

你可以执行下面操作 任意 次：

  • 将 nums 中 任意 元素增加或者减小 1 。

对第 i 个元素执行一次操作的开销是 cost[i] 。

请你返回使 nums 中所有元素 相等 的 最少 总开销。

 

示例 1：

输入：nums = [1,3,5,2], cost = [2,3,1,14]
输出：8
解释：我们可以执行以下操作使所有元素变为 2 ：
- 增加第 0 个元素 1 次，开销为 2 。
- 减小第 1 个元素 1 次，开销为 3 。
- 减小第 2 个元素 3 次，开销为 1 + 1 + 1 = 3 。
总开销为 2 + 3 + 3 = 8 。
这是最小开销。

示例 2：

输入：nums = [2,2,2,2,2], cost = [4,2,8,1,3]
输出：0
解释：数组中所有元素已经全部相等，不需要执行额外的操作。

 

提示：

  • n == nums.length == cost.length
  • 1 <= n <= 105
  • 1 <= nums[i], cost[i] <= 106
  • 测试用例确保输出不超过 253-1。

解法一：加权中位数

思路：总开销函数 f(v) = Σ cost[i] × |nums[i] - v| 是凸函数，其最小值在加权中位数处取得。将 (nums, cost) 按 nums 排序后，从左累加 cost 权重，当累加值超过总权重的一半时，对应的 nums 值即为最优目标值。

func minCost(nums []int, cost []int) int64 {
    n := len(nums)
    pairs := make([]struct{ num, c int }, n)
    for i := range nums {
        pairs[i].num = nums[i]
        pairs[i].c = cost[i]
    }
    sort.Slice(pairs, func(i, j int) bool {
        return pairs[i].num < pairs[j].num
    })

    // 计算总权重，找加权中位数
    totalWeight := 0
    for _, p := range pairs {
        totalWeight += p.c
    }
    half := totalWeight / 2
    cumWeight := 0
    target := 0
    for _, p := range pairs {
        cumWeight += p.c
        if cumWeight > half {
            target = p.num
            break
        }
    }

    // 计算总开销
    ans := int64(0)
    for i := range nums {
        diff := nums[i] - target
        if diff < 0 {
            diff = -diff
        }
        ans += int64(diff) * int64(cost[i])
    }
    return ans
}

复杂度：时间 O(n log n)，空间 O(n)

解法二：凸函数二分求最小值

思路：f(v) = Σ cost[i] × |nums[i] - v| 是关于 v 的凸函数（整数域上单调递减再递增）。利用凸性，比较 f(mid) 与 f(mid+1)：若 f(mid) ≤ f(mid+1) 则最小值在左侧，否则在右侧。二分搜索即可定位最小值点，无需排序。

func minCost(nums []int, cost []int) int64 {
    // 确定搜索范围
    lo, hi := nums[0], nums[0]
    for _, x := range nums {
        if x < lo { lo = x }
        if x > hi { hi = x }
    }

    // 计算总开销的辅助函数
    calc := func(v int) int64 {
        res := int64(0)
        for i := range nums {
            diff := nums[i] - v
            if diff < 0 { diff = -diff }
            res += int64(diff) * int64(cost[i])
        }
        return res
    }

    // 凸函数上二分找最小值
    for lo < hi {
        mid := (lo + hi) / 2
        if calc(mid) <= calc(mid+1) {
            hi = mid
        } else {
            lo = mid + 1
        }
    }
    return calc(lo)
}

复杂度：时间 O(n · log(max(nums) - min(nums))) ≈ O(20n)，空间 O(1)

总结：解法一利用加权中数的数学性质，直观且排序后只需一次线性扫描；解法二利用凸性二分，不需要排序但每轮需重新计算开销。两者时间复杂度相近，解法一在实际中通常更优（排序后扫描更紧凑），解法二则思路简洁、无需额外空间。
