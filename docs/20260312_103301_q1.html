题目描述

给你单链表的头结点 head ，请你找出并返回链表的中间结点。

如果有两个中间结点，则返回第二个中间结点。

 

示例 1：

输入：head = [1,2,3,4,5]
输出：[3,4,5]
解释：链表只有一个中间结点，值为 3 。

示例 2：

输入：head = [1,2,3,4,5,6]
输出：[4,5,6]
解释：该链表有两个中间结点，值分别为 3 和 4 ，返回第二个结点。

 

提示：

  • 链表的结点数范围是 [1, 100]
  • 1 <= Node.val <= 100

链表的中间结点 - 解题分析

1️⃣ 题意理解

核心要求： 给定一个单链表的头节点 head，需要找到并返回链表的中间节点。如果链表有偶数个节点（即存在两个中间节点），则返回第二个中间节点。

关键约束： - 链表节点数范围：[1, 100] - 节点值范围：[1, 100] - 链表非空（至少有一个节点）

边界情况： 1. 链表只有一个节点：直接返回该节点 2. 链表有两个节点：返回第二个节点 3. 链表有偶数个节点：返回第二个中间节点 4. 链表有奇数个节点：返回唯一的中间节点

输入输出格式： - 输入：单链表的头节点 head - 输出：中间节点（从该节点开始的后半部分链表）

2️⃣ 解题思路

解法一：两次遍历法

核心思想：先遍历一次链表获取长度，再遍历到中间位置 - 第一次遍历：计算链表长度 n - 第二次遍历：移动到第 n/2 个节点（从0开始计数）

解法二：快慢指针法（推荐）

核心思想：使用两个指针，快指针每次走两步，慢指针每次走一步 - 当快指针到达链表末尾时，慢指针正好在中间位置 - 对于偶数个节点的情况，快指针会停在最后一个节点，慢指针停在第二个中间节点

解法三：数组存储法

核心思想：将链表节点存储到数组中，直接通过索引访问中间节点 - 遍历链表，将每个节点存入数组 - 通过数组长度计算中间索引：len(arr)/2

3️⃣ 代码实现

解法一：两次遍历法

/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func middleNode(head *ListNode) *ListNode {
    // 第一次遍历：计算链表长度
    length := 0
    current := head
    for current != nil {
        length++
        current = current.Next
    }

    // 第二次遍历：移动到中间位置
    middleIndex := length / 2
    current = head
    for i := 0; i < middleIndex; i++ {
        current = current.Next
    }

    return current
}

解法二：快慢指针法（推荐）

/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func middleNode(head *ListNode) *ListNode {
    // 初始化快慢指针
    slow, fast := head, head

    // 快指针每次走两步，慢指针每次走一步
    // 当快指针到达末尾时，慢指针就在中间
    for fast != nil && fast.Next != nil {
        slow = slow.Next      // 慢指针走一步
        fast = fast.Next.Next // 快指针走两步
    }

    return slow
}

解法三：数组存储法

/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func middleNode(head *ListNode) *ListNode {
    // 将链表节点存储到数组中
    nodes := make([]*ListNode, 0)
    current := head

    for current != nil {
        nodes = append(nodes, current)
        current = current.Next
    }

    // 直接通过索引访问中间节点
    return nodes[len(nodes)/2]
}

4️⃣ 复杂度分析

解法一：两次遍历法

  • ⏱️ 时间复杂度：O(n)
    • 第一次遍历：O(n) 计算长度
    • 第二次遍历：O(n/2) ≈ O(n) 移动到中间
    • 总时间复杂度：O(n) + O(n) = O(n)
  • 💾 空间复杂度：O(1)
    • 只使用了常数级别的额外空间（几个指针变量）

解法二：快慢指针法

  • ⏱️ 时间复杂度：O(n)
    • 只需要遍历一次链表
    • 快指针走到末尾时，慢指针正好在中间
  • 💾 空间复杂度：O(1)
    • 只使用了两个指针变量，常数空间

解法三：数组存储法

  • ⏱️ 时间复杂度：O(n)
    • 遍历一次链表存储节点
    • 索引访问是O(1)
  • 💾 空间复杂度：O(n)
    • 需要额外的数组存储所有节点
    • 空间复杂度与链表长度成正比

5️⃣ 解法对比

解法 优点 缺点 适用场景
两次遍历法 思路简单直观，容易实现 需要遍历两次链表，效率较低 教学示例，小规模数据
快慢指针法 只需遍历一次，效率高，空间最优 指针移动逻辑需要理解 推荐使用，各种规模数据
数组存储法 实现简单，直接索引访问 需要额外O(n)空间 需要多次访问中间节点的情况

推荐解法：快慢指针法 - 时间复杂度最优（只需一次遍历） - 空间复杂度最优（O(1)） - 代码简洁优雅 - 是链表中间节点问题的标准解法
