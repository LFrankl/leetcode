1️⃣ 题意理解

题目重述

从课程完成记录表中，筛选出顶尖学生（完成至少5门课程且平均评分≥4分），然后按时间顺序分析每个顶尖学生的课程完成序列，找出所有连续的课程对（A→B），统计这些课程对在所有顶尖学生中出现的频率，最后按频率降序、课程名称升序返回结果。

关键约束

  1. 顶尖学生条件：
    • 完成课程数 ≥ 5
    • 平均评分 ≥ 4.0
  2. 课程对定义：按completion_date时间顺序的连续课程
  3. 排序规则：
    • 主要按transition_count降序
    • 次要根据first_course和second_course升序

输入输出格式

输入：course_completions表 - user_id：用户ID - course_id：课程ID - course_name：课程名称 - completion_date：完成日期 - course_rating：评分（1-5）

输出：包含三列的表 - first_course：前导课程名称 - second_course：后续课程名称 - transition_count：课程对出现次数

2️⃣ 解题思路

解法一：窗口函数法（推荐）

核心思想：使用SQL窗口函数按用户和时间排序，生成课程序列，然后自连接形成课程对。

步骤： 1. 筛选顶尖学生：按用户分组，计算课程数和平均分 2. 为每个顶尖学生的课程按时间排序 3. 使用LEAD()获取下一门课程，形成课程对 4. 统计课程对频率并排序

注意事项： - 同一用户可能多次完成同一课程（但题目中(userid, courseid)组合唯一） - 日期可能相同，需要确保排序稳定

解法二：自连接法

核心思想：通过自连接同一用户的相邻课程记录来形成课程对。

步骤： 1. 筛选顶尖学生 2. 为每个用户的课程按时间排序并编号 3. 自连接：将编号n的记录与编号n+1的记录连接 4. 统计课程对频率

优缺点： - 更直观，但需要两次排序和连接 - 在数据量大时可能性能稍差

3️⃣ 代码实现

解法一：窗口函数法

WITH top_students AS (
    -- 1. 筛选顶尖学生
    SELECT user_id
    FROM course_completions
    GROUP BY user_id
    HAVING COUNT(*) >= 5 
       AND AVG(course_rating) >= 4.0
),
student_courses AS (
    -- 2. 获取顶尖学生的课程记录并按时间排序
    SELECT 
        cc.user_id,
        cc.course_name,
        cc.completion_date,
        -- 为每个学生的课程按时间排序
        ROW_NUMBER() OVER (
            PARTITION BY cc.user_id 
            ORDER BY cc.completion_date
        ) as course_order
    FROM course_completions cc
    INNER JOIN top_students ts 
        ON cc.user_id = ts.user_id
),
course_pairs AS (
    -- 3. 使用LEAD获取下一门课程，形成课程对
    SELECT 
        user_id,
        course_name as first_course,
        LEAD(course_name) OVER (
            PARTITION BY user_id 
            ORDER BY course_order
        ) as second_course
    FROM student_courses
)
-- 4. 统计课程对频率并排序
SELECT 
    first_course,
    second_course,
    COUNT(*) as transition_count
FROM course_pairs
WHERE second_course IS NOT NULL  -- 排除最后一门课程
GROUP BY first_course, second_course
ORDER BY 
    transition_count DESC,
    first_course ASC,
    second_course ASC;

解法二：自连接法

WITH top_students AS (
    -- 1. 筛选顶尖学生
    SELECT user_id
    FROM course_completions
    GROUP BY user_id
    HAVING COUNT(*) >= 5 
       AND AVG(course_rating) >= 4.0
),
ranked_courses AS (
    -- 2. 为课程排序编号
    SELECT 
        cc.user_id,
        cc.course_name,
        cc.completion_date,
        ROW_NUMBER() OVER (
            PARTITION BY cc.user_id 
            ORDER BY cc.completion_date
        ) as rn
    FROM course_completions cc
    INNER JOIN top_students ts 
        ON cc.user_id = ts.user_id
)
-- 3. 自连接形成课程对并统计
SELECT 
    rc1.course_name as first_course,
    rc2.course_name as second_course,
    COUNT(*) as transition_count
FROM ranked_courses rc1
INNER JOIN ranked_courses rc2 
    ON rc1.user_id = rc2.user_id 
    AND rc1.rn + 1 = rc2.rn  -- 连接相邻课程
GROUP BY rc1.course_name, rc2.course_name
ORDER BY 
    transition_count DESC,
    first_course ASC,
    second_course ASC;

4️⃣ 复杂度分析

解法一：窗口函数法

  • 时间复杂度：O(n log n)
    • 筛选顶尖学生：O(n)扫描 + O(1)聚合
    • 窗口函数排序：O(n log n)，n为顶尖学生的课程记录数
    • 分组统计：O(m)，m为课程对数量
  • 空间复杂度：O(n)
    • 存储中间结果：需要存储顶尖学生的所有课程记录

解法二：自连接法

  • 时间复杂度：O(n log n + m)
    • 排序：O(n log n)
    • 自连接：O(n)（有索引时），最坏O(n²)
    • 分组统计：O(m)
  • 空间复杂度：O(n)
    • 需要存储排序后的课程记录
