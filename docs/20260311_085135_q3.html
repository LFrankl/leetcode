题目描述

给你一个字符串 s ，找出它的所有子串并按字典序排列，返回排在最后的那个子串。

 

示例 1：

输入：s = "abab"
输出："bab"
解释：我们可以找出 7 个子串 ["a", "ab", "aba", "abab", "b", "ba", "bab"]。按字典序排在最后的子串是 "bab"。

输入：s = "leetcode"
输出："tcode"

 

提示：

  • 1 <= s.length <= 4 * 105
  • s 仅含有小写英文字符。

1️⃣ 题意理解

题目重述
给定一个字符串 s，找出它的所有子串中字典序最大的那个子串，并返回该子串。

关键约束与边界
- 字符串长度可达 4×10⁵，要求高效算法（O(n) 或 O(n log n)）。 - 仅包含小写英文字母。 - 子串是连续的字符序列。 - 字典序比较：从第一个字符开始逐字符比较 ASCII 码，直到出现不同或结束。

输入输出格式
- 输入：字符串 s - 输出：字典序最大的子串

2️⃣ 解题思路

解法一：暴力枚举（不可行）

  • 枚举所有子串，比较字典序。
  • 子串数量 O(n²)，比较 O(n)，总 O(n³)，不可行。

解法二：双指针（最大后缀）

核心思想
字典序最大的子串一定是从某个位置开始到末尾的后缀。
证明：若非后缀，可在其后追加字符使字典序更大。

因此问题转化为：找出所有后缀中字典序最大的那个。

优化思路
直接比较所有后缀仍为 O(n²)，需优化。
使用双指针 i 和 j，其中 i 是当前候选最大后缀起点，j 是待比较后缀起点，k 是当前比较的偏移量。

步骤
1. 初始化 i=0, j=1, k=0。 2. 比较 s[i+k] 和 s[j+k]： - 若相等，k++，继续比较。 - 若 s[i+k] < s[j+k]：说明 i 开头的后缀小于 j 开头的后缀，更新 i = max(i+k+1, j)，j = i+1，k=0。 - 若 s[i+k] > s[j+k]：说明 j 开头的后缀更小，更新 j = j+k+1，k=0。 3. 重复直到 j+k >= n。 4. 返回 s[i:]。

注意事项
- 更新 i 时取 max(i+k+1, j) 避免回退。 - 保证 i < j。

解法三：后缀数组（SA）

  • 构建后缀数组，最后一个后缀即为答案。
  • 可用 DC3 或 SA-IS 算法 O(n) 构建，但实现复杂。
  • 本题可简化为求最大后缀，无需完整后缀数组。

3️⃣ 代码实现

解法二：双指针（最大后缀）

func lastSubstring(s string) string {
    n := len(s)
    i, j, k := 0, 1, 0
    for j+k < n {
        if s[i+k] == s[j+k] {
            k++
            continue
        }
        if s[i+k] < s[j+k] {
            // i 开头的后缀更小，更新 i
            i = max(i+k+1, j)
            j = i + 1
        } else {
            // j 开头的后缀更小，更新 j
            j = j + k + 1
        }
        k = 0
    }
    return s[i:]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

4️⃣ 复杂度分析

解法二：双指针

  • 时间复杂度：O(n)
    每次比较至少移动 i 或 j，且 i 和 j 只增不减，最多比较 2n 次。
  • 空间复杂度：O(1)
    仅使用常数变量。

5️⃣ 解法对比

解法 优点 缺点 适用场景
双指针 O(n) 时间，O(1) 空间，实现简单 逻辑较绕，需仔细处理边界 本题最佳选择
后缀数组 通用性强，可解多种后缀问题 实现复杂，空间 O(n) 需要完整后缀排序时
