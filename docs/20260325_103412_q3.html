2263. 数组变为有序的最小操作次数

代码模板

提示

  1. To find the minimum cost to make nums non-decreasing, use a max heap that contains all of the elements that you have seen so far.
  2. If the largest number in the heap is greater than the current number, calculate the cost to make the two numbers equal. Then, in the heap, replace the largest number with the current number.
  3. Now that we have found the minimum cost to make nums non-decreasing, we can set every nums[i] to -nums[i] and do the same process to find the minimum cost to make nums non-increasing.

1️⃣ 题意理解

题目重述： 给定一个整数数组，每次操作可以交换数组中任意两个元素的位置。问最少需要多少次交换操作，才能使数组变为非递减有序（即每个元素都小于等于下一个元素）。

关键约束： - 操作只能交换任意两个元素的位置 - 目标是使数组变为非递减有序 - 数组长度 n 最大可能达到 10^5 - 数组元素可能包含重复值

边界情况： - 数组已经有序：需要 0 次操作 - 数组完全逆序：需要 n-1 次操作 - 数组包含重复元素 - 数组长度为 0 或 1：需要 0 次操作

输入输出： - 输入：整数数组 arr - 输出：最少交换次数（整数）

2️⃣ 解题思路

解法一：基于排序的贪心算法（推荐）

核心思想： 1. 将原数组排序得到目标数组 2. 通过交换操作将原数组变为目标数组 3. 每个元素应该被交换到它最终应该在的位置 4. 使用哈希表记录每个元素的位置，处理重复元素

解题步骤： 1. 复制原数组并排序，得到目标数组 2. 建立哈希表，记录每个值在原数组中的位置（使用队列处理重复值） 3. 遍历原数组，对于每个位置 i： - 如果当前位置的元素已经是目标元素，跳过 - 否则，从哈希表中取出目标元素的位置，进行交换 - 更新哈希表中两个元素的位置信息 4. 统计交换次数

注意事项： - 需要处理重复元素，使用队列存储每个值的所有位置 - 交换后要及时更新位置信息 - 时间复杂度要控制在 O(n log n) 以内

解法二：基于环分解的贪心算法（最优）

核心思想： 1. 将数组排序，得到每个元素最终应该在的位置 2. 将交换操作视为环的分解 3. 对于每个环，需要的交换次数是环的大小减 1 4. 总交换次数 = 所有环的 (环大小 - 1) 之和

解题步骤： 1. 复制原数组并排序，得到目标数组 2. 建立值到索引的映射（处理重复值） 3. 遍历原数组，对于每个未访问的位置： - 如果当前位置的元素已经在正确位置，跳过 - 否则，沿着环追踪，直到回到起点 - 统计环的大小 4. 计算总交换次数

3️⃣ 代码实现

解法一：基于排序的贪心算法

func minSwapsToSort(arr []int) int {
    n := len(arr)
    if n <= 1 {
        return 0
    }

    // 创建排序后的目标数组
    target := make([]int, n)
    copy(target, arr)
    sort.Ints(target)

    // 建立值到位置的映射（使用队列处理重复值）
    posMap := make(map[int][]int)
    for i, num := range arr {
        posMap[num] = append(posMap[num], i)
    }

    swaps := 0
    // 遍历数组，将每个元素放到正确位置
    for i := 0; i < n; i++ {
        // 如果当前位置已经是目标值，跳过
        if arr[i] == target[i] {
            continue
        }

        // 找到目标值在原数组中的位置
        targetNum := target[i]
        positions := posMap[targetNum]

        // 取出第一个位置（队列）
        j := positions[0]
        posMap[targetNum] = positions[1:]

        // 交换 arr[i] 和 arr[j]
        arr[i], arr[j] = arr[j], arr[i]
        swaps++

        // 更新被交换元素的位置信息
        swappedNum := arr[j]
        // 从原位置列表中移除 j，添加 i
        posList := posMap[swappedNum]
        for k, pos := range posList {
            if pos == j {
                posList[k] = i
                break
            }
        }
    }

    return swaps
}

解法二：基于环分解的贪心算法（最优解）

func minSwapsToSort(arr []int) int {
    n := len(arr)
    if n <= 1 {
        return 0
    }

    // 创建排序后的目标数组
    target := make([]int, n)
    copy(target, arr)
    sort.Ints(target)

    // 建立值到索引的映射（处理重复值）
    // 使用队列存储每个值的所有位置
    posMap := make(map[int][]int)
    for i, num := range arr {
        posMap[num] = append(posMap[num], i)
    }

    // 访问标记数组
    visited := make([]bool, n)
    swaps := 0

    for i := 0; i < n; i++ {
        // 如果已经访问过或已经在正确位置，跳过
        if visited[i] || arr[i] == target[i] {
            continue
        }

        // 开始追踪一个环
        cycleSize := 0
        j := i

        for !visited[j] {
            visited[j] = true
            cycleSize++

            // 找到 arr[j] 在目标数组中的位置
            targetNum := target[j]
            // 从队列中取出一个位置
            positions := posMap[targetNum]
            nextPos := positions[0]
            posMap[targetNum] = positions[1:]

            j = nextPos
        }

        // 环的大小为 cycleSize，需要 cycleSize-1 次交换
        if cycleSize > 1 {
            swaps += cycleSize - 1
        }
    }

    return swaps
}

4️⃣ 复杂度分析

解法一：基于排序的贪心算法

  • ⏱️ 时间复杂度：O(n log n + n²)
    • 排序：O(n log n)
    • 遍历交换：最坏情况 O(n²)（每次交换都需要在列表中查找位置）
  • 💾 空间复杂度：O(n)
    • 存储目标数组：O(n)
    • 存储位置映射：O(n)

解法二：基于环分解的贪心算法

  • ⏱️ 时间复杂度：O(n log n)
    • 排序：O(n log n)
    • 环分解：O(n)（每个元素只访问一次）
  • 💾 空间复杂度：O(n)
    • 存储目标数组：O(n)
    • 存储位置映射：O(n)
    • 访问标记数组：O(n)
