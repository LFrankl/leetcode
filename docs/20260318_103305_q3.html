1458. 两个子序列的最大点积

难度: Hard

标签: 数组, 动态规划

题目描述

给你两个数组 nums1 和 nums2 。

请你返回 nums1 和 nums2 中两个长度相同的 非空 子序列的最大点积。

数组的非空子序列是通过删除原数组中某些元素（可能一个也不删除）后剩余数字组成的序列，但不能改变数字间相对顺序。比方说，[2,3,5] 是 [1,2,3,4,5] 的一个子序列而 [1,5,3] 不是。

 

示例 1：

输入：nums1 = [2,1,-2,5], nums2 = [3,0,-6]
输出：18
解释：从 nums1 中得到子序列 [2,-2] ，从 nums2 中得到子序列 [3,-6] 。
它们的点积为 (2*3 + (-2)*(-6)) = 18 。

示例 2：

输入：nums1 = [3,-2], nums2 = [2,-6,7]
输出：21
解释：从 nums1 中得到子序列 [3] ，从 nums2 中得到子序列 [7] 。
它们的点积为 (3*7) = 21 。

示例 3：

输入：nums1 = [-1,-1], nums2 = [1,1]
输出：-1
解释：从 nums1 中得到子序列 [-1] ，从 nums2 中得到子序列 [1] 。
它们的点积为 -1 。

 

提示：

  • 1 <= nums1.length, nums2.length <= 500
  • -1000 <= nums1[i], nums2[i] <= 1000

1️⃣ 题意理解

题目重述
给定两个数组 nums1 和 nums2，需要从每个数组中分别选出一个长度相同的非空子序列，使得这两个子序列的点积（对应位置元素乘积之和）最大。

关键约束
- 子序列：保持原数组顺序，但不要求连续
- 两个子序列长度必须相同
- 子序列必须非空
- 数组长度 ≤ 500，元素值范围 [-1000, 1000]
- 点积可能为负数（如示例3）

输入输出格式
- 输入：两个整数数组 nums1 和 nums2
- 输出：一个整数，表示最大点积

2️⃣ 解题思路

解法一：动态规划（二维DP）

核心思想
类似最长公共子序列（LCS）的DP思路，但状态转移时计算点积。
定义 dp[i][j] 表示 nums1[0..i-1] 和 nums2[0..j-1] 中选出的两个等长子序列的最大点积。
状态转移考虑四种情况： 1. 选择 nums1[i-1] 和 nums2[j-1] 配对，点积为 nums1[i-1]*nums2[j-1]，可能加上之前的最大点积 dp[i-1][j-1]（若为正） 2. 不选 nums1[i-1]，即 dp[i-1][j] 3. 不选 nums2[j-1]，即 dp[i][j-1] 4. 只选当前两个数配对（相当于重新开始选子序列）

注意事项
- 初始化时 dp[0][j] 和 dp[i][0] 设为负无穷（因为子序列必须非空）
- 最终答案可能为负数（如示例3）
- 需要处理负数的乘积可能使点积变大的情况

解法二：动态规划（空间优化）

核心思想
由于 dp[i][j] 只依赖于 dp[i-1][j-1]、dp[i-1][j] 和 dp[i][j-1]，可以用滚动数组优化空间。

注意事项
- 需要保存 dp[i-1][j-1] 的值，因为会被覆盖
- 注意更新顺序

3️⃣ 代码实现

解法一：二维动态规划

func maxDotProduct(nums1 []int, nums2 []int) int {
    m, n := len(nums1), len(nums2)
    // dp[i][j] 表示 nums1[0..i-1] 和 nums2[0..j-1] 的最大点积
    dp := make([][]int, m+1)
    for i := range dp {
        dp[i] = make([]int, n+1)
        for j := range dp[i] {
            dp[i][j] = -1 << 31 // 初始化为极小值
        }
    }

    for i := 1; i <= m; i++ {
        for j := 1; j <= n; j++ {
            // 情况1：只选当前两个数配对
            cur := nums1[i-1] * nums2[j-1]
            // 情况2：当前两个数配对，并加上之前的最大点积（如果为正）
            if dp[i-1][j-1] > 0 {
                cur += dp[i-1][j-1]
            }
            // 情况3：不选 nums1[i-1]
            // 情况4：不选 nums2[j-1]
            dp[i][j] = max(cur, max(dp[i-1][j], dp[i][j-1]))
        }
    }
    return dp[m][n]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

解法二：空间优化动态规划

func maxDotProduct(nums1 []int, nums2 []int) int {
    m, n := len(nums1), len(nums2)
    // 使用一维数组，dp[j] 表示当前行
    dp := make([]int, n+1)
    for j := range dp {
        dp[j] = -1 << 31
    }

    for i := 1; i <= m; i++ {
        // pre 保存 dp[i-1][j-1]
        pre := dp[0]
        dp[0] = -1 << 31 // 每行开始重置
        for j := 1; j <= n; j++ {
            tmp := dp[j] // 保存当前 dp[i-1][j] 给下一轮作为 pre
            cur := nums1[i-1] * nums2[j-1]
            if pre > 0 {
                cur += pre
            }
            dp[j] = max(cur, max(dp[j], dp[j-1]))
            pre = tmp // 更新 pre 为 dp[i-1][j]
        }
    }
    return dp[n]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

4️⃣ 复杂度分析

解法一：二维DP

  • ⏱️ 时间复杂度：O(m×n)，其中 m 和 n 分别为两个数组的长度。需要填充 m×n 的DP表。
  • 💾 空间复杂度：O(m×n)，用于存储DP表。

解法二：空间优化DP

  • ⏱️ 时间复杂度：O(m×n)，与解法一相同。
  • 💾 空间复杂度：O(n)，只使用一维数组，n 为第二个数组的长度。

5️⃣ 解法对比

解法 优点 缺点 适用场景
二维DP 逻辑清晰，易于理解和调试 空间占用较大 数组长度较小（≤500）时完全可行
空间优化DP 空间效率高，适合内存敏感场景 代码稍复杂，状态转移需要小心处理 数组长度较大或内存受限时

推荐
本题数组长度最大为500，二维DP的空间复杂度 O(500×500) = O(250k) 在可接受范围内。推荐使用解法一，因为它逻辑更清晰，不易出错。在实际面试中，可以先实现解法一，再优化到解法二，展示优化能力。
