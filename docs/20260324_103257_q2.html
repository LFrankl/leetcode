1937. 扣分后的最大得分

难度: Medium

标签: 数组, 动态规划, 矩阵

链接: https://leetcode.cn/problems/maximum-number-of-points-with-cost/

题目描述

给你一个 m x n 的整数矩阵 points （下标从 0 开始）。一开始你的得分为 0 ，你想最大化从矩阵中得到的分数。

你的得分方式为：每一行 中选取一个格子，选中坐标为 (r, c) 的格子会给你的总得分 增加 points[r][c] 。

然而，相邻行之间被选中的格子如果隔得太远，你会失去一些得分。对于相邻行 r 和 r + 1 （其中 0 <= r < m - 1），选中坐标为 (r, c1) 和 (r + 1, c2) 的格子，你的总得分 减少 abs(c1 - c2) 。

请你返回你能得到的 最大 得分。

abs(x) 定义为：

  • 如果 x >= 0 ，那么值为 x 。
  • 如果 x < 0 ，那么值为 -x 。

 

示例 1：

输入：points = [[1,2,3],[1,5,1],[3,1,1]]
输出：9
解释：
蓝色格子是最优方案选中的格子，坐标分别为 (0, 2)，(1, 1) 和 (2, 0) 。
你的总得分增加 3 + 5 + 3 = 11 。
但是你的总得分需要扣除 abs(2 - 1) + abs(1 - 0) = 2 。
你的最终得分为 11 - 2 = 9 。

示例 2：

输入：points = [[1,5],[2,3],[4,2]]
输出：11
解释：
蓝色格子是最优方案选中的格子，坐标分别为 (0, 1)，(1, 1) 和 (2, 0) 。
你的总得分增加 5 + 3 + 4 = 12 。
但是你的总得分需要扣除 abs(1 - 1) + abs(1 - 0) = 1 。
你的最终得分为 12 - 1 = 11 。

 

提示：

  • m == points.length
  • n == points[r].length
  • 1 <= m, n <= 105
  • 1 <= m * n <= 105
  • 0 <= points[r][c] <= 105

1️⃣ 题意理解

题目重述

我们有一个 m x n 的矩阵 points，需要从每一行中选取一个格子，选取 (r, c) 会获得 points[r][c] 的分数。但是，相邻两行选取的格子列坐标差的绝对值会作为扣分。目标是最大化总得分（每行得分之和减去相邻行列坐标差的绝对值之和）。

关键约束

  • 必须从每一行选且只选一个格子
  • 相邻行之间的扣分 = abs(c1 - c2)
  • 最终得分 = 所有选中格子的 points 值之和 - 所有相邻行扣分之和

输入输出

  • 输入：二维整数数组 points，m 行 n 列
  • 输出：一个整数，表示最大得分

2️⃣ 解题思路

解法一：动态规划（朴素版）

核心思想：
定义 dp[i][j] 表示处理到第 i 行，且第 i 行选择第 j 列时的最大得分。状态转移需要考虑上一行的所有可能列选择，并扣除列坐标差的绝对值。

步骤： 1. 初始化 dp[0][j] = points[0][j] 2. 对于 i = 1 to m-1，对于每个 j = 0 to n-1： - dp[i][j] = points[i][j] + max_{k=0..n-1} (dp[i-1][k] - abs(j - k)) 3. 最终答案为 max(dp[m-1][j])

坑点： - 直接计算需要三重循环，时间复杂度 O(m·n²)，在 n 较大时会超时

解法二：动态规划 + 左右预处理优化

核心思想：
对于固定的 j，我们需要计算 max(dp[i-1][k] - abs(j - k))。
将绝对值拆分为两种情况： - 当 k ≤ j 时，abs(j - k) = j - k，所以表达式为 dp[i-1][k] + k - j - 当 k ≥ j 时，abs(j - k) = k - j，所以表达式为 dp[i-1][k] - k + j

因此，我们可以预处理两个数组： - left_max[k] = max_{t ≤ k} (dp[i-1][t] + t) - right_max[k] = max_{t ≥ k} (dp[i-1][t] - t)

那么对于每个 j： max_val = max(left_max[j] - j, right_max[j] + j)

步骤： 1. 初始化 dp 为 points[0] 2. 对于每一行 i = 1 to m-1： - 计算 left_max 和 right_max 数组 - 对于每个 j，dp[i][j] = points[i][j] + max(left_max[j] - j, right_max[j] + j) 3. 返回 max(dp[m-1])

注意事项： - 注意数组索引边界 - 使用滚动数组可优化空间复杂度

3️⃣ 代码实现

解法一：朴素动态规划（会超时，仅作理解用）

func maxPoints(points [][]int) int64 {
    m, n := len(points), len(points[0])
    dp := make([][]int64, m)
    for i := range dp {
        dp[i] = make([]int64, n)
    }
    // 初始化第一行
    for j := 0; j < n; j++ {
        dp[0][j] = int64(points[0][j])
    }
    // 动态规划
    for i := 1; i < m; i++ {
        for j := 0; j < n; j++ {
            best := int64(-1 << 63)
            for k := 0; k < n; k++ {
                val := dp[i-1][k] - int64(abs(j-k))
                if val > best {
                    best = val
                }
            }
            dp[i][j] = int64(points[i][j]) + best
        }
    }
    // 找出最后一行最大值
    ans := dp[m-1][0]
    for j := 1; j < n; j++ {
        if dp[m-1][j] > ans {
            ans = dp[m-1][j]
        }
    }
    return ans
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}

解法二：优化动态规划（推荐）

func maxPoints(points [][]int) int64 {
    m, n := len(points), len(points[0])
    // 使用滚动数组，prev 表示上一行的 dp 值
    prev := make([]int64, n)
    for j := 0; j < n; j++ {
        prev[j] = int64(points[0][j])
    }

    for i := 1; i < m; i++ {
        // 计算 left_max: left_max[j] = max_{k<=j} (prev[k] + k)
        leftMax := make([]int64, n)
        leftMax[0] = prev[0] // 注意：这里应该是 prev[0] + 0，但0可以省略
        for j := 1; j < n; j++ {
            leftMax[j] = max(leftMax[j-1], prev[j]+int64(j))
        }

        // 计算 right_max: right_max[j] = max_{k>=j} (prev[k] - k)
        rightMax := make([]int64, n)
        rightMax[n-1] = prev[n-1] - int64(n-1)
        for j := n-2; j >= 0; j-- {
            rightMax[j] = max(rightMax[j+1], prev[j]-int64(j))
        }

        // 计算当前行的 dp 值
        curr := make([]int64, n)
        for j := 0; j < n; j++ {
            // 两种情况取最大值
            best := max(leftMax[j]-int64(j), rightMax[j]+int64(j))
            curr[j] = int64(points[i][j]) + best
        }

        // 滚动数组
        prev = curr
    }

    // 找出最大值
    ans := prev[0]
    for j := 1; j < n; j++ {
        if prev[j] > ans {
            ans = prev[j]
        }
    }
    return ans
}

func max(a, b int64) int64 {
    if a > b {
        return a
    }
    return b
}

4️⃣ 复杂度分析

解法一：朴素动态规划

  • ⏱️ 时间复杂度：O(m·n²)
    三重循环：外层 m-1 次，中层 n 次，内层 n 次
  • 💾 空间复杂度：O(m·n)
    需要存储整个 dp 表

解法二：优化动态规划

  • ⏱️ 时间复杂度：O(m·n)
    对每一行：计算 leftMax 需要 O(n)，计算 rightMax 需要 O(n)，计算 curr 需要 O(n)
  • 💾 空间复杂度：O(n)
    使用滚动数组，只需要存储 prev、leftMax、rightMax、curr 四个长度为 n 的数组

5️⃣ 解法对比

解法 优点 缺点 适用场景
朴素DP 思路直观，易于理解 时间复杂度高，n 较大时超时 仅适用于 n 很小的情况
优化DP 时间复杂度最优，可处理较大数据 实现稍复杂，需要理解绝对值拆分技巧 推荐使用，适用于所有数据范围

推荐解法二：
在 LeetCode 的约束下（m, n ≤ 10^5），朴素解法必然超时。优化解法将时间复杂度从 O(m·n²) 降为 O(m·n)，是解决此问题的标准方法。
