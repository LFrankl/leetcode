LCP 33. 蓄水

难度: Easy

标签: 贪心, 数组, 堆（优先队列）

链接: https://leetcode.cn/problems/o8SXZn/

题目描述

给定 N 个无限容量且初始均空的水缸，每个水缸配有一个水桶用来打水，第 i 个水缸配备的水桶容量记作 bucket[i]。小扣有以下两种操作： - 升级水桶：选择任意一个水桶，使其容量增加为 bucket[i]+1 - 蓄水：将全部水桶接满水，倒入各自对应的水缸

每个水缸对应最低蓄水量记作 vat[i]，返回小扣至少需要多少次操作可以完成所有水缸蓄水要求。

注意：实际蓄水量 达到或超过 最低蓄水量，即完成蓄水要求。

示例 1：

输入：bucket = [1,3], vat = [6,8]

输出：4

解释： 第 1 次操作升级 bucket[0]； 第 2 ~ 4 次操作均选择蓄水，即可完成蓄水要求。

示例 2：

输入：bucket = [9,0,1], vat = [0,2,2]

输出：3

解释： 第 1 次操作均选择升级 bucket[1] 第 2~3 次操作选择蓄水，即可完成蓄水要求。

提示： - 1 <= bucket.length == vat.length <= 100 - 0 <= bucket[i], vat[i] <= 10^4

1️⃣ 题意理解

核心要求：
我们有 N 个水缸，每个水缸有一个对应的水桶。水桶初始容量为 bucket[i]，水缸需要达到的最低蓄水量为 vat[i]。允许两种操作： 1. 升级水桶：将某个水桶容量 +1（每次只能升级一个水桶） 2. 蓄水：将所有水桶接满水，倒入对应水缸（一次性全部倒水）

求完成所有水缸蓄水要求（每个水缸实际水量 ≥ vat[i]）的 最少操作次数。

关键约束： - 水桶容量可以无限升级 - 蓄水操作是全局的，一次性倒所有水桶 - 水缸容量无限，只需达到最低蓄水量即可 - 数据范围：N ≤ 100，vat[i] ≤ 10^4

边界情况： - 如果 vat[i] = 0，则该水缸无需蓄水 - 如果 bucket[i] = 0 且 vat[i] > 0，则必须先升级水桶（否则蓄水操作无效） - 操作次数可能为 0（所有 vat[i] = 0）

输入输出： - 输入：bucket []int, vat []int - 输出：最少操作次数 int

2️⃣ 解题思路

解法一：枚举蓄水次数 + 贪心计算升级次数

核心思想： - 如果我们确定最终要进行 k 次蓄水操作，那么对于第 i 个水缸： - 每次蓄水能倒入 bucket[i] 的水 - 经过 k 次蓄水后总水量为 k * bucket[i] - 需要满足 k * bucket[i] >= vat[i] - 如果 bucket[i] 不够大，我们需要先升级水桶 - 对于给定的 k，计算每个水缸需要的最小水桶容量： - 最小需要容量 need = ceil(vat[i] / k) - 升级次数 = max(0, need - bucket[i]) - 总操作次数 = k + sum(升级次数) - 枚举所有可能的 k（从 1 到 max(vat)），取最小值

注意事项： - 当 k = 0 时，表示不蓄水，此时需要所有 vat[i] = 0 - 当 vat[i] = 0 时，不需要升级和蓄水 - 需要处理 bucket[i] = 0 的情况

解法二：优先队列（更优的枚举）

核心思想： - 初始时，如果 bucket[i] = 0 且 vat[i] > 0，必须至少升级一次 - 每次操作可以选择： 1. 升级当前“瓶颈”水桶（对总次数减少最有利） 2. 直接蓄水 - 使用优先队列维护每个水缸的“当前所需蓄水次数”，每次选择升级能最大程度减少总操作次数的水桶 - 但实现较复杂，且最坏情况仍需枚举

3️⃣ 代码实现

解法一：枚举蓄水次数

func storeWater(bucket []int, vat []int) int {
    n := len(bucket)
    maxVat := 0
    for _, v := range vat {
        if v > maxVat {
            maxVat = v
        }
    }

    // 特殊情况：所有水缸都不需要水
    if maxVat == 0 {
        return 0
    }

    ans := math.MaxInt32

    // 枚举蓄水次数 k
    for k := 1; k <= maxVat; k++ {
        // 如果 k 已经大于当前最优解，可以提前结束
        if k >= ans {
            break
        }

        upgrade := 0
        for i := 0; i < n; i++ {
            // 当前水缸需要的最小桶容量
            need := (vat[i] + k - 1) / k // 向上取整
            if need > bucket[i] {
                upgrade += need - bucket[i]
            }
        }
        total := k + upgrade
        if total < ans {
            ans = total
        }
    }

    return ans
}

解法二：优化枚举（减少枚举范围）

func storeWater(bucket []int, vat []int) int {
    n := len(bucket)
    maxVat := 0
    for _, v := range vat {
        if v > maxVat {
            maxVat = v
        }
    }

    if maxVat == 0 {
        return 0
    }

    ans := math.MaxInt32

    // 优化：k 不需要枚举到 maxVat，因为当 k 很大时，升级次数会很多
    // k 的上界可以设为 maxVat（当某个 bucket=1 时）
    for k := 1; k <= maxVat && k < ans; k++ {
        upgrade := 0
        for i := 0; i < n; i++ {
            if vat[i] == 0 {
                continue
            }
            // 需要的最小桶容量
            need := (vat[i] + k - 1) / k
            if need > bucket[i] {
                upgrade += need - bucket[i]
            }
        }
        if k+upgrade < ans {
            ans = k + upgrade
        }
    }

    // 特殊情况：如果 ans 还是 MaxInt32，说明需要处理 bucket[i]=0 的情况
    // 实际上这种情况已经被上面的循环覆盖了

    return ans
}

4️⃣ 复杂度分析

解法一：

  • ⏱️ 时间复杂度：O(N × M)，其中 N = len(bucket)，M = max(vat)
    • 外层循环最多枚举 M 次（M ≤ 10^4）
    • 内层循环 N 次（N ≤ 100）
    • 总操作数 ≤ 10^4 × 100 = 10^6，可接受
  • 💾 空间复杂度：O(1)，只使用了常数额外空间

解法二：

  • ⏱️ 时间复杂度：O(N × M)，与解法一相同，但实际枚举次数可能更少
  • 💾 空间复杂度：O(1)
