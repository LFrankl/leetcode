274. H 指数

难度: Medium

标签: 数组, 计数排序, 排序

链接: https://leetcode.cn/problems/h-index/

题目描述

给你一个整数数组 citations ，其中 citations[i] 表示研究者的第 i 篇论文被引用的次数。计算并返回该研究者的 h 指数。

根据维基百科上 h 指数的定义：h 代表“高引用次数” ，一名科研人员的 h 指数 是指他（她）至少发表了 h 篇论文，并且 至少 有 h 篇论文被引用次数大于等于 h 。如果 h 有多种可能的值，h 指数 是其中最大的那个。

 

示例 1：

输入：citations = [3,0,6,1,5]
输出：3 
解释：给定数组表示研究者总共有 5 篇论文，每篇论文相应的被引用了 3, 0, 6, 1, 5 次。
     由于研究者有 3 篇论文每篇 至少 被引用了 3 次，其余两篇论文每篇被引用 不多于 3 次，所以她的 h 指数是 3。

示例 2：

输入：citations = [1,3,1]
输出：1

 

提示：

  • n == citations.length
  • 1 <= n <= 5000
  • 0 <= citations[i] <= 1000

AI 解答 (DeepSeek)

解法一：排序法

思路：将 citations 从大到小排序，然后从前往后扫描，找到最大的 h 使得至少有 h 篇论文的引用次数 ≥ h。排序后，第 i 个位置（从0开始）表示第 i+1 篇最高引用的论文，若 citations[i] ≥ i+1，说明前 i+1 篇论文都满足引用次数 ≥ i+1，继续增大 h；否则 break，当前 h 即为答案。

func hIndex(citations []int) int {
    sort.Sort(sort.Reverse(sort.IntSlice(citations)))
    h := 0
    for i, c := range citations {
        if c >= i+1 {
            h = i + 1
        } else {
            break
        }
    }
    return h
}

复杂度：时间 O(n log n)，空间 O(1)（如果排序不算额外空间）/ O(log n)（排序栈空间）

解法二：计数排序法（最优）

思路：h 指数最大不会超过论文总数 n，所以对于引用次数 > n 的论文，统一归入 n 这个桶即可。用长度为 n+1 的数组 count 记录每个引用次数对应的论文数量，然后从大到小累加论文数，当累计论文数 ≥ 当前引用次数时，该引用次数即为 h 指数。

func hIndex(citations []int) int {
    n := len(citations)
    count := make([]int, n+1) // count[i]: 引用次数为i的论文数
    for _, c := range citations {
        if c >= n {
            count[n]++
        } else {
            count[c]++
        }
    }
    total := 0 // 累计论文数
    for i := n; i >= 0; i-- {
        total += count[i]
        if total >= i {
            return i
        }
    }
    return 0
}

复杂度：时间 O(n)，空间 O(n)

核心直觉：h 指数的定义是"至少 h 篇论文引用 ≥ h"，排序法把"引用 ≥ h"直观呈现，计数法则利用 h ≤ n 的性质避免了排序，将复杂度从 O(n log n) 优化到 O(n)。
