1697. 检查边长度限制的路径是否存在

难度: Hard

标签: 并查集, 图, 数组, 双指针, 排序

链接: https://leetcode.cn/problems/checking-existence-of-edge-length-limited-paths/

题目描述

给你一个 n 个点组成的无向图边集 edgeList ，其中 edgeList[i] = [ui, vi, disi] 表示点 ui 和点 vi 之间有一条长度为 disi 的边。请注意，两个点之间可能有 超过一条边 。

给你一个查询数组queries ，其中 queries[j] = [pj, qj, limitj] ，你的任务是对于每个查询 queries[j] ，判断是否存在从 pj 到 qj 的路径，且这条路径上的每一条边都 严格小于 limitj 。

请你返回一个 布尔数组 answer ，其中 answer.length == queries.length ，当 queries[j] 的查询结果为 true 时， answer 第 j 个值为 true ，否则为 false 。

 

示例 1：

输入：n = 3, edgeList = [[0,1,2],[1,2,4],[2,0,8],[1,0,16]], queries = [[0,1,2],[0,2,5]]
输出：[false,true]
解释：上图为给定的输入数据。注意到 0 和 1 之间有两条重边，分别为 2 和 16 。
对于第一个查询，0 和 1 之间没有小于 2 的边，所以我们返回 false 。
对于第二个查询，有一条路径（0 -> 1 -> 2）两条边都小于 5 ，所以这个查询我们返回 true 。

示例 2：

输入：n = 5, edgeList = [[0,1,10],[1,2,5],[2,3,9],[3,4,13]], queries = [[0,4,14],[1,4,13]]
输出：[true,false]
解释：上图为给定数据。

 

提示：

  • 2 <= n <= 105
  • 1 <= edgeList.length, queries.length <= 105
  • edgeList[i].length == 3
  • queries[j].length == 3
  • 0 <= ui, vi, pj, qj <= n - 1
  • ui != vi
  • pj != qj
  • 1 <= disi, limitj <= 109
  • 两个点之间可能有 多条 边。

1️⃣ 题意理解

题目重述
给定一个无向图（可能有重边），每个查询要求判断：是否存在一条从节点 p 到节点 q 的路径，使得路径上每条边的长度都严格小于给定的限制值 limit。

关键约束
- 两个节点之间可能有多条边（重边），查询时只关心边权是否小于 limit
- 路径不要求最短，只要求存在性
- 边权严格小于 limit（不能等于）
- 节点数 n ≤ 10^5，边数 m ≤ 10^5，查询数 q ≤ 10^5，要求高效处理

输入输出格式
- 输入：
- n：节点数
- edgeList：边列表，每条边 [u, v, dis]
- queries：查询列表，每个查询 [p, q, limit]
- 输出：
- answer：布尔数组，长度等于查询数，每个元素表示对应查询是否存在满足条件的路径

2️⃣ 解题思路

解法一：离线查询 + 并查集（Union-Find）

核心思想
将查询按 limit 从小到大排序，同时将边按边权从小到大排序。
按顺序处理查询：对于当前查询的 limit，将所有边权小于 limit 的边加入并查集，然后判断 p 和 q 是否连通。
因为查询是离线的，所以可以复用并查集状态，避免重复计算。

步骤
1. 将边按边权升序排序
2. 将查询按 limit 升序排序，并记录原始索引
3. 初始化并查集
4. 遍历排序后的查询：
- 将边权小于当前 limit 的边加入并查集（双指针推进）
- 检查 p 和 q 是否连通
- 将结果存入原始索引位置
5. 返回结果数组

注意事项
- 查询排序后要记录原始索引，以便正确输出
- 边权严格小于 limit，所以加入并查集的边权必须 < limit
- 并查集需要路径压缩和按秩合并以保证效率

解法二：在线查询 + 二分 + BFS/DFS（不推荐）

核心思想
对每个查询，构建一个只包含边权 < limit 的子图，然后检查 p 和 q 是否连通。
由于查询数多，每次重新建图会超时，但可以结合二分优化：
- 预处理所有边权，对每个查询二分查找满足条件的边集
- 但连通性检查仍需遍历，整体复杂度高，不适用于本题规模

步骤
1. 将边按边权排序
2. 对每个查询：
- 二分找到所有边权 < limit 的边
- 用 BFS/DFS 检查连通性
3. 返回结果

缺点
- 每个查询都要重新建图或遍历，复杂度 O(q * (m + n))，会超时

3️⃣ 代码实现

解法一：离线查询 + 并查集

func distanceLimitedPathsExist(n int, edgeList [][]int, queries [][]int) []bool {
    m := len(edgeList)
    q := len(queries)

    // 1. 将边按边权升序排序
    sort.Slice(edgeList, func(i, j int) bool {
        return edgeList[i][2] < edgeList[j][2]
    })

    // 2. 将查询按 limit 升序排序，并记录原始索引
    queryIdx := make([][]int, q)
    for i := range queries {
        queryIdx[i] = []int{queries[i][0], queries[i][1], queries[i][2], i}
    }
    sort.Slice(queryIdx, func(i, j int) bool {
        return queryIdx[i][2] < queryIdx[j][2]
    })

    // 3. 初始化并查集
    parent := make([]int, n)
    rank := make([]int, n)
    for i := 0; i < n; i++ {
        parent[i] = i
    }

    // 4. 处理查询
    ans := make([]bool, q)
    edgeIdx := 0
    for _, query := range queryIdx {
        p, q, limit, idx := query[0], query[1], query[2], query[3]
        // 将所有边权小于 limit 的边加入并查集
        for edgeIdx < m && edgeList[edgeIdx][2] < limit {
            u, v := edgeList[edgeIdx][0], edgeList[edgeIdx][1]
            union(parent, rank, u, v)
            edgeIdx++
        }
        // 检查 p 和 q 是否连通
        ans[idx] = find(parent, p) == find(parent, q)
    }
    return ans
}

// 并查集查找（带路径压缩）
func find(parent []int, x int) int {
    if parent[x] != x {
        parent[x] = find(parent, parent[x])
    }
    return parent[x]
}

// 并查集合并（带按秩合并）
func union(parent, rank []int, x, y int) {
    rootX := find(parent, x)
    rootY := find(parent, y)
    if rootX != rootY {
        if rank[rootX] < rank[rootY] {
            parent[rootX] = rootY
        } else if rank[rootX] > rank[rootY] {
            parent[rootY] = rootX
        } else {
            parent[rootY] = rootX
            rank[rootX]++
        }
    }
}

4️⃣ 复杂度分析

解法一：离线查询 + 并查集

  • ⏱️ 时间复杂度：O((m + q) * α(n) + m log m + q log q)
    • 排序边：O(m log m)
    • 排序查询：O(q log q)
    • 并查集操作：O((m + q) * α(n))，其中 α(n) 是反阿克曼函数，近似常数
  • 💾 空间复杂度：O(n + m + q)
    • 并查集数组：O(n)
    • 排序查询的额外存储：O(q)
    • 边列表和查询列表：O(m + q)

解法二：在线查询 + 二分 + BFS/DFS（不推荐）

  • ⏱️ 时间复杂度：O(q * (m + n))
    • 每个查询都要遍历边和节点
  • 💾 空间复杂度：O(m + n)
    • 存储图和访问数组

5️⃣ 解法对比

解法 优点 缺点 适用场景
离线查询 + 并查集 高效，复用并查集状态，适合大规模数据 需要离线处理，不能实时回答查询 查询可离线，n, m, q 都很大（10^5 级别）
在线查询 + 二分 + BFS/DFS 直观，可实时回答查询 每个查询都要重新检查，超时 查询数很少或数据规模小

推荐解法：离线查询 + 并查集
原因：本题 n, m, q 最大可达 10^5，必须使用高效算法。离线查询将复杂度从 O(q * (m + n)) 降为 O((m + q) log (m + q))，能够通过所有测试。

输入：

解释：
- limit=1：没有边权 < 1 的边，false
- limit=2：有边权 1 的边，true
- limit=3：有边权 1,2 的边，true
- limit=4：所有边都满足，true

- 图分为两个连通分量 {0,1} 和 {2,3}
- 任何查询都无法跨分量连通
