2243. 计算字符串的数字和

难度: Easy

标签: 字符串, 模拟

链接: https://leetcode.cn/problems/calculate-digit-sum-of-a-string/

题目描述

给你一个由若干数字（0 - 9）组成的字符串 s ，和一个整数。

如果 s 的长度大于 k ，则可以执行一轮操作。在一轮操作中，需要完成以下工作：

  1. 将 s 拆分 成长度为 k 的若干 连续数字组 ，使得前 k 个字符都分在第一组，接下来的 k 个字符都分在第二组，依此类推。注意，最后一个数字组的长度可以小于 k 。
  2. 用表示每个数字组中所有数字之和的字符串来 替换 对应的数字组。例如，"346" 会替换为 "13" ，因为 3 + 4 + 6 = 13 。
  3. 合并 所有组以形成一个新字符串。如果新字符串的长度大于 k 则重复第一步。

返回在完成所有轮操作后的 s 。

 

示例 1：

输入：s = "11111222223", k = 3
输出："135"
解释：
- 第一轮，将 s 分成："111"、"112"、"222" 和 "23" 。
  接着，计算每一组的数字和：1 + 1 + 1 = 3、1 + 1 + 2 = 4、2 + 2 + 2 = 6 和 2 + 3 = 5 。 
  这样，s 在第一轮之后变成 "3" + "4" + "6" + "5" = "3465" 。
- 第二轮，将 s 分成："346" 和 "5" 。
  接着，计算每一组的数字和：3 + 4 + 6 = 13 、5 = 5 。
  这样，s 在第二轮之后变成 "13" + "5" = "135" 。 
现在，s.length <= k ，所以返回 "135" 作为答案。

示例 2：

输入：s = "00000000", k = 3
输出："000"
解释：
将 "000", "000", and "00".
接着，计算每一组的数字和：0 + 0 + 0 = 0 、0 + 0 + 0 = 0 和 0 + 0 = 0 。 
s 变为 "0" + "0" + "0" = "000" ，其长度等于 k ，所以返回 "000" 。

 

提示：

  • 1 <= s.length <= 100
  • 2 <= k <= 100
  • s 仅由数字（0 - 9）组成。

解法一：模拟迭代

思路：直接按照题意模拟每一轮操作。每轮将字符串按 k 长度分组，每组计算数字之和并转为字符串，拼接后得到新字符串。若新字符串长度仍大于 k，则继续下一轮，直到长度 ≤ k为止返回结果。

func digitSum(s string, k int) string {
    while len(s) > k {
        var next string
        for i := 0; i < len(s); i += k {
            end := min(i+k, len(s))
            sum := 0
            for j := i; j < end; j++ {
                sum += int(s[j] - '0')
            }
            next += strconv.Itoa(sum)
        }
        s = next
    }
    return s
}

复杂度：时间 O(n·r)，其中 n 为字符串长度，r 为操作轮数（每轮字符串长度显著缩短，实际 r 很小）；空间 O(n)

解法二：递归

思路：与迭代本质相同，只是用递归表达。若当前字符串长度 ≤ k 直接返回；否则执行一轮分组求和拼接，再对结果递归调用。

func digitSum(s string, k int) string {
    if len(s) <= k {
        return s
    }
    var next string
    for i := 0; i < len(s); i += k {
        end := min(i+k, len(s))
        sum := 0
        for j := i; j < end; j++ {
            sum += int(s[j] - '0')
        }
        next += strconv.Itoa(sum)
    }
    return digitSum(next, k)
}

复杂度：时间 O(n·r)，空间 O(n·r)（递归栈深度为 r）
