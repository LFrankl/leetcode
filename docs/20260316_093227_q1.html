2656. K 个元素的最大和

难度: Easy

标签: 贪心, 数组

链接: https://leetcode.cn/problems/maximum-sum-with-exactly-k-elements/

题目描述

给你一个下标从 0 开始的整数数组 nums 和一个整数 k 。你需要执行以下操作 恰好 k 次，最大化你的得分：

  1. 从 nums 中选择一个元素 m 。
  2. 将选中的元素 m 从数组中删除。
  3. 将新元素 m + 1 添加到数组中。
  4. 你的得分增加 m 。

请你返回执行以上操作恰好 k 次后的最大得分。

 

示例 1：

输入：nums = [1,2,3,4,5], k = 3
输出：18
解释：我们需要从 nums 中恰好选择 3 个元素并最大化得分。
第一次选择 5 。和为 5 ，nums = [1,2,3,4,6] 。
第二次选择 6 。和为 6 ，nums = [1,2,3,4,7] 。
第三次选择 7 。和为 5 + 6 + 7 = 18 ，nums = [1,2,3,4,8] 。
所以我们返回 18 。
18 是可以得到的最大答案。

示例 2：

输入：nums = [5,5,5], k = 2
输出：11
解释：我们需要从 nums 中恰好选择 2 个元素并最大化得分。
第一次选择 5 。和为 5 ，nums = [5,5,6] 。
第二次选择 6 。和为 6 ，nums = [5,5,7] 。
所以我们返回 11 。
11 是可以得到的最大答案。

 

提示：

  • 1 <= nums.length <= 100
  • 1 <= nums[i] <= 100
  • 1 <= k <= 100

1️⃣ 题意理解

题目重述：
给定一个整数数组 nums 和一个整数 k，需要执行恰好 k 次操作，每次操作： 1. 从数组中选择一个元素 m 2. 删除该元素 3. 向数组中添加 m + 1 4. 得分增加 m

目标是最大化 k 次操作后的总得分。

关键约束： - 每次操作后，数组长度不变（删除一个元素，添加一个更大的元素） - 操作次数 k 固定，必须恰好执行 k 次 - 数组元素和操作次数规模较小（nums.length ≤ 100, k ≤ 100）

边界情况： - 数组长度为 1 时，每次只能选择同一个位置（但元素值会递增） - 元素值可能重复 - k 可能远大于数组长度

输入输出格式： - 输入：nums []int, k int - 输出：int 类型的最大得分

2️⃣ 解题思路

解法一：贪心法（每次选最大值）

核心思想：
由于每次操作后，被选中的元素会变成 m+1 重新加入数组，因此为了最大化总得分，每次都应该选择当前数组中的最大值。这样每次得到的 m 都是当前可能的最大值，且后续操作中这个值还会继续增大。

数学推导：
设初始最大值为 maxVal，第一次得分为 maxVal，第二次得分为 maxVal+1，第三次得分为 maxVal+2，...，第 k 次得分为 maxVal + k - 1。
总得分 = k * maxVal + (0 + 1 + 2 + ... + (k-1)) = k * maxVal + k*(k-1)/2

步骤： 1. 找到数组中的最大值 maxVal 2. 计算总得分：k * maxVal + k*(k-1)/2

注意事项： - 需要处理整数溢出问题（但本题约束下不会溢出） - 注意 k 可能为 0 的情况（但题目约束 k ≥ 1）

解法二：模拟法（显式模拟操作过程）

核心思想：
直接模拟题目描述的操作过程，每次循环： 1. 找到当前数组最大值 2. 累加得分 3. 将最大值加 1 后放回数组

步骤： 1. 初始化得分 score = 0 2. 循环 k 次： - 找到当前数组最大值及其索引 - score += maxVal - 将 maxVal+1 放回原位置 3. 返回 score

注意事项： - 时间复杂度较高（O(k*n)），但数据规模小可以接受 - 每次找最大值需要遍历数组

3️⃣ 代码实现

解法一：贪心法（数学公式）

func maximizeSum(nums []int, k int) int {
    // 1. 找到数组中的最大值
    maxVal := nums[0]
    for _, num := range nums {
        if num > maxVal {
            maxVal = num
        }
    }

    // 2. 计算总得分：k * maxVal + (0+1+2+...+(k-1))
    // 等差数列求和公式：k*(k-1)/2
    return k*maxVal + k*(k-1)/2
}

解法二：模拟法

func maximizeSum(nums []int, k int) int {
    score := 0
    n := len(nums)

    for i := 0; i < k; i++ {
        // 找到当前最大值及其索引
        maxIdx := 0
        for j := 1; j < n; j++ {
            if nums[j] > nums[maxIdx] {
                maxIdx = j
            }
        }

        // 累加得分
        score += nums[maxIdx]

        // 将最大值加1后放回
        nums[maxIdx] = nums[maxIdx] + 1
    }

    return score
}

4️⃣ 复杂度分析

解法一：贪心法

  • ⏱️ 时间复杂度：O(n)，其中 n 是数组长度。只需要一次遍历找到最大值。
  • 💾 空间复杂度：O(1)，只使用了常数个额外变量。

解法二：模拟法

  • ⏱️ 时间复杂度：O(k*n)，每次操作需要遍历数组找最大值，共 k 次操作。
  • 💾 空间复杂度：O(1)，在原数组上修改，没有使用额外空间。
