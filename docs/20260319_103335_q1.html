1️⃣ 题意理解

题目重述
给定一个字符串 s，判断在最多删除一个字符的情况下，能否将其变为回文串。删除操作只能进行一次（或不进行）。

关键约束
- 字符串长度范围：1 ≤ s.length ≤ 10^5
- 字符串仅由小写英文字母组成
- 最多只能删除一个字符（即允许跳过一对不匹配的字符）

边界情况
- 字符串本身已经是回文串 → 直接返回 true
- 字符串长度为 1 → 一定是回文串
- 需要删除的字符可能在左侧或右侧

输入输出格式
- 输入：字符串 s
- 输出：布尔值 true 或 false

2️⃣ 解题思路

解法一：双指针 + 递归/回溯

核心思想
使用双指针从两端向中间检查，遇到不匹配时，尝试删除左字符或右字符，然后递归检查剩余部分是否回文。

步骤
1. 定义辅助函数 isPalindrome(l, r) 检查子串 s[l:r+1] 是否回文
2. 主函数中使用双指针 left 和 right 从两端向中间移动
3. 当 s[left] != s[right] 时，尝试两种情况：
- 删除左字符：检查 s[left+1:right+1] 是否回文
- 删除右字符：检查 s[left:right] 是否回文
4. 如果两种情况有一种成立，则返回 true，否则 false

注意事项
- 递归深度最多一层（因为只允许删除一次）
- 注意索引边界

解法二：双指针 + 直接判断

核心思想
在双指针遍历过程中，遇到不匹配时，直接尝试跳过左或右字符，继续检查剩余部分，避免显式递归。

步骤
1. 双指针 i, j 从两端向中间移动
2. 当 s[i] != s[j] 时：
- 调用 isPalindrome(i+1, j) 检查跳过左字符
- 调用 isPalindrome(i, j-1) 检查跳过右字符
3. 如果两种情况有一种成立，则返回 true
4. 如果遍历完成未遇到不匹配，返回 true

注意事项
- 辅助函数 isPalindrome 只需简单循环检查
- 时间复杂度仍为 O(n)，但常数较小

3️⃣ 代码实现

解法一：双指针 + 递归/回溯

func validPalindrome(s string) bool {
    // 辅助函数：判断子串 s[l:r+1] 是否为回文
    isPalindrome := func(l, r int) bool {
        for l < r {
            if s[l] != s[r] {
                return false
            }
            l++
            r--
        }
        return true
    }

    left, right := 0, len(s)-1
    for left < right {
        if s[left] != s[right] {
            // 尝试删除左字符或右字符
            return isPalindrome(left+1, right) || isPalindrome(left, right-1)
        }
        left++
        right--
    }
    return true
}

解法二：双指针 + 直接判断（优化版）

func validPalindrome(s string) bool {
    // 辅助函数：判断子串 s[l:r+1] 是否为回文
    isPalindrome := func(l, r int) bool {
        for l < r {
            if s[l] != s[r] {
                return false
            }
            l++
            r--
        }
        return true
    }

    i, j := 0, len(s)-1
    for i < j {
        if s[i] != s[j] {
            // 尝试跳过左字符或右字符
            return isPalindrome(i+1, j) || isPalindrome(i, j-1)
        }
        i++
        j--
    }
    return true
}

4️⃣ 复杂度分析

解法一 & 解法二

  • ⏱️ 时间复杂度：O(n)
    最坏情况下需要遍历整个字符串两次（一次主循环，一次辅助函数检查），因此是 O(2n) = O(n)
  • 💾 空间复杂度：O(1)
    只使用了常数级别的额外空间（指针变量）

5️⃣ 解法对比

解法 优点 缺点 适用场景
解法一（递归/回溯） 逻辑清晰，易于理解 递归调用可能带来额外开销 教学或代码可读性优先
解法二（直接判断） 常数时间更优，无递归开销 与解法一差异不大 实际编码推荐

推荐解法
解法二（双指针 + 直接判断），因为：
1. 代码简洁，无递归栈开销
2. 时间复杂度相同，常数更小
3. 更符合实际工程实现
