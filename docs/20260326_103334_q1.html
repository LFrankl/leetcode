1484. 按日期分组销售产品

难度: Easy

标签: 数据库

链接: https://leetcode.cn/problems/group-sold-products-by-the-date/

题目描述

表 Activities：

+-------------+---------+
| 列名         | 类型    |
+-------------+---------+
| sell_date   | date    |
| product     | varchar |
+-------------+---------+
该表没有主键(具有唯一值的列)。它可能包含重复项。
此表的每一行都包含产品名称和在市场上销售的日期。

 

编写解决方案找出每个日期、销售的不同产品的数量及其名称。
每个日期的销售产品名称应按词典序排列。
返回按 sell_date 排序的结果表。
结果表结果格式如下例所示。

 

示例 1:

输入：
Activities 表：
+------------+-------------+
| sell_date  | product     |
+------------+-------------+
| 2020-05-30 | Headphone   |
| 2020-06-01 | Pencil      |
| 2020-06-02 | Mask        |
| 2020-05-30 | Basketball  |
| 2020-06-01 | Bible       |
| 2020-06-02 | Mask        |
| 2020-05-30 | T-Shirt     |
+------------+-------------+
输出：
+------------+----------+------------------------------+
| sell_date  | num_sold | products                     |
+------------+----------+------------------------------+
| 2020-05-30 | 3        | Basketball,Headphone,T-shirt |
| 2020-06-01 | 2        | Bible,Pencil                 |
| 2020-06-02 | 1        | Mask                         |
+------------+----------+------------------------------+
解释：
对于2020-05-30，出售的物品是 (Headphone, Basketball, T-shirt)，按词典序排列，并用逗号 ',' 分隔。
对于2020-06-01，出售的物品是 (Pencil, Bible)，按词典序排列，并用逗号分隔。
对于2020-06-02，出售的物品是 (Mask)，只需返回该物品名。

1️⃣ 题意理解

题目核心要求： - 统计每个销售日期（sell_date）的以下信息： 1. 当天销售的不同产品数量（num_sold） 2. 当天销售的所有不同产品名称（products），按字典序排序并用逗号连接 - 结果按 sell_date 升序排列

关键约束和边界情况： - 表中可能包含重复的 (sell_date, product) 记录，需要去重 - 产品名称需要按字典序排序（不区分大小写？示例中 T-shirt 输出为 T-shirt，但输入是 T-Shirt，说明可能区分大小写，按 ASCII 排序） - 结果中 products 列是字符串，用逗号分隔 - 如果某天只有一个产品，直接返回该产品名（不加逗号）

输入输出格式： - 输入：Activities 表，包含 sell_date（日期）和 product（字符串） - 输出：包含三列的结果表：sell_date、num_sold（整数）、products（字符串）

2️⃣ 解题思路

解法一：基础分组聚合（标准 SQL）

核心思想： - 使用 GROUP BY 按日期分组 - 在分组内： - 用 COUNT(DISTINCT product) 计算不同产品数 - 用 GROUP_CONCAT(DISTINCT product ORDER BY product) 连接产品名 - 最后按日期排序

步骤： 1. 按 sell_date 分组 2. 对每个分组： - 计算 num_sold = COUNT(DISTINCT product) - 生成 products = GROUP_CONCAT(DISTINCT product ORDER BY product SEPARATOR ',') 3. 按 sell_date 排序结果

注意事项： - MySQL 中 GROUP_CONCAT 默认有长度限制（1024），如果产品很多可能被截断 - 需要显式使用 DISTINCT 去重

解法二：先子查询去重再聚合

核心思想： - 先通过子查询或 CTE 获取每个日期的不同产品 - 再对去重后的结果进行聚合

步骤： 1. 先查询 SELECT DISTINCT sell_date, product FROM Activities 2. 对上述结果按 sell_date 分组，进行聚合操作 3. 按日期排序

优点： - 逻辑更清晰，先去重再聚合 - 在某些数据库（如不支持 COUNT(DISTINCT) 和 GROUP_CONCAT(DISTINCT) 同时使用）中更通用

3️⃣ 代码实现

解法一：基础分组聚合

SELECT 
    sell_date,
    COUNT(DISTINCT product) AS num_sold,
    GROUP_CONCAT(DISTINCT product ORDER BY product SEPARATOR ',') AS products
FROM Activities
GROUP BY sell_date
ORDER BY sell_date;

解法二：先子查询去重

WITH DistinctActivities AS (
    SELECT DISTINCT sell_date, product
    FROM Activities
)
SELECT 
    sell_date,
    COUNT(product) AS num_sold,
    GROUP_CONCAT(product ORDER BY product SEPARATOR ',') AS products
FROM DistinctActivities
GROUP BY sell_date
ORDER BY sell_date;

4️⃣ 复杂度分析

解法一

  • ⏱️ 时间复杂度：O(n log n)，其中 n 是记录数
    • GROUP BY 需要排序或哈希，通常 O(n log n)
    • GROUP_CONCAT 在每个分组内排序产品名，最坏情况 O(m log m)，m 是分组大小
  • 💾 空间复杂度：O(n)
    • 需要存储分组中间结果

解法二

  • ⏱️ 时间复杂度：O(n log n)
    • 子查询去重需要排序或哈希：O(n log n)
    • 外层分组聚合：O(k log k)，k 是去重后的记录数
  • 💾 空间复杂度：O(n)
    • 需要存储去重后的中间结果
