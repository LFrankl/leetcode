2295. 替换数组中的元素

难度: Medium

标签: 数组, 哈希表, 模拟

链接: https://leetcode.cn/problems/replace-elements-in-an-array/

题目描述

给你一个下标从 0 开始的数组 nums ，它包含 n 个 互不相同 的正整数。请你对这个数组执行 m 个操作，在第 i 个操作中，你需要将数字 operations[i][0] 替换成 operations[i][1] 。

题目保证在第 i 个操作中：

  • operations[i][0] 在 nums 中存在。
  • operations[i][1] 在 nums 中不存在。

请你返回执行完所有操作后的数组。

 

示例 1：

输入：nums = [1,2,4,6], operations = [[1,3],[4,7],[6,1]]
输出：[3,2,7,1]
解释：我们对 nums 执行以下操作：
- 将数字 1 替换为 3 。nums 变为 [3,2,4,6] 。
- 将数字 4 替换为 7 。nums 变为 [3,2,7,6] 。
- 将数字 6 替换为 1 。nums 变为 [3,2,7,1] 。
返回最终数组 [3,2,7,1] 。

示例 2：

输入：nums = [1,2], operations = [[1,3],[2,1],[3,2]]
输出：[2,1]
解释：我们对 nums 执行以下操作：
- 将数字 1 替换为 3 。nums 变为 [3,2] 。
- 将数字 2 替换为 1 。nums 变为 [3,1] 。
- 将数字 3 替换为 2 。nums 变为 [2,1] 。
返回最终数组 [2,1] 。

 

提示：

  • n == nums.length
  • m == operations.length
  • 1 <= n, m <= 105
  • nums 中所有数字 互不相同 。
  • operations[i].length == 2
  • 1 <= nums[i], operations[i][0], operations[i][1] <= 106
  • 在执行第 i 个操作时，operations[i][0] 在 nums 中存在。
  • 在执行第 i 个操作时，operations[i][1] 在 nums 中不存在。

1️⃣ 题意理解

  • 核心要求：给定一个互不相同的正整数数组 nums 和一系列替换操作 operations，每个操作 [old, new] 表示将数组中的 old 值替换为 new 值。保证每次操作时 old 存在于数组中，new 不存在于数组中。执行所有操作后返回最终数组。
  • 关键约束：
    • 所有数字互不相同
    • 每次操作时 old 一定存在，new 一定不存在
    • 操作可能形成链式替换（如示例2中 1→3→2）
    • 数据规模：n, m ≤ 10^5
  • 输入输出：
    • 输入：nums []int, operations [][]int
    • 输出：执行所有操作后的 []int

2️⃣ 解题思路

解法一：直接模拟（暴力替换）

  • 核心思想：对于每个操作，遍历数组找到 old 值并替换为 new 值
  • 问题：每次查找需要 O(n)，总复杂度 O(n×m) 会超时（10^10 级别）
  • 不适用：数据规模大时不可行

解法二：哈希映射法

  • 核心思想：使用哈希表记录每个值在数组中的索引，通过索引直接访问并修改
  • 步骤：
    1. 创建 valueToIndex 哈希表，记录每个值在 nums 中的索引
    2. 遍历每个操作 [old, new]：
      • 通过哈希表找到 old 的索引 idx
      • 将 nums[idx] 更新为 new
      • 更新哈希表：删除 old 的映射，添加 new → idx 的映射
    3. 返回 nums
  • 注意事项：
    • 需要同时更新数组和哈希表
    • 操作可能形成链式关系，但哈希表能正确处理

解法三：并查集/映射链法（优化链式操作）

  • 核心思想：对于链式替换 a→b→c，最终是 a→c，可以跳过中间映射
  • 步骤：
    1. 创建 mapping 哈希表，初始为空
    2. 逆序遍历操作（从最后一个操作向前）：
      • 对于 [old, new]，如果 new 在 mapping 中有映射，则建立 old → mapping[new] 的映射
      • 否则建立 old → new 的映射
    3. 遍历 nums，如果值在 mapping 中，则替换为最终映射值
  • 优势：避免多次更新数组，适合链式替换多的情况

3️⃣ 代码实现

解法二：哈希映射法

func arrayChange(nums []int, operations [][]int) []int {
    // 创建值到索引的映射
    indexMap := make(map[int]int, len(nums))
    for i, num := range nums {
        indexMap[num] = i
    }

    // 执行所有操作
    for _, op := range operations {
        oldVal, newVal := op[0], op[1]
        idx := indexMap[oldVal]  // 获取旧值的索引

        // 更新数组
        nums[idx] = newVal

        // 更新映射：删除旧映射，添加新映射
        delete(indexMap, oldVal)
        indexMap[newVal] = idx
    }

    return nums
}

解法三：映射链法

func arrayChange(nums []int, operations [][]int) []int {
    // 创建最终映射表
    finalMap := make(map[int]int)

    // 逆序遍历操作，建立最终映射关系
    for i := len(operations) - 1; i >= 0; i-- {
        oldVal, newVal := operations[i][0], operations[i][1]

        // 如果newVal有后续映射，则oldVal直接映射到最终值
        if finalVal, exists := finalMap[newVal]; exists {
            finalMap[oldVal] = finalVal
        } else {
            finalMap[oldVal] = newVal
        }
    }

    // 根据最终映射更新数组
    for i, num := range nums {
        if newVal, exists := finalMap[num]; exists {
            nums[i] = newVal
        }
    }

    return nums
}

4️⃣ 复杂度分析

解法二：哈希映射法

  • ⏱️ 时间复杂度：O(n + m)
    • 构建初始哈希表：O(n)
    • 执行 m 个操作：每个操作 O(1) 的查找和更新
    • 总计：O(n + m)
  • 💾 空间复杂度：O(n)
    • 哈希表存储 n 个元素的索引映射

解法三：映射链法

  • ⏱️ 时间复杂度：O(n + m)
    • 逆序遍历 m 个操作：O(m)
    • 遍历 n 个元素更新：O(n)
    • 总计：O(n + m)
  • 💾 空间复杂度：O(m)
    • 最坏情况下需要存储 m 个映射关系
