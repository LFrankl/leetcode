2347. 最好的扑克手牌

难度: Easy

标签: 数组, 哈希表, 计数

链接: https://leetcode.cn/problems/best-poker-hand/

题目描述

给你一个整数数组 ranks 和一个字符数组 suit 。你有 5 张扑克牌，第 i 张牌大小为 ranks[i] ，花色为 suits[i] 。

下述是从好到坏你可能持有的 手牌类型 ：

  1. "Flush"：同花，五张相同花色的扑克牌。
  2. "Three of a Kind"：三条，有 3 张大小相同的扑克牌。
  3. "Pair"：对子，两张大小一样的扑克牌。
  4. "High Card"：高牌，五张大小互不相同的扑克牌。

请你返回一个字符串，表示给定的 5 张牌中，你能组成的 最好手牌类型 。

注意：返回的字符串 大小写 需与题目描述相同。

 

示例 1：

输入：ranks = [13,2,3,1,9], suits = ["a","a","a","a","a"]
输出："Flush"
解释：5 张扑克牌的花色相同，所以返回 "Flush" 。

示例 2：

输入：ranks = [4,4,2,4,4], suits = ["d","a","a","b","c"]
输出："Three of a Kind"
解释：第一、二和四张牌组成三张相同大小的扑克牌，所以得到 "Three of a Kind" 。
注意我们也可以得到 "Pair" ，但是 "Three of a Kind" 是更好的手牌类型。
有其他的 3 张牌也可以组成 "Three of a Kind" 手牌类型。

示例 3：

输入：ranks = [10,10,2,12,9], suits = ["a","b","c","a","d"]
输出："Pair"
解释：第一和第二张牌大小相同，所以得到 "Pair" 。
我们无法得到 "Flush" 或者 "Three of a Kind" 。

 

提示：

  • ranks.length == suits.length == 5
  • 1 <= ranks[i] <= 13
  • 'a' <= suits[i] <= 'd'
  • 任意两张扑克牌不会同时有相同的大小和花色。

解法一：枚举判断

思路：按手牌优先级从高到低依次判断。先检查5张牌花色是否全部相同（Flush），再统计每个rank出现的次数，看是否有rank出现3次以上（Three of a Kind），是否有rank出现2次以上（Pair），否则就是High Card。

func bestHand(ranks []int, suits []char) string {
    // 判断 Flush：所有花色相同
    flush := true
    for i := 1; i < 5; i++ {
        if suits[i] != suits[0] {
            flush = false
            break
        }
    }
    if flush {
        return "Flush"
    }

    // 统计每个 rank 出现次数
    count := make(map[int]int)
    for _, r := range ranks {
        count[r]++
    }

    hasThree, hasPair := false, false
    for _, c := range count {
        if c >= 3 {
            hasThree = true
        }
        if c >= 2 {
            hasPair = true
        }
    }

    if hasThree {
        return "Three of a Kind"
    }
    if hasPair {
        return "Pair"
    }
    return "High Card"
}

复杂度：时间O(n)，空间O(n)，其中n=5（固定），实际可视为O(1)。

解法二：排序+紧凑判断

思路：对ranks排序后，通过连续相同值的最大长度判断三条/对子；花色判断与解法一相同。排序后只需扫描一次即可得到最大连续重复长度。

func bestHand(ranks []int, suits []char) string {
    // 判断 Flush
    flush := true
    for i := 1; i < 5; i++ {
        if suits[i] != suits[0] {
            flush = false
            break
        }
    }
    if flush {
        return "Flush"
    }

    // 排序 ranks
    sort.Ints(ranks)

    // 找最大连续重复长度
    maxLen := 1
    curLen := 1
    for i := 1; i < 5; i++ {
        if ranks[i] == ranks[i-1] {
            curLen++
            if curLen > maxLen {
                maxLen = curLen
            }
        } else {
            curLen = 1
        }
    }

    switch maxLen {
    case 5, 4, 3:
        return "Three of a Kind"
    case 2:
        return "Pair"
    default:
        return "High Card"
    }
}

复杂度：时间O(n log n)（排序），空间O(1)。因n=5固定，实际也为O(1)。

总结：解法一用哈希表计数更直观，解法二排序后扫描更紧凑。由于n恒为5，两者性能差异可忽略，推荐解法一更易理解。
