题目描述

给你一棵二叉树，它的根为 root 。请你删除 1 条边，使二叉树分裂成两棵子树，且它们子树和的乘积尽可能大。

由于答案可能会很大，请你将结果对 10^9 + 7 取模后再返回。

 

示例 1：

输入：root = [1,2,3,4,5,6]
输出：110
解释：删除红色的边，得到 2 棵子树，和分别为 11 和 10 。它们的乘积是 110 （11*10）

示例 2：

输入：root = [1,null,2,3,4,null,null,5,6]
输出：90
解释：移除红色的边，得到 2 棵子树，和分别是 15 和 6 。它们的乘积为 90 （15*6）

示例 3：

输入：root = [2,3,9,10,7,8,6,5,4,11,1]
输出：1025

示例 4：

输入：root = [1,1]
输出：1

 

提示：

  • 每棵树最多有 50000 个节点，且至少有 2 个节点。
  • 每个节点的值在 [1, 10000] 之间。

1️⃣ 题意理解

核心要求：
给定一棵二叉树，通过删除一条边将其分裂成两棵子树，使得两棵子树的节点值之和的乘积最大化，结果对 (10^9+7) 取模。

关键约束： - 树节点数：(2 \leq n \leq 50000) - 节点值范围：([1, 10000]) - 只能删除一条边 - 结果需要取模 (10^9+7)

边界情况： - 树为链状结构 - 节点值较大，求和可能溢出 - 乘积可能超过 64 位整数范围

输入输出： - 输入：二叉树的根节点 root - 输出：最大乘积对 (10^9+7) 取模的结果

2️⃣ 解题思路

解法一：两次遍历（先求总和，再枚举切割）

核心思想： 1. 第一次遍历计算整棵树的总和 totalSum 2. 第二次遍历，对于每个节点，计算以该节点为根的子树和 subtreeSum 3. 删除该节点与其父节点之间的边时，分裂出的两棵子树和分别为： - subtreeSum - totalSum - subtreeSum 4. 乘积为 subtreeSum * (totalSum - subtreeSum)，取最大值

注意事项： - 使用 int64 避免中间结果溢出 - 取模只在最后结果进行，比较最大值时用原始值 - 遍历时注意空节点处理

解法二：一次遍历（DFS 同时计算）

核心思想： 在 DFS 过程中同时计算子树和并更新最大乘积，避免两次遍历。

步骤： 1. 先计算整棵树的总和（必须的，因为需要知道总和） 2. 实际上还是需要先知道总和，所以严格来说无法完全一次遍历完成总和计算和切割枚举

优化： 可以在 DFS 返回子树和的同时，用闭包或全局变量记录 totalSum 并更新最大乘积。

3️⃣ 代码实现

解法一：两次遍历（推荐）

/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

func maxProduct(root *TreeNode) int {
    const mod = 1_000_000_007

    // 第一步：计算整棵树的总和
    var totalSum int64
    var dfsSum func(node *TreeNode) int64
    dfsSum = func(node *TreeNode) int64 {
        if node == nil {
            return 0
        }
        left := dfsSum(node.Left)
        right := dfsSum(node.Right)
        return left + right + int64(node.Val)
    }
    totalSum = dfsSum(root)

    // 第二步：遍历每个节点，计算删除该节点与父节点之间边时的乘积
    var maxProd int64 = 0
    var dfsProd func(node *TreeNode) int64
    dfsProd = func(node *TreeNode) int64 {
        if node == nil {
            return 0
        }
        // 计算当前子树和
        left := dfsProd(node.Left)
        right := dfsProd(node.Right)
        subtreeSum := left + right + int64(node.Val)

        // 计算乘积：subtreeSum * (totalSum - subtreeSum)
        product := subtreeSum * (totalSum - subtreeSum)
        if product > maxProd {
            maxProd = product
        }

        return subtreeSum
    }
    dfsProd(root)

    return int(maxProd % mod)
}

解法二：一次遍历（优化版）

func maxProduct(root *TreeNode) int {
    const mod = 1_000_000_007

    // 先计算总和
    var totalSum int64
    var sumDFS func(node *TreeNode) int64
    sumDFS = func(node *TreeNode) int64 {
        if node == nil {
            return 0
        }
        return sumDFS(node.Left) + sumDFS(node.Right) + int64(node.Val)
    }
    totalSum = sumDFS(root)

    var maxProd int64 = 0
    // 第二次遍历同时计算乘积
    var dfs func(node *TreeNode) int64
    dfs = func(node *TreeNode) int64 {
        if node == nil {
            return 0
        }
        left := dfs(node.Left)
        right := dfs(node.Right)
        subtreeSum := left + right + int64(node.Val)

        product := subtreeSum * (totalSum - subtreeSum)
        if product > maxProd {
            maxProd = product
        }

        return subtreeSum
    }
    dfs(root)

    return int(maxProd % mod)
}

4️⃣ 复杂度分析

解法一：

  • ⏱️ 时间复杂度：O(n)
    • 第一次遍历计算总和：O(n)
    • 第二次遍历枚举切割：O(n)
    • 总时间复杂度 O(2n) = O(n)
  • 💾 空间复杂度：O(h)
    • h 为树的高度，递归栈空间
    • 最坏情况（链状树）O(n)，平均 O(log n)

解法二：

  • ⏱️ 时间复杂度：O(n)
    • 计算总和：O(n)
    • 遍历计算乘积：O(n)
    • 总时间复杂度 O(2n) = O(n)
  • 💾 空间复杂度：O(h)
    • 与解法一相同

5️⃣ 解法对比

解法 优点 缺点 适用场景
解法一 逻辑清晰，易于理解 需要两次遍历 通用场景
解法二 代码稍简洁 实际仍需先计算总和 同解法一

推荐解法一：
虽然两种方法时间复杂度相同，但解法一逻辑更清晰，便于理解和调试。在实际编码中，两种方法性能差异极小，选择更易读的代码更重要。
