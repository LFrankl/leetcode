2787. 将一个数字表示成幂的和的方案数

难度: Medium

题目描述

给你两个 正 整数 n 和 x 。

请你返回将 n 表示成一些 互不相同 正整数的 x 次幂之和的方案数。换句话说，你需要返回互不相同整数 [n1, n2, ..., nk] 的集合数目，满足 n = n1x + n2x + ... + nkx 。

由于答案可能非常大，请你将它对 109 + 7 取余后返回。

比方说，n = 160 且 x = 3 ，一个表示 n 的方法是 n = 23 + 33 + 53 。

 

示例 1：

输入：n = 10, x = 2
输出：1
解释：我们可以将 n 表示为：n = 32 + 12 = 10 。
这是唯一将 10 表达成不同整数 2 次方之和的方案。

示例 2：

输入：n = 4, x = 1
输出：2
解释：我们可以将 n 按以下方案表示：
- n = 41 = 4 。
- n = 31 + 11 = 4 。

 

提示：

  • 1 <= n <= 300
  • 1 <= x <= 5

1️⃣ 题意理解

核心要求：
给定正整数 n 和 x，计算将 n 表示为 互不相同 的正整数的 x 次幂之和的方案数。
例如：n = 10, x = 2，可以表示为 3² + 1² = 10，且每个底数必须不同。

关键约束： - 1 ≤ n ≤ 300 - 1 ≤ x ≤ 5 - 底数必须互不相同（即每个正整数最多使用一次） - 方案数可能很大，需要对 10⁹ + 7 取模

边界情况： - x = 1 时，问题退化为“将 n 表示为不同正整数之和”的方案数（类似整数划分但要求互不相同） - n 较小但 x 较大时，可选的底数很少（例如 n=5, x=3 只有 1³=1 和 2³=8，8>5 不可用）

输入输出： - 输入：int n, int x - 输出：int 方案数（模 10⁹+7）

2️⃣ 解题思路

解法一：动态规划（0-1背包变种）

核心思想：
将每个底数的 x 次幂视为一个物品，其“重量”为幂值，价值为方案数。
问题转化为：从这些物品中选取若干，总重量恰好为 n，且每个物品最多选一次（0-1背包）。
由于底数互不相同，相当于物品编号就是底数本身。

步骤： 1. 预处理所有可能的底数 i，计算 power = i^x，直到 power > n 停止。 2. 定义 dp[j] 表示总幂和为 j 的方案数。 3. 遍历每个底数 i，从 n 向下遍历 j，更新 dp[j] += dp[j - power]（避免重复使用同一底数）。 4. 最终 dp[n] 即为答案。

注意事项： - 需要模运算避免溢出。 - 遍历顺序必须从 n 到 power，确保每个底数只用一次（0-1背包）。

解法二：记忆化搜索（DFS + 剪枝）

核心思想：
递归枚举每个底数选或不选，用记忆化缓存 (idx, sum) 的状态，避免重复计算。

步骤： 1. 预处理所有底数的幂值列表 powers。 2. 定义 dfs(idx, sum) 表示从第 idx 个底数开始，当前和为 sum 的方案数。 3. 递归时，可以选择跳过当前底数，或选择当前底数（如果 sum + powers[idx] ≤ n）。 4. 用哈希表或数组缓存结果。

注意事项： - 递归深度可能较大，但 n ≤ 300 且 x ≥ 1 时底数最多约 √n 或 n^(1/x) 个，可接受。 - 需要剪枝：当 sum > n 时直接返回 0。

3️⃣ 代码实现

解法一：动态规划（0-1背包）

func numberOfWays(n int, x int) int {
    const MOD = 1_000_000_007
    // dp[j] 表示总幂和为 j 的方案数
    dp := make([]int, n+1)
    dp[0] = 1 // 空集方案数为1

    // 遍历所有可能的底数 i
    for i := 1; ; i++ {
        power := pow(i, x)
        if power > n {
            break
        }
        // 0-1背包：从后向前更新，避免重复使用同一底数
        for j := n; j >= power; j-- {
            dp[j] = (dp[j] + dp[j-power]) % MOD
        }
    }
    return dp[n]
}

// 快速幂计算 i^x
func pow(a, b int) int {
    res := 1
    for b > 0 {
        if b&1 == 1 {
            res *= a
        }
        a *= a
        b >>= 1
    }
    return res
}

解法二：记忆化搜索（DFS + 剪枝）

func numberOfWays(n int, x int) int {
    const MOD = 1_000_000_007
    // 预处理所有可能的幂值
    powers := []int{}
    for i := 1; ; i++ {
        val := pow(i, x)
        if val > n {
            break
        }
        powers = append(powers, val)
    }

    // 记忆化缓存
    memo := make([][]int, len(powers))
    for i := range memo {
        memo[i] = make([]int, n+1)
        for j := range memo[i] {
            memo[i][j] = -1
        }
    }

    var dfs func(idx, sum int) int
    dfs = func(idx, sum int) int {
        if sum == n {
            return 1
        }
        if idx == len(powers) || sum > n {
            return 0
        }
        if memo[idx][sum] != -1 {
            return memo[idx][sum]
        }
        // 不选当前底数
        res := dfs(idx+1, sum)
        // 选当前底数
        if sum+powers[idx] <= n {
            res = (res + dfs(idx+1, sum+powers[idx])) % MOD
        }
        memo[idx][sum] = res
        return res
    }

    return dfs(0, 0)
}

func pow(a, b int) int {
    res := 1
    for b > 0 {
        if b&1 == 1 {
            res *= a
        }
        a *= a
        b >>= 1
    }
    return res
}

4️⃣ 复杂度分析

解法一：动态规划

  • 时间复杂度：O(n * m)
    其中 m 是满足 i^x ≤ n 的底数个数，最大约为 n^(1/x)。
    当 x=1 时，m ≈ n，复杂度 O(n²)；当 x=5 时，m ≈ n^(1/5) ≈ 3（n≤300），复杂度接近 O(n)。
    实际最坏情况 O(n * n^(1/x))。
  • 空间复杂度：O(n)
    只使用一维 dp 数组。

解法二：记忆化搜索

  • 时间复杂度：O(m * n)
    状态数为 (m+1)*(n+1)，每个状态计算一次。
  • 空间复杂度：O(m * n)
    用于存储记忆化表。

5️⃣ 解法对比

解法 优点 缺点 适用场景
动态规划 代码简洁，效率高，空间优化好 状态转移需要小心遍历顺序 推荐，尤其 n 较大时
记忆化搜索 思路直观，易于理解 递归开销，空间占用较大 适合教学或小规模数据

推荐解法一（动态规划）：
- 本题 n ≤ 300，动态规划效率高且代码简单。 - 0-1背包模型是经典解法，不易出错。
