1️⃣ 题意理解

题目重述： 给定一个长度为 n 的整数数组 nums 和一个整数 k，我们需要将数组向右循环移动 k 次（每次移动一个位置），然后计算移动后有多少个索引 i 满足 nums[i] == i（即元素值等于其索引值）。要求返回所有可能的 k（0 ≤ k < n）中，能够得到的最大匹配索引数。

关键约束： - 数组长度 n 最大为 10^5 - 数组元素为整数，但未指定范围 - 需要计算所有可能的 k 对应的匹配数，并返回最大值

边界情况： - 数组长度为 1 时，只有 k=0 一种情况 - 数组元素可能为负数或大于 n-1，这些元素永远不可能匹配 - 当 n 很大时，需要高效算法

输入输出： - 输入：整数数组 nums - 输出：整数，表示最大匹配索引数

2️⃣ 解题思路

解法一：暴力法（不可行）

  • 对每个 k 进行右移，然后统计匹配数
  • 时间复杂度 O(n²)，对于 n=10^5 会超时

解法二：数学分析 + 差分数组

核心思想： 1. 对于每个元素 nums[i]，分析它在哪些 k 值下能够匹配 2. 元素 nums[i] 在右移 k 次后，会移动到位置 (i + k) % n 3. 当 nums[i] == (i + k) % n 时匹配 4. 可以解出 k ≡ (nums[i] - i) mod n 5. 但需要注意负数取模的处理

解题步骤： 1. 遍历数组，对于每个元素 nums[i]： - 如果 nums[i] < 0 或 nums[i] >= n，跳过（永远不可能匹配） - 计算 diff = (nums[i] - i) % n - 由于 k 需要是非负整数，需要调整 diff 到 [0, n-1] 范围 - 确定这个元素在哪些 k 值下能够匹配 2. 使用差分数组记录每个 k 对应的匹配数变化 3. 最后计算前缀和得到每个 k 的匹配数，取最大值

注意事项： - 负数取模需要特殊处理 - 差分数组大小应为 n+1 以便处理边界

解法三：优化版差分

核心思想： - 每个元素最多能匹配 2 个 k 值（因为 k 的范围是连续的） - 通过分析 k 的连续区间来更新

3️⃣ 代码实现

解法二：差分数组法

func maximumMatchingIndex(nums []int) int {
    n := len(nums)
    if n == 0 {
        return 0
    }

    // 差分数组，diff[k] 表示 k 相对于 k-1 的匹配数变化
    diff := make([]int, n+1)

    for i := 0; i < n; i++ {
        val := nums[i]
        // 如果值不在有效范围内，永远不可能匹配
        if val < 0 || val >= n {
            continue
        }

        // 计算使得 nums[i] == (i + k) % n 的 k
        // 即 k ≡ (val - i) mod n
        // 但 k 需要在 [0, n-1] 范围内

        // 情况1：k = (val - i) mod n，但需要确保非负
        k1 := (val - i) % n
        if k1 < 0 {
            k1 += n
        }

        // 情况2：由于取模，实际上只有一个 k 值
        // 但我们需要考虑 k 的连续区间

        // 实际上，对于每个 i，匹配条件为：
        // nums[i] == (i + k) % n
        // 即 k ≡ (nums[i] - i) mod n
        // 所以只有一个 k 值

        // 更新差分数组
        diff[k1] += 1
        // 注意：k1 只会出现一次，不需要减1
    }

    // 计算前缀和得到每个 k 的匹配数
    maxMatch := 0
    current := 0

    // 注意：k 从 0 到 n-1
    for k := 0; k < n; k++ {
        current += diff[k]
        if current > maxMatch {
            maxMatch = current
        }
    }

    return maxMatch
}

解法三：优化版（处理连续区间）

func maximumMatchingIndex(nums []int) int {
    n := len(nums)
    if n == 0 {
        return 0
    }

    diff := make([]int, n+1)

    for i := 0; i < n; i++ {
        val := nums[i]
        if val < 0 || val >= n {
            continue
        }

        // 计算匹配的 k 值
        // 需要满足：val == (i + k) % n
        // 所以 k ≡ (val - i) mod n

        // 计算基础 k 值
        baseK := val - i

        // 调整到 [0, n-1] 范围
        k := ((baseK % n) + n) % n

        // 这个元素在 k 值下匹配
        diff[k] += 1
    }

    // 计算最大值
    maxMatch := 0
    current := 0
    for k := 0; k < n; k++ {
        current += diff[k]
        if current > maxMatch {
            maxMatch = current
        }
    }

    return maxMatch
}

4️⃣ 复杂度分析

解法二/三：差分数组法

  • ⏱️ 时间复杂度：O(n)
    • 遍历数组一次：O(n)
    • 计算前缀和一次：O(n)
    • 总时间复杂度：O(2n) = O(n)
  • 💾 空间复杂度：O(n)
    • 需要额外的差分数组，大小为 n+1
    • 空间复杂度为 O(n)
