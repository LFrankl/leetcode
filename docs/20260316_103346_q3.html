3530. 有向无环图中合法拓扑排序的最大利润

难度: Hard

标签: 位运算, 图, 拓扑排序, 数组, 动态规划, 位掩码

链接: https://leetcode.cn/problems/maximum-profit-from-valid-topological-order-in-dag/

题目描述

给你一个由 n 个节点组成的有向无环图（DAG），节点编号从 0 到 n - 1，通过二维数组 edges 表示，其中 edges[i] = [ui, vi] 表示一条从节点 ui 指向节点 vi 的有向边。每个节点都有一个对应的 得分 ，由数组 score 给出，其中 score[i] 表示节点 i 的得分。

你需要以 有效的拓扑排序 顺序处理这些节点。每个节点在处理顺序中被分配一个编号从 1 开始的位置。

将每个节点的得分乘以其在拓扑排序中的位置，然后求和，得到的值称为 利润。

请返回在所有合法拓扑排序中可获得的 最大利润 。

拓扑排序 是一个对 DAG 中所有节点的线性排序，使得每条有向边 u → v 中，节点 u 都出现在 v 之前。

 

示例 1：

输入： n = 2, edges = [[0,1]], score = [2,3]

输出： 8

解释：

节点 1 依赖于节点 0，因此一个合法顺序是 [0, 1]。

节点 处理顺序 得分 乘数 利润计算
0 第 1 个 2 1 2 × 1 = 2
1 第 2 个 3 2 3 × 2 = 6

所有合法拓扑排序中可获得的最大总利润是 2 + 6 = 8。

示例 2：

输入： n = 3, edges = [[0,1],[0,2]], score = [1,6,3]

输出： 25

解释：

节点 1 和 2 都依赖于节点 0，因此最优的合法顺序是 [0, 2, 1]。

节点 处理顺序 得分 乘数 利润计算
0 第 1 个 1 1 1 × 1 = 1
2 第 2 个 3 2 3 × 2 = 6
1 第 3 个 6 3 6 × 3 = 18

所有合法拓扑排序中可获得的最大总利润是 1 + 6 + 18 = 25。

 

提示：

  • 1 <= n == score.length <= 22
  • 1 <= score[i] <= 105
  • 0 <= edges.length <= n * (n - 1) / 2
  • edges[i] == [ui, vi] 表示一条从 ui 到 vi 的有向边。
  • 0 <= ui, vi < n
  • ui != vi
  • 输入图 保证 是一个 DAG。
  • 不存在重复的边。

1️⃣ 题意理解

核心要求： 给定一个有向无环图（DAG），每个节点有一个得分。我们需要找到一个合法的拓扑排序顺序，使得每个节点的得分乘以其在排序中的位置（从1开始）的总和（称为“利润”）最大。

关键约束： - 节点数 n ≤ 22（很小，暗示可能使用状态压缩动态规划） - 得分值 score[i] ≥ 1 - 图保证是 DAG，无环 - 边无重复 - 拓扑排序必须满足所有边的方向约束（u 必须在 v 之前）

输入输出： - 输入：n（节点数），edges（边列表），score（得分数组） - 输出：最大利润（整数）

2️⃣ 解题思路

解法一：状态压缩 DP（推荐）

核心思想： - 由于 n ≤ 22，可以用一个整数 mask 的二进制位表示已选择的节点集合。 - 定义 dp[mask] 表示已选择节点集合为 mask 时的最大利润。 - 转移时，考虑下一个可以选择的节点（即入度为0且未选择的节点），计算将其加入排序后的利润增量。 - 利润计算：每个节点的位置 = 已选节点数 + 1（因为位置从1开始）。

步骤： 1. 预处理每个节点的入度（原始入度）。 2. 初始化 dp[0] = 0。 3. 遍历所有 mask 状态（0 到 (1<

注意事项： - 需要动态维护每个节点的入度：当选择节点 u 后，所有 u 指向的节点入度减1。 - 利润增量 = score[node] * (cnt + 1)。

解法二：基于优先队列的贪心（不可行）

  • 贪心思路：每次选择入度为0且得分最高的节点。
  • 反例：得分高的节点可能有很多后继，提前选择会限制后续选择。
  • 结论：贪心不能保证全局最优，因为位置权重随时间增加。

3️⃣ 代码实现

解法一：状态压缩 DP

func maxProfit(n int, edges [][]int, score []int) int64 {
    // 1. 预处理邻接表和入度
    indeg := make([]int, n)
    graph := make([][]int, n)
    for _, e := range edges {
        u, v := e[0], e[1]
        graph[u] = append(graph[u], v)
        indeg[v]++
    }

    // 2. DP 数组，dp[mask] 表示选择节点集合 mask 时的最大利润
    totalStates := 1 << n
    dp := make([]int64, totalStates)
    for i := range dp {
        dp[i] = -1
    }
    dp[0] = 0

    // 3. 遍历所有状态
    for mask := 0; mask < totalStates; mask++ {
        if dp[mask] < 0 {
            continue // 不可达状态
        }
        cnt := bits.OnesCount(uint(mask)) // 已选节点数
        // 4. 计算当前状态下每个节点的入度（考虑已选节点的影响）
        curIndeg := make([]int, n)
        copy(curIndeg, indeg)
        for u := 0; u < n; u++ {
            if mask>>u&1 == 1 { // u 已选
                for _, v := range graph[u] {
                    curIndeg[v]-- // u 已选，v 入度减1
                }
            }
        }
        // 5. 找出所有可选节点（入度为0且未选）
        for node := 0; node < n; node++ {
            if mask>>node&1 == 0 && curIndeg[node] == 0 {
                nextMask := mask | (1 << node)
                profit := dp[mask] + int64(score[node])*int64(cnt+1)
                if profit > dp[nextMask] {
                    dp[nextMask] = profit
                }
            }
        }
    }

    return dp[totalStates-1]
}

4️⃣ 复杂度分析

解法一：状态压缩 DP

  • 时间复杂度：O(2^n * n^2)
    • 状态数：2^n（n ≤ 22，最多约 4e6）
    • 每个状态需要 O(n) 计算当前入度（复制数组 O(n)，遍历邻接表 O(n)）
    • 总复杂度 O(2^n * n^2) ≈ 4e6 * 484 ≈ 1.9e9（最坏情况）
    • 实际运行中由于很多状态不可达，且 n=22 时 2^n=4e6 可接受（Go 约 1-2 秒）
  • 空间复杂度：O(2^n)
    • dp 数组大小 2^n
    • 辅助数组 O(n)
