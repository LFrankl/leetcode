1221. 分割平衡字符串

难度: Easy

标签: 贪心, 字符串, 计数

链接: https://leetcode.cn/problems/split-a-string-in-balanced-strings/

题目描述

平衡字符串 中，'L' 和 'R' 字符的数量是相同的。

给你一个平衡字符串 s，请你将它分割成尽可能多的子字符串，并满足：

  • 每个子字符串都是平衡字符串。

返回可以通过分割得到的平衡字符串的 最大数量 。

 

示例 1：

输入：s = "RLRRLLRLRL"
输出：4
解释：s 可以分割为 "RL"、"RRLL"、"RL"、"RL" ，每个子字符串中都包含相同数量的 'L' 和 'R' 。

示例 2：

输入：s = "RLRRRLLRLL"
输出：2
解释：s 可以分割为 "RL"、"RRRLLRLL"，每个子字符串中都包含相同数量的 'L' 和 'R' 。
注意，s 无法分割为 "RL"、"RR"、"RL"、"LR"、"LL" 因为第 2 个和第 5 个子字符串不是平衡字符串。

示例 3：

输入：s = "LLLLRRRR"
输出：1
解释：s 只能保持原样 "LLLLRRRR" 。

 

提示：

  • 2 <= s.length <= 1000
  • s[i] = 'L' 或 'R'
  • s 是一个 平衡 字符串

AI 解答 (DeepSeek)

解法一：贪心计数法

思路：遍历字符串，用一个计数器记录 L 和 R 的平衡状态。遇到 R 时计数器+1，遇到 L 时-1。每当计数器归零时，说明当前位置恰好构成了一个平衡子字符串，立即分割（贪心策略：越早分割，能得到越多子串）。

func balancedStringSplit(s string) int {
    count, result := 0, 0
    for _, ch := range s {
        if ch == 'R' {
            count++
        } else {
            count--
        }
        if count == 0 {
            result++
        }
    }
    return result
}

复杂度：时间 O(n)，空间 O(1)

贪心正确性说明：由于原字符串 s 本身是平衡的（L和R总数相等），计数器最终一定会归零。而在遍历过程中，每次计数器归零就意味着从上一个分割点到当前位置的子串是平衡的。贪心地在此处分割是最优的——因为如果延迟分割，只会把两个平衡子串合并成一个，减少总数，而不会让后续产生更多的分割点。
