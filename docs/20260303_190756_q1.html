1179. 重新格式化部门表

难度: Easy

标签: 数据库

链接: https://leetcode.cn/problems/reformat-department-table/

题目描述

表 Department：

+---------------+---------+
| Column Name   | Type    |
+---------------+---------+
| id            | int     |
| revenue       | int     |
| month         | varchar |
+---------------+---------+
在 SQL 中，(id, month) 是表的联合主键。
这个表格有关于每个部门每月收入的信息。
月份（month）可以取下列值 ["Jan","Feb","Mar","Apr","May","Jun","Jul","Aug","Sep","Oct","Nov","Dec"]。

 

重新格式化表格，使得 每个月 都有一个部门 id 列和一个收入列。

以 任意顺序 返回结果表。

结果格式如以下示例所示。

 

示例 1：

输入：
Department table:
+------+---------+-------+
| id   | revenue | month |
+------+---------+-------+
| 1    | 8000    | Jan   |
| 2    | 9000    | Jan   |
| 3    | 10000   | Feb   |
| 1    | 7000    | Feb   |
| 1    | 6000    | Mar   |
+------+---------+-------+
输出：
+------+-------------+-------------+-------------+-----+-------------+
| id   | Jan_Revenue | Feb_Revenue | Mar_Revenue | ... | Dec_Revenue |
+------+-------------+-------------+-------------+-----+-------------+
| 1    | 8000        | 7000        | 6000        | ... | null        |
| 2    | 9000        | null        | null        | ... | null        |
| 3    | null        | 10000       | null        | ... | null        |
+------+-------------+-------------+-------------+-----+-------------+
解释：四月到十二月的收入为空。 
请注意，结果表共有 13 列（1 列用于部门 ID，其余 12 列用于各个月份）。

1️⃣ 题意理解

核心要求： 将部门收入表的行结构（每个部门每月一行记录）转换为列结构（每个部门一行，包含12个月份的收入列）。

关键约束： 1. 原始表的主键是 (id, month)，意味着同一部门同一月份最多只有一条记录 2. 月份值固定为12个英文缩写 3. 结果表需要13列：1列部门ID + 12列月份收入（JanRevenue 到 DecRevenue） 4. 如果某部门在某月没有收入，对应单元格为 NULL 5. 输出顺序任意

输入输出格式： - 输入：Department 表，包含 id, revenue, month 三列 - 输出：转换后的宽表，包含 id 和 JanRevenue 到 DecRevenue 共13列

2️⃣ 解题思路

解法一：条件聚合（CASE WHEN + GROUP BY）

核心思想： 使用 CASE WHEN 语句将每个月份的收入值提取到单独的列中，然后通过 GROUP BY id 聚合，确保每个部门只有一行记录。

解题步骤： 1. 对每个部门ID进行分组 2. 为每个月份创建条件列：当月份匹配时取revenue值，否则为NULL 3. 使用聚合函数（如SUM或MAX）合并数据 4. 按部门ID排序（可选）

注意事项： - 必须使用聚合函数，因为GROUP BY后需要处理多行数据 - 使用SUM或MAX都可以，因为同一部门同一月份最多一条记录

解法二：PIVOT操作（如果数据库支持）

核心思想： 使用数据库的PIVOT语法直接进行行列转换（但MySQL不支持，SQL Server/Oracle支持）。

解题步骤： 1. 使用PIVOT关键字指定要转换的列 2. 指定月份值作为新列名 3. 指定聚合函数

注意事项： - 不是所有数据库都支持PIVOT语法 - LeetCode的SQL环境通常支持标准SQL语法

解法一：条件聚合

SELECT 
    id,
    MAX(CASE WHEN month = 'Jan' THEN revenue END) AS Jan_Revenue,
    MAX(CASE WHEN month = 'Feb' THEN revenue END) AS Feb_Revenue,
    MAX(CASE WHEN month = 'Mar' THEN revenue END) AS Mar_Revenue,
    MAX(CASE WHEN month = 'Apr' THEN revenue END) AS Apr_Revenue,
    MAX(CASE WHEN month = 'May' THEN revenue END) AS May_Revenue,
    MAX(CASE WHEN month = 'Jun' THEN revenue END) AS Jun_Revenue,
    MAX(CASE WHEN month = 'Jul' THEN revenue END) AS Jul_Revenue,
    MAX(CASE WHEN month = 'Aug' THEN revenue END) AS Aug_Revenue,
    MAX(CASE WHEN month = 'Sep' THEN revenue END) AS Sep_Revenue,
    MAX(CASE WHEN month = 'Oct' THEN revenue END) AS Oct_Revenue,
    MAX(CASE WHEN month = 'Nov' THEN revenue END) AS Nov_Revenue,
    MAX(CASE WHEN month = 'Dec' THEN revenue END) AS Dec_Revenue
FROM Department
GROUP BY id
ORDER BY id;

Go语言模拟实现（如果需要在应用层处理）：

package main

import (
    "fmt"
)

// 模拟数据库记录
type DepartmentRecord struct {
    ID      int
    Revenue int
    Month   string
}

// 转换后的结果行
type ResultRow struct {
    ID          int
    JanRevenue  *int
    FebRevenue  *int
    MarRevenue  *int
    AprRevenue  *int
    MayRevenue  *int
    JunRevenue  *int
    JulRevenue  *int
    AugRevenue  *int
    SepRevenue  *int
    OctRevenue  *int
    NovRevenue  *int
    DecRevenue  *int
}

func reformatDepartment(records []DepartmentRecord) []ResultRow {
    // 使用map按部门ID分组
    departmentMap := make(map[int]*ResultRow)

    // 月份到字段的映射
    monthMapping := map[string]func(*ResultRow, int){
        "Jan": func(r *ResultRow, val int) { r.JanRevenue = &val },
        "Feb": func(r *ResultRow, val int) { r.FebRevenue = &val },
        "Mar": func(r *ResultRow, val int) { r.MarRevenue = &val },
        "Apr": func(r *ResultRow, val int) { r.AprRevenue = &val },
        "May": func(r *ResultRow, val int) { r.MayRevenue = &val },
        "Jun": func(r *ResultRow, val int) { r.JunRevenue = &val },
        "Jul": func(r *ResultRow, val int) { r.JulRevenue = &val },
        "Aug": func(r *ResultRow, val int) { r.AugRevenue = &val },
        "Sep": func(r *ResultRow, val int) { r.SepRevenue = &val },
        "Oct": func(r *ResultRow, val int) { r.OctRevenue = &val },
        "Nov": func(r *ResultRow, val int) { r.NovRevenue = &val },
        "Dec": func(r *ResultRow, val int) { r.DecRevenue = &val },
    }

    // 处理每条记录
    for _, record := range records {
        // 获取或创建该部门的行
        if _, exists := departmentMap[record.ID]; !exists {
            departmentMap[record.ID] = &ResultRow{ID: record.ID}
        }

        // 根据月份设置对应的收入字段
        if setter, ok := monthMapping[record.Month]; ok {
            setter(departmentMap[record.ID], record.Revenue)
        }
    }

    // 将map转换为切片
    result := make([]ResultRow, 0, len(departmentMap))
    for _, row := range departmentMap {
        result = append(result, *row)
    }

    return result
}

func main() {
    // 测试数据
    records := []DepartmentRecord{
        {1, 8000, "Jan"},
        {2, 9000, "Jan"},
        {3, 10000, "Feb"},
        {1, 7000, "Feb"},
        {1, 6000, "Mar"},
    }

    result := reformatDepartment(records)

    // 打印结果
    for _, row := range result {
        fmt.Printf("ID: %d\n", row.ID)
        fmt.Printf("Jan: %v, Feb: %v, Mar: %v\n", 
            row.JanRevenue, row.FebRevenue, row.MarRevenue)
        fmt.Println("---")
    }
}

解法二：PIVOT语法（SQL Server示例）

SELECT *
FROM Department
PIVOT (
    MAX(revenue)
    FOR month IN (
        [Jan], [Feb], [Mar], [Apr], [May], [Jun],
        [Jul], [Aug], [Sep], [Oct], [Nov], [Dec]
    )
) AS PivotTable
ORDER BY id;

4️⃣ 复杂度分析

解法一：条件聚合

  • 时间复杂度：O(n)，其中n是表中的记录数。需要扫描一次表并进行分组聚合。
  • 空间复杂度：O(m)，其中m是不同部门的数量。需要为每个部门存储一行中间结果。

解法二：PIVOT操作

  • 时间复杂度：O(n)，与解法一类似，需要扫描并转换数据。
  • 空间复杂度：O(m)，需要存储转换后的结果。
