1️⃣ 题意理解

核心要求：
给定一个键盘，有 26 个字母（a~z），每个字母最多可以按 k 次。现在要按 n 次键，问能按出多少种不同的字符串（内容）。结果对 1e9+7 取模。

关键约束： - 每个字母最多按 k 次（0 ≤ 使用次数 ≤ k） - 总按键次数为 n - 字母顺序不同视为不同内容（即排列问题） - k 很小（1 ≤ k ≤ 5），n 最大为 26*k（≤ 130）

边界情况： - n = 0：只有空字符串一种（但题目 n ≥ 1） - k = 1：每个字母最多用一次，相当于从 26 个字母中选 n 个排列 - n > 26*k：不可能，因为最多 26*k 次按键

输入输出： - 输入：k（每个字母最大次数），n（总按键次数） - 输出：可能的字符串数量 mod 1e9+7

2️⃣ 解题思路

解法一：动态规划（DP）

核心思想：
将问题转化为：从 26 个字母中，每个字母可以选择 0~k 次，总次数为 n 的排列数。
设 dp[i][j] 表示使用前 i 个字母，总按键次数为 j 时的方案数。
对于第 i 个字母，可以选择使用 t 次（0 ≤ t ≤ min(k, j)），则：

dp[i][j] = Σ dp[i-1][j-t] * C(j, t)

其中 C(j, t) 是从 j 个位置中选 t 个位置放置当前字母的组合数。

步骤： 1. 初始化 dp[0][0] = 1 2. 预计算组合数 C[n][m] 3. 三重循环： - 遍历字母 i = 1..26 - 遍历总次数 j = 0..n - 遍历当前字母使用次数 t = 0..min(k, j) 4. 状态转移：dp[i][j] += dp[i-1][j-t] * C[j][t] 5. 最终答案 dp[26][n]

注意事项： - 组合数取模需用模逆元或预计算 - 三重循环但 k ≤ 5，实际可行

解法二：带组合数的 DP（优化空间）

核心思想：
与解法一相同，但使用滚动数组优化空间，因为 dp[i] 只依赖 dp[i-1]。

步骤： 1. 预计算组合数 2. 使用一维数组 dp[j] 表示当前字母考虑前的状态 3. 新数组 ndp[j] 表示考虑当前字母后的状态 4. 每考虑一个字母后，dp = ndp 5. 最终 dp[n] 即为答案

解法三：生成函数（数学）

核心思想：
每个字母的生成函数为 Σ_{t=0}^k x^t / t!，因为排列数需要考虑顺序。
26 个字母的生成函数为 (Σ_{t=0}^k x^t / t!)^26，展开后 x^n 的系数乘以 n! 即为答案。
由于 k 很小，可手动展开计算。

步骤： 1. 计算每个字母的生成函数多项式（系数为 1/t!） 2. 计算 26 次幂（多项式乘法） 3. 取 x^n 系数，乘以 n! 并取模

注意事项： - 需要模逆元计算阶乘逆元 - 多项式乘法复杂度较高，但 n ≤ 130 可行

3️⃣ 代码实现

解法一：动态规划（完整版）

func keyboard(k int, n int) int {
    const mod = 1_000_000_007

    // 预计算组合数 C[n][m]
    C := make([][]int, n+1)
    for i := range C {
        C[i] = make([]int, n+1)
        C[i][0], C[i][i] = 1, 1
        for j := 1; j < i; j++ {
            C[i][j] = (C[i-1][j-1] + C[i-1][j]) % mod
        }
    }

    // dp[i][j]: 使用前 i 个字母，总次数为 j 的方案数
    dp := make([][]int, 27)
    for i := range dp {
        dp[i] = make([]int, n+1)
    }
    dp[0][0] = 1

    for i := 1; i <= 26; i++ {
        for j := 0; j <= n; j++ {
            // 当前字母使用 t 次
            maxT := min(k, j)
            for t := 0; t <= maxT; t++ {
                // 从 j 个位置中选 t 个放当前字母
                dp[i][j] = (dp[i][j] + dp[i-1][j-t]*C[j][t]) % mod
            }
        }
    }

    return dp[26][n]
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}

解法二：滚动数组优化

func keyboard(k int, n int) int {
    const mod = 1_000_000_007

    // 预计算组合数
    C := make([][]int, n+1)
    for i := range C {
        C[i] = make([]int, n+1)
        C[i][0], C[i][i] = 1, 1
        for j := 1; j < i; j++ {
            C[i][j] = (C[i-1][j-1] + C[i-1][j]) % mod
        }
    }

    dp := make([]int, n+1)
    dp[0] = 1

    for i := 1; i <= 26; i++ {
        ndp := make([]int, n+1)
        for j := 0; j <= n; j++ {
            maxT := min(k, j)
            for t := 0; t <= maxT; t++ {
                ndp[j] = (ndp[j] + dp[j-t]*C[j][t]) % mod
            }
        }
        dp = ndp
    }

    return dp[n]
}

4️⃣ 复杂度分析

解法一：

  • ⏱️ 时间复杂度：O(26 * n * k)
    外层字母 26，中间总次数 n，内层使用次数 k（≤5），实际约 261305 ≈ 16900 次操作。
  • 💾 空间复杂度：O(26 * n)
    dp 数组大小 27(n+1)，组合数数组 (n+1)(n+1)，n ≤ 130，内存很小。

解法二：

  • ⏱️ 时间复杂度：O(26 * n * k)
    与解法一相同。
  • 💾 空间复杂度：O(n)
    使用滚动数组，只需两个一维数组。
