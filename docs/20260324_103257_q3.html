2218. 从栈中取出 K 个硬币的最大面值和

难度: Hard

标签: 数组, 动态规划, 前缀和

链接: https://leetcode.cn/problems/maximum-value-of-k-coins-from-piles/

题目描述

一张桌子上总共有 n 个硬币 栈 。每个栈有 正整数 个带面值的硬币。

每一次操作中，你可以从任意一个栈的 顶部 取出 1 个硬币，从栈中移除它，并放入你的钱包里。

给你一个列表 piles ，其中 piles[i] 是一个整数数组，分别表示第 i 个栈里 从顶到底 的硬币面值。同时给你一个正整数 k ，请你返回在 恰好 进行 k 次操作的前提下，你钱包里硬币面值之和 最大为多少 。

 

示例 1：

输入：piles = [[1,100,3],[7,8,9]], k = 2
输出：101
解释：
上图展示了几种选择 k 个硬币的不同方法。
我们可以得到的最大面值为 101 。

示例 2：

输入：piles = [[100],[100],[100],[100],[100],[100],[1,1,1,1,1,1,700]], k = 7
输出：706
解释：
如果我们所有硬币都从最后一个栈中取，可以得到最大面值和。

 

提示：

  • n == piles.length
  • 1 <= n <= 1000
  • 1 <= piles[i][j] <= 105
  • 1 <= k <= sum(piles[i].length) <= 2000

1️⃣ 题意理解

题目重述
我们有 n 个栈，每个栈里从上到下放着若干枚硬币，每枚硬币有一个面值。我们可以从任意一个栈的顶部取出一枚硬币（移除并放入钱包），最多进行 k 次操作。目标是：在恰好取 k 枚硬币的情况下，使钱包中硬币面值之和最大。

关键约束
- 每个栈的硬币数量可能不同，但总硬币数不超过 2000。 - 只能从栈顶取硬币，即每个栈取硬币的顺序是固定的（从上到下）。 - k 不超过总硬币数。 - 硬币面值均为正整数。

输入输出格式
- 输入：piles 是一个二维整数数组，piles[i] 表示第 i 个栈从上到下的硬币面值；k 为整数。 - 输出：一个整数，表示最大面值和。

2️⃣ 解题思路

核心思想

这是一个分组背包问题： - 每个栈是一个“组”，我们可以从这个组中取 0 到 min(len(piles[i]), k) 枚硬币。 - 从同一个栈取 t 枚硬币的收益是固定的（取前 t 枚的面值和）。 - 我们需要从每组中至多选择一个“物品”（取 t 枚硬币），使得总硬币数恰好为 k 时总收益最大。

解法一：动态规划（分组背包）

算法步骤： 1. 预处理每个栈的前缀和，方便快速得到取 t 枚硬币的收益。 2. 定义 dp[j] 表示取 j 枚硬币时的最大面值和。 3. 遍历每个栈（组），对于每个栈，我们考虑取 0 到 min(len(piles[i]), k) 枚硬币的所有可能，并更新 dp。 4. 注意更新顺序：为了避免同一组被重复取，需要从 k 到 0 逆序更新 dp。

注意事项： - 初始化 dp[0] = 0，其余为负无穷（或一个很小的负数），因为要求恰好取 k 枚。 - 每个栈最多取 min(len(piles[i]), k) 枚，超过栈长的部分不考虑。

解法二：动态规划（二维 DP）

  • 定义 dp[i][j] 表示从前 i 个栈中取 j 枚硬币的最大面值和。
  • 状态转移：dp[i][j] = max(dp[i-1][j-t] + sum[i][t])，其中 t 为从第 i 个栈取的硬币数。
  • 可以优化为一维数组（即解法一）。

3️⃣ 代码实现

解法一：动态规划（分组背包，一维优化）

func maxValueOfCoins(piles [][]int, k int) int {
    // dp[j] 表示取 j 枚硬币时的最大面值和
    dp := make([]int, k+1)
    for i := 1; i <= k; i++ {
        dp[i] = -1 << 31 // 初始化为极小值，表示不可达
    }
    dp[0] = 0

    // 遍历每个栈（组）
    for _, pile := range piles {
        m := len(pile)
        // 计算前缀和，prefix[t] 表示取该栈前 t 枚硬币的面值和
        prefix := make([]int, m+1)
        for t := 1; t <= m; t++ {
            prefix[t] = prefix[t-1] + pile[t-1]
        }

        // 逆序更新 dp，避免同一组重复取
        for j := k; j >= 0; j-- {
            // 从当前栈取 t 枚硬币
            for t := 1; t <= m && t <= j; t++ {
                if dp[j-t] != -1<<31 {
                    dp[j] = max(dp[j], dp[j-t]+prefix[t])
                }
            }
        }
    }
    return dp[k]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

4️⃣ 复杂度分析

解法一

  • ⏱️ 时间复杂度：O(n * k * min(m, k))，其中 n 是栈的数量，m 是每个栈的平均长度。由于总硬币数不超过 2000，且 k 不超过总硬币数，因此最坏情况下约为 O(2000 * 2000) = O(4e6)，可接受。
  • 💾 空间复杂度：O(k)，使用一维 DP 数组。

复杂度来源： - 外层循环遍历每个栈（n 次）。 - 内层循环遍历 j 从 k 到 0（k 次）。 - 最内层循环遍历从当前栈取的硬币数 t（最多 min(m, k) 次）。

5️⃣ 解法对比

解法 优点 缺点 适用场景
一维 DP（分组背包） 空间优化好，代码简洁 状态转移需逆序，理解稍难 推荐，适用于大多数情况
二维 DP 直观，易于理解 空间复杂度 O(n*k) 当需要记录完整路径时

推荐解法一：因为空间复杂度低，且运行效率高，适合本题的数据规模。

输出：10
解释：从第二个栈取顶部 1，或从第一个栈取 5，最大为 10（取第二个栈的顶部 1？不对，第二个栈顶部是 1，最大是 5？等等，第二个栈顶部是 1，但我们可以取任意栈的顶部，所以最大是 max(5,1)=5？等等，题目要求取 k 枚，但每个栈只能从顶部取，所以对于第二个栈，如果取 1 枚就是 1，取 2 枚才是 1+10=11，但 k=1，所以只能取 1 枚，所以最大是 max(5,1)=5。但示例中 k=2 时才能取到 100。所以这里 k=1 时，最大是 5。)
