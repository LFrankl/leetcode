788. 旋转数字

难度: Medium

标签: 数学, 动态规划

链接: https://leetcode.cn/problems/rotated-digits/

题目描述

我们称一个数 X 为好数, 如果它的每位数字逐个地被旋转 180 度后，我们仍可以得到一个有效的，且和 X 不同的数。要求每位数字都要被旋转。

如果一个数的每位数字被旋转以后仍然还是一个数字， 则这个数是有效的。0, 1, 和 8 被旋转后仍然是它们自己；2 和 5 可以互相旋转成对方（在这种情况下，它们以不同的方向旋转，换句话说，2 和 5 互为镜像）；6 和 9 同理，除了这些以外其他的数字旋转以后都不再是有效的数字。

现在我们有一个正整数 N, 计算从 1 到 N 中有多少个数 X 是好数？

 

示例：

输入: 10
输出: 4
解释: 
在[1, 10]中有四个好数： 2, 5, 6, 9。
注意 1 和 10 不是好数, 因为他们在旋转之后不变。

 

提示：

  • N 的取值范围是 [1, 10000]。

1️⃣ 题意理解

题目重述：
给定正整数 N，统计 1 到 N 中“好数”的个数。
- 好数定义：将数字每一位旋转 180° 后，得到的新数字是有效的且与原数字不同。
- 旋转规则：
- 0、1、8 旋转后仍为自身
- 2 和 5 互相旋转
- 6 和 9 互相旋转
- 其他数字（3、4、7）旋转后无效
- 要求：每一位数字都必须旋转后有效，且最终结果不能与原数相同。

关键约束：
- N ∈ [1, 10000]
- 数字旋转后必须每一位都有效，且整体不能与原数相同
- 数字中不能包含 3、4、7，否则无效
- 数字中必须至少包含一个 2、5、6、9 中的一个，否则旋转后与原数相同

输入输出：
- 输入：整数 N
- 输出：1 到 N 中好数的个数

2️⃣ 解题思路

解法一：暴力枚举 + 逐位判断

  • 核心思想：遍历 1 到 N 的每个数，检查其每一位是否符合旋转规则，并判断旋转后是否与原数不同。
  • 步骤：
    1. 定义旋转映射表：0→0, 1→1, 8→8, 2→5, 5→2, 6→9, 9→6，其他数字无效。
    2. 对每个数，逐位旋转：
      • 若遇到无效数字（3、4、7），直接跳过该数
      • 否则记录旋转后的数字
    3. 比较旋转后的数字与原数字，若不同则计数加一。
  • 注意事项：
    • 旋转后数字可能位数不同？不会，因为每一位都映射到另一个有效数字。
    • 注意数字为 0 的情况，但题目从 1 开始。

解法二：数位 DP（动态规划）

  • 核心思想：使用动态规划统计满足条件的数字个数，适用于 N 很大时（本题 N ≤ 10000，暴力足够，但 DP 可扩展）。
  • 步骤：
    1. 将 N 转为字符串，从高位向低位 DP。
    2. 状态设计：dp[pos][hasDiff][tight] 表示：
      • pos：当前处理到的位置
      • hasDiff：是否已包含 2/5/6/9（保证旋转后不同）
      • tight：是否受 N 的限制（前缀是否与 N 相同）
    3. 递归或迭代计算，只选择有效数字（0、1、2、5、6、8、9），并更新 hasDiff。
    4. 最终统计 hasDiff = true 的数字个数。
  • 注意事项：
    • 注意前导零的处理，因为数字不能有前导零（除了 0 本身，但题目从 1 开始）。

3️⃣ 代码实现

解法一：暴力枚举

func rotatedDigits(N int) int {
    count := 0
    for i := 1; i <= N; i++ {
        if isValid(i) {
            count++
        }
    }
    return count
}

func isValid(num int) bool {
    rotateMap := map[int]int{
        0: 0, 1: 1, 8: 8,
        2: 5, 5: 2,
        6: 9, 9: 6,
    }
    original := num
    rotated := 0
    base := 1
    for num > 0 {
        digit := num % 10
        if newDigit, ok := rotateMap[digit]; ok {
            rotated = newDigit*base + rotated
            base *= 10
            num /= 10
        } else {
            return false // 包含无效数字
        }
    }
    return rotated != original
}

解法二：数位 DP

func rotatedDigits(N int) int {
    s := strconv.Itoa(N)
    n := len(s)
    memo := make([][2][2]int, n)
    for i := range memo {
        memo[i] = [2][2]int{{-1, -1}, {-1, -1}}
    }
    var dfs func(pos int, hasDiff bool, tight bool) int
    dfs = func(pos int, hasDiff bool, tight bool) int {
        if pos == n {
            if hasDiff {
                return 1
            }
            return 0
        }
        if memo[pos][boolToInt(hasDiff)][boolToInt(tight)] != -1 {
            return memo[pos][boolToInt(hasDiff)][boolToInt(tight)]
        }
        limit := 9
        if tight {
            limit = int(s[pos] - '0')
        }
        res := 0
        for d := 0; d <= limit; d++ {
            if d == 3 || d == 4 || d == 7 {
                continue
            }
            nextTight := tight && (d == limit)
            nextHasDiff := hasDiff || d == 2 || d == 5 || d == 6 || d == 9
            res += dfs(pos+1, nextHasDiff, nextTight)
        }
        memo[pos][boolToInt(hasDiff)][boolToInt(tight)] = res
        return res
    }
    return dfs(0, false, true)
}

func boolToInt(b bool) int {
    if b {
        return 1
    }
    return 0
}

4️⃣ 复杂度分析

解法一：暴力枚举

  • ⏱️ 时间复杂度：O(N * log N)
    遍历 N 个数，每个数处理其位数（约 log₁₀ N 位），故总复杂度 O(N log N)。
    当 N=10000 时，log N ≈ 4，完全可行。
  • 💾 空间复杂度：O(1)
    只使用常数空间。

解法二：数位 DP

  • ⏱️ 时间复杂度：O(log N * 2 * 2 * 10) ≈ O(log N)
    状态数：位置数 × hasDiff(2) × tight(2) ≈ 4 * log N，每个状态枚举 0~9 共 10 种可能。
  • 💾 空间复杂度：O(log N)
    用于存储 memo 数组。

5️⃣ 解法对比

解法 优点 缺点 适用场景
暴力枚举 实现简单，直观易懂 N 很大时效率低 N ≤ 10⁵ 左右
数位 DP 高效，可处理超大 N（如 10¹⁸） 实现复杂，状态设计需谨慎 N 非常大时

推荐：
本题 N ≤ 10000，暴力枚举完全足够，且代码简洁，易于理解和调试。
数位 DP 可作为扩展学习，适用于更一般化的问题（如统计区间内满足某种数位条件的数字个数）。
